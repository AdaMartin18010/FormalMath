## 1. 类型推导算法

### 问题描述

实现简单类型λ演算的类型推导算法。

### 解决方案

#### 1.1 Hindley-Milner算法

```python
def type_infer(expr, gamma=None):
    """Hindley-Milner类型推导"""
    if gamma is None:
        gamma = {}
    
    if isinstance(expr, Var):
        # 查找环境
        return instantiate(gamma[expr.name])
    
    elif isinstance(expr, Lambda):
        # 生成新的类型变量
        arg_type = fresh_type_var()
        new_gamma = gamma.copy()
        new_gamma[expr.arg] = arg_type
        body_type = type_infer(expr.body, new_gamma)
        return Arrow(arg_type, body_type)
    
    elif isinstance(expr, Apply):
        # 应用规则
        func_type = type_infer(expr.func, gamma)
        arg_type = type_infer(expr.arg, gamma)
        result_type = fresh_type_var()
        # 统一: func_type = arg_type → result_type
        unify(func_type, Arrow(arg_type, result_type))
        return result_type
    
    elif isinstance(expr, Let):
        # 多态let
        val_type = type_infer(expr.value, gamma)
        val_type_general = generalize(val_type, gamma)
        new_gamma = gamma.copy()
        new_gamma[expr.name] = val_type_general
        return type_infer(expr.body, new_gamma)
```

#### 1.2 统一算法

```python
def unify(t1, t2):
    """Robinson统一算法"""
    if isinstance(t1, TypeVar):
        if t1 != t2:
            if occurs_in(t1, t2):
                raise UnificationError("Occurs check failed")
            t1.instance = t2
    elif isinstance(t2, TypeVar):
        unify(t2, t1)
    elif isinstance(t1, Arrow) and isinstance(t2, Arrow):
        unify(t1.arg, t2.arg)
        unify(t1.ret, t2.ret)
    else:
        raise UnificationError(f"Cannot unify {t1} with {t2}")
```

#### 1.3 复杂度

**定理**: Hindley-Milner类型推断是DEXPTIME完全的。

## 2. 向量类型安全操作

### 问题描述

使用依赖类型实现安全的向量操作。

### 解决方案

#### 2.1 向量定义与操作

```
-- 向量类型定义
inductive Vec (A : Type) : ℕ → Type
  | nil : Vec A 0
  | cons : ∀ n, A → Vec A n → Vec A (n+1)

-- 安全head：类型保证非空

def head {A} {n} (v : Vec A (n+1)) : A :=
  match v with
  | cons _ x _ => x
  -- nil情况被类型系统排除！

-- 安全tail
def tail {A} {n} (v : Vec A (n+1)) : Vec A n :=
  match v with
  | cons _ _ xs => xs

-- 安全索引
def get {A} {n} (v : Vec A n) (i : Fin n) : A :=
  match v, i with
  | cons _ x _, zero _ => x
  | cons _ _ xs, succ _ i' => get xs i'
```

#### 2.2 向量连接与长度证明

```
def append {A m n} (xs : Vec A m) (ys : Vec A n) : Vec A (m + n) :=
  match xs with
  | nil => ys
  | cons _ x xs' => cons _ x (append xs' ys)

-- 编译时保证: length(append xs ys) = length(xs) + length(ys)
```

## 3. 同伦类型论证明

### 问题描述

在HoTT中证明基本性质。

### 解决方案

#### 3.1 路径归纳应用

```
-- 路径的对称性
def path_symm {A} {x y : A} (p : x = y) : y = x :=
  match p with
  | refl => refl

-- 路径的传递性  
def path_trans {A} {x y z : A} (p : x = y) (q : y = z) : x = z :=
  match p, q with
  | refl, refl => refl

-- 路径的函数作用 (ap)
def ap {A B} (f : A → B) {x y : A} (p : x = y) : f x = f y :=
  match p with
  | refl => refl
```

#### 3.2 基于路径的证明

```
-- 证明 (p · q) · r = p · (q · r)
def path_assoc {A} {x y z w : A} 
  (p : x = y) (q : y = z) (r : z = w) :
  (p · q) · r = p · (q · r) :=
  match p with
  | refl => 
    match q with
    | refl => 
      match r with
      | refl => refl
```

#### 3.3 等价与Univalence

```
-- 类型等价
def equiv (A B : Type) :=
  Σ (f : A → B), is_equiv f

-- Univalence: (A = B) ≃ (A ≃ B)
axiom univalence : ∀ A B, (A = B) ≃ (A ≃ B)

-- 应用: 结构传递
example (A B C : Type) (e1 : A ≃ B) (e2 : B ≃ C) : A ≃ C :=
  transport (λ X, X ≃ C) 
    (univalence⁻¹ e1) e2
```

## 4. 归纳类型与递归模式

### 问题描述

定义递归函数并证明其性质。

### 解决方案

#### 4.1 结构递归

```
-- 自然数加法
def add (m n : ℕ) : ℕ :=
  match n with
  | zero => m
  | succ n' => succ (add m n')

-- 证明加法交换律
theorem add_comm (m n : ℕ) : add m n = add n m :=
  match n with
  | zero => 
    -- add m 0 = m = add 0 m
    by simp [add]
  | succ n' =>
    -- add m (succ n') = succ (add m n')
    -- = succ (add n' m) = add (succ n') m
    by simp [add, add_comm m n']
```

#### 4.2 良基递归

```
-- Ackermann函数（非结构递归）
def ackermann : ℕ → ℕ → ℕ
  | 0, n => n + 1
  | m+1, 0 => ackermann m 1
  | m+1, n+1 => ackermann m (ackermann (m+1) n)

-- 证明终止性需要良基归纳
```

## 6. 类型级编程

### 问题描述

使用类型系统实现编译时计算。

### 解决方案

#### 6.1 类型级自然数

```
inductive Nat : Type
  | zero : Nat
  | succ : Nat → Nat

-- 类型级加法
def Add : Nat → Nat → Nat
  | zero, n => n
  | succ m, n => succ (Add m n)

-- 使用：Vec A (Add m n) 表示长度m+n的向量
```

#### 6.2 类型安全的状态机

```
-- 状态作为类型
inductive DoorState

  | Closed
```

## 7. 类型论中的数学结构

### 问题描述

形式化代数结构。

### 解决方案

#### 7.1 群的形式化

```
class Group (G : Type) where
  mul : G → G → G
  one : G
  inv : G → G
  
  mul_assoc : ∀ a b c, mul (mul a b) c = mul a (mul b c)
  one_mul : ∀ a, mul one a = a
  mul_one : ∀ a, mul a one = a
  mul_left_inv : ∀ a, mul (inv a) a = one

-- 群同态
def GroupHom (G H : Type) [Group G] [Group H] :=
  { f : G → H // 
    ∀ x y, f (mul x y) = mul (f x) (f y) }
```

#### 7.2 范畴的形式化

```
class Category (Obj : Type) (Hom : Obj → Obj → Type) where
  id : ∀ X, Hom X X
  comp : ∀ {X Y Z}, Hom Y Z → Hom X Y → Hom X Z
  
  id_comp : ∀ {X Y} (f : Hom X Y), comp (id Y) f = f
  comp_id : ∀ {X Y} (f : Hom X Y), comp f (id X) = f
  assoc : ∀ {W X Y Z} (f : Hom W X) (g : Hom X Y) (h : Hom Y Z),
    comp (comp h g) f = comp h (comp g f)
```
