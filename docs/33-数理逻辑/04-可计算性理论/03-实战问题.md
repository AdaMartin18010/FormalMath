# 可计算性理论实战问题

## 一、归约构造实例

### 问题描述

证明3-SAT ≤_p CLIQUE（3-SAT可归约到团问题）。

### 解决方案

#### 1.1 归约构造

**输入**: 3-CNF公式 φ = C₁ ∧ C₂ ∧ ... ∧ C_k，每个Cᵢ = (lᵢ₁ ∨ lᵢ₂ ∨ lᵢ₃)

**输出**: 图G = (V, E) 和整数k

**构造**:

```

顶点集 V:
- 对每个子句Cᵢ，添加3个顶点vᵢ₁, vᵢ₂, vᵢ₃
- 标记: vᵢⱼ 标记为 (i, j, lᵢⱼ)

边集 E:
- 连接不同子句的顶点
- 不连接: 
  - 同一子句内的顶点
  - 矛盾的文字（x和¬x）

```

#### 1.2 正确性证明

**引理**: φ可满足 ⟺ G有k-团

**证明**:
- (⇒) 取满足赋值，从每个子句选一个满足的文字，对应顶点形成团
- (⇐) k-团必含每个子句恰好一个顶点，对应文字无矛盾，构造满足赋值

## 二、NP完全性证明：顶点覆盖

### 问题描述

证明顶点覆盖是NP完全的。

### 解决方案

#### 2.1 问题定义

**顶点覆盖**: 图G = (V, E)的子集S ⊆ V使得每条边至少有一个端点在S中。

**决策问题**: 给定(G, k)，是否存在大小≤k的顶点覆盖？

#### 2.2 NP成员性

**验证器**: 给定S，检查|S| ≤ k且S覆盖所有边。

**时间**: O(|V| + |E|)，多项式。

#### 2.3 从3-SAT归约

**构造**:

```

对每个变量xᵢ: 创建边(xᵢ, ¬xᵢ)的图（变量小工具）
对每个子句Cⱼ = (a ∨ b ∨ c): 创建三角形(a', b', c')（子句小工具）
连接: 将子句文字与对应变量文字连接
设置k = n + 2m（n个变量，m个子句）

```

**正确性**: φ可满足 ⟺ G有大小为k的顶点覆盖

## 三、图灵机模拟实现

### 问题描述

实现通用图灵机(UTM)的概念模型。

### 解决方案

#### 3.1 编码方案

**状态编码**: 

```

q₀ → 0
q₁ → 00
q_accept → 000
...

```

**转移函数编码**:

```

δ(q, a) = (p, b, D) → 0^q 1 0^a 1 0^p 1 0^b 1 0^D
其中 D=0 (L) 或 D=00 (R)

```

**图灵机编码**: 所有转移的序列，用11分隔

#### 3.2 模拟算法

```python
class UniversalTM:
    def __init__(self, tm_description):
        self.parse_tm(tm_description)
    
    def simulate(self, input_string, max_steps=10000):
        tape = list(input_string) + ['⊔'] * 100
        head = 0
        state = 'q₀'
        steps = 0
        
        while state not in ['q_accept', 'q_reject']:
            if steps > max_steps:
                return 'TIMEOUT'
            
            symbol = tape[head]
            transition = self.find_transition(state, symbol)
            
            if transition is None:
                return 'REJECT'
            
            new_state, write_symbol, direction = transition
            tape[head] = write_symbol
            state = new_state
            
            if direction == 'R':
                head += 1
            else:
                head -= 1
            
            steps += 1
        
        return 'ACCEPT' if state == 'q_accept' else 'REJECT'

```

## 四、P与NP问题分析

### 问题描述

分析某个具体问题是否可能在P中。

### 解决方案

#### 4.1 2-SAT ∈ P

**问题**: 每个子句至多2个文字的SAT。

**算法**: 
1. 构建暗示图
2. 检查强连通分量
3. 不存在x和¬x在同一SCC中

**时间**: O(n + m)，线性。

#### 4.2 Horn-SAT ∈ P

**问题**: 每个子句至多一个正文字。

**算法**: 单元传播（贪心）

#### 4.3 整数规划 ∉ P（可能）

**对比**:
- 线性规划 ∈ P（内点法、椭球法）
- 整数规划 NP难

**原因**: 整数约束引入组合爆炸。

## 五、近似算法分析

### 问题描述

为顶点覆盖设计近似算法并分析。

### 解决方案

#### 5.1 2-近似算法

```python
def approx_vertex_cover(G):
    C = set()
    E = set(G.edges())
    
    while E:
        (u, v) = E.pop()
        C.add(u)
        C.add(v)
        # 移除与u或v关联的所有边
        E = {e for e in E if u not in e and v not in e}
    
    return C

```

#### 5.2 近似比分析

**定理**: 算法返回的覆盖C满足 |C| ≤ 2|OPT|。

**证明**:
1. 算法选择的边集M是匹配（无公共顶点）
2. |C| = 2|M|

3. 任何顶点覆盖必须包含M中每条边至少一个端点
4. 因此 |OPT| ≥ |M|
5. 故 |C| = 2|M| ≤ 2|OPT|

#### 5.3 近似难度

**定理**: 若P ≠ NP，则顶点覆盖没有(2-ε)-近似（UFG假设下）。

## 六、概率算法设计

### 问题描述

设计并分析随机化算法。

### 解决方案

#### 6.1 随机快速排序

```python
import random

def randomized_quicksort(arr):
    if len(arr) <= 1:
        return arr
    
    pivot = random.choice(arr)
    left = [x for x in arr if x < pivot]
    middle = [x for x in arr if x == pivot]
    right = [x for x in arr if x > pivot]
    
    return randomized_quicksort(left) + middle + randomized_quicksort(right)

```

**期望时间**: O(n log n)

#### 6.2 多项式恒等检验

**问题**: 检验多项式P(x₁, ..., xₙ)是否恒等于0。

**算法** (Schwartz-Zippel):

```

随机选择有限集S中的值a₁, ..., aₙ
若 P(a₁, ..., aₙ) = 0，返回"可能是零"
否则返回"非零"

```

**错误概率**: ≤ d/|S|（d为多项式次数）

**应用**: 二分图完美匹配检验。

## 七、计算复杂性下界

### 问题描述

证明某个问题的计算下界。

### 解决方案

#### 7.1 比较排序下界

**定理**: 任何基于比较的排序算法在最坏情况下需要Ω(n log n)次比较。

**证明**:
1. 决策树模型
2. n!个叶子（所有排列）
3. 树高 ≥ log(n!) = Ω(n log n)

#### 7.2 通信复杂性

**模型**: Alice有x，Bob有y，计算f(x, y)。

**下界技术**:
- 划分数
- 秩方法
- 差异方法

**例子**: 相等函数EQ(x, y) = [x = y]需要n位通信。
