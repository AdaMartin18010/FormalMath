## I. Project Vision and Goals

### 1.1 Vision

**"Enabling every mathematics learner to find their own learning path"**

FormalMath is committed to building a comprehensive educational knowledge base covering mathematics from fundamentals to modern frontiers. Through structured content, the unique perspective of mathematician ideology systems, close alignment with internationally authoritative courses, and support from intelligent learning systems, we provide personalized mathematics learning experiences for learners from diverse backgrounds.

### 1.2 Goals

| Goal Level | Specific Goal | Completion Status |
|------------|---------------|-------------------|
| **Basic Goals** | Establish complete documentation system for 15 mathematical branches | ✅ 100% |
| **Core Goals** | Build ideology systems for 62 mathematicians | ✅ 100% |
| **Alignment Goals** | Establish mappings with 96+ international courses from seven universities | ✅ 100% |
| **Technical Goals** | Develop three intelligent learning systems | ✅ 100% |
| **Formalization Goals** | 20 Lean4 theorem formalizations | ✅ 100% |

### 1.3 Project Value

1. **Academic Value**: Systematically organize mathematicians' thoughts and establish knowledge association networks
2. **Educational Value**: Provide structured learning paths and lower the barrier to mathematics learning
3. **Technical Value**: Explore possibilities of AI-assisted mathematics education
4. **International Value**: Align with world-class mathematics courses to promote academic exchange

### 2.4 Intelligent Learning Systems

The project has developed three interconnected intelligent learning systems:

#### Cognitive Diagnosis System (T2.1)
- **Tech Stack**: Python + FastAPI + PostgreSQL + React + TypeScript
- **Core Function**: HCD diagnosis algorithm, 100+ diagnostic questions
- **Diagnosis Dimensions**: Knowledge mastery, cognitive ability, learning style

#### Assessment System (T2.2)
- **Tech Stack**: Python + FastAPI + SQLAlchemy + React + Recharts
- **Core Function**: Multi-dimensional evaluation, feedback engine, report generation
- **Evaluation Dimensions**: Knowledge, skills, thinking, innovation

#### Adaptive Learning System (T3.1)
- **Tech Stack**: Python + FastAPI + React + JavaScript
- **Core Function**: A* path algorithm, real-time adjustment, knowledge graph
- **Adaptation Dimensions**: Learning progress, cognitive state, interest preferences

### 4.1 How to Choose Your Starting Point

Select an appropriate learning starting point based on your mathematics background:

#### Beginner Path (High School → Freshman)

| Stage | Document | Estimated Time |
|-------|----------|----------------|
| 1 | `docs/01-基础数学/集合论/01-集合论基础-国际标准版.md` | 1 week |
| 2 | `docs/01-基础数学/集合论/02-数系与运算-深度扩展版.md` | 1 week |
| 3 | `docs/02-代数结构/线性代数/线性代数-深度扩展版.md` | 3 weeks |

#### Intermediate Path (Sophomore → Junior)

| Stage | Document | Estimated Time |

### 4.2 How to Utilize International Course Alignment

1. **Find Your Course**: Locate the corresponding school/course in the [International Course Alignment Table](../project/00-国际课程与机构对齐对照表-2026年4月.md)
2. **View Mapping Relationships**: Understand which project documents correspond to each chapter of the course
3. **Learn in Order**: Follow the recommended learning sequence
4. **Reference Authoritative Resources**: Utilize Stacks tags and textbook chapters referenced in the documents

### 4.3 How to Use the Mathematician Ideology System

1. **Select an Interested Mathematician**: Start from the Mathematician Index
2. **Read START-HERE.md**: Quickly understand the mathematician's core contributions
3. **Deep Dive into 02-DeepMathematicalAnalysis**: Learn specific mathematical theories
4. **Explore 08-KnowledgeAssociation**: Understand connections between concepts

### 4.4 How to Run Lean4 Code

```bash
# Install Lean4 and Mathlib4
# See docs/09-形式化证明/Lean4入门指南.md

# Compile project theorems
cd docs/09-形式化证明/lean4-examples/
lean --make
```
