# FormalMath项目文档格式规范

**制定日期**: 2025年12月29日
**版本**: v1.0
**适用范围**: 项目所有Markdown文档
**状态**: 正式执行

---

## 📋 规范概述

本文档定义FormalMath项目中所有Markdown文档的标准格式，确保文档格式的一致性、可读性和专业性。

---

## 🎯 格式原则

1. **一致性**: 同类内容使用统一格式
2. **可读性**: 格式清晰易读
3. **专业性**: 符合学术文档标准
4. **可维护性**: 便于后续维护和更新
5. **国际化**: 符合国际标准

---

## 📝 Markdown格式规范

### 1. 文档头部

**格式**: 每个文档应包含标准头部

```markdown
# [文档标题]

**创建日期**: YYYY年MM月DD日
**最后更新**: YYYY年MM月DD日
**版本**: vX.X
**状态**: [草稿/进行中/完成/归档]

---

## 📋 [概述/简介]

[文档概述内容]

---
```

**规则**:

- 使用一级标题作为文档标题
- 包含元数据（日期、版本、状态）
- 使用分隔线分隔头部和正文

### 2. 标题层级

**格式**: 使用标准的标题层级

```markdown
# 一级标题（文档标题）
## 二级标题（主要章节）
### 三级标题（子章节）
#### 四级标题（小节）
##### 五级标题（子小节）
```

**规则**:

- 一级标题仅用于文档标题
- 二级标题用于主要章节
- 三级标题用于子章节
- 避免使用超过五级标题
- 标题后空一行

### 3. 列表格式

#### 3.1 无序列表

**格式**:

```markdown
- 项目1
- 项目2
  - 子项目2.1
  - 子项目2.2
- 项目3
```

**规则**:

- 使用`-`作为列表标记
- 子项目使用2个空格缩进
- 列表项后空一行（如果后面有段落）

#### 3.2 有序列表

**格式**:

```markdown
1. 第一项
2. 第二项
   1. 子项2.1
   2. 子项2.2
3. 第三项
```

**规则**:

- 使用数字作为列表标记
- 子项目使用3个空格缩进
- 保持数字连续

#### 3.3 任务列表

**格式**:

```markdown
- [ ] 未完成任务
- [x] 已完成任务
```

**规则**:

- 使用`- [ ]`表示未完成
- 使用`- [x]`表示已完成
- 用于任务跟踪和检查清单

### 4. 表格格式

**格式**:

```markdown
| 列1 | 列2 | 列3 |
|-----|-----|-----|
| 数据1 | 数据2 | 数据3 |
| 数据4 | 数据5 | 数据6 |
```

**规则**:

- 使用管道符`|`分隔列
- 表头和数据行之间使用分隔行
- 对齐方式：左对齐（默认）
- 表格前后空一行

**对齐方式**:

```markdown
| 左对齐 | 居中 | 右对齐 |
|:-------|:----:|-------:|
| 数据1  | 数据2 | 数据3  |
```

### 5. 代码格式

#### 5.1 行内代码

**格式**:

```markdown
使用`代码`表示行内代码。
```

**规则**:

- 使用反引号`` ` ``包裹代码
- 用于文件名、函数名、变量名等

#### 5.2 代码块

**格式**:

````markdown
```language
代码内容
```
````

**规则**:

- 使用三个反引号开始和结束
- 指定语言类型（如`python`, `powershell`, `markdown`）
- 代码块前后空一行

**示例**:

````markdown
```python
def hello():
    print("Hello, World!")
```
````

### 6. 数学公式格式

#### 6.1 行内公式

**格式**:

```markdown
这是一个行内公式：$E = mc^2$
```

**规则**:

- 使用单个`$`包裹公式
- 用于段落中的数学表达式

#### 6.2 块级公式

**格式**:

```markdown
$$
E = mc^2
$$
```

**规则**:

- 使用双`$$`包裹公式
- 用于独立的数学公式
- 公式前后空一行

### 7. 链接格式

#### 7.1 内联链接

**格式**:

```markdown
[链接文本](链接地址)
```

**示例**:

```markdown
[项目索引](docs/项目索引.md)
```

#### 7.2 引用链接

**格式**:

```markdown
[链接文本][引用标识]

[引用标识]: 链接地址 "可选标题"
```

**示例**:

```markdown
[项目索引][index]

[index]: docs/项目索引.md "项目索引文档"
```

**规则**:

- 优先使用相对路径
- 链接文本应清晰描述目标
- 引用链接放在文档末尾

### 8. 图片格式

**格式**:

```markdown
![图片描述](图片路径)
```

**规则**:

- 使用相对路径
- 提供有意义的图片描述
- 图片文件放在`images/`目录

### 9. 引用格式

**格式**:

```markdown
> 引用内容
>
> 可以包含多行
```

**规则**:

- 使用`>`作为引用标记
- 引用前后空一行
- 用于重要说明或引用

### 10. 强调格式

**格式**:

```markdown
**粗体文本**
*斜体文本*
***粗斜体文本***
~~删除线文本~~
```

**规则**:

- `**粗体**`用于强调
- `*斜体*`用于轻微强调
- `~~删除线~~`用于标记删除内容

## 🚫 格式禁止项

### 1. 禁止的格式

- ❌ 使用HTML标签（除非必要）
- ❌ 混合使用不同的列表标记（`-`, `*`, `+`）
- ❌ 使用Tab缩进（使用空格）
- ❌ 行尾空格
- ❌ 连续空行超过2行
- ❌ 标题后缺少空行

### 2. 禁止的内容

- ❌ 使用全角标点符号（使用半角）
- ❌ 中英文之间缺少空格
- ❌ 数字和单位之间缺少空格
- ❌ 不规范的日期格式
