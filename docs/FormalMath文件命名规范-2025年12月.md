# FormalMath项目文件命名规范

**制定日期**: 2025年12月29日
**版本**: v1.0
**适用范围**: 项目所有Markdown文件
**状态**: 正式执行

---

## 📋 规范概述

本文档定义FormalMath项目中所有文件的命名规范，确保项目文件命名的一致性、可读性和可维护性。

---

## 🎯 命名原则

1. **清晰性**: 文件名应清晰表达文件内容
2. **一致性**: 同类文件使用统一的命名模式
3. **可排序**: 使用数字前缀实现自然排序
4. **可读性**: 使用中文和英文混合，避免特殊字符
5. **简洁性**: 文件名不宜过长（建议不超过50个字符）

---

## 📝 命名规范

### 1. 管理文档（00-前缀）

**格式**: `00-[文档类型]-[日期].md`

**文档类型**:

- `项目状态` - 项目状态文档
- `文档索引` - 文档索引
- `README` - 说明文档
- `完成报告` - 完成报告
- `进度报告` - 进度报告
- `工作总结` - 工作总结
- `执行计划` - 执行计划
- `评价报告` - 评价报告

**日期格式**: `YYYY年MM月DD日` 或 `YYYY-MM-DD`

**示例**:

```
00-项目状态.md
00-文档索引.md
00-README.md
00-完成报告-2025年12月29日.md
00-进度报告-2025年12月29日.md
00-项目全面批判性评价与改进计划-2025年12月29日.md
```

**规则**:

- 管理文档统一使用`00-`前缀
- 日期可选，仅在需要区分版本时使用
- 最新版本不使用日期，历史版本归档时添加日期

### 2. 内容文档（数字前缀）

**格式**: `[序号]-[文档标题].md`

**序号规则**:

- `01-` 到 `99-`: 按内容顺序编号
- 使用两位数，不足补零
- 预留编号空间（如01-05, 10-15）

**文档标题**:

- 使用中文
- 简洁明了
- 避免特殊字符

**示例**:

```
01-核心理论.md
02-数学内容深度分析.md
03-教育与影响.md
04-历史与传记.md
05-现代应用与拓展.md
```

**规则**:

- 内容文档不使用`00-`前缀
- 序号按逻辑顺序排列
- 标题应准确反映内容

### 3. 子目录文档

**格式**: `[父目录序号]-[子目录序号]-[文档标题].md`

**示例**:

```
01-01-傅里叶级数的定义与性质.md
01-02-傅里叶变换理论.md
02-01-热传导方程的建立.md
02-02-分离变量法.md
```

**规则**:

- 继承父目录的序号
- 子目录使用独立的序号
- 保持层级关系清晰

### 4. 归档文档

**格式**: `[原文件名]-归档-YYYY年MM月DD日.md`

**示例**:

```
00-完成报告-2025年12月11日-归档-2025年12月29日.md
```

**规则**:

- 归档文档添加`-归档-`标记
- 添加归档日期
- 保留原文件名

### 5. 特殊文档

#### 5.1 索引文档

**格式**: `00-文档索引.md` 或 `00-[模块名]索引.md`

**示例**:

```
00-文档索引.md
00-术语词典总索引.md
00-傅里叶数学理念索引.md
```

#### 5.2 README文档

**格式**: `README.md` 或 `00-README.md`

**规则**:

- 根目录使用`README.md`
- 子目录可使用`README.md`或`00-README.md`

#### 5.3 脚本文件

**格式**: `[功能名].ps1` 或 `[功能名].sh`

**示例**:

```
归档脚本.ps1
检查命名.ps1
格式化文档.ps1
```

## 🚫 禁止使用的命名

### 1. 禁止使用的字符

- 特殊字符: `<>:"/\|?*`
- 空格（使用`-`或`_`代替）
- 连续的点号: `..`
- 控制字符

### 2. 禁止使用的模式

- ❌ `完成报告.md` （缺少前缀）
- ❌ `项目状态-2025-12-29.md` （日期格式不一致）
- ❌ `00 项目状态.md` （包含空格）
- ❌ `00-项目状态(最新).md` （包含括号）
- ❌ `00-项目状态_v2.md` （版本号不规范）

### 3. 推荐使用的模式

- ✅ `00-项目状态.md`
- ✅ `00-完成报告-2025年12月29日.md`
- ✅ `01-核心理论.md`
- ✅ `README.md`

## 📊 命名检查规则

### 自动检查规则

1. **前缀检查**: 管理文档必须以`00-`开头
2. **序号检查**: 内容文档必须使用数字前缀
3. **字符检查**: 不允许使用禁止字符
4. **长度检查**: 文件名不超过50个字符
5. **格式检查**: 日期格式必须一致

### 检查工具

使用PowerShell脚本检查命名规范：

```powershell
# 检查文件命名是否符合规范
# 输出不符合规范的文件列表
```
