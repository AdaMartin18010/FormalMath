# FormalMath 认知诊断系统使用指南

## 系统概述

FormalMath认知诊断系统基于**HCD（层次约束感知认知诊断）框架**，旨在：

- 准确评估学生在L0-L3四个知识层次的能力表现
- 识别学习优势与薄弱环节
- 生成个性化学习路径建议
- 为自适应学习系统提供数据支持

## 知识层次定义

### L0: 基础概念层

- **目标**: 建立对数学概念的直观理解和基本认知
- **能力要求**: 概念识别、直观理解、例子识别、基本操作
- **典型任务**: 识别群、举群的例子、基本集合运算

### L1: 定义理解层

- **目标**: 掌握严格的形式化定义和基本定理
- **能力要求**: 形式化定义理解、定理理解、基本证明、标准应用
- **典型任务**: 用ε-δ语言定义极限、证明群的基本性质

### L2: 定理应用层

- **目标**: 掌握深入定理，能够解决复杂问题
- **能力要求**: 深入定理掌握、复杂证明、综合应用、问题转化
- **典型任务**: Galois理论应用、Sylow定理证明

### L3: 综合证明层

- **目标**: 能够进行创新性研究和综合证明
- **能力要求**: 开放问题理解、研究探索、创新证明、综合构建
- **典型任务**: 概形理论理解、研究计划设计

## 使用流程

### 1. 开始诊断

1. 登录系统，进入仪表盘
2. 点击"开始诊断测试"
3. 选择测试参数：
   - 目标层次（可选）
   - 重点领域（可选）
   - 题目数量（默认30题）

### 2. 完成测试

1. 系统根据您的选择生成个性化题目
2. 逐题作答，支持：
   - 跳题（稍后返回）
   - 标记（不确定的题目）
   - 查看已用时间
3. 完成所有题目后提交

### 3. 查看报告

诊断报告包含：

- **综合评估**: 整体水平和置信度
- **L0-L3能力分析**: 各层次得分和等级
- **知识领域分布**: 各数学分支掌握情况
- **薄弱环节**: 需要重点提升的知识点
- **优势领域**: 掌握较好的知识点
- **个性化建议**: 学习路径和练习推荐

### 4. 跟随学习路径

系统基于诊断结果生成个性化学习路径：

- 推荐学习内容的顺序
- 每个节点的预计用时
- 先修知识要求
- 配套资源（视频、阅读、练习）

## 题型说明

### 单选题 (SC)

- 只有一个正确答案
- 自动评分
- 适合概念识别和基础计算

### 多选题 (MC)

- 可能有多个正确答案
- 自动评分（需全部选对）
- 适合综合判断

### 填空题 (FB)

- 填写答案
- 自动评分
- 适合公式和简单计算

### 简答题 (SA)

- 文字作答
- 需人工评分
- 适合定义理解和概念解释

### 证明题 (PR)

- 完整证明过程
- 需人工评分
- 适合高级层次诊断

## 学习建议解读

### 建议类型

| 类型 | 含义 | 优先级 |
|------|------|--------|
| foundation | 基础层次薄弱 | 高 |
| concept | 具体知识点强化 | 中 |
| level_up | 层次进阶建议 | 低 |
| method | 学习方法建议 | 参考 |

### 行动指南

1. **优先处理薄弱环节**: 按优先级顺序学习
2. **巩固优势领域**: 定期复习保持
3. **遵循学习路径**: 按推荐顺序学习
4. **定期复测**: 建议每2-4周进行一次诊断

## 常见问题

### Q: 测试需要多长时间？

A: 默认30题约需30-45分钟，具体时间因难度而异。

### Q: 诊断结果准确吗？

A: 系统基于HCD算法，诊断置信度通常在80%以上。建议定期复测以跟踪进展。

### Q: 可以只做特定层次的题目吗？

A: 可以。在开始诊断时选择目标层次即可。

### Q: 简答题和证明题如何评分？

A: 这两类题目需要教师人工评分，系统会提供参考答案和评分标准。

### Q: 如何查看历史诊断记录？

A: 在个人中心的"诊断历史"页面可以查看所有历史记录和能力变化趋势。

## 技术说明

### HCD算法特点

- **层次感知**: 显式建模L0-L3层次依赖关系
- **约束优化**: 使用层次约束指导知识状态估计
- **多维度输出**: 同时输出知识点掌握度和层次能力
- **可解释性**: 诊断结果有明确的层次结构解释

### 诊断准确度

- 知识状态估计准确度: >85%
- 能力水平评估准确度: >80%
- 诊断一致性: >85%
