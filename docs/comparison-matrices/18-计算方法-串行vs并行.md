## 核心概念对比

### 基本信息

| 维度 | 内容 |
|:---:|:---|
| **对比主题** | 计算执行的两种基本模式 |
| **核心问题** | 如何有效利用多核/多机资源 |
| **硬件背景** | 从单核到众核的演进（频率墙→并行墙） |
| **理论基础** | Flynn分类法、并行计算理论 |

### 核心特征矩阵

| 特征维度 | 串行计算 (Serial) | 并行计算 (Parallel) |
|:---:|:---:|:---:|
| **执行模型** | 单指令流单数据流 (SISD) | 多指令流多数据流 (MIMD) |
| **硬件假设** | 单一处理器核心 | 多处理器/多核/GPU/集群 |
| **编程模型** | 顺序执行、单一控制流 | 任务并行/数据并行/流水线 |
| **内存模型** | 统一地址空间 | 共享内存/分布式内存/混合 |
| **同步需求** | 无 | 需要同步原语（锁、屏障） |
| **调试难度** | ⭐⭐ 相对较低 | ⭐⭐⭐⭐⭐ 高（竞态、死锁） |
| **优化重点** | 算法效率、缓存利用 | 负载均衡、通信最小化 |

### Flynn分类法

```
指令流 \ 数据流 | 单数据流(S) | 多数据流(M)
----------------|-------------|------------
单指令流(S)     | SISD        | SIMD
                | (串行)      | (向量/GPU)
----------------|-------------|------------
多指令流(M)     | MISD        | MIMD
                | (罕见)      | (多核/集群)
```

|:---:|:---:|:---:|:---:|:---:|

### 代码示例对比

**串行矩阵乘法：**
```python
for i in range(n):
    for j in range(n):
        C[i,j] = sum(A[i,k] * B[k,j] for k in range(n))
```

**OpenMP并行（共享内存）：**
```cpp
#pragma omp parallel for collapse(2)
for (int i = 0; i < n; i++) {
    for (int j = 0; j < n; j++) {
        C[i][j] = 0;
        for (int k = 0; k < n; k++)
            C[i][j] += A[i][k] * B[k][j];
    }
}
```

**MPI并行（分布式内存）：**
```cpp
// 分发数据到各进程
MPI_Scatter(A, n*n/size, MPI_DOUBLE, local_A, ...);
MPI_Bcast(B, n*n, MPI_DOUBLE, 0, MPI_COMM_WORLD);

// 本地计算
for (int i = 0; i < local_n; i++)
    for (int j = 0; j < n; j++)
        for (int k = 0; k < n; k++)
            local_C[i][j] += local_A[i][k] * B[k][j];

// 收集结果
MPI_Gather(local_C, n*local_n, MPI_DOUBLE, C, ...);
```

**CUDA并行（GPU）：**
```cuda
__global__ void matmul(double* A, double* B, double* C, int n) {
    int i = blockIdx.y * blockDim.y + threadIdx.y;
    int j = blockIdx.x * blockDim.x + threadIdx.x;
    
    double sum = 0;
    for (int k = 0; k < n; k++)
        sum += A[i * n + k] * B[k * n + j];
    C[i * n + j] = sum;
}
```

## 并行性能理论

### Amdahl定律

**公式：**
$$S(N) = \frac{1}{(1 - P) + \frac{P}{N}}$$

其中：
- $S(N)$ = N个处理器的加速比
- $P$ = 可并行化部分的比例
- $N$ = 处理器数量

**关键洞察：**

| 可并行比例 P | 最大加速比 (N→∞) | 实用意义 |
|:---:|:---:|:---|
| 50% | 2× | 并行价值有限 |
| 90% | 10× | 需要关注串行部分 |
| 99% | 100× | 大规模并行可行 |
| 99.9% | 1000× | 接近理想扩展 |

```
加速比 vs 处理器数（不同P值）

加速比
  │
10├───────┐
  │       ╲ P=0.99
  │        ╲
  │         ╲
5 ├───┐     ╲
  │   ╲ P=0.9  ╲
  │    ╲        ╲
  │     ╲        ╲
2 ├┐     ╲        ╲ P=0.99
  │╲ P=0.5        ╲
  │ ╲              ╲
1 ├─┴───────────────► 处理器数
  1   10    100   1000
```

### Gustafson定律

**观点：** 问题规模随处理器增加而增长

$$S(N) = N - (N-1) \cdot (1 - P) = N \cdot P + (1 - P)$$

**适用场景：**
- 弱扩展（weak scaling）：固定每个处理器的负载
- 大数据处理
- 科学计算中的大模型

### 扩展性类型

| 类型 | 定义 | 目标 | 衡量指标 |
|:---:|:---:|:---:|:---:|
| **强扩展** | 固定问题规模，增加处理器 | 减少时间 | 加速比 |
| **弱扩展** | 固定每个处理器的负载 | 解决更大问题 | 效率保持 |

## 并行算法设计

### 设计模式

| 模式 | 描述 | 适用问题 | 示例 |
|:---:|:---:|:---:|:---|
| **分治并行** | 递归分解子问题 | 排序、FFT、矩阵乘法 | 并行归并排序 |
| **数据并行** | 同一操作应用于不同数据 | 向量运算、图像处理 | GPU矩阵运算 |
| **流水线** | 任务分阶段连续执行 | 信号处理、编译器 | 视频编码 |
| **主从模式** | 主进程分发任务 | 任务农场、搜索 | 蒙特卡洛模拟 |
| **工作窃取** | 空闲线程窃取任务 | 不规则任务 | Cilk任务并行 |
| **MapReduce** | 映射+归约 | 大数据处理 | Hadoop, Spark |

### 并行复杂度分析

| 复杂度类型 | 定义 | 关注点 |
|:---:|:---:|:---|
| **工作量 (Work)** | 总操作数 | 与最优串行算法比较 |
| **跨度 (Span)** | 关键路径长度 | 最大并行加速上限 |
| **并行度** | Work/Span | 所需理想处理器数 |

** Brent定理：**
$$T_P \leq T_1/P + T_\infty$$

其中：
- $T_1$ = 串行时间（Work）
- $T_P$ = P个处理器的时间
- $T_\infty$ = 关键路径（Span）

## 数学计算中的并行策略

### 线性代数运算

| 运算 | 并行策略 | 可扩展性 | 库支持 |
|:---:|:---:|:---:|:---:|
| **矩阵乘法** | 分块并行、SUMMA算法 | 高 | ScaLAPACK, cuBLAS |
| **LU分解** | 块算法、流水线 | 中 | ScaLAPACK |
| **QR分解** | Householder并行 | 中 | Elemental |
| **特征值** | 分而治之、MRRR | 中 | SLEPc, PRIMME |
| **稀疏求解** | 域分解、代数多重网格 | 中-高 | PETSc, Trilinos |
| **FFT** | Cooley-Tukey并行 | 高 | FFTW, cuFFT |

### 微分方程求解

| 方法 | 并行策略 | 粒度 |
|:---:|:---:|:---:|
| **显式ODE** | 时间并行（Parareal） | 粗粒度 |
| **隐式ODE** | 线性求解器并行 | 细粒度 |
| **有限差分** | 空间域分解 | 中粒度 |
| **有限元** | 网格划分、代数多重网格 | 多粒度 |
| **谱方法** | FFT并行 | 中粒度 |
| **蒙特卡洛** | 样本独立并行 |  embarrassingly并行 |

### 优化算法并行化

| 算法 | 并行策略 | 通信需求 |
|:---:|:---:|:---:|
| **梯度下降** | 数据并行（大批量） | 低 |
| **牛顿法** | 线性求解器并行 | 高 |
| **遗传算法** | 种群并行 | 低 |
| **模拟退火** | 多链并行 | 低 |
| **ADMM** | 变量分解 | 中 |

## 并行化挑战与解决方案

### 主要挑战

| 挑战 | 描述 | 影响 | 解决方案 |
|:---:|:---:|:---:|:---|
| **负载不均衡** | 任务分配不均，部分处理器空闲 | 效率下降 | 动态调度、工作窃取 |
| **数据竞争** | 并发访问共享数据，结果不确定 | 错误结果 | 锁、原子操作、无锁结构 |
| **假共享** | 不同核心修改同一缓存行的不同部分 | 性能下降 | 填充、数据对齐 |
| **通信开销** | 数据传输延迟和带宽限制 | 扩展性差 | 计算通信重叠、局部性优化 |
| **同步开销** | 等待其他线程到达同步点 | 效率下降 | 异步算法、减少同步 |
| **死锁** | 循环等待资源 | 程序挂起 | 锁顺序、超时、无锁设计 |
| **可扩展性瓶颈** | 串行部分或通信限制 | 加速比饱和 | 算法重新设计 |

## 总结与展望

### 核心对比总结

| 维度 | 串行计算 | 并行计算 |
|:---:|:---:|:---:|
| **核心优势** | 简单、可预测、易调试 | 性能扩展、解决大规模问题 |
| **主要挑战** | 性能天花板（频率墙） | 编程复杂度、可扩展性 |
| **现代趋势** | 异构编程、自动并行化 | 众核、加速器、云计算 |
| **不可替代** | 小规模问题、原型开发 | HPC、AI训练、大数据分析 |

### 发展趋势

| 趋势 | 描述 | 影响 |
|:---:|:---:|:---:|
| **异构计算** | CPU+GPU+FPGA协同 | 编程更复杂，性能更高 |
| **自动并行化** | 编译器自动识别并行 | 降低编程门槛 |
| **云原生HPC** | 云计算平台提供HPC | 普及高性能计算 |
| **量子计算** | 新型并行范式 | 根本性变革 |
| **神经形态计算** | 脑启发计算 | 低功耗AI |

### 未来展望

```
理想并行环境：
├── 自动并行化
│   └── 编译器自动识别和优化并行
├── 性能可移植
│   └── 同一代码在不同硬件高效运行
├── 容错能力
│   └── 大规模并行下的可靠性
├── 能效优化
│   └── 在功耗限制下最大化性能
└── 易用性
    └── 降低并行编程门槛
```
