|------|------|

## 核心特征矩阵

| 特征维度 | 串行计算 (Serial) | 并行计算 (Parallel) |
|----------|-------------------|---------------------|
| **执行模型** | 单指令流单数据流 (SISD) | 多指令流多数据流 (MIMD) |
| **硬件假设** | 单一处理器 | 多处理器/多核/加速器 |
| **编程模型** | 顺序执行 | 任务并行/数据并行 |
| **内存模型** | 统一地址空间 | 共享内存/分布式内存 |
| **同步需求** | 无 | 需要同步机制 |
| **调试难度** | 相对较低 | 较高（竞态条件）|

---

## 技术特征详细对比

| 技术指标 | 串行计算 | 并行计算 |
|----------|----------|----------|
| **峰值性能** | 单核频率受限 | 随核心数扩展 |
| **内存带宽** | 单一通道 | 聚合带宽 |
| **扩展性** | 有限（频率墙） | 理论上线性 |
| **能耗效率** | 随频率超线性增长 | 更优（低频率多核）|
| **编程复杂度** | ⭐⭐ 低 | ⭐⭐⭐⭐⭐ 高 |
| **可移植性** | ⭐⭐⭐⭐⭐ 高 | ⭐⭐⭐ 中等 |

## 并行算法设计原则

### Amdahl定律

```
加速比 = 1 / ((1 - P) + P/N)

其中：
- P = 可并行化部分比例
- N = 处理器数量
```

**关键洞察**：即使99%的代码可并行，最大加速比也只有100倍。

### Gustafson定律

```
加速比 = N - (N-1) * (1 - P)

观点：问题规模随处理器增加而增长
```

## 并行化挑战

| 挑战 | 描述 | 解决方案 |
|------|------|----------|
| **负载均衡** | 任务分配不均 | 动态调度、任务窃取 |
| **数据竞争** | 并发访问冲突 | 锁、原子操作、无锁结构 |
| **假共享** | 缓存行冲突 | 填充、数据对齐 |
| **通信开销** | 数据传输延迟 | 计算通信重叠、局部性优化 |
| **同步开销** | 等待其他线程 | 异步算法、减少同步点 |

## 性能对比案例

### 矩阵乘法（N×N）

| N | 串行时间 | 并行时间(8核) | 加速比 |
|---|----------|---------------|--------|
| 1000 | 1.2s | 0.18s | 6.7x |
| 5000 | 150s | 22s | 6.8x |
| 10000 | 1200s | 175s | 6.9x |

*注：接近线性加速因计算密集且易并行化*

### 快速排序

| 规模 | 串行 | 并行(8核) | 加速比 |
|------|------|-----------|--------|
| 10⁶ | 0.8s | 0.15s | 5.3x |
| 10⁸ | 120s | 22s | 5.5x |

*注：低于矩阵乘法因递归和划分开销*

## 选择决策矩阵

| 问题特征 | 推荐模式 | 理由 |
|----------|----------|------|
| 小规模数据 | 串行 | 并行开销不划算 |
| 数据依赖强 | 串行/流水线 | 难以并行化 |
| 大规模独立任务 | 并行 | 理想并行场景 |
| 实时性要求高 | 并行 | 降低延迟 |
| 内存受限 | 串行/分布式 | 避免内存瓶颈 |
