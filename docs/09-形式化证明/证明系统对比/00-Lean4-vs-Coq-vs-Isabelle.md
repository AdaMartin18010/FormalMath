# 证明助手系统对比

## 概述

本文档对比四个主流的定理证明系统：Lean4、Coq、Isabelle/HOL和Agda。

## 详细对比

| 特性 | Lean4 | Coq | Isabelle/HOL | Agda |
|------|-------|-----|--------------|------|
| **类型理论** | 依赖类型 | 归纳构造演算 | 简单类型+HOL | 依赖类型 |
| **逻辑基础** | 构造性+经典 | 构造性 | 经典 | 构造性 |
| **编程范式** | 函数式 | 函数式 | 函数式+命令式 | 纯函数式 |
| **自动化** | 中等 | 强 | 很强 | 弱 |
| **性能** | 优秀 | 良好 | 良好 | 良好 |
| **数学库规模** | 极大(Mathlib4) | 大 | 大(Archive) | 中等 |
| **学习曲线** | 中等 | 陡峭 | 平缓 | 中等 |
| **IDE支持** | VS Code | CoqIDE/VS Code | Isabelle/jEdit | Emacs/VS Code |
| **元编程** | 优秀 | 良好 | 中等 | 良好 |

## 各系统特点

### Lean4

- **优势**: 性能优秀，数学库庞大，社区活跃
- **劣势**: 相对较新，部分工具链不成熟
- **适用场景**: 大规模数学形式化，教学

### Coq

- **优势**: 历史悠久，自动化强，工具体系成熟
- **劣势**: 学习曲线陡峭，数学库分散
- **适用场景**: 软件验证，类型理论研究

### Isabelle/HOL

- **优势**: 自动化极强，文档丰富，LCF架构
- **劣势**: HOL逻辑限制，依赖类型支持弱
- **适用场景**: 软件验证，硬件验证

### Agda

- **优势**: 依赖类型纯粹，与Haskell生态结合
- **劣势**: 自动化弱，库较小
- **适用场景**: 类型理论研究，编程语言研究

## 互操作性

```
Lean4 ←→ Coq: 通过导出/导入语法
Lean4 ←→ Isabelle: 有限支持
Coq ←→ Isabelle: 通过Dedukti
Agda ←→ 其他: 通过编译到Haskell
```

## 选择建议

| 应用场景 | 推荐系统 |
|----------|----------|
| 纯数学形式化 | Lean4 |
| 软件验证 | Isabelle/HOL |
| 类型理论研究 | Coq/Agda |
| 教学入门 | Lean4/Isabelle |
| 工业应用 | Isabelle/HOL |
