# 各证明系统优势分析

## 概述

现代定理证明系统的发展为数学形式化、软件验证和逻辑研究提供了强大的工具。本文档从数学理论深度、工程实践广度与形式化案例三个层面，深入分析 Lean4、Coq、Isabelle/HOL 与 Agda 四大系统的核心优势，并辅以严格的定义、定理与证明思路。

## 1. Lean4

### 1.1 理论基础

**定义 1.1（依赖类型演算）**. Lean4 的核心逻辑为归纳构造演算（CIC），其项由以下语法生成：

$$t, u ::= x \mid \lambda x:A.\, t \mid t\,u \mid \Pi x:A.\, B \mid \mathsf{Type}_i \mid \mathsf{Sort}\, u \mid \mathsf{inductive}\, \{\, C_i \,\}$$

其中 $C_i$ 为归纳类型的构造子（Constructors）。Lean4 引入不可变宇宙（Non-cumulative Universes），要求若 $A : \mathsf{Type}_i$ 且 $B : A \to \mathsf{Type}_j$，则 $\Pi x:A.\, B(x) : \mathsf{Type}_{\max(i,j)}$。

**定理 1.1（Lean4 中 Church-Rosser 性质）**. 对任意良类型项 $t$，若 $t \twoheadrightarrow u$ 且 $t \twoheadrightarrow v$，则存在 $w$ 使得 $u \twoheadrightarrow w$ 且 $v \twoheadrightarrow w$。

*证明概要*：通过 Tait-Girard 的可约性候选（Reducibility Candidates）或正交闭包（Orthogonal Closure）证明。该性质保证了类型检查中 $\beta\eta$-等价的合流性，从而确保类型归属判断的确定性。∎

### 1.2 核心优势

1. **统一的设计语言**：编程与证明共享同一套依赖类型语法，使得算法实现与其正确性证明可以在同一模块中协同开发。

2. **高性能运行时**：Lean4 编译器将源码翻译为 C 代码，再经 GCC/Clang 编译为机器码。其基准测试显示，纯数值计算性能接近手写 C 代码。

3. **Mathlib4**：截至 2025 年，Mathlib4 已涵盖超过 100 万行形式化代码，覆盖：
   - **代数**：群、环、域、模、Galois 理论
   - **分析**：测度论、积分论、复分析、泛函分析
   - **拓扑**：点集拓扑、代数拓扑基础（同伦群计算）
   - **数论**：素数分布、椭圆曲线、类域论基础

4. **元编程能力**：Lean4 的宏系统（`macro`）与扩展器（`elab`）允许用户在语言层面定义新的证明策略与 DSL。

5. **现代化类型理论**：支持 Quotient Types、结构继承（Structure Inheritance）、类型类（Type Classes）与自动隐式参数推断。

### 1.3 典型形式化案例

**定理 1.2（球体同伦群）**. $\pi_4(\mathbb{S}^3) \cong \mathbb{Z}/2\mathbb{Z}$。

*形式化背景*：该结果由 Brunerie（2016）在 Homotopy Type Theory 中首次构造性证明，后由 Mathlib 社区在 Lean 中形式化。证明核心依赖于 James 构造（James Construction）与 Hopf 不变量（Hopf Invariant）的代数拓扑工具。

**定理 1.3（正则素数的 Fermat 大定理）**. 若素数 $p$ 为正则素数（Regular Prime），则方程 $x^p + y^p = z^p$ 无非零整数解。

*形式化背景*：该定理的证明由 Kummer 在 19 世纪给出，涉及分圆域 $\mathbb{Q}(\zeta_p)$ 的类数理论。Lean4 的 Mathlib 已完整形式化分圆域的基本理论，为最终证明奠定了代数基础。

**定理 1.4（Liquid Tensor Experiment）**. 令 $S$ 为 profinite 集合，$V$ 为 p-完备局部凸向量空间，则特定 Ext 群满足凝聚性条件（Liquid Condition）。

*形式化背景*：此结果由 Clausen 与 Scholze 提出，Dahmen 等人在 Lean 中完成形式化，是当代代数几何与泛函分析交叉领域的里程碑式工作。

## 2. Coq

### 2.1 理论基础

**定义 2.1（构造性逻辑）**. 在构造性命题逻辑中，证明 $\exists x:A,\; P(x)$ 必须提供一个具体的 witness $a:A$ 及 $P(a)$ 的证明；证明 $P \lor Q$ 必须指明哪一支成立并提供相应证明。排中律 $P \lor \neg P$ 不成立。

**定理 2.1（Brouwer-Heyting-Kolmogorov 解释）**. 命题 $P$ 的构造性证明对应一个程序（算法），该程序满足 $P$ 的计算规范：

- $P \land Q$ 的证明为对 $(p, q)$，其中 $p$ 证 $P$，$q$ 证 $Q$；
- $P \Rightarrow Q$ 的证明为函数 $f$，将 $P$ 的证明映射为 $Q$ 的证明；
- $\forall x:A,\; P(x)$ 的证明为函数 $f$，将 $a:A$ 映射为 $P(a)$ 的证明。

### 2.2 核心优势

1. **构造性证明即程序**：Coq 的 Curry-Howard 同构实现最为纯粹，任何存在性证明均可提取为可执行程序。

2. **程序提取（Extraction）**：Coq 的提取机制通过移除证明内容（Proof Irrelevance）保留计算内容，生成 OCaml、Haskell 或 Scheme 代码。例如，排序算法的正确性证明可直接提取为经过验证的排序函数。

3. **SSReflect 方法论**：由 Gonthier 发展的 SSReflect（Small-Scale Reflection）通过将计算反射（Computational Reflection）引入证明过程，大幅压缩了证明长度。其核心思想是将命题 $P$ 翻译为布尔值 $b_P$，通过计算 $b_P = \mathsf{true}$ 替代逻辑推理。

**定义 2.2（计算反射）**. 设 $D$ 为决策过程（Decision Procedure），$D : \mathsf{Formula} \to \mathsf{Bool}$。若对任意公式 $\phi$ 有

$$D(\phi) = \mathsf{true} \;\Longleftrightarrow\; \vdash \phi,$$

则证明 $\vdash \phi$ 可转化为计算 $D(\phi)$ 并验证其等于 $\mathsf{true}$。

4. **成熟的数学成分类**：Coq 的 Mathematical Components（MathComp）库采用严格的代数层次结构，已形式化四色定理、奇阶定理等重量级结果。

### 2.3 典型形式化案例

**定理 2.2（四色定理）**. 任何平面图至多需要四种颜色即可正常着色。

*形式化详情*：Gonthier（2008）在 Coq 中的证明约 6 万行代码。核心步骤包括：
1. 将问题归约为平面图的可着色性；
2. 证明任何可避构型集（Unavoidable Set）必然包含可约构型；
3. 通过计算验证 633 个构型的可约性（使用计算反射加速）。

**定理 2.3（CompCert 编译器正确性）**. CompCert 是一个经过形式化验证的 C 编译器，其编译过程保持程序语义不变。

*形式化详情*：Leroy 等人在 Coq 中证明了从 CompCert C 到汇编代码的每一编译阶段的语义保持性（Simulation）。该编译器已在航空电子等高可靠性领域投入商业应用。

**定理 2.4（Feit-Thompson 奇阶定理）**. 任何奇数阶有限群都是可解群。

*形式化详情*：MathComp 团队完成了约 15 万行的形式化证明，涵盖有限群表示论、特征标理论与局部群论的完整形式化。这是迄今最长的形式化证明之一。

## 3. Isabelle/HOL

### 3.1 理论基础

**定义 3.1（经典高阶逻辑）**. Isabelle/HOL 的逻辑基础为 Church 的简单类型 $\lambda$-演算扩展：

$$\sigma, \tau ::= \iota \mid o \mid \sigma \to \tau$$

其中 $\iota$ 为个体类型，$o$ 为布尔类型。项的构造遵循类型规则：

$$\frac{\Gamma \vdash t : \sigma \to \tau \quad \Gamma \vdash u : \sigma}{\Gamma \vdash t\,u : \tau}$$

**定理 3.1（Isabelle 的可靠性）**. 若 Isabelle/HOL 导出 $\vdash \phi$，则 $\phi$ 在所有标准模型中为真。

*证明思路*：Isabelle 的核心推理引擎仅包含 Hilbert 风格的推理规则（假言推理、全称概括、等式替换等）。所有高阶工具（Sledgehammer、Nitpick、Auto）最终产生的证明必须经核心引擎验证。因此只要核心引擎实现正确，整个系统即是可靠的。∎

### 3.2 核心优势

1. **经典逻辑的直接支持**：Isabelle/HOL 默认支持排中律、选择公理与无穷公理，与数学家日常使用的逻辑完全一致，无需额外调整。

2. **强大的自动化**：Sledgehammer 工具可自动调用外部 ATP（E、Vampire、Z3 等）与 SMT 求解器。其工作流程为：
   - 将 Isabelle 目标翻译为 TPTP/FOF 或 SMT-LIB 格式；
   - 调用外部求解器搜索证明；
   - 使用 MaSh（Machine Learning for Sledgehammer）或 Metis 将外部证明重构为 Isabelle 的可靠证明。

3. **Isar 证明语言**：Isar（Intelligible Semi-Automated Reasoning）允许编写结构化的、可读的证明文本，接近传统数学写作风格。

4. **Archive of Formal Proofs（AFP）**：AFP 是 Isabelle 的形式化数学库，采用同行评审机制，已收录超过 800 篇文章，涵盖数论、图论、概率论与密码学。

### 3.3 典型形式化案例

**定理 3.2（seL4 操作系统内核正确性）**. seL4 微内核的 C 实现满足功能正确性、信息安全性与信息完整性。

*形式化详情*：Klein 等人在 Isabelle/HOL 中证明了 seL4 的 C 代码与抽象规范之间的精化关系（Refinement）。这是首个经过完整形式化验证的操作系统内核，已在无人机、自动驾驶与高安全通信系统中部署。

**定理 3.3（素数定理）**. $\pi(x) \sim \dfrac{x}{\ln x}$。

*形式化详情*：Harrison（1996）在 HOL Light 中完成了基于复分析的证明，涉及 Riemann $\zeta$ 函数的解析延拓、非平凡零点的分布与围道积分。

**定理 3.4（Kepler 猜想 / Flyspeck 项目）**. 在三维欧几里得空间中，相同球体的最密堆积密度为 $\dfrac{\pi}{\sqrt{18}} \approx 0.74048$。

*形式化详情*：Hales 等人发起的 Flyspeck 项目在 Isabelle/HOL 与 HOL Light 中完成了约 20 万行的形式化验证，将原证明中的大量 case analysis 转化为可靠的计算验证。

## 4. Agda

### 4.1 理论基础

**定义 4.1（Martin-Löf 类型理论）**. Agda 基于直觉主义类型理论（Intuitionistic Type Theory, ITT），其核心判断为：
- $A \;\mathsf{type}$：$A$ 是类型
- $a : A$：$a$ 属于类型 $A$
- $a \equiv b : A$：$a$ 与 $b$ 在类型 $A$ 中定义相等

**定理 4.1（Agda 中函数外延性）**. 在 Agda 中，函数外延性（Function Extensionality）可作为 postulate 引入：

$$\mathsf{funext} : \prod_{A:\mathcal{U}} \prod_{B:A\to\mathcal{U}} \prod_{f,g:\prod_{x:A} B(x)} \Big( \prod_{x:A} f(x) = g(x) \Big) \to f = g.$$

禁用公理 K 时，该 postulate 与 Univalence Axiom 相容，为 HoTT 研究提供实验平台。

### 4.2 核心优势

1. **纯粹的依赖类型**：Agda 的类型系统直接对应 Martin-Löf 的原始设计，无额外逻辑层，使得类型与命题的对应关系最为清晰。

2. **归纳族（Inductive Families）**：Agda 支持以任意索引族定义的归纳类型，例如长度索引向量（Length-Indexed Vectors）：

$$\mathsf{Vec}\,A\,0 = \mathsf{nil}, \qquad \mathsf{Vec}\,A\,(n+1) = A \times \mathsf{Vec}\,A\,n.$$

3. **与 Haskell 生态的融合**：Agda 编译器可生成 Haskell 代码，利用 Haskell 庞大的库生态系统进行 I/O 与系统交互。

4. **同伦类型论的实验平台**：通过 `--without-K` 选项，Agda 成为研究高阶归纳类型（HITs）、Univalence 与立方类型论（Cubical Type Theory）的重要工具。

### 4.3 典型形式化案例

**定理 4.2（长度索引向量的安全头操作）**. 定义 $\mathsf{head} : \prod_{A:\mathcal{U}} \prod_{n:\mathbb{N}} \mathsf{Vec}\,A\,(n+1) \to A$，则 $\mathsf{head}$ 在类型层面排除了空向量的访问，无需运行时检查。

*证明思路*：由于 $\mathsf{Vec}\,A\,(n+1)$ 的构造子仅包含 $\mathsf{cons}$，模式匹配直接获得头部元素，不存在 $\mathsf{nil}$ 分支。∎

## 5. 数学理论对比总结

| 特性 | Lean4 | Coq | Isabelle/HOL | Agda |
|------|-------|-----|--------------|------|
| **逻辑范式** | 构造性/经典可选 | 构造性 | 经典 | 构造性 |
| **Curry-Howard** | 完全支持 | 完全支持 | 部分支持（通过提取） | 完全支持 |
| **高阶归纳类型** | 部分支持（实验性） | 部分支持（HoTT 库） | 不支持 | 支持（Cubical Agda） |
| **证明可读性** | 中等 | 中等（SSReflect 压缩） | 高（Isar） | 高 |
| **形式化数学规模** | 极大 | 大 | 大 | 中等 |
