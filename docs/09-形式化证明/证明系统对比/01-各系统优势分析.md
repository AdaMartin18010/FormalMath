# 各证明系统优势分析

## 1. Lean4

### 1.1 核心优势

1. **统一的设计**: 编程和证明使用同一语言（依赖类型）
2. **高性能**: 编译到 C，执行速度快
3. **Mathlib4**: 世界上最大的数学形式化库，涵盖代数、分析、拓扑、数论等
4. **元编程**: 强大的宏系统和策略编写能力
5. **现代化**: 最新的类型理论研究成果

### 1.2 典型案例

- 球体同伦群计算
- 正则素数的 Fermat 大定理
- Liquid Tensor Experiment（凝聚态数学）

## 2. Coq

### 2.1 核心优势

1. **构造性**: 每个证明对应可执行程序
2. **程序提取**: 从证明自动提取 OCaml/Haskell/Scheme 代码
3. **SSReflect**: 强大的小规模反射证明方法
4. **数学成分类**: 已形式化四色定理、奇序定理等

### 2.2 典型案例

- 四色定理（Gonthier, 2008）
- CompCert（验证的 C 编译器）
- Feit-Thompson 奇阶定理

## 3. Isabelle/HOL

### 3.1 核心优势

1. **经典逻辑**: 直接支持排中律、选择公理
2. **自动化**: Sledgehammer 工具可自动调用外部 ATP
3. **可读性**: Isar 证明语言接近数学文本
4. ** Archive of Formal Proofs**: 庞大的形式化数学库

### 3.2 典型案例

- seL4（验证的操作系统内核）
- Prime Number Theorem
- Kepler 猜想（Hales, Flyspeck 项目）

## 4. Agda

### 4.1 核心优势

1. **依赖类型**: 与 Lean/Coq 类似的强大表达力
2. **证明即程序**: 强调构造性证明
3. **归纳族**: 丰富的归纳定义能力
4. **无公理 K**: 支持同伦类型论探索
