# Lean4证明策略的数学原理与系统指南

## 1. 引言

Lean4的证明策略（tactics）是交互式定理证明的核心界面。策略本质上是元程序，它们操作证明状态（proof state）——即一组待证目标（goals）及其假设上下文（local context）——通过逐步应用推理规则来构造底层的类型论证明项。理解策略背后的数学原理，不仅能提高形式化证明的效率，更能深化对证明论与类型论之间对应关系的认识。

本文从自然演绎（natural deduction）的 sequent calculus 视角出发，系统分类Lean4的证明策略，阐明每条策略对应的逻辑规则，并通过实例展示策略的组合与自动化技术。

## 2. 严格数学定义

### 2.1 证明状态的形式描述

**定义 2.1（目标 / Goal）**
**目标**是一个 sequent：
$$\Gamma \vdash \Delta$$
其中 $\Gamma = h_1: A_1, \ldots, h_n: A_n$ 为假设上下文，$\Delta$ 为待证命题（Lean中通常为单结论 $B$）。

**定义 2.2（策略的语义）**
**策略** $T$ 是从证明状态到证明状态的偏函数。若当前状态为 $(\Gamma \vdash B)$，应用 $T$ 后得到子目标集 $\{ (\Gamma_i \vdash B_i) \}_{i=1}^k$ 及证明项构造函数 $f: \prod_{i=1}^k \text{Pf}(\Gamma_i \vdash B_i) \to \text{Pf}(\Gamma \vdash B)$。

### 2.2 自然演绎规则

**定义 2.3（蕴含引入 $\\supset I$）**
$$\frac{\Gamma, A \vdash B}{\Gamma \vdash A \to B}(\\supset I)$$

**定义 2.4（蕴含消去 $\\supset E$ / Modus Ponens）**
$$\frac{\Gamma \vdash A \to B \quad \Gamma \vdash A}{\Gamma \vdash B}(\\supset E)$$

**定义 2.5（合取引入 $\wedge I$）**
$$\frac{\Gamma \vdash A \quad \Gamma \vdash B}{\Gamma \vdash A \wedge B}(\wedge I)$$

**定义 2.6（存在引入 $\exists I$）**
$$\frac{\Gamma \vdash A[t/x]}{\Gamma \vdash \exists x.\, A}(\exists I)$$

## 3. 核心定理与证明

### 定理 3.1（策略的完备性）

Lean4的核心策略集合 $\{intro, apply, exact, cases, use, rw, simp, linarith\}$ 对一阶逻辑（无归纳类型）是**语义完备的**：即对任意有效的一阶 sequent $\Gamma \vdash B$，存在策略序列将其归约为空目标集。

**证明概要**：通过将目标转化为一阶逻辑的标准证明（如归结法或表列法），再将每一步翻译为Lean策略。对命题逻辑，$tauto$ 策略基于DPLL算法，是完备的。对纯等式逻辑，$rw$ 结合对称性与传递性完备。对线性算术，$linarith$ 基于Fourier-Motzkin消元，对实数/整数线性约束完备。$\square$

### 定理 3.2（Curry-Howard同构的策略对应）

每条自然演绎规则对应一个Lean策略及一个 $\lambda$-项构造：

| 自然演绎 | Lean策略 | 项构造 |
|----------|----------|--------|
| $\\supset I$ | `intro h` | $\lambda h: A.\, M$ |
| $\\supset E$ | `apply h` | $h\,M$ |
| $\wedge I$ | `constructor` | $\langle M, N \rangle$ |
| $\wedge E_1$ | `cases h with h1 h2` | $\pi_1(M)$ |
| $\exists I$ | `use t` | $\langle t, M \rangle$ |
| $=_R$ | `rw [h]` | 替换 |

**证明**：通过对策略的元定义归纳。Lean内核最终检查生成的项是否良类型，即证明策略的正确性归结为类型检查。$\square$

## 4. 策略详解与例子

### 4.1 基础策略

| 策略 | 逻辑规则 | 用法示例 |
|------|----------|----------|
| `exact h` | 直接应用 | `exact h1` 当目标恰好为假设 `h1` 的类型 |
| `apply h` | $\\supset E$ | `apply h` 当 `h : A → B` 且目标为 `B` |
| `intro h` | $\\supset I$ | `intro h` 当目标为 `A → B`，引入假设 `h : A` |
| `have h : A := ...` | 引理 | 建立中间结论，分割证明 |

### 4.2 等式推理

| 策略 | 功能 | 示例 |
|------|------|------|
| `rfl` | 自反性 | 证明 `a = a` |
| `rw [h]` | 重写 | 用等式 `h : a = b` 替换目标中的 `a` |
| `simp` | 简化 | 使用 simp-lemmas 重写目标 |
| `ring` | 环代数 | 自动证明多项式等式 |
| `field_simp` | 域代数 | 化简含分式的表达式 |

### 4.3 逻辑策略

| 策略 | 功能 | 示例 |
|------|------|------|
| `constructor` | 分解合取目标 `A ∧ B` | 生成两个子目标 `A` 和 `B` |
| `cases h with h1 h2` | 分解假设 `h : A ∨ B` 或 `h : ∃x, P(x)` | 分情形讨论 |
| `left` / `right` | 选择析取分支 | 证明 `A ∨ B` 时选左/右 |
| `by_contra h` | 反证法 | 假设否定，推出矛盾 |

### 4.4 量词策略

| 策略 | 功能 | 示例 |
|------|------|------|
| `use t` | 提供存在量词证据 | 对目标 `∃x, P(x)` 提供 `t` |
| `obtain ⟨x, hx⟩ := h` | 消去存在量词 | 从 `h : ∃x, P(x)` 提取见证 |
| `specialize h a` | 特化全称量词 | 从 `h : ∀x, P(x)` 得 `P(a)` |

### 4.5 高级自动化

| 策略 | 算法 | 适用范围 |
|------|------|----------|
| `linarith` | Fourier-Motzkin消元 | 线性等式/不等式 |
| `nlinarith` | 非线性扩展 | 多项式不等式 |
| `tauto` | DPLL / 真值表 | 命题逻辑 |
| `finish` | 强力自动搜索 | 一阶逻辑片段 |

## 5. 详细例子

### 例 5.1：策略组合证明传递性

```lean
example (P Q R : Prop) (h1 : P → Q) (h2 : Q → R) (hp : P) : R := by
  have hq : Q := h1 hp
  exact h2 hq
```

**证明项**：$h2\,(h1\,hp) : R$

**对应自然演绎**：
$$\frac{h1: P \to Q \quad hp: P}{Q}(\\supset E) \quad h2: Q \to R}{R}(\\supset E)$$

### 例 5.2：归纳证明的完整策略序列

```lean
theorem add_comm (n m : Nat) : n + m = m + n := by
  induction n with
  | zero =>
    rw [Nat.zero_add, Nat.add_zero]
  | succ n ih =>
    rw [Nat.succ_add, Nat.add_succ, ih]
```

**数学解释**：对 $n$ 的结构归纳。
- **基例**（$n = 0$）：$0 + m = m = m + 0$（由 $Nat.zero_add$ 与 $Nat.add_zero$）
- **归纳步骤**：假设 $n + m = m + n$，证 $(n+1) + m = m + (n+1)$
  - $(n+1) + m = (n + m) + 1$（由 $Nat.succ_add$）
  - $m + (n+1) = (m + n) + 1$（由 $Nat.add_succ$）
  - 由归纳假设 $n + m = m + n$，两边相等

## 6. 策略组合与元编程

### 6.1 复合策略

```lean
-- 顺序组合 <;> ：对所有生成的子目标应用后续策略
example (A B C : Prop) (h : A ∧ B ∧ C) : C := by
  rcases h with ⟨ha, hb, hc⟩
  exact hc

-- 尝试组合 try { tac } ：失败不影响后续
example (n : Nat) : n ≤ n + 1 := by
  simp
  try { linarith }
  apply Nat.le_succ
```

### 6.2 自定义策略宏

```lean
macro "my_simp" : tactic => `(tactic|
  simp only [Nat.add_assoc, Nat.mul_assoc]
  try { ring }
  try { linarith }
)
```
