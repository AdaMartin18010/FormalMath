# Lean4证明策略的数学原理与系统指南

## 1. 证明状态的形式描述

**定义 1.1（目标 / Goal）**. Lean4 中的证明状态由一组待证目标组成，每个目标形如：

$$\Gamma \vdash A$$

其中 $\Gamma$ 为局部上下文（假设集合），$A$ 为目标类型（命题）。

## 2. 核心策略

### 2.1 引入与消去

| 策略 | 逻辑规则 | 数学意义 |
|------|---------|---------|
| `intro` | $\rightarrow$-引入 | 假设前提 |
| `apply` | $\rightarrow$-消去 | 使用蕴含 |
| `exact` | 公理 | 直接给出证明项 |
| `have` | 切割 | 引入中间结论 |

### 2.2 归纳与递归

**定理 2.1（归纳原理）**. 对归纳类型 $T$，为证明 $\forall x: T, P(x)$，只需证明对每个构造子，$P$ 对构造的输出成立（假设对输入成立）。

**策略 `induction`**. 自动应用归纳原理，生成对每个构造子的子目标。

### 2.3 等式推理

| 策略 | 功能 |
|------|------|
| `rw` | 重写（使用等式替换） |
| `simp` | 简化（使用引理库自动重写） |
| `ring` | 环等式自动证明 |
| `linarith` | 线性不等式自动证明 |

## 3. 自动化策略

### 3.1 `simp` 引擎

**定义 3.1（简化集）**. `simp` 使用标记为 `@[simp]` 的引理作为重写规则，通过 Knuth-Bendix 风格的 Completion 算法（近似）将表达式化为范式。

### 3.2 `aesop` 与 `tauto`

- `tauto`: 命题逻辑自动判定器
- `aesop`: 可扩展的自动证明搜索框架

## 4. 例子

### 4.1 例子：自然数加法交换律

```lean
theorem add_comm : ∀ n m : Nat, n + m = m + n
| n, 0 => by simp [Nat.add_zero]
| n, m+1 => by simp [Nat.add_succ, add_comm n m]
```

数学证明：对 $m$ 归纳。基例 $m=0$：$n+0 = n = 0+n$。归纳步：$n+(m+1) = (n+m)+1 = (m+n)+1 = (m+1)+n$。

### 4.2 例子：经典分析定理

```lean
theorem tendsto_add {f g : ℕ → ℝ} {a b : ℝ}
  (hf : Tendsto f a) (hg : Tendsto g b) :
  Tendsto (λ n => f n + g n) (a + b)
```

使用 `intro ε hε` 引入，利用三角不等式 $|f(n)+g(n)-(a+b)| \leq |f(n)-a| + |g(n)-b|$。
