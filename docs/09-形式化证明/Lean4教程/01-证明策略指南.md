# Lean4 证明策略的数学原理与系统指南

## 概述

Lean4 的证明策略（Tactics）是连接数学直觉与形式化验证的桥梁。本文档从 sequent 演算、类型理论与自动推理三个维度，系统阐述 Lean4 核心策略的数学原理，并辅以定义、定理、详细证明与应用实例。

## 1. 证明状态的形式描述

**定义 1.1（目标 / Goal）**. Lean4 中的证明状态由一组待证目标组成，每个目标形如 sequent：

$$\Gamma \vdash A$$

其中 $\Gamma = \{H_1 : P_1,\; H_2 : P_2,\; \dots,\; H_n : P_n\}$ 为局部上下文（假设集合），$A$ 为目标类型（命题）。策略的作用是将此目标转换为若干子目标，直至所有子目标被消解。

**定义 1.2（策略的可靠性）**. 策略 $T$ 是可靠的（Sound），当且仅当对任意目标 $\Gamma \vdash A$，若应用 $T$ 后产生子目标 $\Gamma_1 \vdash A_1,\; \dots,\; \Gamma_k \vdash A_k$，则

$$\bigwedge_{i=1}^k (\Gamma_i \vdash A_i) \;\Longrightarrow\; (\Gamma \vdash A).$$

即子目标全部可证蕴含原目标可证。

**定理 1.1（Lean4 核心策略的可靠性）**. Lean4 核心库（Core / Std / Mathlib）中提供的所有策略均满足定义 1.2 的可靠性条件。

*证明思路*：Lean4 的核心策略最终归结为对项（Proof Terms）的构造。每个策略对应一个高阶函数，该函数在元理论层面构造出一个 $\lambda$-项 $t$ 使得 $\Gamma \vdash t : A$。由于 Lean4 内核的类型检查器验证了该 $\lambda$-项的良类型性，策略的可靠性由类型系统的可靠性保证。∎

## 2. 核心策略的逻辑对应

### 2.1 引入与消去规则

下表列出核心策略与直觉主义自然推演（Natural Deduction）规则的对应关系：

| 策略 | 逻辑规则 | 数学意义 | 形式描述 |
|------|---------|---------|---------|
| `intro` | $\rightarrow$-引入 | 假设前提 | 从 $\Gamma \vdash P \Rightarrow Q$ 到 $\Gamma,\; h:P \vdash Q$ |
| `apply` | $\rightarrow$-消去 | 使用蕴含 | 从 $\Gamma \vdash Q$ 与 $f : P \Rightarrow Q \in \Gamma$ 到 $\Gamma \vdash P$ |
| `exact` | 公理 / 假设 | 直接给出证明项 | 从 $\Gamma \vdash A$ 与 $t : A \in \Gamma$ 到消解目标 |
| `have` | 切割（Cut） | 引入中间结论 | 从 $\Gamma \vdash B$ 插入子目标 $\Gamma \vdash A$ 与 $\Gamma,\; h:A \vdash B$ |

**定理 2.1（切割消除）**. 在直觉主义命题逻辑中，任何使用切割规则的证明均可转化为无切割证明。

*证明概要*：通过对证明结构的双重归纳（对切割公式复杂度与证明高度同时归纳），将切割逐步推向公理层，最终消除。Gentzen 的 Hauptsatz 建立了该结果。在 Lean4 中，`have` 策略虽然引入切割，但生成的证明项最终通过类型检查，其语义等价于无切割的正规形式（Normal Form）。∎

### 2.2 合取、析取与量词策略

**定义 2.1（合取与析取的类型解释）**. 在 Lean4 中：
- 合取 $P \land Q$ 对应积类型 $P \times Q$，其元素为对 $(p, q)$；
- 析取 $P \lor Q$ 对应和类型 $P + Q$，其元素为 $\mathsf{inl}\,p$ 或 $\mathsf{inr}\,q$；
- 存在量词 $\exists x:A,\; P(x)$ 对应依赖和类型 $\Sigma_{x:A} P(x)$。

| 策略 | 规则 | 形式描述 |
|------|------|---------|
| `constructor` | $\land$-引入 | 将 $\Gamma \vdash P \land Q$ 分解为 $\Gamma \vdash P$ 与 $\Gamma \vdash Q$ |
| `cases` | $\lor$-消去 / 归纳消去 | 对 $h : P \lor Q$ 分情况讨论 |
| `left` / `right` | $\lor$-引入 | 选择证明左支或右支 |
| `exists` | $\exists$-引入 | 提供 witness $a:A$ 并证明 $P(a)$ |
| `rcases` | 模式匹配消去 | 对复杂假设进行结构化解构 |

### 2.3 归纳与递归

**定理 2.2（归纳原理）**. 对归纳类型 $T$，令 $C_1, \dots, C_m$ 为其构造子。为证明 $\forall x:T,\; P(x)$，只需证明对每个构造子 $C_i$（设其递归参数类型为 $T$），在归纳假设 $P(\text{参数})$ 成立的前提下，证明 $P(C_i(\dots))$ 成立。

*形式表述*：设 $T$ 的 eliminator 为

$$\mathsf{rec}_T : \prod_{P:T\to\mathcal{U}} \Big( \prod_{i} \mathsf{Motive}_i(P) \Big) \to \prod_{x:T} P(x),$$

其中 $\mathsf{Motive}_i(P)$ 为构造子 $C_i$ 对应的归纳步。

**策略 `induction`**：自动应用 $\mathsf{rec}_T$，生成对每个构造子的子目标。用户只需填写归纳基例与归纳步。

**策略 `rw`**：重写（Rewrite）基于 Leibniz 等式原理：若 $a = b$，则对任意谓词 $P$，$P(a) \Leftrightarrow P(b)$。在类型论中，这对应于类型替换（Transport）：

$$\mathsf{transport}^P : \prod_{a,b:A} (a =_A b) \to P(a) \to P(b).$$

### 2.4 等式推理的代数化

| 策略 | 数学原理 | 适用域 |
|------|---------|--------|
| `rw` | Leibniz 替换原理 | 任意等式 $a = b$ |
| `simp` | Knuth-Bendix 完成（近似） | 标记为 `@[simp]` 的引理集合 |
| `ring` | 交换环的 Gröbner 基判定 | 交换环中的多项式等式 |
| `linarith` | Fourier-Motzkin 消元 | 线性不等式组 |
| `nlinarith` | 非线性扩展（平方和 / 单调性） | 多项式不等式 |
| `field_simp` | 域的分式化简 | 域上的有理表达式 |

**定义 2.2（简化集 / Simp-set）**. `simp` 引擎维护一个重写规则集合 $\mathcal{R} = \{ l_i \Rightarrow r_i \}$，其中每个规则由标记 `@[simp]` 的引理提供。`simp` 通过迭代应用这些规则，将表达式化为范式（Normal Form）。

**定理 2.3（`ring` 策略的完备性）**. 对交换环 $R$ 中的任意多项式等式 $p(x_1,\dots,x_n) = q(x_1,\dots,x_n)$，若该等式在所有交换环中成立，则 `ring` 策略可在有限步内证明之。

*证明思路*：`ring` 将两侧多项式展开为标准形式（所有项按字典序排列，系数合并），然后比较标准形式是否相同。该过程本质上是多项式环 $\mathbb{Z}[x_1,\dots,x_n]$ 中 word problem 的判定算法，由交换环的等式理论完备性保证。∎

## 3. 高级策略与自动化

### 3.1 等式链（Calc）

**定义 3.1（传递链）**. 等式链 `calc` 构建如下推理结构：

$$a_1 \;\op_1\; a_2 \;\op_2\; a_3 \;\op_3\; \dots \;\op_{n-1}\; a_n$$

其中每个 $\op_i \in \{=, \leq, <, \subseteq, \dots\}$ 为传递关系。Lean4 要求对每个步骤 $a_i \;\op_i\; a_{i+1}$ 提供显式证明。

**例子 3.1（三角不等式的等式链）**.

$$|x + y| \;\leq\; |x| + |y| \;=\; |y| + |x| \;\leq\; |y| + |z| + |x - z|$$

每一步分别调用三角不等式、加法交换律与三角不等式的变形。

### 3.2 可扩展自动证明框架

**定义 3.2（Aesop 搜索树）**. `aesop` 策略维护一棵搜索树，每个节点为待证目标。扩展规则（Forward / Backward / Destruct）根据目标形状选择适用引理或策略，生成子节点。搜索过程可配置深度限制与启发式评分。

**定理 3.1（Aesop 的可靠性）**. 若 `aesop` 成功关闭目标，则其构造的证明项通过 Lean4 内核的类型检查，因此该目标是定理。

*证明思路*：Aesop 仅使用用户注册的安全规则（Safe Rules）与终端规则（Terminal Rules）。每条规则均为一个已验证的 Lean4 策略或引理，因此搜索树的每个扩展步骤保持可靠性。∎

### 3.3 决策过程

| 策略 | 判定问题 | 算法复杂度 | 理论基础 |
|------|---------|-----------|---------|
| `tauto` | 命题逻辑可满足性 | 多项式时间 | 直觉主义命题逻辑的完备证明搜索 |
| `finish` | 一阶逻辑的特定片段 | 超多项式 | 启发式实例化 + 命题求解 |
| `decide` | 可判定命题（计算结果为 `true`） | 依赖计算 | 反射原理（Reflection Principle） |
| `norm_num` | 数值表达式的化简与比较 | 多项式时间 | 环/域上的规范算术算法 |

**定义 3.3（反射原理）**. 设 $P : \mathbb{N} \to \mathsf{Prop}$ 为可判定谓词，$D_P : \mathbb{N} \to \mathsf{Bool}$ 为其决策过程，满足

$$\forall n:\mathbb{N},\; D_P(n) = \mathsf{true} \;\Longleftrightarrow\; P(n).$$

则证明 $P(42)$ 可转化为计算 $D_P(42)$ 并验证其等于 $\mathsf{true}$。`decide` 策略自动执行此转化。

## 4. 详细证明示例

### 4.1 例子：自然数加法交换律的完整形式化证明

**定理 4.1**（加法交换律）. $\forall n, m \in \mathbb{N},\; n + m = m + n$.

*数学证明*：对 $m$ 进行数学归纳法。

**基例**（$m = 0$）：对任意 $n$，有 $n + 0 = n$（由加法定义）且 $0 + n = n$（由 `Nat.zero_add` 引理），故 $n + 0 = 0 + n$。

**归纳步**：假设对某个 $m$ 有 $n + m = m + n$（归纳假设，IH）。需证 $n + (m+1) = (m+1) + n$。

$$\begin{aligned}
n + (m+1) &= (n + m) + 1 &&\text{（由加法定义 `Nat.add_succ`）} \\
&= (m + n) + 1 &&\text{（由 IH）} \\
&= (m + 1) + n &&\text{（由 `Nat.succ_add`）}
\end{aligned}$$

*Lean4 证明*：

```lean
theorem add_comm : ∀ n m : Nat, n + m = m + n
| n, 0 => by simp [Nat.add_zero, Nat.zero_add]
| n, m+1 => by
  calc
    n + (m + 1) = (n + m) + 1 := by rw [Nat.add_succ]
    _           = (m + n) + 1 := by rw [add_comm n m]
    _           = (m + 1) + n := by rw [Nat.succ_add]
```

### 4.2 例子：极限的线性性质

**定理 4.2**（极限的加法连续性）. 设 $(f_n), (g_n)$ 为实数序列，$\lim_{n\to\infty} f_n = a$，$\lim_{n\to\infty} g_n = b$，则

$$\lim_{n\to\infty} \bigl(f_n + g_n\bigr) = a + b.$$

*数学证明*：给定 $\varepsilon > 0$，取 $\varepsilon/2 > 0$。由极限定义，存在 $N_1$ 使得 $\forall n \geq N_1,\; |f_n - a| < \varepsilon/2$；存在 $N_2$ 使得 $\forall n \geq N_2,\; |g_n - b| < \varepsilon/2$。令 $N = \max(N_1, N_2)$，则对 $n \geq N$：

$$\begin{aligned}
|(f_n + g_n) - (a + b)| &= |(f_n - a) + (g_n - b)| \\
&\leq |f_n - a| + |g_n - b| &&\text{（三角不等式）} \\
&< \frac{\varepsilon}{2} + \frac{\varepsilon}{2} = \varepsilon.
\end{aligned}$$

*Lean4 证明框架*：

```lean
theorem tendsto_add {f g : ℕ → ℝ} {a b : ℝ}
  (hf : Tendsto f a) (hg : Tendsto g b) :
  Tendsto (λ n => f n + g n) (a + b) := by
  intro ε hε
  have hε2 : ε / 2 > 0 := by linarith
  rcases hf (ε / 2) hε2 with ⟨N₁, hN₁⟩
  rcases hg (ε / 2) hε2 with ⟨N₂, hN₂⟩
  use max N₁ N₂
  intro n hn
  have h₁ : |f n - a| < ε / 2 := hN₁ n (le_of_max_le_left hn)
  have h₂ : |g n - b| < ε / 2 := hN₂ n (le_of_max_le_right hn)
  calc
    |(f n + g n) - (a + b)| = |(f n - a) + (g n - b)| := by ring
    _ ≤ |f n - a| + |g n - b| := by apply abs_add_le
    _ < ε / 2 + ε / 2 := by linarith
    _ = ε := by ring
```

### 4.3 例子：归纳法证明算术-几何平均不等式

**定理 4.3**（AM-GM 不等式，二元情形）. 对任意非负实数 $a, b \geq 0$，

$$\frac{a + b}{2} \geq \sqrt{ab}.$$

*数学证明*：由 $(\sqrt{a} - \sqrt{b})^2 \geq 0$ 展开得 $a - 2\sqrt{ab} + b \geq 0$，即 $a + b \geq 2\sqrt{ab}$，整理即得结论。

*Lean4 证明*：

```lean
theorem amgm_two (a b : ℝ) (ha : 0 ≤ a) (hb : 0 ≤ b) :
  (a + b) / 2 ≥ Real.sqrt (a * b) := by
  have h : (Real.sqrt a - Real.sqrt b) ^ 2 ≥ 0 := sq_nonneg _
  have h₁ : Real.sqrt (a * b) = Real.sqrt a * Real.sqrt b :=
    Real.sqrt_mul' a (by linarith)
  nlinarith [Real.sq_sqrt (show 0 ≤ a by linarith),
            Real.sq_sqrt (show 0 ≤ b by linarith),
            Real.sqrt_nonneg a, Real.sqrt_nonneg b, h₁]
```

## 5. 策略组合与证明工程

### 5.1 宏策略（Macro Tactics）

**定义 5.1（策略宏）**. Lean4 允许通过 `macro` 定义策略组合：

```lean
macro "my_tac" : tactic => `(tactic| intro h; rcases h with ⟨a, b⟩; simp at a b; linarith)
```

该宏将 `intro`、`rcases`、`simp` 与 `linarith` 组合为单一策略调用。

### 5.2 自定义引理注册

**定理 5.1（Simp 引理的合流性启发式）**. 若 `simp` 引理集合 $\mathcal{R}$ 满足：
1. 无临界对冲突（Critical Pair Confluence）；
2. 每个规则的左线性（Left-linear）；

则 `simp` 的迭代重写过程在有限步内终止并产生唯一范式。

*应用*：在 Mathlib4 中，代数结构的层级引理（如 `Monoid`、`Group`、`Ring`）通过 careful engineering 保证 `simp` 集合的近似合流性，使得复杂表达式的化简结果可预测。
