# Lean4形式化证明中的类型理论、常见错误与调试方法

---

## 1. 依赖类型基础

**定义 1.1（类型与项）**. Lean4 基于 CIC（Calculus of Inductive Constructions），其中：
- 每个项 $t$ 有类型 $T$，记 $t : T$；
- 类型本身也是项，位于宇宙中：$\text{Type}_0 : \text{Type}_1 : \text{Type}_2 : \dots$；
- 依赖函数类型 $\Pi_{x:A} B(x)$ 允许返回值类型依赖于参数。

**依赖类型的威力**：在 Lean4 中，我们可以定义"长度为 $n$ 的向量"类型：
```lean
def Vec (α : Type) (n : Nat) := { l : List α // l.length = n }
```

这样，`Vec Nat 3` 就是恰好包含 3 个自然数的向量。类型系统保证了越界访问在编译时被拒绝。

**命题即类型 (Curry-Howard 同构)**：
- 命题 $P$ 被编码为类型 `P : Prop`
- 证明 $p$ 是命题 $P$ 的证据，即 `p : P`
- 逻辑蕴含 $P \to Q$ 对应函数类型 `P → Q`
- 合取 $P \land Q$ 对应积类型 `P × Q`

## 2. 常见错误模式

### 2.1 类型不匹配

**错误**: `type mismatch, expected type Foo, got type Bar`

**原因**: 术语的类型与期望类型不一致。常见情形：
- 使用了错误类型的引理
- 隐含参数未正确推断
- 宇宙层级冲突

**调试**: 
1. 使用 `#check` 查看实际类型
2. 使用 `@lemma` 显式提供所有参数
3. 使用 `set_option pp.universes true` 显示宇宙层级

**具体例子**：
```lean
-- 错误示例
example (n : Nat) : n + 0 = 0 + n := by
  rw [Nat.add_comm] -- 错误! add_comm 需要两个参数

-- 修正
example (n : Nat) : n + 0 = 0 + n := by
  rw [Nat.add_comm n 0]
```

`Nat.add_comm` 的类型是 `∀ (n m : Nat), n + m = m + n`。当只提供一个参数时，Lean 尝试将 `n + 0 = 0 + n` 与 `?n + ?m = ?m + ?n` 统一，可能失败或产生意外结果。

### 2.2 证明目标管理

**错误**: `tactic 'apply' failed, failed to unify`

**原因**: `apply` 尝试将引理结论与目标统一，但统一失败。

**调试**: 
1. 使用 `refine` 替代 `apply`，保留未填充的占位符
2. 使用 `convert` 处理几乎匹配的引理
3. 使用 `congr` 处理等式目标的结构匹配

**具体例子**：
```lean
-- 目标: a + b + c = a + c + b
-- apply Nat.add_comm 会失败（add_comm 处理两个数，不是三个）

-- 方法1：分步处理
example (a b c : Nat) : a + b + c = a + c + b := by
  rw [← Nat.add_assoc]
  rw [Nat.add_comm b]
  rw [Nat.add_assoc]

-- 方法2：使用 calc
example (a b c : Nat) : a + b + c = a + c + b := by
  calc
    a + b + c = a + (b + c) := by rw [Nat.add_assoc]
    _ = a + (c + b) := by rw [Nat.add_comm b]
    _ = a + c + b := by rw [Nat.add_assoc]
```

### 2.3 宇宙层级冲突

**错误**: `universe level mismatch`

**原因**: CIC 的宇宙累积规则限制了类型的层级。例如 `Type u` 不能同时属于 `Type v` 和 `Type w`（除非 $u < v$ 且 $u < w$）。

**调试**: 
1. 使用 `universe u v` 声明宇宙变量
2. 使用 `ULift` 提升宇宙层级
3. 使用宇宙多态定义：

```lean
universe u v

def myFunction {α : Type u} {β : Type v} (f : α → β) (x : α) : β := f x
```

**具体例子**：
```lean
-- 错误：尝试将 Type 1 的元素放入 Type 0
def bad : Type 0 := Type 0 -- 错误: Type 0 不能属于自身

-- 正确：使用更高的宇宙
def good : Type 1 := Type 0

-- 使用 ULift 显式提升
def lifted : Type 1 := ULift.{1, 0} (Type 0)
```

### 2.4 证明循环

**错误**: `tactic 'simp' failed, nested error, maximum recursion depth`

**原因**: `simp` 策略可能进入无限重写循环，尤其是当 simp 引理集合中包含相互推导的等式时。

**调试**:
1. 使用 `simp only [lemma1, lemma2]` 限制引理集
2. 使用 `local attribute [-simp]` 临时禁用某些 `simp` 引理
3. 分步重写代替 `simp`

## 3. 性能优化

### 3.1 简化集控制

**问题**: `simp` 可能因引理过多而超时。

**解决**: 
- 使用 `simp only [lemma1, lemma2]` 限制引理集
- 使用 `local attribute [-simp]` 临时禁用某些 `simp` 引理
- 使用 `simp_all` 代替多次 `simp` 调用

**最佳实践**：
```lean
-- 不好的做法（可能超时）
example : ... := by simp

-- 好的做法（显式控制）
example : ... := by simp only [Nat.add_comm, Nat.mul_assoc]
```

### 3.2 类型类推断

**问题**: 类型类推断（type class resolution）可能搜索空间爆炸。

**解决**:
- 为类型类实例设置优先级 `@[instance 100]`
- 使用 `attribute [local instance]` 限制实例范围
- 显式提供实例参数

**具体例子**：
```lean
-- 类型类搜索可能很慢
example (n m : Nat) : n + m = m + n := by
  exact Nat.add_comm n m -- 直接提供实例，避免搜索
```

## 4. 调试技巧与工具

### 4.1 查看证明状态

在证明的任何位置插入以下命令查看当前状态：

```lean
-- 查看当前目标
example (n : Nat) : n + 0 = n := by
  have h : n + 0 = n := by rfl
  exact h
```

### 4.2 使用 `set_option` 诊断

```lean
set_option pp.all true -- 显示完整内部表示
set_option pp.universes true -- 显示宇宙层级
set_option pp.implicit true -- 显示隐式参数
set_option trace.Meta.synthInstance true -- 追踪类型类推断
```

### 4.3 例子：统一失败调试

```lean
theorem example (n m : Nat) : n + m = m + n := by
  -- 错误: apply Nat.add_comm 可能失败，因参数顺序
  -- Nat.add_comm 的目标形式是 ?n + ?m = ?m + ?n
  -- 当前目标是 n + m = m + n，应该匹配
  -- 但如果上下文复杂，统一可能失败
  rw [Nat.add_comm] -- 正确: rw 处理统一更灵活
```

### 4.4 例子：宇宙问题

```lean
-- 这是可以的
def foo : Type 1 := Type 0

-- 这是错误的
def bar : Type 0 := Type 0 -- 错误: Type 0 不能属于自身

-- 修正方案：提升层级或使用不同的类型
def bar' : Type 1 := Type 0
```

## 5. 高级调试策略

### 5.1 分而治之

当面对复杂目标时，将其分解为引理：

```lean
-- 不好的做法：一次性证明所有内容
example (a b c d : Nat) :
    (a + b) + (c + d) = (a + c) + (b + d) := by
  simp [Nat.add_assoc, Nat.add_comm]

-- 好的做法：提取中间引理
lemma add_rearrange (a b c d : Nat) :
    (a + b) + (c + d) = (a + c) + (b + d) := by
  rw [← Nat.add_assoc]
  rw [Nat.add_assoc b]
  rw [Nat.add_comm b]
  rw [Nat.add_assoc]
  rw [Nat.add_assoc]
```

### 5.2 利用 `by_cases` 处理分支

```lean
example (n : Nat) : n = 0 ∨ n > 0 := by
  by_cases h : n = 0
  · left; exact h
  · right
    have : n ≥ 1 := by
      omega -- 自动线性整数算术
    linarith
```
