# Lean4形式化证明中的类型理论、常见错误与调试方法

## 1. 依赖类型基础

**定义 1.1（类型与项）**. Lean4 基于 CIC（Calculus of Inductive Constructions），其中：
- 每个项 $t$ 有类型 $T$，记 $t : T$；
- 类型本身也是项，位于宇宙中：$\text{Type}_0 : \text{Type}_1 : \text{Type}_2 : \dots$；
- 依赖函数类型 $\Pi_{x:A} B(x)$ 允许返回值类型依赖于参数。

## 2. 常见错误模式

### 2.1 类型不匹配

**错误**: `type mismatch, expected type Foo, got type Bar`

**原因**: 术语的类型与期望类型不一致。常见情形：
- 使用了错误类型的引理
- 隐含参数未正确推断
- 宇宙层级冲突

**调试**: 使用 `#check` 查看实际类型；使用 `@lemma` 显式提供所有参数；使用 `set_option pp.universes true` 显示宇宙层级。

### 2.2 证明目标管理

**错误**: `tactic 'apply' failed, failed to unify`

**原因**: `apply` 尝试将引理结论与目标统一，但统一失败。

**调试**: 使用 `refine` 替代 `apply`，保留未填充的占位符；使用 `convert` 处理几乎匹配的引理。

### 2.3 宇宙层级冲突

**错误**: `universe level mismatch`

**原因**: CIC 的宇宙累积规则限制了类型的层级。例如 `Type u` 不能同时属于 `Type v` 和 `Type w`（除非 $u < v$ 且 $u < w$）。

**调试**: 使用 `universe u v` 声明宇宙变量；使用 `ULift` 提升宇宙层级。

## 3. 性能优化

### 3.1 简化集控制

**问题**: `simp` 可能因引理过多而超时。

**解决**: 
- 使用 `simp only [lemma1, lemma2]` 限制引理集
- 使用 `local attribute [-simp]` 临时禁用某些 `simp` 引理

### 3.2 类型类推断

**问题**: 类型类推断（type class resolution）可能搜索空间爆炸。

**解决**:
- 为类型类实例设置优先级 `@[instance 100]`
- 使用 `attribute [local instance]` 限制实例范围

## 4. 例子

### 4.1 例子：统一失败调试

```lean
theorem example (n m : Nat) : n + m = m + n := by
  -- 错误: apply Nat.add_comm 可能失败，因参数顺序
  rw [Nat.add_comm] -- 正确: rw 处理统一更灵活
```

### 4.2 例子：宇宙问题

```lean
-- 错误
def foo : Type 1 := Type 0
-- Type 0 : Type 1, 但 foo 的类型是 Type 1，而 Type 0 不是 Type 1 的元素（它是 Type 1 的元素）
-- 实际上这可以，但下面的不行：

-- def bar : Type 0 := Type 0 -- 错误: Type 0 不能属于自身
```
