## 一、如何浏览数学家理念体系

### 1.1 数学家理念体系简介

数学家理念体系是FormalMath项目的**核心特色**，包含62位数学家的深度分析，从古希腊的阿基米德到当代的肖尔策。

### 1.2 标准目录结构

每个数学家的理念体系遵循以下标准结构：

```

数学家姓名数学理念/
├── 00-文档索引.md              # 该数学家所有文档的完整索引
├── 00-项目状态.md              # 完成度统计与待办事项
├── README.md                   # 快速入门指南
├── START-HERE.md              # 新用户首选阅读文档
├── 01-核心理论/               # 核心理论贡献概览
├── 02-数学内容深度分析/       # 主要数学内容（核心）
├── 03-教育与影响/             # 教育思想、学生与学派
├── 04-历史与传记/             # 生平、著作、学术合作
├── 05-现代应用与拓展/         # 现代数学/物理/CS应用
├── 06-对比研究/               # 与同时代/前人/后继者对比
├── 07-现代视角与评价/         # 现代数学家观点与未解决问题
└── 08-知识关联分析/           # 概念网络（含Mermaid图）

```

### 1.3 推荐浏览路径

#### 新用户快速入门

1. **选择感兴趣的数学家**：从 [数学家索引](../数学家理念体系/00-数学家索引-2025年12月.md) 开始
2. **阅读README.md**：了解该数学家的核心贡献
3. **阅读START-HERE.md**：获取推荐的阅读顺序
4. **深入02-数学内容深度分析**：学习具体数学理论
5. **探索08-知识关联分析**：理解概念间的联系

#### 深度研究路径

1. **阅读所有标准模块**：从01到08系统学习
2. **关注知识关联分析**：理解该数学家工作的上下文
3. **参考国际课程映射**：了解与现代课程的对应关系
4. **查看跨数学家链接**：探索不同数学家的思想联系

## 二、如何使用形式化证明模块

### 2.1 形式化证明简介

FormalMath项目包含**20个经过编译验证的Lean4定理**，涵盖代数、分析、数论、拓扑、逻辑五大领域。

### 2.2 环境要求

- Lean 4.29.0
- Mathlib4 v4.29.0
- 支持平台：Windows、macOS、Linux

### 2.3 快速开始

#### 安装Lean4

```bash
# 安装elan（Lean版本管理器）
curl https://raw.githubusercontent.com/leanprover/elan/master/elan-init.sh -sSf | sh

# 安装Lean4
elan install leanprover/lean4:v4.29.0

# 设置默认版本
elan default leanprover/lean4:v4.29.0

```

## 编译项目定理

```bash
# 进入形式化证明目录
cd docs/09-形式化证明/lean4-examples/

# 编译所有定理
lake build

# 或编译单个文件
lean --make GroupTheory/Basic.lean

```

## 三、如何使用智能学习系统

FormalMath项目开发了三个智能学习系统，形成完整的学习支持闭环。

### 3.1 认知诊断系统

**功能**：评估学习者的知识掌握度和认知能力

#### 使用步骤

1. **访问系统**：进入 `project/cognitive-diagnosis-system/`
2. **启动服务**：
   ```bash
   cd project/cognitive-diagnosis-system/
   docker-compose up
   ```

3. **访问界面**：浏览器打开 `http://localhost:3000`
4. **完成诊断测试**：约20-30分钟
5. **查看诊断报告**：获取知识图谱和能力评估

#### 诊断维度

| 维度 | 说明 |
|------|------|
| 知识掌握度 | 各数学概念的掌握程度 |
| 认知能力 | 抽象思维、逻辑推理、空间想象 |
| 学习风格 | 视觉/听觉/动觉偏好 |

#### 系统架构

- **后端**：Python + FastAPI + PostgreSQL
- **前端**：React + TypeScript
- **算法**：HCD（Hierarchical Cognitive Diagnosis）诊断模型
- **部署**：Docker容器化

### 3.2 评估系统

**功能**：多维度评价学习过程和成果

#### 使用步骤

1. **访问系统**：进入 `project/assessment-system/`
2. **启动服务**：
   ```bash
   cd project/assessment-system/
   docker-compose up
   ```

3. **访问界面**：浏览器打开 `http://localhost:3001`
4. **选择评估类型**：
   - 形成性评估（学习过程中）
   - 总结性评估（阶段结束时）
   - 诊断性评估（针对性测试）
5. **查看评估报告**：获取详细反馈

#### 评估维度

| 维度 | 权重 | 说明 |
|------|------|------|
| 知识掌握 | 30% | 概念理解和记忆 |
| 技能应用 | 25% | 问题解决能力 |
| 数学思维 | 25% | 逻辑推理和抽象思维 |
| 创新能力 | 20% | 创造性解决问题 |

#### 反馈机制

- **即时反馈**：答题后立即给出提示
- **详细报告**：学习结束后生成完整报告
- **改进建议**：针对性学习建议

### 3.3 自适应学习路径系统

**功能**：根据学习者特征推荐个性化学习路径

#### 使用步骤

1. **访问系统**：进入 `project/adaptive-learning-system/`
2. **启动服务**：
   ```bash
   cd project/adaptive-learning-system/
   docker-compose up
   ```

3. **访问界面**：浏览器打开 `http://localhost:3002`
4. **输入学习目标**：选择想要学习的主题
5. **系统生成路径**：基于A*算法生成最优路径
6. **开始学习**：跟随推荐路径学习
7. **实时调整**：系统根据学习表现动态调整

#### 个性化维度

| 维度 | 说明 |
|------|------|
| 学习进度 | 已掌握 vs 待学习内容 |
| 认知状态 | 当前知识掌握度 |
| 兴趣偏好 | 感兴趣的数学领域 |
| 时间约束 | 可用学习时间 |

#### 路径优化算法

- **核心算法**：A*搜索算法
- **启发函数**：综合考虑难度、先修关系、学习目标
- **动态调整**：根据学习反馈实时优化

### 3.4 三系统协同使用

```

┌─────────────────┐
│  认知诊断系统   │ ← 了解当前状态
│   (初始评估)    │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 自适应学习系统  │ ← 制定学习路径
│  (路径规划)     │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│    学习过程     │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│   评估系统      │ ← 检验学习效果
│   (过程评估)    │
└────────┬────────┘
         │
         └──────→ 反馈到认知诊断，循环优化

```

### 4.2 查找步骤

1. 打开 [国际课程与机构对齐对照表](../project/00-国际课程与机构对齐对照表-2026年4月.md)

3. 查看课程列表和对应的项目文档
4. 点击链接进入详细映射表

#### 方法二：通过课程代码查找

以MIT 18.726为例：

1. 在总表中找到MIT部分
2. 定位18.726（代数几何）
3. 查看对应项目内容：
   - 主要对应：`数学家理念体系/格洛腾迪克数学理念/02-概形理论/`
   - 详细周主题映射：[00-标杆课周主题篇目映射表-MIT18.726.md](../project/00-标杆课周主题篇目映射表-MIT18.726.md)

#### 方法三：通过主题反向查找

如果想学习"层上同调"：

1. 在总表中搜索"层上同调"
2. 发现对应：
   - MIT 18.726（第X周）
   - Harvard Math 232br
   - Stanford FOAG Ch 20
   - Oxford C2.2
3. 选择适合的课程进行学习

## 五、常见问题FAQ

### Q1: 我是初学者，应该从哪里开始？

**A**: 推荐从以下文档开始：
1. `docs/01-基础数学/集合论/01-集合论基础-国际标准版.md`
2. `docs/01-基础数学/集合论/02-数系与运算-深度扩展版.md`
3. `docs/02-代数结构/线性代数/线性代数-深度扩展版.md`

同时参考 [从入门到精通学习路径](00-从入门到精通学习路径.md) 中的"初学者路径"。

### Q2: 如何找到与国际课程对应的资源？

**A**: 有三种方法：
1. 查阅 [国际课程与机构对齐对照表](../project/00-国际课程与机构对齐对照表-2026年4月.md)
2. 查看各学校的详细映射表（如 [MIT-course-mapping-detailed.md](../project/MIT-course-mapping-detailed.md)）
3. 使用搜索功能在文档中查找课程代码

### Q3: 数学家理念体系如何使用？

**A**: 
1. 从 [数学家索引](../数学家理念体系/00-数学家索引-2025年12月.md) 选择感兴趣的数学家
2. 阅读该数学家的README.md和START-HERE.md
3. 深入02-数学内容深度分析学习具体理论
4. 探索08-知识关联分析理解概念联系

### Q4: Lean4代码如何运行？

**A**: 
1. 安装Lean 4.29.0和Mathlib4 v4.29.0
2. 进入 `docs/09-形式化证明/lean4-examples/`
3. 运行 `lake build` 编译所有定理
4. 使用VS Code + Lean插件进行交互式探索

详见 [Lean4入门指南](09-形式化证明/Lean4入门指南.md)

### Q5: 内容深度如何选择？

**A**: 根据您的背景选择：
- **基础版**：初学者，概念介绍
- **增强版**：中级学习者，定理+例子
- **深度扩展版**：高级学习者，完整证明
- **国际标准版**：所有水平，国际对齐
