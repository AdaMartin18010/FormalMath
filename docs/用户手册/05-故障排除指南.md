## 一、文档访问问题

### 问题1.1: 无法打开 Markdown 文档

**症状**: 双击 .md 文件无法打开或显示乱码

**解决方案**:

| 方案 | 操作步骤 |
|------|----------|
| **方案A** | 使用 VS Code 安装 Markdown 扩展 |
| **方案B** | 使用 Typora 等专业 Markdown 编辑器 |
| **方案C** | 使用浏览器插件（如 Markdown Viewer） |
| **方案D** | 转换为 HTML: `pandoc file.md -o file.html` |

**推荐工具**:

- Windows: Typora, VS Code
- macOS: Typora, iA Writer
- Linux: ReText, VS Code

---

### 问题1.2: 文档中的公式显示不正确

**症状**: LaTeX 公式显示为源代码或乱码

**解决方案**:

1. **使用支持 MathJax 的阅读器**
   - VS Code + Markdown Preview Enhanced
   - Typora
   - GitHub/GitLab 网页版

2. **在线渲染**

   ```bash
   # 使用 pandoc 转换
   pandoc file.md -o file.html --mathjax
   ```

3. **浏览器插件**
   - Markdown Viewer (Chrome/Firefox)
   - 开启 MathJax 支持

---

### 问题1.3: 中文显示乱码

**症状**: 中文内容显示为乱码或方框

**解决方案**:

1. **检查文件编码**

   ```bash
   # 确认文件为 UTF-8 编码
   file -i document.md
   ```

2. **转换编码**

   ```bash
   # 使用 iconv 转换
   iconv -f GBK -t UTF-8 input.md > output.md
   ```

3. **编辑器设置**
   - VS Code: 右下角点击编码 → 通过编码保存 → UTF-8
   - Notepad++: 编码 → 转为 UTF-8 编码

---

### 问题1.4: 文档链接失效

**症状**: 点击链接提示"文件不存在"

**原因与解决**:

| 原因 | 解决方案 |
|------|----------|
| 相对路径错误 | 从项目根目录打开文档 |
| 文件被移动 | 使用搜索功能查找新位置 |
| 文件名更改 | 查阅 `00-FormalMath项目内容索引-完整版.md` |
| 文档未下载 | 确保完整克隆/下载仓库 |

**快速修复**:

```bash
# 在项目根目录搜索
grep -r "目标文件名" . --include="*.md"
```

## 二、智能学习系统问题

### 问题2.1: 认知诊断系统无法启动

**症状**: 运行诊断系统时出错

**排查步骤**:

1. **检查 Python 版本**

   ```bash
   python --version  # 需要 3.9+
   ```

2. **安装依赖**

   ```bash
   pip install -r requirements.txt
   ```

3. **检查端口占用**

   ```bash
   # Windows
   netstat -ano | findstr :8000

   # macOS/Linux
   lsof -i :8000
   ```

4. **查看日志**

   ```bash
   tail -f logs/diagnosis.log
   ```

---

### 问题2.2: 诊断测试提交失败

**症状**: 完成测试后无法提交

**解决方案**:

| 可能原因 | 解决方案 |
|----------|----------|
| 网络连接问题 | 检查网络，稍后重试 |
| 服务器超时 | 分段完成测试 |
| 数据格式错误 | 刷新页面重新测试 |
| 浏览器兼容性 | 使用 Chrome/Firefox |

**数据恢复**:

```javascript
// 在控制台执行恢复数据
localStorage.getItem('diagnosis_progress')
```

---

### 问题2.3: 学习路径生成失败

**症状**: 点击生成路径后无响应或报错

**排查步骤**:

1. **检查学习者档案**
   - 确保已完成认知诊断
   - 确认学习目标设置正确

2. **检查知识图谱**

   ```bash
   # 验证 Neo4j 连接
   curl http://localhost:7474
   ```

3. **简化目标**
   - 减少目标概念数量
   - 缩短时间范围

4. **查看错误日志**

   ```bash
   cat logs/path_generation.log
   ```

---

### 问题2.4: 评估系统显示异常

**症状**: 雷达图或图表不显示

**解决方案**:

1. **检查前端依赖**

   ```bash
   cd frontend
   npm install
   npm start
   ```

2. **清除缓存**

   ```bash
   # 浏览器控制台
   localStorage.clear()
   ```

3. **检查浏览器控制台错误**
   - F12 打开开发者工具
   - 查看 Console 错误信息

## 三、Lean4 形式化证明问题

### 问题3.1: Lean 安装失败

**症状**: elan 安装命令执行失败

**解决方案**:

**Windows**:

```powershell
# 使用 PowerShell 管理员权限
Invoke-WebRequest -Uri "https://raw.githubusercontent.com/leanprover/elan/master/elan-init.ps1" -OutFile "elan-init.ps1"
./elan-init.ps1
```

**macOS/Linux**:

```bash
# 安装依赖
sudo apt-get install curl git

# 重新安装
curl https://raw.githubusercontent.com/leanprover/elan/master/elan-init.sh -sSf | sh
source $HOME/.elan/env
```

---

### 问题3.2: lake build 失败

**症状**: 编译 Lean4 项目时出错

**常见错误与解决**:

| 错误信息 | 解决方案 |
|----------|----------|
| `unknown package` | 运行 `lake update` |
| `toolchain mismatch` | `elan default leanprover/lean4:v4.29.0` |
| `timeout` | 增加超时时间或检查网络 |
| `out of memory` | 关闭其他程序或增加交换空间 |
| `git clone failed` | 配置 git 代理 |

**完整重建步骤**:

```bash
cd docs/09-形式化证明/Lean4/
rm -rf .lake lake-packages
lake update
lake build
```

---

### 问题3.3: VS Code Lean 扩展不工作

**症状**: 无语法高亮、无目标视图

**解决方案**:

1. **安装 Lean4 扩展**
   - 扩展商店搜索 "Lean4"
   - 安装leanprover/lean4

2. **检查 Lean 路径**

   ```bash
   which lean  # 确认 lean 在 PATH 中
   ```

3. **重启 Lean 服务器**
   - Ctrl+Shift+P → "Lean4: Restart Server"

4. **检查项目配置**

   ```bash
   cat lakefile.lean  # 确认配置正确
   cat lean-toolchain  # 确认版本正确
   ```

---

### 问题3.4: 证明检查超时

**症状**: 复杂证明检查时间过长

**优化建议**:

1. **增加超时时间**

   ```lean
   set_option maxHeartbeats 1000000
   ```

2. **简化证明**
   - 分解复杂证明为引理
   - 使用 `by`  tactic 替代自动化

3. **更新 Mathlib**

   ```bash
   lake update
   ```

## 四、搜索与导航问题

### 问题4.1: grep 搜索无结果

**症状**: 确认存在的内容搜索不到

**排查步骤**:

1. **检查搜索范围**

   ```bash
   # 确保在正确目录
   pwd

   # 扩大搜索范围
   grep -r "关键词" /path/to/FormalMath/
   ```

2. **检查文件编码**

   ```bash
   # 使用 -a 选项处理二进制文件
   grep -a "关键词" file.md
   ```

3. **忽略大小写**

   ```bash
   grep -ri "关键词" docs/
   ```

4. **使用正则表达式**

   ```bash
   grep -rE "(关键词1|关键词2)" docs/
   ```

---

### 问题4.2: 文档索引加载缓慢

**症状**: 打开内容索引文档卡顿

**优化方案**:

1. **使用轻量编辑器**
   - 避免使用重型 IDE
   - 推荐使用 VS Code 或 Sublime Text

2. **分页加载**
   - 不要一次性打开整个索引
   - 使用搜索定位到具体章节

3. **生成子索引**

   ```bash
   # 提取特定分支的索引
   grep -A 5 "02-代数结构" 00-FormalMath项目内容索引-完整版.md > algebra_index.md
   ```

---

### 问题4.3: 无法找到相关文档

**症状**: 知道概念存在但找不到文档

**解决策略**:

1. **多关键词搜索**

   ```bash
   # 尝试不同表达
   grep -r "正规子群" docs/
   grep -r "normal subgroup" docs/
   grep -r "不变子群" docs/
   ```

2. **查看概念映射**
   - 查阅 `multilang_concept_table.json`
   - 使用术语对照表

3. **按 MSC 分类查找**
   - 确定概念的 MSC 编码
   - 查找对应分类的文档

## 五、内容理解问题

### 问题5.1: 前置知识不足

**症状**: 阅读文档时遇到不熟悉的概念

**解决策略**:

1. **追溯前置链接**
   - 点击文档中的概念链接
   - 先学习前置内容

2. **降低文档深度**
   - 切换到 `-基础版.md`
   - 或 `-国际标准版.md`

3. **使用认知诊断**
   - 运行诊断测试
   - 系统会指出知识缺口

4. **查阅外部资源**
   - 文档末尾的参考文献
   - Wikipedia/MathWorld 对照

---

### 问题5.2: 证明难以理解

**症状**: 证明步骤跳跃太大

**建议方法**:

1. **分步理解**
   - 将证明分解为小步骤
   - 验证每一步的正确性

2. **动手验证**
   - 用具体例子验证
   - 画图辅助理解

3. **寻找替代证明**
   - 查阅不同教材的证明
   - 参考数学家理念体系

4. **寻求帮助**
   - 在讨论区提问
   - 参考 StackExchange

---

### 问题5.3: 符号系统混乱

**症状**: 不同文档使用不同符号

**解决方案**:

1. **查阅符号对照表**
   - `docs/00-数学符号对照表.md`
   - `docs/00-数学符号速查表.md`

2. **查看文档符号说明**
   - 大多数文档开头有符号说明
   - 附录中通常有符号汇总

3. **理解上下文**
   - 符号含义通常在上下文中明确
   - 注意作者的符号约定

## 六、性能问题

### 问题6.1: 文档打开缓慢

**症状**: 大型文档打开时间长

**优化建议**:

1. **使用高效编辑器**
   - VS Code
   - Sublime Text
   - Vim/Neovim

2. **禁用实时预览**
   - 关闭自动渲染
   - 手动触发预览

3. **分割大文档**

   ```bash
   # 使用脚本分割
   split -l 500 large.md segment_
   ```

---

### 问题6.2: 搜索速度慢

**症状**: grep 搜索大型仓库缓慢

**优化方案**:

1. **限制搜索范围**

   ```bash
   # 只搜索特定分支
   grep -r "关键词" docs/02-代数结构/

   # 只搜索特定深度
   find docs/ -name "*深度扩展版.md" -exec grep -l "关键词" {} \;
   ```

2. **使用 ripgrep (rg)**

   ```bash
   # rg 比 grep 更快
   rg "关键词" docs/
   ```

3. **建立索引**

   ```bash
   # 使用 ctags 建立索引
   ctags -R docs/
   ```

---

### 问题6.3: 内存占用过高

**症状**: 打开多个文档后系统变慢

**解决方案**:

1. **关闭不需要的文档**
   - 一次只打开几个相关文档
   - 及时关闭已读完的文档

2. **使用轻量阅读器**
   - 避免在浏览器中打开大量标签
   - 使用专用 Markdown 阅读器

3. **分页浏览索引**
   - 不要一次性加载整个索引
   - 使用搜索定位
