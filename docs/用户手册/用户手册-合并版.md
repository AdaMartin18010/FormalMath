## 第一步：选择您的起点

根据您的数学背景，选择合适的学习起点：

### 🌱 初学者（高中/大一水平）

**如果您符合以下情况**：
- 刚接触高等数学
- 需要巩固数学基础
- 准备大学数学课程

**推荐学习路径**：

1. **集合论基础** → `docs/01-基础数学/集合论/01-集合论基础-国际标准版.md`

**预计学习时间**: 3-4个月

**如果您符合以下情况**：

**推荐学习路径**：

## 第三步：开始使用智能学习系统

FormalMath 提供三大智能学习系统，帮助您高效学习：

### 1. 认知诊断系统

**功能**: 评估您在 L0-L3 四个知识层次的能力

**使用步骤**：

1. 进入 `docs/00-认知诊断系统使用指南.md`
2. 完成诊断测试（约30-45分钟）
3. 查看诊断报告，了解薄弱环节
4. 根据建议开始学习

**知识层次说明**：

| 层次 | 名称 | 能力要求 |
|------|------|----------|
| L0 | 基础概念层 | 概念识别、直观理解 |
| L1 | 定义理解层 | 形式化定义、基本证明 |
| L2 | 定理应用层 | 复杂证明、综合应用 |
| L3 | 综合证明层 | 创新性研究、开放问题 |

---

### 2. 自适应学习路径系统

**功能**: 根据您的特征生成个性化学习路径

**使用步骤**：

1. 导入认知诊断结果
2. 设定学习目标（如"掌握群论基础"）
3. 系统生成最优学习路径
4. 跟随路径学习，系统实时调整

**路径特点**：
- 基于 A* 算法优化
- 考虑您的学习风格（视觉/听觉/动手/阅读）
- 根据进度动态调整难度

---

### 3. 评估系统

**功能**: 多维度评价学习效果

**评估维度**：

| 维度 | 权重 | 说明 |
|------|------|------|
| 知识掌握度 | 30% | 概念理解、定理掌握 |
| 技能熟练度 | 25% | 计算、证明、建模 |
| 问题解决能力 | 25% | 分析、策略、验证 |
| 创新思维能力 | 20% | 发散性、创造性 |

## 常见问题快速解答

### Q: 我是纯初学者，完全没有高等数学基础，该从哪里开始？

**A**: 从 `docs/01-基础数学/集合论/01-集合论基础-国际标准版.md` 开始，这是最基础的内容。

### Q: 如何找到与我想学的课程对应的内容？

**A**: 
- MIT课程 → 查阅 `project/MIT-course-mapping-detailed.md`
- Harvard课程 → 查阅 `project/Harvard-course-mapping-detailed.md`
- Stanford FOAG → 查阅 `project/Stanford-FOAG-mapping-detailed.md`

### Q: 文档太多了，如何快速找到我需要的？

**A**: 查阅 `00-FormalMath项目内容索引-完整版.md`，这是全项目的索引。

### Q: 内容深度不合适怎么办？

**A**: 每个主题通常有多个版本，选择适合您的后缀：`-基础版`、`-增强版`、`-深度扩展版`。

### Q: 学习过程中遇到问题如何求助？

### 2.1 认知诊断系统

#### 系统概述

基于 HCD（层次约束感知认知诊断）框架，评估 L0-L3 四个知识层次的能力。

#### 使用流程

**步骤1：进入诊断系统**

打开 `docs/00-认知诊断系统使用指南.md`

**步骤2：配置诊断参数**

```yaml
诊断配置：
  目标层次: L1-L2          # 可选：L0, L1, L2, L3
  重点领域: 代数结构       # 可选：全部、代数、分析、几何
  题目数量: 30             # 默认30题
```

**步骤3：完成测试**

- 单选题：自动评分
- 多选题：需全部选对
- 填空题：精确匹配
- 简答题/证明题：人工评分

**步骤4：解读诊断报告**

报告包含：

```
诊断报告
├── 综合评估
│   ├── 整体水平: L1.5
│   └── 置信度: 87%
├── L0-L3能力分析
│   ├── L0: 0.92 (优秀)
│   ├── L1: 0.78 (良好)
│   ├── L2: 0.45 (需提升)
│   └── L3: 0.12 (初学)
├── 知识领域分布
│   ├── 代数结构: 75%
│   ├── 分析学: 60%
│   └── 几何学: 45%
├── 薄弱环节
│   ├── 环论同态
│   ├── 拓扑连续性
│   └── 测度论基础
└── 学习建议
    ├── 优先: 强化L1基础
    ├── 次要: 补充先修知识
    └── 长期: 逐步提升层次
```

#### 诊断算法原理

HCD 算法的核心特点：

1. **层次感知**：显式建模 L0→L1→L2→L3 的依赖关系
2. **约束优化**：使用层次约束指导知识状态估计
3. **多维度输出**：同时输出知识点掌握度和层次能力
4. **可解释性**：诊断结果有明确的层次结构解释

### 2.2 自适应学习路径系统

#### 系统概述

基于知识图谱和 A* 算法，生成个性化学习路径。

#### 核心功能

**1. 学习者特征分析**

```python
学习者档案 = {
    "认知诊断结果": {...},
    "学习风格": "visual",      # visual/auditory/kinesthetic/reading
    "先验知识": {...},
    "学习目标": ["群", "环", "域"],
    "可用时间": {
        "daily_hours": 2,
        "weekly_days": 5
    }
}
```

**2. 路径生成**

系统使用 A* 算法寻找最优路径：

```
输入: 当前知识状态 → 目标概念
输出: 最优学习路径

评估指标:
├── 平滑度: 难度曲线连续性
├── 覆盖率: 目标概念覆盖程度
├── 效率: 单位时间学习效果
└── 多样性: MSC分类分布
```

**3. 实时调整**

| 检测指标 | 阈值 | 调整策略 |
|---------|------|----------|
| 时间比例 | > 2.0 | 降低难度 |
| 错误率 | > 50% | 插入复习 |
| 帮助请求 | ≥ 3次 | 添加支持内容 |
| 掌握度 | ≥ 0.9 | 提高难度 |

**4. 间隔重复复习**

基于艾宾浩斯遗忘曲线：

```
掌握度 >= 0.9: [7, 14, 30, 60, 120] 天
掌握度 >= 0.8: [3, 7, 14, 30, 60] 天
掌握度 >= 0.7: [1, 3, 7, 14, 30] 天
掌握度 < 0.7:  [1, 2, 4, 7, 14] 天
```

#### 使用示例

    "path_id": "path-001",

        {"concept": "集合", "estimated_time": 2},
        {"concept": "二元运算", "estimated_time": 3},
        {"concept": "群", "estimated_time": 8},
        ...
    ]
}

## 一、入门问题

### Q1: 我完全没有任何高等数学基础，可以学习FormalMath吗？

**A**: 可以！FormalMath 提供从基础到前沿的完整内容。

**推荐起点**：

1. `docs/01-基础数学/集合论/01-集合论基础-国际标准版.md`
2. `docs/01-基础数学/集合论/02-数系与运算-深度扩展版.md`
3. `docs/02-代数结构/线性代数/线性代数-深度扩展版.md`

**学习建议**：从带有 `-基础版` 或 `-国际标准版` 后缀的文档开始。

---

### Q2: FormalMath 适合什么水平的学习者？

**A**: FormalMath 涵盖 L0-L3 四个知识层次：

| 层次 | 适合人群 | 内容深度 |
|------|----------|----------|
| L0 | 高中/大一 | 概念介绍、直观理解 |
| L1 | 大二/大三 | 形式化定义、基本定理 |
| L2 | 研究生 | 复杂证明、综合应用 |
| L3 | 研究者 | 前沿内容、开放问题 |

---

### Q3: 使用 FormalMath 需要先学习编程吗？

**A**: 不需要。

- **阅读文档**: 完全不需要编程
- **形式化证明**: 需要学习 Lean4（可选）
- **智能学习系统**: 通过界面使用（无需编程）

---

### Q4: 我应该从哪里开始学习？

**A**: 根据您的背景选择：

**情况A：高中毕业生/大一新生**

```
路径: 集合论 → 线性代数 → 微积分 → 群论
时间: 约6个月
```

**情况B：大二/大三数学系学生**

```
路径: 实分析 → 抽象代数 → 拓扑学
时间: 约4个月
```

**情况C：研究生/研究人员**

```
路径: 根据研究方向选择专题
例如: 概形理论 → 层上同调 → 导出范畴
时间: 约6-12个月
```

---

### Q5: 每天需要花多少时间学习？

**A**: 取决于您的目标：

| 学习目标 | 每日时间 | 完成基础内容 |
|----------|----------|--------------|
| 兴趣学习 | 30分钟 | 1-2年 |
| 专业提升 | 1-2小时 | 6-12个月 |
| 全职学习 | 4-6小时 | 3-6个月 |

## 四、搜索与导航问题

### 问题4.1: grep 搜索无结果

**症状**: 确认存在的内容搜索不到

**排查步骤**:

1. **检查搜索范围**

   ```bash
   # 确保在正确目录
   pwd

   # 扩大搜索范围
   grep -r "关键词" /path/to/FormalMath/
   ```

2. **检查文件编码**

   ```bash
   # 使用 -a 选项处理二进制文件
   grep -a "关键词" file.md
   ```

3. **忽略大小写**

   ```bash
   grep -ri "关键词" docs/
   ```

4. **使用正则表达式**

   ```bash
   grep -rE "(关键词1|关键词2)" docs/
   ```

---

### 问题4.2: 文档索引加载缓慢

**症状**: 打开内容索引文档卡顿

**优化方案**:

1. **使用轻量编辑器**
   - 避免使用重型 IDE
   - 推荐使用 VS Code 或 Sublime Text

2. **分页加载**
   - 不要一次性打开整个索引
   - 使用搜索定位到具体章节

3. **生成子索引**

   ```bash
   # 提取特定分支的索引
   grep -A 5 "02-代数结构" 00-FormalMath项目内容索引-完整版.md > algebra_index.md
   ```

**症状**: 知道概念存在但找不到文档

**解决策略**:

1. **多关键词搜索**

   ```bash
   # 尝试不同表达
   grep -r "正规子群" docs/
   grep -r "normal subgroup" docs/
   grep -r "不变子群" docs/
   ```

2. **查看概念映射**
   - 查阅 `multilang_concept_table.json`
   - 使用术语对照表

3. **按 MSC 分类查找**
   - 确定概念的 MSC 编码
   - 查找对应分类的文档

## 六、性能问题

### 问题6.1: 文档打开缓慢

**症状**: 大型文档打开时间长

**优化建议**:

1. **使用高效编辑器**
   - VS Code
   - Sublime Text
   - Vim/Neovim

2. **禁用实时预览**
   - 关闭自动渲染
   - 手动触发预览

3. **分割大文档**

   ```bash
   # 使用脚本分割
   split -l 500 large.md segment_
   ```

---

### 问题6.2: 搜索速度慢

**症状**: grep 搜索大型仓库缓慢

**优化方案**:

1. **限制搜索范围**

   ```bash
   # 只搜索特定分支
   grep -r "关键词" docs/02-代数结构/

   # 只搜索特定深度
   find docs/ -name "*深度扩展版.md" -exec grep -l "关键词" {} \;
   ```

2. **使用 ripgrep (rg)**

   ```bash
   # rg 比 grep 更快
   rg "关键词" docs/
   ```

3. **建立索引**

   ```bash
   # 使用 ctags 建立索引
   ctags -R docs/
   ```

---

### 问题6.3: 内存占用过高

**症状**: 打开多个文档后系统变慢

**解决方案**:

1. **关闭不需要的文档**
   - 一次只打开几个相关文档
   - 及时关闭已读完的文档

2. **使用轻量阅读器**
   - 避免在浏览器中打开大量标签
   - 使用专用 Markdown 阅读器

3. **分页浏览索引**
   - 不要一次性加载整个索引
   - 使用搜索定位
