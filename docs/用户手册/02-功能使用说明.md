## 一、文档系统使用

### 1.1 文档结构

FormalMath 的文档按数学分支组织，采用 MSC2020 分类标准：

```
docs/
├── 01-基础数学/          # 集合论、数理逻辑基础
├── 02-代数结构/          # 群论、环论、域论、线性代数
├── 03-分析学/            # 实分析、复分析、泛函分析
├── 04-几何学/            # 欧氏几何、非欧几何
├── 05-拓扑学/            # 点集拓扑、代数拓扑
├── 06-数论/              # 初等数论、解析数论、代数数论
├── 07-逻辑学/            # 命题逻辑、一阶逻辑、证明论
├── 08-计算数学/          # 数值分析、优化算法
├── 09-形式化证明/        # Lean4、证明系统
├── 10-语义模型/          # 模型论、类型论
├── 11-组合数学/          # 计数原理、图论
├── 12-应用数学/          # 概率论、统计学、微分方程
├── 13-代数几何/          # 概形理论、层上同调
├── 14-微分几何/          # 流形、黎曼几何
├── 15-同调代数/          # 链复形、导出函子
└── 18-数学物理/          # 量子力学数学基础
```

### 1.2 文档格式规范

每个文档采用统一的 YAML 头部格式：

```yaml
---
msc_primary: "20-XX"        # 主要MSC分类

author: "FormalMath Team"   # 作者

### 1.3 阅读技巧

#### 使用文档层级

每个数学主题通常包含多个深度层次：

```
主题：群论
group_theory/
├── 群论-基础版.md          # L0-L1: 概念和例子
├── 群论-增强版.md          # L1-L2: 定理和应用
└── 群论-深度扩展版.md      # L2-L3: 完整证明

**建议**：

- 首次接触主题 → 从基础版开始
- 需要深入理解 → 阅读深度扩展版
- 备考或应用 → 参考增强版

## 二、智能学习系统

### 2.1 认知诊断系统

#### 系统概述

基于 HCD（层次约束感知认知诊断）框架，评估 L0-L3 四个知识层次的能力。

#### 使用流程

**步骤1：进入诊断系统**

打开 `docs/00-认知诊断系统使用指南.md`

**步骤2：配置诊断参数**

```yaml
诊断配置：
  目标层次: L1-L2          # 可选：L0, L1, L2, L3
  重点领域: 代数结构       # 可选：全部、代数、分析、几何
  题目数量: 30             # 默认30题
```

**步骤3：完成测试**

- 单选题：自动评分
- 多选题：需全部选对
- 填空题：精确匹配
- 简答题/证明题：人工评分

**步骤4：解读诊断报告**

报告包含：

```
诊断报告
├── 综合评估
│   ├── 整体水平: L1.5
│   └── 置信度: 87%
├── L0-L3能力分析
│   ├── L0: 0.92 (优秀)
│   ├── L1: 0.78 (良好)
│   ├── L2: 0.45 (需提升)
│   └── L3: 0.12 (初学)
├── 知识领域分布
│   ├── 代数结构: 75%
│   ├── 分析学: 60%
│   └── 几何学: 45%
├── 薄弱环节
│   ├── 环论同态
│   ├── 拓扑连续性
│   └── 测度论基础
└── 学习建议
    ├── 优先: 强化L1基础
    ├── 次要: 补充先修知识
    └── 长期: 逐步提升层次
```

#### 诊断算法原理

HCD 算法的核心特点：

1. **层次感知**：显式建模 L0→L1→L2→L3 的依赖关系
2. **约束优化**：使用层次约束指导知识状态估计
3. **多维度输出**：同时输出知识点掌握度和层次能力
4. **可解释性**：诊断结果有明确的层次结构解释

### 2.2 自适应学习路径系统

#### 系统概述

基于知识图谱和 A* 算法，生成个性化学习路径。

#### 核心功能

**1. 学习者特征分析**

```python
学习者档案 = {
    "认知诊断结果": {...},
    "学习风格": "visual",      # visual/auditory/kinesthetic/reading
    "先验知识": {...},
    "学习目标": ["群", "环", "域"],
    "可用时间": {
        "daily_hours": 2,
        "weekly_days": 5
    }
}
```

**2. 路径生成**

系统使用 A* 算法寻找最优路径：

```
输入: 当前知识状态 → 目标概念
输出: 最优学习路径

评估指标:
├── 平滑度: 难度曲线连续性
├── 覆盖率: 目标概念覆盖程度
├── 效率: 单位时间学习效果
└── 多样性: MSC分类分布
```

**3. 实时调整**

| 检测指标 | 阈值 | 调整策略 |
|---------|------|----------|
| 时间比例 | > 2.0 | 降低难度 |
| 错误率 | > 50% | 插入复习 |
| 帮助请求 | ≥ 3次 | 添加支持内容 |
| 掌握度 | ≥ 0.9 | 提高难度 |

**4. 间隔重复复习**

基于艾宾浩斯遗忘曲线：

```
掌握度 >= 0.9: [7, 14, 30, 60, 120] 天
掌握度 >= 0.8: [3, 7, 14, 30, 60] 天
掌握度 >= 0.7: [1, 3, 7, 14, 30] 天
掌握度 < 0.7:  [1, 2, 4, 7, 14] 天
```

#### 使用示例

```bash
# 生成学习路径
POST /api/v1/learning-path/generate
{
    "learner_id": "learner-001",
    "goal_concepts": ["群", "环", "域"],
    "max_time_hours": 100
}

# 响应
{
    "path_id": "path-001",
    "total_nodes": 15,
    "estimated_hours": 45.5,
    "match_score": 0.85,
    "nodes": [
        {"concept": "集合", "estimated_time": 2},
        {"concept": "二元运算", "estimated_time": 3},
        {"concept": "群", "estimated_time": 8},
        ...
    ]
}
```

### 2.3 评估系统

#### 系统概述

多维度、过程化、发展性的学习评估平台。

#### 四维评价模型

| 维度 | 权重 | 子指标 |
|------|------|--------|
| 知识掌握度 | 30% | 概念理解、定理掌握、方法熟练 |
| 技能熟练度 | 25% | 计算能力、证明能力、建模能力 |
| 问题解决能力 | 25% | 问题分析、策略选择、执行验证 |
| 创新思维能力 | 20% | 发散性思维、创造性、批判性思维 |

#### 评价类型

**1. 综合评价**

综合评估当前能力水平：

```json
{
  "learner_id": "learner-001",
  "assessment_type": "comprehensive",
  "scores": {
    "knowledge": {"concept": 85, "theorem": 80, "method": 75},
    "skill": {"calculation": 78, "proof": 72, "modeling": 70},
    "problem_solving": {"analysis": 82, "strategy": 80, "verification": 85},
    "creative": {"divergent": 75, "creativity": 78, "critical": 72}
  }
}
```

**2. 过程性评价**

基于学习行为数据的持续评估：

```
评价指标：
├── 参与度 (35%): 学习时长、频率
├── 主动性 (25%): 主动学习行为占比
├── 持续性 (20%): 学习规律性
└── 投入度 (20%): 平均学习时长
```

**3. 增值评价**

对比期初期末，量化学习进步：

```
增值指标：
├── 绝对增值: 期末分 - 期初分
├── 相对增值: (期末 - 期初) / 期初 × 100%
├── 增长百分位 (SGP): 同侪排名
└── 增长等级: 快速增长/稳健增长/缓慢增长/停滞/退步
```

#### 评价等级

| 等级 | 分数范围 | 描述 |
|------|----------|------|
| 专家 | 90-100 | 能够创造新知和指导他人 |
| 高级 | 80-89 | 能够灵活应用和迁移知识 |
| 熟练 | 60-79 | 能够独立应用知识和技能 |
| 发展中 | 40-59 | 正在发展理解和应用能力 |
| 初级 | 0-39 | 正在建立基础概念和技能 |

### 4.3 环境配置

**安装 Lean 4**：

```bash
# 使用 elan 安装 Lean
curl https://raw.githubusercontent.com/leanprover/elan/master/elan-init.sh -sSf | sh

# 安装特定版本
elan toolchain install leanprover/lean4:v4.29.0
elan default leanprover/lean4:v4.29.0
```

**安装 Mathlib4**：

```bash
cd docs/09-形式化证明/Lean4/
lake update
lake build
```

### 4.4 运行证明

```bash
# 编译所有证明
lake build

# 运行特定证明
lean LagrangeTheorem.lean

# 在 VS Code 中打开
# 安装 Lean4 扩展后打开项目文件夹
```
