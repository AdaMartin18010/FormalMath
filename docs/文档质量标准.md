## 1. 概述

本文档定义了FormalMath项目中文档质量的严格标准，旨在确保所有数学概念文档具有：

- **完整性**: 包含定义、示例、证明等必需元素
- **准确性**: 数学内容严谨、无误
- **可读性**: 结构清晰、格式规范
- **一致性**: 符合项目整体风格和规范

所有贡献者在提交文档前，必须通过质量检查工具验证。

## 2. 质量检查清单

### 2.1 最小字数要求

| 文档类型 | 最小字节数 | 建议字节数 |
|---------|-----------|-----------|
| 核心概念文档 (concept/) | 2000 | 5000+ |
| 分支文档 (docs/分支/) | 2000 | 4000+ |
| 索引/导航文档 | 500 | 1000+ |
| 报告/总结文档 | 1000 | 2000+ |

**说明**:
- 字节数使用UTF-8编码计算
- 少于最小字节数的文档将被标记为低质量
- 建议使用 `tools/quality_checker.py` 自动检查

**例外情况**:
- 纯索引文件（如README.md）可适当放宽
- 目录文件（仅包含目录列表）需单独评估

### 2.2 必需内容元素

#### A. 核心概念文档 (concept/核心概念/)

必须包含以下章节：

| 序号 | 章节 | 要求 | 说明 |
|-----|------|-----|------|
| 1 | **Frontmatter** | 必须 | 包含 `msc_primary` 字段 |
| 2 | **概述** | 必须 | 简要介绍概念及其重要性 |
| 3 | **严格定义** | 必须 | 形式化数学定义，包含符号表示 |
| 4 | **示例** | 必须 | 至少2个具体示例 |
| 5 | **证明/推导** | 必须 | 至少1个定理的完整证明 |
| 6 | **历史背景** | 推荐 | 发展脉络、关键人物 |
| 7 | **应用实例** | 推荐 | 理论与实际应用 |
| 8 | **关联概念** | 推荐 | 依赖关系、等价关系等 |
| 9 | **参考文献** | 推荐 | 经典教材、研究论文 |

#### B. 分支文档 (docs/XX-分支/)

必须包含以下章节：

| 序号 | 章节 | 要求 | 说明 |
|-----|------|-----|------|
| 1 | **Frontmatter** | 推荐 | 包含 `title`, `msc_primary` 等 |
| 2 | **概述** | 必须 | 主题介绍和范围说明 |
| 3 | **定义** | 必须 | 核心概念的定义 |
| 4 | **定理/性质** | 必须 | 重要定理及其证明 |
| 5 | **示例** | 必须 | 说明性示例 |

#### C. 内容深度要求

**定义要求**:
- 必须包含形式化定义（使用数学符号）
- 建议包含直观解释（L0层次）
- 建议包含等价定义（如适用）

**示例要求**:
- 至少1个简单示例（L0层次）
- 至少1个复杂示例（L1-L2层次）
- 示例应包含完整的解题过程

**证明要求**:
- 至少1个定理的完整证明
- 证明应包含：前提、证明步骤、结论
- 建议使用Lean4代码或自然演绎格式

### 2.3 格式规范

#### A. Frontmatter 规范

**必需字段** (concept/目录):

```yaml
---
msc_primary: "03E99"           # 5位MSC编码
msc_secondary: ["00A05"]       # 可选，次要MSC编码
---
```

**推荐字段** (docs/目录):

```yaml
---
title: "文档标题"
msc_primary: "03E99"
msc_secondary: ["03E30", "03B30"]
processed_at: "2026-04-05"
---
```

**MSC编码格式**:
- 必须是5位格式：2位数字 + 1位大写字母 + 2位数字
- 示例: `03E99`, `11A41`, `55N99`
- 参考: [Mathematics Subject Classification](https://mathscinet.ams.org/mathscinet/msc/msc2020.html)

#### B. 标题层级规范

```markdown
# 一级标题 - 文档主标题（仅1个）

## 二级标题 - 主要章节

### 三级标题 - 子章节

#### 四级标题 - 细节内容
```

**规则**:
- 标题层级必须连续，不得跳跃（如H1直接到H3）
- 一级标题只能有1个
- 标题符号 `#` 后必须有空格

#### C. 数学公式规范

**行内公式**:
```markdown
集合 $A$ 包含元素 $x$，记作 $x \in A$。
```

**块级公式**:
```markdown
$$\forall x \forall y [\forall z(z \in x \leftrightarrow z \in y) \rightarrow x = y]$$
```

**要求**:
- 数学概念文档必须包含数学公式
- 使用标准LaTeX语法
- 重要公式应编号

#### D. 文档结构模板

```markdown
---
msc_primary: "03E99"
---

# 概念名称

**概念编号**: C.CORE.XXX
**知识层次**: L0-L2
**知识领域**: D1 (基础数学)
```

### 2.4 禁止内容

#### A. 占位符和标记

以下内容是**禁止**的：

| 禁止内容 | 示例 | 处理方式 |
|---------|------|---------|
| TODO标记 | `TODO: 待补充` | 完成内容后删除 |
| FIXME标记 | `FIXME: 需要修正` | 立即修正 |
| XXX标记 | `XXX: 检查这里` | 处理后删除 |
| 待补充标记 | `[待补充]` | 补充实际内容 |
| 待完善标记 | `[待完善]` | 完善后删除 |
| 待续标记 | `（待续）` | 完成内容 |

#### B. 空章节

禁止包含只有标题而无内容的章节：

```markdown
<!-- 禁止 -->
## 证明

<!-- 没有内容 -->

## 示例

<!-- 空章节 -->
```

#### C. 重复内容

- 禁止大段复制粘贴相同内容
- 如果需要引用，使用链接或引用标记
- 重复段落长度超过50字符将被标记

#### D. 格式问题

| 问题类型 | 说明 | 修复方法 |
|---------|------|---------|
| 空链接 | `[文本]()` | 补充链接目标或删除 |
| 空标题 | `## ` | 添加标题内容 |
| 过多空行 | 连续3个以上换行 | 减少到最多2个 |
| 标题后无空格 | `##标题` | 改为 `## 标题` |

## 3. 文档评分标准

### 3.1 评分维度

| 维度 | 权重 | 说明 |
|-----|------|------|
| 内容完整性 | 30% | 必需元素是否齐全 |
| 格式规范 | 25% | Frontmatter、标题、格式 |
| 结构完整性 | 25% | 章节组织、层级结构 |
| 内容质量 | 20% | 公式、示例、证明质量 |

### 3.2 质量等级

| 等级 | 分数 | 状态 | 处理建议 |
|-----|------|------|---------|
| 🟢 优秀 | 90-100 | 通过 | 可直接发布 |
| 🟡 良好 | 80-89 | 通过 | 建议微调 |
| 🟠 及格 | 60-79 | 警告 | 需要改进 |
| 🔴 需改进 | 0-59 | 失败 | 必须重写 |

### 3.3 扣分细则

**内容完整性扣分**:
- 文件大小不足: -20分
- 缺少定义部分: -15分
- 缺少示例部分: -10分
- 缺少证明部分: -10分

**格式规范扣分**:
- 缺少Frontmatter: -20分（concept/目录）
- 缺少必需字段: -10分/字段
- MSC格式错误: -5分
- YAML语法错误: -15分

**结构完整性扣分**:
- 无标题结构: -30分
- 标题层级跳跃: -5分/处
- 缺少必需章节: -10分/章节

**内容质量扣分**:
- 无数学公式: -10分
- 检测到TODO等标记: -15分/处
- 检测到重复内容: -10分
- 检测到禁止内容: -15分/处

## 4. 质量检查工具使用

### 4.1 基本使用

```bash
# 检查默认路径（docs/ 和 concept/）
python tools/quality_checker.py

# 检查特定目录
python tools/quality_checker.py --path docs/01-基础数学

# 严格模式检查
python tools/quality_checker.py --strict

# 指定最小文件大小
python tools/quality_checker.py --min-size 3000

# 指定输出报告
python tools/quality_checker.py --output reports/quality_report.md
```

### 4.2 在CI/CD中集成

建议在提交前自动运行质量检查：

```yaml
# .github/workflows/quality-check.yml
name: Document Quality Check

on: [push, pull_request]

jobs:
  quality-check:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      - name: Run quality checker
        run: python tools/quality_checker.py --strict
```

### 4.3 报告解读

质量报告包含以下信息：

1. **统计概览**: 文件总数、平均得分、问题统计
2. **质量评级分布**: 各等级文档数量
3. **低质量文档列表**: 需要优先修复的文档
4. **详细检查结果**: 每个文档的具体问题和建议

## 5. 低质量文档修复指南

### 5.1 修复流程

```
1. 运行质量检查工具
   ↓
2. 查看质量报告，识别低质量文档
   ↓
3. 根据问题类型进行修复
   ↓
4. 重新运行检查，确认问题已解决
   ↓
5. 提交修复后的文档
```

### 5.2 常见问题修复方法

**问题1: 文件大小不足**

解决方法:
- 添加更多示例
- 补充历史背景
- 添加更多应用实例
- 补充相关定理和证明

**问题2: 缺少Frontmatter**

解决方法:
```markdown
---
msc_primary: "03E99"
msc_secondary: ["00A05"]
---

# 原有内容...
```

**问题3: 缺少定义**

解决方法:
```markdown
## 2. 🎯 严格定义

### 2.1 基础定义
**定义**: 设 $X$ 是一个...则称 $X$ 为...

### 2.2 形式化定义
**形式化**: 在ZFC公理系统中...
$$\forall x \phi(x)$$
```

**问题4: 缺少示例**

解决方法:
```markdown
## 6. 💡 应用实例

### 示例1: 简单例子
...

### 示例2: 复杂应用
...
```

**问题5: 缺少证明**

解决方法:
```markdown
## 5. 🔬 形式化证明

**定理**: ...

**证明**:
1. 步骤1...
2. 步骤2...
3. 结论...

证毕。∎
```

### 5.3 批量修复工具

对于大量低质量文档，可以使用批量修复脚本：

```bash
# 批量添加Frontmatter
python tools/fix_doc_format.py --add-frontmatter

# 批量修复格式问题
python tools/fix_doc_format.py --fix-formatting
```
