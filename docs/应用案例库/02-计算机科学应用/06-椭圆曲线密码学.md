# 椭圆曲线密码学的群结构

## 应用领域

**学科**: 密码学 / 代数几何 / 计算数论
**具体应用**: 比特币/区块链、TLS握手、移动设备加密、物联网安全
**MSC分类**: 94A60 (Cryptography), 11G05 (Elliptic curves), 14H52 (Elliptic curves)

---

## 数学概念

### 核心数学工具

1. **椭圆曲线**: Weierstrass方程 $y^2 = x^3 + ax + b$
2. **群结构**: 点加法、无穷远点为单位元
3. **有限域**: $\mathbb{F}_p$、$\mathbb{F}_{2^m}$、$\mathbb{F}_{p^m}$
4. **离散对数问题**: ECDLP的困难性
5. **Hasse定理**: 点数的估计

### 关键定义

- **判别式**: $\Delta = -16(4a^3 + 27b^2) \neq 0$（保证曲线非奇异）
- **阶**: 曲线上点的个数 $\#E(\mathbb{F}_p)$
- **生成元**: 生成子循环群的点

---

## 问题描述

### 椭圆曲线上的离散对数问题(ECDLP)

给定椭圆曲线 $E$ 上的点 $P$ 和 $Q$，找整数 $k$ 使得 $Q = kP$。

**核心性质**: 对于精心选择的曲线，ECDLP比整数分解(DLP)更困难。

### ECC优势

相同安全级别下，ECC使用更短的密钥：

| 安全级别 | RSA密钥 | ECC密钥 | 速度比 |
|----------|---------|---------|--------|
| 80位 | 1024位 | 160位 | ECC ~5× |
| 128位 | 3072位 | 256位 | ECC ~10× |
| 256位 | 15360位 | 512位 | ECC ~50× |

## 数学模型

### 椭圆曲线定义

**Weierstrass方程**（在域 $\mathbb{F}_p$ 上，$p > 3$）:

$$E: y^2 \equiv x^3 + ax + b \pmod{p}$$

**条件**: $4a^3 + 27b^2 \not\equiv 0 \pmod{p}$（非奇异）

### 群运算

**点加法几何**:

给定 $P = (x_1, y_1)$，$Q = (x_2, y_2)$：

**情况1**: $P \neq Q$（点加）
$$\lambda = \frac{y_2 - y_1}{x_2 - x_1}$$

**情况2**: $P = Q$（倍点）
$$\lambda = \frac{3x_1^2 + a}{2y_1}$$

**结果**:
$$x_3 = \lambda^2 - x_1 - x_2$$
$$y_3 = \lambda(x_1 - x_3) - y_1$$

**单位元**: 无穷远点 $\mathcal{O}$

**逆元**: $-(x, y) = (x, -y)$

### Hasse定理

$$|p + 1 - \#E(\mathbb{F}_p)| \leq 2\sqrt{p}$$

曲线点数约为 $p$。

## 求解过程

### 步骤1：标量乘法算法

**目标**: 高效计算 $kP$

**双倍-加算法**（类似平方-乘）:

```

scalar_mult(k, P):
    Q = O
    while k > 0:
        if k is odd:
            Q = Q + P
        P = 2P  # 倍点
        k = k / 2
    return Q

```

复杂度: $O(\log k)$ 次点加

### 步骤2：曲线选择标准

**安全要求**:

1. **大素数阶**: $\#E(\mathbb{F}_p)$ 包含大素因子
2. **非异常**: $\#E \neq p$（避免Smart攻击）
3. **MOV条件**: $\#E$ 不整除 $p^k - 1$ 对小 $k$（避免MOV攻击）
4. **随机性**: 系数应随机选择且可验证

**标准曲线**:

- NIST曲线（P-192, P-224, P-256, P-384, P-521）
- Curve25519/Ed25519（现代推荐）
- secp256k1（比特币使用）

### 步骤3：ECDH密钥交换

**公共参数**: 曲线 $E$，生成元 $G$，阶 $n$

**Alice**:

- 选择私钥 $a \in [1, n-1]$
- 计算公钥 $A = aG$

**Bob**:

- 选择私钥 $b \in [1, n-1]$
- 计算公钥 $B = bG$

**共享密钥**:

- Alice计算: $S = aB = a(bG) = (ab)G$
- Bob计算: $S = bA = b(aG) = (ab)G$

**安全性**: 从 $A$ 或 $B$ 求 $a$ 或 $b$ 即ECDLP。

### 步骤4：ECDSA签名

**密钥生成**: 私钥 $d$，公钥 $Q = dG$

**签名消息 $m$**:

1. 计算 $e = \text{hash}(m)$
2. 选择随机数 $k \in [1, n-1]$
3. 计算 $R = kG = (x_1, y_1)$
4. 计算 $r = x_1 \mod n$，若 $r = 0$ 重来
5. 计算 $s = k^{-1}(e + dr) \mod n$，若 $s = 0$ 重来
6. 签名为 $(r, s)$

**验证**:

1. 验证 $r, s \in [1, n-1]$
2. 计算 $e = \text{hash}(m)$
3. 计算 $w = s^{-1} \mod n$
4. 计算 $u_1 = ew \mod n$，$u_2 = rw \mod n$
5. 计算 $R = u_1 G + u_2 Q$
6. 若 $R = \mathcal{O}$ 拒绝
7. 验证 $x_R \mod n = r$

## 结果分析

### ECDSA安全性

**安全假设**: ECDLP在选定曲线上困难。

**最佳已知算法**:

- Pollard's rho: $O(\sqrt{n})$，并行化
- 对于256位曲线，约需 $2^{128}$ 次运算

### 实现考量

| 问题 | 解决方案 |
|------|----------|
| **侧信道攻击** | 常数时间标量乘法 |
| **无效曲线攻击** | 验证输入点在曲线上 |
| **小子群攻击** | 验证点在正确子群中 |
| **随机数重用** | 使用RFC 6979确定性签名 |

### Curve25519优势

**设计原则** (Daniel Bernstein):

1. **抵抗时序攻击**: 所有运算恒定时间
2. **简化实现**: 无需验证输入点
3. **无陷阱门**: 系数可验证随机
4. **高性能**: Montgomery ladder标量乘法

**方程**: $y^2 = x^3 + 486662x^2 + x$ 在 $\mathbb{F}_{2^{255}-19}$
