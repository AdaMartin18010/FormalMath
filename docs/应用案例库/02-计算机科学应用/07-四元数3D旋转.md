# 3D渲染中的四元数旋转

## 应用领域

**学科**: 计算机图形学 / 计算几何
**具体应用**: 3D游戏引擎、航空航天模拟、骨骼动画、VR/AR
**MSC分类**: 68U05 (Computer graphics), 15A66 (Clifford algebras), 53A04 (Curves in Euclidean space)

---

## 数学概念

### 核心数学工具

1. **四元数代数**: $\mathbb{H}$，非交换除环
2. **单位四元数**: 3D旋转的表示
3. **球面线性插值(SLERP)**: 平滑旋转插值
4. **欧拉角与万向节锁**: 旋转表示的局限性
5. **旋转矩阵**: 3×3正交矩阵

### 关键定义

- **四元数**: $q = w + xi + yj + zk = w + \mathbf{v}$
- **共轭**: $q^* = w - xi - yj - zk$
- **模**: $|q| = \sqrt{qq^*} = \sqrt{w^2 + x^2 + y^2 + z^2}$

## 问题描述

### 3D旋转表示的挑战

**欧拉角的问题**:

1. **万向节锁(Gimbal Lock)**: 失去一个旋转自由度
2. **顺序依赖**: 不同旋转顺序产生不同结果
3. **插值困难**: 直接插值欧拉角产生非自然旋转路径

**旋转矩阵的问题**:

1. **存储开销**: 9个参数（只有3个自由度）
2. **数值漂移**: 累积误差使矩阵不正交
3. **插值复杂**: 矩阵插值不保持正交性

### 四元数解决方案

- **无万向节锁**
- **紧凑表示**（4个参数）
- **平滑插值**（SLERP）
- **数值稳定**

## 数学模型

### 四元数基础

**定义**: $q = w + xi + yj + zk$，其中 $i^2 = j^2 = k^2 = ijk = -1$

**乘法规则**:

- $ij = k$, $ji = -k$
- $jk = i$, $kj = -i$
- $ki = j$, $ik = -j$

**乘法公式**:

设 $q_1 = w_1 + \mathbf{v}_1$, $q_2 = w_2 + \mathbf{v}_2$:

$$q_1 q_2 = (w_1 w_2 - \mathbf{v}_1 \cdot \mathbf{v}_2) + (w_1 \mathbf{v}_2 + w_2 \mathbf{v}_1 + \mathbf{v}_1 \times \mathbf{v}_2)$$

### 旋转表示

**轴角到四元数**: 绕单位轴 $\mathbf{u} = (u_x, u_y, u_z)$ 旋转角度 $\theta$

$$q = \cos\frac{\theta}{2} + \sin\frac{\theta}{2}(u_x i + u_y j + u_z k)$$

**单位四元数**: $|q| = 1$，即 $\cos^2\frac{\theta}{2} + \sin^2\frac{\theta}{2} = 1$ ✓

### 旋转向量

对向量 $\mathbf{p} = (p_x, p_y, p_z)$，表示为纯虚四元数 $p = 0 + p_x i + p_y j + p_z k$。

**旋转公式**:

$$\boxed{p' = q p q^{-1} = q p q^*}$$

（对单位四元数，$q^{-1} = q^*$）

**展开**:

$$p' = p_{\parallel} + p_{\perp}'$$

其中:

- $p_{\parallel} = (\mathbf{p} \cdot \mathbf{u})\mathbf{u}$（平行于旋转轴的分量，不变）
- $p_{\perp}' = \cos\theta \, p_{\perp} + \sin\theta \, (\mathbf{u} \times \mathbf{p})$（垂直分量旋转）

### 四元数到旋转矩阵

$$R = \begin{pmatrix}
1 - 2(y^2 + z^2) & 2(xy - wz) & 2(xz + wy) \\
2(xy + wz) & 1 - 2(x^2 + z^2) & 2(yz - wx) \\
2(xz - wy) & 2(yz + wx) & 1 - 2(x^2 + y^2)
\end{pmatrix}$$

## 求解过程

### 步骤1：四元数乘法实现

```

quat_mult(q1, q2):
    w = q1.w*q2.w - q1.x*q2.x - q1.y*q2.y - q1.z*q2.z
    x = q1.w*q2.x + q1.x*q2.w + q1.y*q2.z - q1.z*q2.y
    y = q1.w*q2.y - q1.x*q2.z + q1.y*q2.w + q1.z*q2.x
    z = q1.w*q2.z + q1.x*q2.y - q1.y*q2.x + q1.z*q2.w
    return (w, x, y, z)

```

12次乘法，8次加法。

### 步骤2：向量旋转实现

```

rotate_vector(q, v):
    // q必须是单位四元数
    qv = (0, v.x, v.y, v.z)
    q_conj = (q.w, -q.x, -q.y, -q.z)
    result = quat_mult(quat_mult(q, qv), q_conj)
    return (result.x, result.y, result.z)

```

约30次乘法（可优化）。

### 步骤3：球面线性插值(SLERP)

**问题**: 在 $q_1$ 和 $q_2$ 之间平滑插值参数 $t \in [0, 1]$。

**直接插值的问题**: $q(t) = (1-t)q_1 + tq_2$ 不保持单位长度，且速度不均匀。

**SLERP公式**:

设 $\omega$ 为 $q_1$ 和 $q_2$ 之间的夹角： $\cos\omega = q_1 \cdot q_2$

$$\text{SLERP}(q_1, q_2, t) = \frac{\sin((1-t)\omega)}{\sin\omega} q_1 + \frac{\sin(t\omega)}{\sin\omega} q_2$$

**常数速度**: 在单位球面上沿大圆弧匀速移动。

### 步骤4：四元数规范化

数值累积误差后，四元数可能偏离单位长度：

```

normalize(q):
    norm = sqrt(q.w^2 + q.x^2 + q.y^2 + q.z^2)
    return (q.w/norm, q.x/norm, q.y/norm, q.z/norm)

```

## 结果分析

### 表示效率比较

| 表示方法 | 参数数 | 存储 | 插值 | 万向节锁 |
|----------|--------|------|------|----------|
| **欧拉角** | 3 | 12字节 | 困难 | 有 |
| **旋转矩阵** | 9 (3 DOF) | 36字节 | 复杂 | 无 |
| **轴角** | 4 (3+1) | 16字节 | 困难 | 无 |
| **四元数** | 4 (1约束) | 16字节 | SLERP | 无 |

### 游戏引擎应用

**Unity**:
- `Quaternion` 类是主要旋转表示
- `Quaternion.Slerp()` 用于平滑旋转
- `transform.rotation` 内部使用四元数

**Unreal Engine**:
- `FQuat` 类
- 动画系统广泛使用SLERP

**骨骼动画**:
- 关节旋转用四元数存储
- 关键帧间SLERP插值
- 比欧拉角更自然的关节运动

### 航天应用

**姿态控制**:
- 卫星姿态确定与控制系统(ADCS)
- 避免万向节锁的姿态机动规划
- 四元数作为状态变量进行滤波
