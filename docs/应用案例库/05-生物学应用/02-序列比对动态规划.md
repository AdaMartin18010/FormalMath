# 序列比对的动态规划算法

## 应用领域

**学科**: 生物信息学 / 计算生物学  
**具体应用**: DNA序列比对、蛋白质结构预测、进化分析、基因组注释  
**MSC分类**: 92D20 (Protein sequences, DNA sequences), 68W40 (Analysis of algorithms), 90C39 (Dynamic programming)

---

## 数学概念

### 核心数学工具

1. **动态规划**: 最优子结构、重叠子问题
2. **编辑距离**: Levenshtein距离
3. **计分矩阵**: 替换矩阵（PAM、BLOSUM）
4. **仿射间隙罚分**: 间隙开启与延伸的不同惩罚
5. **回溯**: 重建最优比对

### 关键定义

- **序列比对**: 在两个序列间插入间隙使相似性最大化
- **全局比对**: 比对整个序列
- **局部比对**: 寻找最相似子串

---

## 问题描述

### 生物学动机

**序列相似性暗示**: 
- 功能相似性
- 进化关系（同源）
- 结构相似性

**核心问题**: 给定两个序列，找到最优比对。

### 比对类型

**全局比对**（Needleman-Wunsch）:
- 比对整个序列
- 适合相似度高的序列

**局部比对**（Smith-Waterman）:
- 寻找最佳匹配子串
- 适合相似度低的序列

---

## 数学模型

### 编辑距离模型

**操作**:
- **匹配**: $a_i = b_j$，得分 $+s(a_i, b_j)$
- **错配**: $a_i \neq b_j$，得分 $s(a_i, b_j)$（可能为负）
- **插入/删除(间隙)**: 得分 $-d$

**目标**: 最大化比对得分

### Needleman-Wunsch算法（全局比对）

**动态规划矩阵** $F(i, j)$: 序列 $a[1..i]$ 和 $b[1..j]$ 的最优比对得分

**递推关系**:

$$F(i, j) = \max\begin{cases}
F(i-1, j-1) + s(a_i, b_j) & \text{匹配/错配} \\
F(i-1, j) - d & \text{删除（在$a$中插入间隙）} \\
F(i, j-1) - d & \text{插入（在$b$中插入间隙）}
\end{cases}$$

**边界条件**:
- $F(0, 0) = 0$
- $F(i, 0) = -i \cdot d$（在$b$中插入$i$个间隙）
- $F(0, j) = -j \cdot d$（在$a$中插入$j$个间隙）

**复杂度**: $O(mn)$ 时间和空间，其中 $m, n$ 是序列长度。

### Smith-Waterman算法（局部比对）

**修改递推**:

$$F(i, j) = \max\begin{cases}
0 & \text{（重新开始）} \\
F(i-1, j-1) + s(a_i, b_j) \\
F(i-1, j) - d \\
F(i, j-1) - d
\end{cases}$$

**结果**: 最大得分可能来自任意子串对比对。

### 仿射间隙罚分

**更现实模型**: 开启间隙比延伸更昂贵

- **间隙开启**: $-d_{open}$
- **间隙延伸**: $-d_{extend}$（$d_{extend} < d_{open}$）

**修改递推**（需要三个矩阵）:

$$M(i, j) = \max\begin{cases}
M(i-1, j-1) + s(a_i, b_j) \\
I_x(i-1, j-1) + s(a_i, b_j) \\
I_y(i-1, j-1) + s(a_i, b_j)
\end{cases}$$

$$I_x(i, j) = \max\begin{cases}
M(i-1, j) - d_{open} \\
I_x(i-1, j) - d_{extend}
\end{cases}$$

$$I_y(i, j) = \max\begin{cases}
M(i, j-1) - d_{open} \\
I_y(i, j-1) - d_{extend}
\end{cases}$$

**复杂度**: 仍为 $O(mn)$，但常数因子增大。

## 求解过程

### 步骤1：选择计分矩阵

**DNA比对**: 简单计分
- 匹配: +2
- 错配: -1
- 间隙: -2

**蛋白质比对**: 替换矩阵（考虑氨基酸理化性质）

| 矩阵 | 基础 | 适用 |
|------|------|------|
| **PAM** | 进化模型 | 远缘蛋白 |
| **BLOSUM** | 保守块 | 近缘蛋白 |
| **BLOSUM62** | 62%相似性聚类 | 默认选择 |

### 步骤2：填充动态规划矩阵

```

for i = 1 to m:
    for j = 1 to n:
        match = F[i-1][j-1] + score(a[i], b[j])
        delete = F[i-1][j] - gap_penalty
        insert = F[i][j-1] - gap_penalty
        F[i][j] = max(match, delete, insert)
        # 记录哪个选择产生最大值（用于回溯）
        traceback[i][j] = argmax

```

### 步骤3：回溯重建比对

```

alignment_a = ""
alignment_b = ""
i, j = m, n
while i > 0 or j > 0:
    if traceback[i][j] == "match":
        alignment_a = a[i] + alignment_a
        alignment_b = b[j] + alignment_b
        i, j = i-1, j-1
    elif traceback[i][j] == "delete":
        alignment_a = a[i] + alignment_a
        alignment_b = "-" + alignment_b
        i = i-1
    else:  # insert
        alignment_a = "-" + alignment_a
        alignment_b = b[j] + alignment_b
        j = j-1

```

### 步骤4：空间优化

**Hirschberg算法**: 线性空间 $O(m+n)$ 的比对算法

**思想**: 分治法，找到中间行的最优分割点

**复杂度**: 
- 时间: $O(mn)$（常数因子约2）
- 空间: $O(m+n)$

## 结果分析

### 比对得分解释

**原始得分**: 依赖于计分矩阵和序列长度

**归一化得分**:
- **位得分**: $\frac{\lambda S - \ln K}{\ln 2}$（bits）
- **E值**: 期望在数据库中随机找到该得分或更高的比对数

$$E = Kmn e^{-\lambda S}$$

$E < 0.01$: 比对显著

### BLAST启发式

**问题**: 对长序列和大数据库，$O(mn)$ 太慢

**BLAST策略**:
1. 寻找高得分短词（种子）
2. 向两端延伸
3. 仅对高得分区域进行动态规划

**速度提升**: 100-1000倍，牺牲少量敏感性。

### 多序列比对

**扩展**: 同时比对 $k > 2$ 条序列

**复杂性**: NP-hard（精确算法指数时间）

**启发式方法**:
- **渐进式**: ClustalW（先两两比对，再逐步合并）
- **迭代式**: MUSCLE, MAFFT
