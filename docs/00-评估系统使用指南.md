## 1. 系统概述

### 1.1 系统简介

FormalMath 评估系统是一个多维度、过程化、发展性的学习评估平台，专为数学学习设计。系统提供：

- **多维度评价**：知识掌握度、技能熟练度、问题解决能力、创新思维能力
- **过程性评价**：跟踪学习过程，评估参与度和行为
- **增值评价**：对比期初期末，量化学习进步
- **实时反馈**：即时反馈机制，错误分析引擎
- **报告系统**：个人/群体报告，多格式导出

### 1.2 系统架构

## 2. 快速开始

### 2.1 环境要求

- Python 3.8+
- Node.js 16+
- PostgreSQL (可选，默认使用SQLite)

### 2.2 后端安装与启动

```bash
# 进入后端目录
cd project/assessment-system/backend

# 创建虚拟环境
python -m venv venv

# 激活虚拟环境
# Windows:
venv\Scripts\activate
# macOS/Linux:
source venv/bin/activate

# 安装依赖
pip install -r requirements.txt

# 启动服务
python main.py
```

服务将启动在 `http://localhost:8000`

API文档地址：`http://localhost:8000/docs`

### 2.3 前端安装与启动

```bash
# 进入前端目录
cd project/assessment-system/frontend

# 安装依赖
npm install

# 启动开发服务器
npm run dev
```

前端将启动在 `http://localhost:3000`

### 2.4 快速测试

```bash
# 测试API是否正常运行
curl http://localhost:8000/api/v1/system/health

# 创建测试学习者
curl -X POST http://localhost:8000/api/v1/learners \
  -H "Content-Type: application/json" \
  -d '{"name": "测试用户", "email": "test@example.com"}'
```


#### 更新知识状态

```http
PUT /api/v1/learners/{learner_id}/knowledge
Content-Type: application/json

{
  "集合论基础": 90,
  "函数与映射": 85,
  "数理逻辑": 70
}
```

### 3.2 评估功能

#### 综合评价

四维评价模型，包含：

- **知识掌握度** (30%): 概念理解、定理掌握、方法熟练
- **技能熟练度** (25%): 计算能力、证明能力、建模能力
- **问题解决能力** (25%): 问题分析、策略选择、执行验证
- **创新思维能力** (20%): 发散性、创造性、批判性

```http
POST /api/v1/evaluations/comprehensive
Content-Type: application/json

{
  "learner_id": "learner-uuid",
  "assessment_type": "comprehensive",
  "scores": {
    "knowledge": {
      "concept_understanding": 85,
      "theorem_mastery": 80,
      "method_proficiency": 75
    },
    "skill": {
      "calculation": 78,
      "proof": 72,
      "modeling": 70
    },
    "problem_solving": {
      "problem_analysis": 82,
      "strategy_selection": 80,
      "execution_verification": 85
    },
    "creative": {
      "divergent_thinking": 75,
      "creativity": 78,
      "critical_thinking": 72
    }
  }
}
```

#### 过程性评价

基于学习行为数据进行评价：

```http
POST /api/v1/evaluations/process?learner_id={learner_id}&period_days=7
```

评价指标：

- 参与度 (35%): 学习时长、频率
- 主动性 (25%): 主动学习行为占比
- 持续性 (20%): 学习规律性和稳定性
- 投入度 (20%): 平均学习时长、互动深度

#### 增值评价

对比期初和期末数据：

```http
POST /api/v1/evaluations/value-added?learner_id={learner_id}&period_days=30
```

返回数据：

- 绝对增值: 期末分 - 期初分
- 相对增值: (期末分 - 期初分) / 期初分 × 100%
- 增长百分位 (SGP): 在同侪中的百分位排名
- 增长等级: 退步/停滞/缓慢增长/稳健增长/快速增长

### 3.3 学习行为记录

```http
POST /api/v1/behaviors
Content-Type: application/json

{
  "learner_id": "learner-uuid",
  "behavior_type": "content_view",
  "duration": 300,
  "content_id": "concept-set-theory",
  "metadata": {
    "page": "定义与例子",
    "scroll_depth": 0.8
  }
}
```

行为类型：

- `content_view`: 内容查看
- `video_watch`: 视频观看
- `exercise_start`: 练习开始
- `exercise_submit`: 练习提交
- `note_create`: 笔记创建
- `search`: 搜索
- `discussion`: 讨论参与
- `reflection`: 反思记录

### 3.4 反馈系统

#### 获取反馈列表

```http
GET /api/v1/feedback/{learner_id}?unread_only=true&limit=10
```

#### 实时反馈

```http
POST /api/v1/feedback/real-time
Content-Type: application/json

{
  "learner_id": "learner-uuid",
  "consecutive_errors": 3,
  "consecutive_correct": 0,
  "time_spent": 600,
  "hint_used": false
}
```

触发条件：

- 连续错误 ≥ 3次：提示回顾知识点
- 用时 > 10分钟：建议使用提示
- 连续正确 ≥ 5次：鼓励性反馈

#### 错误分析

```http
POST /api/v1/feedback/error-analysis
Content-Type: application/json

{
  "learner_id": "learner-uuid",
  "question_id": "question-123",
  "user_answer": "错误答案",
  "correct_answer": "正确答案",
  "concept_id": "concept-algebra"
}
```

错误类型：

- 概念理解错误
- 计算错误
- 逻辑错误
- 方法选择错误
- 粗心错误

### 3.5 报告生成

#### 生成个人报告

```http
POST /api/v1/reports/generate
Content-Type: application/json

{
  "learner_id": "learner-uuid",
  "report_type": "personal_learning",
  "period_days": 30,
  "format": "html"

```

#### 生成群体报告


#### 报告格式

支持格式：

- `json`: JSON数据格式

## 6. 常见问题

### Q1: 如何初始化数据库？

系统首次启动时会自动创建所有数据表。如需重置数据：

```python
from models import DatabaseManager
db = DatabaseManager("sqlite:///./assessment.db")
db.drop_tables()  # 删除所有表
db.create_tables()  # 重新创建
```

### Q2: 如何修改评价权重？

在 `evaluation_engine.py` 中修改权重配置：

```python
class AssessmentEngine:
    DIMENSION_WEIGHTS = {
        "knowledge_mastery": 0.30,  # 修改此处
        "skill_proficiency": 0.25,
        "problem_solving": 0.25,
        "creative_thinking": 0.20
    }
```

### Q3: 如何添加新的评价维度？

1. 在 `evaluation_engine.py` 中添加新维度计算方法
2. 更新前端评估表单
3. 修改数据库模型（如需存储新维度）

### Q4: 报告导出失败怎么办？

- 检查文件权限
- 确认存储目录存在且可写
- 查看后端日志获取详细错误信息

### Q5: 如何对接外部系统？

使用标准REST API进行对接：

```python
import requests

# 创建学习者
response = requests.post(
    "http://localhost:8000/api/v1/learners",
    json={"name": "外部用户", "email": "user@external.com"}
)
learner_id = response.json()["id"]

# 提交评价
evaluation = requests.post(
    "http://localhost:8000/api/v1/evaluations/comprehensive",
    json={
        "learner_id": learner_id,
        "assessment_type": "comprehensive",
        "scores": {...}
    }
)
```

## 附录

### A. 评价等级说明

| 等级 | 分数范围 | 描述 |
|------|----------|------|
| 专家 | 90-100 | 能够创造新知和指导他人 |
| 高级 | 80-89 | 能够灵活应用和迁移知识 |
| 熟练 | 60-79 | 能够独立应用知识和技能 |
| 发展中 | 40-59 | 正在发展理解和应用能力 |
| 初级 | 0-39 | 正在建立基础概念和技能 |

### B. 增值等级说明

| 等级 | 绝对增值 | 描述 |
|------|----------|------|
| 快速增长 | > 30 | 学习效果显著 |
| 稳健增长 | 15-30 | 持续稳定进步 |
| 缓慢增长 | 5-15 | 有进步但较慢 |
| 停滞 | -5 ~ 5 | 基本持平 |
| 退步 | < -5 | 需要关注 |

### C. 反馈优先级

- **高优先级**: 需要立即关注的问题
- **中优先级**: 建议近期处理的问题
- **低优先级**: 一般性提示和鼓励
