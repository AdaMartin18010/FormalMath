# 可计算性理论实战问题

## 一、归约构造

### 问题描述

构造从已知不可判定问题到新问题的多项式时间归约。

### 从3-SAT到顶点覆盖

**问题**: 证明顶点覆盖是NP完全的。

**归约**: 3-SAT ≤_p VERTEX-COVER

**构造**:

给定3-CNF公式φ = C₁ ∧ C₂ ∧ ... ∧ C_m，其中每个C_i = (l_{i1} ∨ l_{i2} ∨ l_{i3})。

**图G构造**:

1. **变量小工具**: 对每个变量x_i，添加边 (x_i, ¬x_i)
2. **子句小工具**: 对每个子句C_j = (a ∨ b ∨ c)，添加三角形 {a, b, c}
3. **连接**: 将子句中的文字连接到对应的变量节点

**证明**:

- φ可满足 ⟺ G有大小为n+2m的顶点覆盖

### 从停机问题到Post对应问题

**问题**: 证明Post对应问题(PCP)是不可判定的。

**归约**: HALT_TM ≤_m PCP

**思路**:

1. 模拟图灵机的计算历史
2. 将配置编码为字符串
3. PCP有解当且仅当计算历史有效且停机

### 实战技巧

**归约设计原则**:

1. 保持问题结构对应
2. 确保多项式时间构造
3. 验证等价性（双向）

## 二、NP完全性证明

### 问题：证明子集和问题是NP完全的

### 证明结构

**第一部分**: SUBSET-SUM ∈ NP

- 证书: 子集本身
- 验证器: 检查子集和是否等于目标值
- 多项式时间: O(n)

**第二部分**: 3-SAT ≤_p SUBSET-SUM

**构造**:

给定3-CNF φ，构造子集和实例：

1. 对每个变量x_i，创建两个数v_i和v'_i
2. 对每个子句C_j，创建两个数s_j和s'_j
3. 目标值编码每个子句必须满足

**关键**: 用十进制编码确保每个变量选一个值，每个子句至少满足一次。

### 更多NP完全问题证明

**哈密尔顿路径**:

- 从3-SAT归约
- 使用专用小工具(gadget)表示变量和子句

**图着色**:

- 从3-SAT归约
- 每个变量对应三角形的一个顶点
- 子句对应额外的约束结构

## 三、近似算法分析

### 问题：顶点覆盖的近似算法

### 2-近似算法

**算法**:

```
APPROX-VERTEX-COVER(G):
  C = ∅
  E' = G.E
  while E' ≠ ∅:
    取任意边 (u,v) ∈ E'
    C = C ∪ {u, v}
    从E'删除所有与u或v关联的边
  return C
```

**分析**:

1. C是顶点覆盖（显然）
2. 设A为算法选择的边集合
3. |C| = 2|A|
4. A是匹配，任何顶点覆盖至少需要|A|个顶点
5. 因此 |C| ≤ 2|OPT|

### 不可近似性结果

**定理**: 若P ≠ NP，则不存在(2-ε)-近似算法对顶点覆盖。

**更强的**: 某些问题（如团）甚至不存在任何常数因子近似。

### 实战：集合覆盖

**贪心算法**:

```
GREEDY-SET-COVER(X, F):
  U = X
  C = ∅
  while U ≠ ∅:
    选择S ∈ F最大化|S ∩ U|
    U = U - S
    C = C ∪ {S}
  return C
```

**近似比**: H_n（调和数），即O(log n)

**紧性**: 除非P = NP，这是最优的。

## 四、P vs NP问题分析

### 问题：P vs NP的研究进展

### 已知结果

**包含关系**:

```
P ⊆ NP ⊆ PSPACE ⊆ EXPTIME
```

**真包含**: P ⊊ EXPTIME

**开放问题**:

- P = NP?
- NP = PSPACE?
- PSPACE = EXPTIME?

### 研究路线

**对角化**:

- 证明时间层次定理
- 但似乎不能直接证明P ≠ NP
- 相对化障碍（Baker-Gill-Solovay）

**电路复杂性**:

- 尝试证明NP ∉ P/poly
- 自然证明障碍（Razborov-Rudich）

**几何复杂性**:

- Mulmuley-Sohoni的GCT计划
- 长期研究项目

### 若P = NP的影响

**正面**:

- 高效优化问题解决
- 密码学突破
- 数学定理自动发现

**负面**:

- 现代密码系统崩溃
- 隐私安全威胁

## 五、可计算性理论应用

### 问题：Rice定理的应用

### 实战例子

**问题**: 判断程序是否计算常值函数是否可判定？

**解答**:

1. 性质P = {L : L是单元素集或空集}
2. P是非平凡的（存在满足和不满足的程序）
3. 由Rice定理，问题不可判定

**问题**: 判断程序是否对所有输入停机是否可判定？

**解答**:

1. 性质P = {L : L是全函数}
2. P是非平凡的
3. 由Rice定理，不可判定

### 自我引用程序

**问题**: 构造打印自身的程序

**解答** (Quine):

```python
s = 's = %r\nprint(s %% s)'
print(s % s)
```

**递归定理**: 这样的程序总是存在。
