## 📋 概述

密码学是研究安全通信的数学科学，其核心建立在计算复杂性理论、数论和代数几何的严格数学基础之上。现代密码学从信息论安全演进为计算安全，强调在计算资源受限假设下的安全性证明。

## 一、计算复杂性基础

### 1.1 P vs NP与密码学

**定义 1.1（复杂性类）**:

- $\mathbf{P}$: 确定性图灵机在多项式时间内可判定的语言类
- $\mathbf{NP}$: 非确定性图灵机在多项式时间内可判定的语言类
- $\mathbf{BPP}$: 概率图灵机在多项式时间内以有界错误概率可判定的语言类

**猜想 1.2（P ≠ NP）**:
$$
\mathbf{P} \neq \mathbf{NP}
$$

这是现代密码学的根本假设。若$\mathbf{P} = \mathbf{NP}$，则所有基于NP困难问题的密码系统都将不安全。

**定义 1.3（单向函数）**:
函数$f: \{0,1\}^* \to \{0,1\}^*$称为**单向函数**，若满足：

1. **易计算**: 存在多项式时间算法计算$f(x)$
2. **难逆**: 对所有PPT（概率多项式时间）算法$\mathcal{A}$，存在可忽略函数$\mathsf{negl}$使得：

$$
\Pr_{x \leftarrow \{0,1\}^n}[\mathcal{A}(f(x), 1^n) \in f^{-1}(f(x))] \leq \mathsf{negl}(n)
$$

**定理 1.4（单向函数存在蕴含P ≠ NP）**:
若单向函数存在，则$\mathbf{P} \neq \mathbf{NP}$。

*证明概要*: 若$\mathbf{P} = \mathbf{NP}$，则对任意$y = f(x)$，可在多项式时间内找到前像$x'$。∎

### 1.2 困难问题假设

**定义 1.5（离散对数问题 - DLP）**:
设$\mathbb{G}$为阶为$q$的循环群，生成元为$g$。给定$h = g^x$，求$x \in \mathbb{Z}_q$称为离散对数问题。

**定义 1.6（计算Diffie-Hellman假设 - CDH）**:
给定$(g, g^a, g^b)$，计算$g^{ab}$在计算上是困难的。

**定义 1.7（判定Diffie-Hellman假设 - DDH）**:
区分$(g, g^a, g^b, g^{ab})$与$(g, g^a, g^b, g^c)$（其中$c$随机）在计算上是困难的。

**定义 1.8（整数分解假设）**:
给定合数$N = pq$（$p,q$为大素数），分解$N$在计算上是困难的。

## 二、对称加密的数学结构

### 2.1 完美保密性

**定义 2.1（香农完美保密）**:
加密方案$(\mathsf{Gen}, \mathsf{Enc}, \mathsf{Dec})$具有**完美保密性**，若对所有消息$m_0, m_1 \in \mathcal{M}$和所有密文$c \in \mathcal{C}$：
$$
\Pr[C = c \mid M = m_0] = \Pr[C = c \mid M = m_1]
$$

**定理 2.2（香农定理）**:
完美保密加密满足$|\mathcal{K}| \geq |\mathcal{M}|$，且一次一密(OTP)是达到完美保密的充要条件。

**一次一密(OTP)构造**:

- 密钥空间: $\mathcal{K} = \{0,1\}^n$
- 消息空间: $\mathcal{M} = \{0,1\}^n$
- 加密: $c = m \oplus k$
- 解密: $m = c \oplus k$

### 2.2 Feistel网络

**定义 2.3（Feistel网络）**:
设$F: \{0,1\}^n \times \{0,1\}^n \to \{0,1\}^n$为轮函数。$r$轮Feistel网络$\Psi_F^r$定义如下：

对输入$(L_0, R_0) \in \{0,1\}^{2n}$和第$i$轮密钥$k_i$：
$$
\begin{aligned}
L_i &= R_{i-1} \\
R_i &= L_{i-1} \oplus F(R_{i-3}, k_i)
\end{aligned}
$$

**定理 2.4（Luby-Rackoff）**:
若$F$是伪随机函数(PRF)，则3轮Feistel网络是伪随机置换(PRP)。

### 2.3 代换-置换网络(SPN)

**定义 2.5（SPN结构）**:
SPN由以下组件构成：

1. **代换层(S-box)**: 非线性变换$\pi_S: \{0,1\}^n \to \{0,1\}^n$
2. **置换层(P-box)**: 线性变换$\pi_P: \{0,1\}^{mn} \to \{0,1\}^{mn}$
3. **密钥混合**: 轮密钥加

**安全性分析**:
SPN的安全性依赖于：

- **混淆(Confusion)**: S-box提供的非线性
- **扩散(Diffusion)**: P-box提供的比特混合

## 三、RSA的数论基础

### 3.1 Euler定理与模幂运算

**定义 3.1（Euler函数）**:
$$
\varphi(n) = |\{a \in \mathbb{Z}_n^* : \gcd(a,n) = 1\}|

$$

对$n = pq$（$p,q$为不同素数）：
$$
\varphi(n) = (p-1)(q-1)
$$

**定理 3.2（Euler定理）**:
若$\gcd(a,n) = 1$，则：
$$
a^{\varphi(n)} \equiv 1 \pmod{n}
$$

**定理 3.3（RSA正确性）**:
设$ed \equiv 1 \pmod{\varphi(n)}$，则对所有$m \in \mathbb{Z}_n^*$：
$$
(m^e)^d \equiv m \pmod{n}
$$

*证明*:
由$ed = 1 + k\varphi(n)$，有：
$$
(m^e)^d = m^{ed} = m^{1+k\varphi(n)} = m \cdot (m^{\varphi(n)})^k \equiv m \cdot 1^k = m \pmod{n}
$$
∎

### 3.2 RSA的安全性分析

**安全假设**:

- **RSA假设**: 给定$(N, e, m^e \bmod N)$，计算$m$是困难的
- **与分解的关系**: RSA $\Leftarrow$ FACTORING（但等价性未证明）

**攻击方法**:

1. **直接分解**: 使用GNFS（一般数域筛法），复杂度$L_N[1/3, c]$
2. **共模攻击**: 同一$N$不同$e$可恢复明文
3. **低指数攻击**: 小$e$时广播攻击有效
4. **侧信道攻击**: 时序、功耗分析

**安全参数建议**（NIST SP 800-57）:

| 安全级别 | 模数长度 |
|---------|---------|
| 80-bit | 1024-bit（不推荐） |
| 112-bit | 2048-bit |
| 128-bit | 3072-bit |
| 256-bit | 15360-bit |

## 四、椭圆曲线密码学

### 4.1 椭圆曲线的代数结构

**定义 4.1（Weierstrass方程）**:
定义在域$\mathbb{F}_q$上的椭圆曲线：
$$
E: y^2 = x^3 + ax + b
$$
其中$4a^3 + 27b^2 \neq 0$（非奇异条件）。

**定义 4.2（椭圆曲线点群）**:
点集$E(\mathbb{F}_q) = \{(x,y) \in \mathbb{F}_q^2 : y^2 = x^3 + ax + b\} \cup \{\mathcal{O}\}$构成Abel群，群运算定义为弦切法则。

**群运算公式**:
设$P = (x_1, y_1)$, $Q = (x_2, y_2)$，$P \neq Q$：
$$
\lambda = \frac{y_2 - y_1}{x_2 - x_1}, \quad x_3 = \lambda^2 - x_1 - x_2, \quad y_3 = \lambda(x_1 - x_3) - y_1
$$

### 4.2 ECDSA原理

**密钥生成**:

- 私钥: $d \leftarrow_R [1, n-1]$
- 公钥: $Q = dG$，其中$G$为基点

**签名算法**:

1. 选随机$k \in [1, n-1]$
2. 计算$R = kG = (x_1, y_1)$，$r = x_1 \bmod n$
3. 计算$s = k^{-1}(H(m) + dr) \bmod n$
4. 签名为$(r, s)$

**验证算法**:

1. 计算$u_1 = H(m) \cdot s^{-1} \bmod n$，$u_2 = r \cdot s^{-1} \bmod n$
2. 计算$R' = u_1G + u_2Q = (x_1', y_1')$
3. 验证$r \equiv x_1' \pmod{n}$

**安全性**:

- 基于椭圆曲线离散对数问题(ECDLP)
- 160-bit ECDLP ≈ 1024-bit RSA安全性
- 256-bit ECDLP ≈ 3072-bit RSA安全性

## 五、零知识证明

### 5.1 基本定义

**定义 5.1（交互式证明系统）**:
语言$L$的交互式证明系统$(P, V)$满足：

1. **完备性**: $x \in L \Rightarrow \Pr[\langle P, V \rangle(x) = 1] \geq 1 - \mathsf{negl}(|x|)$
2. **可靠性**: $x \notin L \Rightarrow \forall P^*: \Pr[\langle P^*, V \rangle(x) = 1] \leq \mathsf{negl}(|x|)$

**定义 5.2（零知识性）**:
证明系统具有**零知识性**，若存在模拟器$\mathcal{S}$使得：
$$
\{\langle P, V \rangle(x)\}_{x \in L} \approx \{\mathcal{S}(x)\}_{x \in L}
$$

### 5.2 Schnorr协议

**协议描述**（证明知道离散对数）:

- 公开: $h = g^x$，证明者知道$x$
- 承诺: 证明者选$r \leftarrow \mathbb{Z}_q$，发送$a = g^r$
- 挑战: 验证者选$e \leftarrow \mathbb{Z}_q$
- 响应: 证明者发送$z = r + ex \pmod{q}$
- 验证: 验证$g^z = a \cdot h^e$

**安全性**:

- **完备性**: $g^{r+ex} = g^r \cdot (g^x)^e = a \cdot h^e$ ✓
- **可靠性**: 从两个接受转录可提取$x = (z_1 - z_2)/(e_1 - e_2)$
- **零知识**: 模拟器可生成不可区分的转录

## 六、Python实现

### 6.1 RSA实现

```python
import random
from typing import Tuple

def is_prime(n: int, k: int = 10) -> bool:
    """Miller-Rabin素性测试"""
    if n < 2: return False
    for p in [2, 3, 5, 7, 11, 13, 17, 19]:
        if n % p == 0: return n == p

    r, d = 0, n - 1
    while d % 2 == 0:
        r += 1
        d //= 2

    for _ in range(k):
        a = random.randrange(2, n - 1)
        x = pow(a, d, n)
        if x == 1 or x == n - 1: continue
        for _ in range(r - 1):
            x = pow(x, 2, n)
            if x == n - 1: break
        else: return False
    return True

def generate_prime(bits: int) -> int:
    """生成指定位数的素数"""
    while True:
        p = random.getrandbits(bits) | 1 | (1 << (bits - 1))

        if is_prime(p): return p

def egcd(a: int, b: int) -> Tuple[int, int, int]:
    """扩展欧几里得算法"""
    if b == 0: return a, 1, 0
    g, x1, y1 = egcd(b, a % b)
    return g, y1, x1 - (a // b) * y1

def modinv(a: int, m: int) -> int:
    """模逆元"""
    g, x, _ = egcd(a % m, m)
    if g != 1: raise ValueError("模逆元不存在")
    return x % m

class RSA:
    """RSA加密系统实现"""

    def __init__(self, key_size: int = 2048):
        # 密钥生成
        p = generate_prime(key_size // 2)
        q = generate_prime(key_size // 2)
        self.n = p * q
        self.phi = (p - 1) * (q - 1)
        self.e = 65537  # 常用公钥指数
        self.d = modinv(self.e, self.phi)

    def encrypt(self, m: int) -> int:
        """加密: c = m^e mod n"""
        return pow(m, self.e, self.n)

    def decrypt(self, c: int) -> int:
        """解密: m = c^d mod n"""
        return pow(c, self.d, self.n)

    def sign(self, m: int) -> int:
        """签名: s = m^d mod n"""
        return pow(m, self.d, self.n)

    def verify(self, m: int, s: int) -> bool:
        """验证: m = s^e mod n"""
        return pow(s, self.e, self.n) == m

# 使用示例
if __name__ == "__main__":
    rsa = RSA(1024)  # 演示用1024位，实际应使用2048+
    message = 123456789

    # 加密解密
    ciphertext = rsa.encrypt(message)
    plaintext = rsa.decrypt(ciphertext)
    print(f"原始消息: {message}")
    print(f"解密结果: {plaintext}")
    print(f"加密正确: {message == plaintext}")

    # 数字签名
    signature = rsa.sign(message)
    valid = rsa.verify(message, signature)
    print(f"签名验证: {valid}")

```

## 6.2 椭圆曲线实现

```python
class EllipticCurve:
    """椭圆曲线 y^2 = x^3 + ax + b over F_p"""

    def __init__(self, a: int, b: int, p: int):
        self.a = a
        self.b = b
        self.p = p
        # 验证非奇异: 4a^3 + 27b^2 != 0
        assert (4 * a**3 + 27 * b**2) % p != 0

    def add(self, P: tuple, Q: tuple) -> tuple:
        """椭圆曲线点加法"""
        if P is None: return Q
        if Q is None: return P
        x1, y1 = P
        x2, y2 = Q

        if x1 == x2 and y1 != y2: return None  # P + (-P) = O

        if x1 == x2:  # P == Q (倍点)
            m = (3 * x1**2 + self.a) * pow(2 * y1, -1, self.p) % self.p
        else:  # P != Q
            m = (y2 - y1) * pow(x2 - x1, -1, self.p) % self.p

        x3 = (m**2 - x1 - x2) % self.p
        y3 = (m * (x1 - x3) - y1) % self.p
        return (x3, y3)

    def mul(self, k: int, P: tuple) -> tuple:
        """标量乘法 [k]P"""
        result = None
        addend = P
        while k:
            if k & 1: result = self.add(result, addend)
            addend = self.add(addend, addend)
            k >>= 1
        return result
```

### 7.2 单向函数定义的形式化

```lean4
-- Definition of negligible function
-- A function μ : ℕ → ℝ is negligible if for all c > 0, μ(n) < n^(-c) for large n
def Negligible (μ : ℕ → ℝ) : Prop :=
  ∀ c > 0, ∃ n₀, ∀ n ≥ n₀, μ n < (n : ℝ) ^ (-c : ℤ)

-- Definition of one-way function
-- f is one-way if it's easy to compute but hard to invert
def OneWayFunction {α β : Type} [Fintype α] [DecidableEq β]
    (f : α → β) (gen : ℕ → α) : Prop :=
  -- Easy to compute: computable in polynomial time
  (∃ poly, ∀ n, ∀ x ∈ gen n, computeTime f x ≤ poly n) ∧
  -- Hard to invert: for all PPT algorithms A, success probability is negligible
  (∀ (A : β → Option α),
    (∀ n, ∀ y, computeTime A y ≤ polynomial n) →
    Negligible (λ n =>
      let x := gen n
      let y := f x
      match A y with

      | some x' => if f x' = y then 1 else 0
      | none => 0))

```

## 八、实际应用

### 8.1 TLS/SSL协议

现代TLS 1.3使用：

- **密钥交换**: ECDHE (Elliptic Curve Diffie-Hellman Ephemeral)
- **认证**: ECDSA或Ed25519签名
- **对称加密**: AES-GCM或ChaCha20-Poly1305
- **哈希**: SHA-256或SHA-384

**握手流程**:

1. 客户端发送支持的密码套件
2. 服务器选择参数并发送证书
3. ECDHE密钥交换
4. 派生会话密钥
5. 加密应用数据传输

### 8.2 区块链应用

**比特币**:

- 地址生成: RIPEMD-160(SHA-256(public_key))
- 交易签名: ECDSA on secp256k1
- 挖矿: SHA-256双哈希工作量证明

**以太坊**:

- 账户地址:  Keccak-256public_key)
- 交易签名: ECDSA with chain_id replay protection
- 智能合约: 以太坊虚拟机执行

### 8.3 安全通信

**Signal协议**:

- X3DH (Extended Triple Diffie-Hellman)密钥协商
- 双棘轮(Double Ratchet)算法实现前向安全
- Curve25519椭圆曲线
