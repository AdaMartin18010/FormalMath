## ✅ 贡献者提交前自查清单

### 1.1 通用检查（所有贡献类型）

```markdown
## 提交前通用检查

### 分支与提交
- [ ] 基于最新的 `main` 分支创建
- [ ] 分支命名符合规范：`type/description`
- [ ] 提交信息符合规范：`type(scope): description`
- [ ] 所有提交已签名（git commit -s）

### 文件检查
- [ ] 只包含相关文件的更改
- [ ] 无敏感信息泄露
- [ ] 无临时文件（.tmp, .log等）
- [ ] 无大文件（>10MB）提交

### 本地验证
- [ ] 本地运行通过所有检查脚本
- [ ] 无明显的格式错误
- [ ] 链接可正常访问
```

### 1.2 内容贡献检查清单

#### 文档内容检查

```markdown
## 内容贡献自查清单

### 元数据 (⭐⭐⭐ 重要)
- [ ] YAML Frontmatter 完整
  - [ ] `msc_primary`: 符合MSC2020主分类
  - [ ] `msc_secondary`: 相关次分类（如适用）
  - [ ] `tags`: 相关标签
  - [ ] `level`: L0-L4 知识层次
  - [ ] `course_alignment`: 相关课程（如适用）

### 数学内容 (⭐⭐⭐ 重要)
- [ ] 定义严格准确
  - [ ] 数学符号使用规范
  - [ ] 术语符合项目词典
  - [ ] 无概念混淆或错误
- [ ] 定理陈述准确
  - [ ] 前提条件完整
  - [ ] 结论明确
  - [ ] 引用来源正确
- [ ] 证明完整或提供证明思路
  - [ ] 逻辑链条清晰
  - [ ] 关键步骤有说明
  - [ ] 引用其他结果时标注来源

### 示例与应用 (⭐⭐ 重要)
- [ ] 至少包含1个示例
- [ ] 示例与概念直接相关
- [ ] 示例解答完整
- [ ] 应用案例（L1+推荐）

### 格式规范 (⭐⭐ 重要)
- [ ] 标题层次正确（# ## ###）
- [ ] 数学公式使用LaTeX格式
  - [ ] 行内公式：`$...$`
  - [ ] 块级公式：`$$...$$`
  - [ ] 编号公式使用equation环境
- [ ] 代码块标明语言类型
- [ ] 列表使用规范

### 引用与链接 (⭐⭐ 重要)
- [ ] 参考文献格式正确
  - [ ] 包含作者、标题、出处、年份
  - [ ] DOI或ISBN可检索
- [ ] 内部链接使用相对路径
- [ ] 外部链接可访问

### 语言质量 (⭐ 重要)
- [ ] 无拼写错误
- [ ] 无语法错误
- [ ] 语言简洁清晰
- [ ] 专业术语使用准确
```

#### 深度等级检查

| 等级 | 必含内容 | 自查要点 |
|:-----|:---------|:---------|
| L0 | 定义+1-2例 | 字数1000-2000，基础概念清晰 |
| L1 | 定义+定理+思路+应用 | 字数2000-4000，逻辑完整 |
| L2 | 完整证明+多例+拓展 | 字数4000-6000，深度适中 |
| L3 | 前沿+研究方向 | 字数6000+，有参考价值 |

### 2.6 审查责任与权限

| 角色 | 审查范围 | 批准权限 |
|:-----|:---------|:---------|
| 社区成员 | 语言表达、格式问题 | 无 |
| 活跃贡献者 | 内容质量、术语规范 | 文档类PR |
| 核心贡献者 | 数学准确性、架构影响 | 文档/代码PR |
| 项目维护者 | 全部 | 全部+最终合并 |

### 2.7 审查响应时间标准

| PR类型 | 首次响应 | 完成审查 |
|:-------|:---------|:---------|
| 简单修复 | 1-2天 | 2-3天 |
| 文档更新 | 2-3天 | 3-5天 |
| 新功能 | 3-5天 | 5-10天 |
| 重大变更 | 5-7天 | 10-14天 |

### 3.3 常见错误速查

| 错误类型 | 常见表现 | 检查方法 |
|:---------|:---------|:---------|
| 术语错误 | 同一概念不同表述 | 对照术语词典 |
| 符号错误 | 公式渲染失败 | 本地预览 |
| 链接错误 | 404或指向错误 | 链接检查脚本 |
| 结构错误 | 标题层级混乱 | Markdown lint |
| 元数据错误 | MSC编码错误 | MSC验证脚本 |
