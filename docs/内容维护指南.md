## 🎯 新文档质量标准

### 1.1 文档创建标准

#### 基本要求

| 检查项 | 标准 | 重要性 |
|:-------|:-----|:------:|
| YAML Frontmatter | 必须包含完整元数据 | ⭐⭐⭐ |
| MSC分类 | 符合MSC2020标准 | ⭐⭐⭐ |
| 知识层次 | 明确标注L0-L4层级 | ⭐⭐⭐ |
| 字数要求 | 符合层级标准 | ⭐⭐⭐ |
| 文档结构 | 遵循标准模板 | ⭐⭐ |
| 术语规范 | 使用项目标准术语 | ⭐⭐⭐ |
| 数学公式 | LaTeX格式正确 | ⭐⭐⭐ |
| 引用规范 | DOI/ISBN可检索 | ⭐⭐ |

#### 内容完整性检查

```markdown
## 新文档必须包含

### 核心内容 (100%必须)
- [ ] 严格数学定义
- [ ] 至少1个示例说明
- [ ] 来源引用（定理/引理）
- [ ] 文档创建日期
- [ ] 最后更新日期

### 推荐内容 (L1+推荐)
- [ ] 证明或证明思路
- [ ] 应用场景说明
- [ ] 相关概念链接
- [ ] 历史背景（如适用）

### 深度内容 (L2+推荐)
- [ ] 完整证明过程
- [ ] 多个复杂示例
- [ ] 现代发展视角
- [ ] 开放问题/研究方向
```

### 1.2 文档分级标准

| 层级 | 字数 | 内容深度 | 适用场景 |
|:-----|:-----|:---------|:---------|
| L0 | 1000-2000 | 基础定义+1-2例 | 快速参考 |
| L1 | 2000-4000 | 完整定义+证明思路+应用 | 核心概念 |
| L2 | 4000-6000 | 深入定理+完整证明+拓展 | 深入学习 |
| L3 | 6000+ | 前沿内容+研究方向 | 研究参考 |

### 1.3 文档命名规范

```
# 标准格式
[序号]-[概念名称]-[版本标识].md

# 示例
01-集合论基础-国际标准版.md
02-群论基础-L1版.md
03-代数几何引论-深度扩展版.md
```

## 🔍 重复内容检测方法

### 3.1 自动化检测工具

```python
# 重复内容检测配置
DUPLICATE_DETECTION = {
    # 相似度阈值
    'similarity_threshold': 0.75,
    
    # 最小检测长度（字符）
    'min_content_length': 200,
    
    # 检测范围
    'scan_paths': [
        'docs/',
        'concept/',
    ],
    
    # 排除模式
    'exclude_patterns': [
        '*-模板.md',
        'README*',
        'CHANGELOG*',
    ]
}
```

### 3.2 检测维度

| 维度 | 检测方法 | 阈值 | 处理建议 |
|:-----|:---------|:-----|:---------|
| 标题相似 | 字符串相似度 | >85% | 合并或重命名 |
| 内容相似 | 文本指纹 | >70% | 整合或引用 |
| 定义重复 | 语义分析 | 相同定义 | 统一术语 |
| 示例重复 | 结构对比 | >80% | 保留最佳版本 |
| MSC冲突 | 分类对比 | 同一文档多分类 | 核实分类 |

### 3.3 人工审查流程

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│ 自动检测    │ -> │ 生成报告    │ -> │ 人工审查    │
└─────────────┘    └─────────────┘    └──────┬──────┘
                                              │
                    ┌─────────────────────────┼─────────────────────────┐
                    │                         │                         │
                    ▼                         ▼                         ▼
            ┌─────────────┐           ┌─────────────┐           ┌─────────────┐
            │ 确认重复    │           │ 假阳性      │           │ 需要讨论    │
            │ -> 合并     │           │ -> 标记忽略 │           │ -> 发起Issue│
            └─────────────┘           └─────────────┘           └─────────────┘
```

### 3.4 重复内容处理决策树

```markdown
## 重复内容处理流程

### Step 1: 判断类型
- [ ] 完全重复（内容几乎相同）
  - 保留最新/最完整版本
  - 其他版本添加重定向
  
- [ ] 部分重复（有重叠内容）
  - 提取公共部分为独立文档
  - 原文档引用公共部分
  
- [ ] 概念重复（不同角度描述）
  - 保留多版本
  - 添加交叉引用说明

### Step 2: 执行操作
- [ ] 创建合并计划
- [ ] 更新相关链接
- [ ] 添加变更记录
- [ ] 通知相关贡献者
```

## ✅ 质量检查流程

### 4.1 三级质量检查体系

```
┌─────────────────────────────────────────────────────────────┐
│                    三级质量检查体系                          │
├─────────────────┬─────────────────┬─────────────────────────┤
│   第一级：自动   │   第二级：提交前  │    第三级：人工审核      │
│   CI/CD检查     │   贡献者自查     │    维护者审查           │
├─────────────────┼─────────────────┼─────────────────────────┤
│ • 格式检查      │ • 本地检查脚本   │ • 数学准确性            │
│ • 链接检查      │ • 自查清单      │ • 术语规范性            │
│ • 代码编译      │ • 同行互查      │ • 教育有效性            │
│ • 基础lint     │ • 预览验证      │ • 格式规范性            │
└─────────────────┴─────────────────┴─────────────────────────┘
```

### 4.2 自动化质量检查

#### CI/CD 检查项

```yaml
# .github/workflows/quality-assurance.yml
quality_checks:
  markdown:
    - lint: markdownlint
    - links: lychee
    - format: prettier
    
  content:
    - yaml_frontmatter: required_fields
    - msc_validation: msc2020_standard
    - terminology: dictionary_check
    
  code:
    - lean: lean4_compiler
    - python: pylint + black
    - tests: pytest
```

#### 本地检查命令

```bash
# 完整检查
python tools/quality/check_all.py

# 分项检查
python tools/quality/check_markdown.py     # Markdown格式
python tools/quality/check_links.py        # 链接有效性
python tools/quality/check_terminology.py  # 术语规范
python tools/quality/check_msc.py          # MSC分类
python tools/quality/check_lean.py         # Lean4代码
```

### 4.3 质量评分标准

| 维度 | 权重 | 评分标准 |
|:-----|:-----|:---------|
| 数学准确性 | 30% | 定义正确、定理无误、证明完整 |
| 术语规范性 | 20% | 符合术语词典、无混用 |
| 格式规范性 | 15% | 符合文档模板、渲染正确 |
| 内容完整性 | 15% | 定义、示例、证明、应用齐全 |
| 引用规范性 | 10% | 来源可检索、引用格式正确 |
| 可读性 | 10% | 结构清晰、语言流畅 |

### 4.4 质量等级判定

| 等级 | 分数 | 处理建议 |
|:-----|:-----|:---------|
| A (优秀) | 90-100 | 直接通过，可作为范例 |
| B (良好) | 75-89 | 通过，建议小改进 |
| C (合格) | 60-74 | 有条件通过，需后续改进 |
| D (待改进) | <60 | 退回修改 |
