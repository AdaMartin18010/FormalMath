# 零知识证明基础

零知识证明（Zero-Knowledge Proof, ZKP）允许证明者向验证者证明某个陈述为真，而不泄露任何超出陈述真值本身的信息。这一反直觉的概念由 Goldwasser、Micali 与 Rackoff 于 1985 年正式定义，现已成为现代密码学、区块链与隐私计算的核心工具。

---

## 1. 定义与性质

### 1.1 交互式证明系统

**定义 1.1（交互式证明系统）**. 对语言 $L$，交互式证明系统由证明者 $P$ 和验证者 $V$ 组成，通过多轮交互使 $V$ 相信 $x   hetacalin L$。

形式化地，一个交互式证明系统是一对概率图灵机 $(P, V)$，其中 $V$ 在多项式时间内运行。对公共输入 $x$，$P$ 和 $V$ 交换消息序列 $(a_1, b_1, a_2, b_2, ..., a_k, b_k)$，最终 $V$ 输出接受或拒绝。

**定义 1.2（完备性）**. 若 $x   hetacalin L$，$V$ 以高概率接受。

形式化：$  hetacal Pr[  hetalangle P, V   hetarangle(x) =   ext{接受}]   hetacalgeq 1 -   hetacal delta(|x|)$，其中 $  hetacal delta$ 是可忽略函数。

**定义 1.3（可靠性）**. 若 $x   hetanot  hetacalin L$，任何欺骗性 $P^*$ 使 $V$ 接受的概率极低。

形式化：$  hetacal Pr[  hetalangle P^*, V   hetarangle(x) =   ext{接受}]   hetacalleq   hetacal epsilon(|x|)$，对任意 $P^*$ 和可忽略函数 $  hetacal epsilon$。

### 1.2 零知识性

**定义 1.4（零知识）**. 对任意验证者 $V^*$，存在模拟器 $S$ 使得 $S(x)$ 与真实交互 $(P, V^*)(x)$ 计算不可区分。

**直观解释**：零知识性的核心思想是"如果我能在不看证明的情况下模拟出整个交互过程，那么证明本身就没有泄露任何额外信息"。模拟器 $S$ 不拥有见证（witness），却能生成与真实交互不可区分的 transcripts。

**定理 1.5（GMR）**. 二次剩余语言有完全零知识证明。

**二次剩余语言**：
$$QR = \{(n, y)   hetamid y   ext{ 是模 } n   ext{ 的二次剩余}\}$$

即存在 $x$ 使 $y   hetacalequiv x^2   hetpmod{n}$。

## 2. zk-SNARKs

### 2.1 特性

zk-SNARK（Zero-Knowledge Succinct Non-Interactive Argument of Knowledge）具有：

- **简洁性 (Succinctness)**：证明大小与验证时间独立于计算量，通常为 $O(1)$ 或 $O(  hetalog |C|)$（$C$ 是电路）；
- **非交互性 (Non-interactivity)**：单条消息即可验证，无需多轮交互；
- **知识论证 (Argument of Knowledge)**：证明者必须"知道"见证，而不仅仅是陈述为真。

**与交互式证明的区别**：zk-SNARK 通过**Fiat-Shamir启发式**将交互式协议转化为非交互式。基本思想是用密码学哈希函数代替验证者的随机挑战：
$$  ext{挑战} = H(  ext{公共输入},   ext{承诺})$$

### 2.2 构造框架

典型构造（Groth16）：

1. **将计算转化为算术电路**：把程序 $f(x, w)$ 编译为加法门和乘法门组成的电路 $C$；
2. **电路转化为 R1CS（Rank-1 Constraint System）**：将电路表示为一组秩-1约束 $A   hetacalcdot B = C$；
3. **R1CS 转化为 QAP（Quadratic Arithmetic Program）**：通过Lagrange插值，将约束组转化为多项式恒等式；
4. **利用双线性配对构造证明**：在椭圆曲线上，利用配对 $e:   hetacalmathbb{G}_1   hetacaltimes   hetacalmathbb{G}_2   hetato   hetacalmathbb{G}_T$ 的线性性质构造简洁证明。

**Groth16证明结构**：证明 $  hetacal pi = (A, B, C)$ 包含三个群元素，验证只需检查一个配对方程：
$$e(A, B) = e(  hetacalalpha   hetacalcdot G_1,   hetacalbeta   hetacalcdot G_2)   hetacalcdot e(C,   hetacaldelta   hetacalcdot G_2)   hetacalcdot e(  ext{pub},   hetacalgamma   hetacalcdot G_2)$$

验证时间为常数（与电路大小无关），证明大小约 200 字节。

## 3. 详细例子

### 3.1 例子：图同构的零知识证明

**协议**：

设 $G_0$ 和 $G_1$ 是两个同构图。证明者知道同构映射 $  hetacalphi: G_0   hetato G_1$。

1. **承诺**：证明者随机选取置换 $  hetacalpi$，发送 $H =   hetacalpi(G_0)$；
2. **挑战**：验证者随机挑战 $b   hetacalin \{0, 1\}$；
3. **响应**：
   - 若 $b = 0$，证明者发送 $  hetacalsigma =   hetacalpi$（使 $  hetacalsigma(G_0) = H$）
   - 若 $b = 1$，证明者发送 $  hetacalsigma =   hetacalpi   hetacalcirc   hetacalphi^{-1}$（使 $  hetacalsigma(G_1) = H$）
4. **验证**：验证者检查 $  hetacalsigma(G_b) = H$。

**零知识性证明**：

**模拟器** $S$ 的工作方式：

- 随机猜测 $b'   hetacalin \{0, 1\}$
- 若 $b' = 0$：选随机 $  hetacalsigma$，设 $H =   hetacalsigma(G_0)$
- 若 $b' = 1$：选随机 $  hetacalsigma$，设 $H =   hetacalsigma(G_1)$
- 发送 $(H, b',   hetacalsigma)$

若 $b' = b$（概率 $1/2$），模拟器的输出分布与真实交互完全相同；若 $b'   hetacalneq b$，模拟器可以重试。通过重复或并行化，可以得到计算不可区分的模拟。

**为什么这是零知识的？** 验证者在交互中只看到了：

- 一个随机图 $H$（与 $G_0, G_1$ 同构）
- 一个将 $G_b$ 送到 $H$ 的随机置换

这些信息完全可以由不知道 $  hetacalphi$ 的模拟器生成。

### 3.2 例子：离散对数的知识证明

证明者知道 $x$ 使 $y = g^x$（在循环群 $  hetacalmathbb{G}$ 中）。Schnorr 协议：

1. **承诺**：证明者选随机 $r$，发送 $t = g^r$；
2. **挑战**：验证者发送随机挑战 $c$；
3. **响应**：证明者发送 $s = r + cx$；
4. **验证**：验证者检查 $g^s = t   hetacalcdot y^c$。

**验证等式推导**：
$$g^s = g^{r+cx} = g^r   hetacalcdot (g^x)^c = t   hetacalcdot y^c$$

**零知识性**：模拟器可以：

1. 随机选 $s$ 和 $c$
2. 计算 $t = g^s / y^c$
3. 输出 $(t, c, s)$

因为 $s$ 和 $c$ 是均匀随机的（在真实协议中 $r$ 随机导致 $s$ 随机），模拟器的输出分布与真实协议相同。

## 4. 应用领域

零知识证明的应用已从纯密码学理论扩展到多个实际领域：

### 4.1 区块链与隐私币

- **Zcash**：使用 zk-SNARKs 实现**屏蔽交易**（shielded transactions），隐藏发送方、接收方和金额
- **zk-Rollups**：在以太坊 Layer 2 上，将数千笔交易压缩为一个零知识证明，极大提升吞吐量

### 4.2 身份认证与凭证

- **匿名凭证**：证明"我是成年人"而不泄露出生日期；证明"我的收入超过某阈值"而不泄露具体数额
- **去中心化身份 (DID)**：用户控制自己的身份属性，选择性披露

### 4.3 机器学习

- **模型完整性证明**：证明推理确实由特定模型执行，而无需暴露模型参数
- **隐私保护推理**：用户的数据对模型提供方保持加密状态

### 4.4 监管合规

- **偿付能力证明**：交易所可以证明其资产大于负债，而无需暴露具体地址和余额
- **反洗钱合规**：在保持用户隐私的同时，证明资金不来自黑名单地址
