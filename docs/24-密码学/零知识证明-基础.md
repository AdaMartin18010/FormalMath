# 零知识证明基础

零知识证明（Zero-Knowledge Proof, ZKP）允许证明者向验证者证明某个陈述为真，而不泄露任何超出陈述真值本身的信息。这一反直觉的概念由 Goldwasser、Micali 与 Rackoff 于 1985 年正式定义，现已成为现代密码学、区块链与隐私计算的核心工具。

## 1. 定义与性质

### 1.1 交互式证明系统

**定义 1.1（交互式证明系统）**. 对语言 $L$，交互式证明系统由证明者 $P$ 和验证者 $V$ 组成，通过多轮交互使 $V$ 相信 $x \in L$。

**定义 1.2（完备性）**. 若 $x \in L$，$V$ 以高概率接受。

**定义 1.3（可靠性）**. 若 $x \notin L$，任何欺骗性 $P^*$ 使 $V$ 接受的概率极低。

### 1.2 零知识性

**定义 1.4（零知识）**. 对任意验证者 $V^*$，存在模拟器 $S$ 使得 $S(x)$ 与真实交互 $(P, V^*)(x)$ 计算不可区分。

**定理 1.5（GMR）**. 二次剩余语言有完全零知识证明。

## 2. zk-SNARKs

### 2.1 特性

zk-SNARK（Zero-Knowledge Succinct Non-Interactive Argument of Knowledge）具有：

- **简洁性**：证明大小与验证时间独立于计算量；
- **非交互性**：单条消息即可验证；
- **知识论证**：证明者必须"知道"见证。

### 2.2 构造框架

典型构造（Groth16）：

1. 将计算转化为算术电路；
2. 电路转化为 R1CS（Rank-1 Constraint System）；
3. R1CS 转化为 QAP（Quadratic Arithmetic Program）；
4. 利用双线性配对构造证明。

## 3. 例子

### 3.1 例子：图同构的零知识证明

**协议**：

1. 证明者随机选取置换 $\pi$，发送 $H = \pi(G_0)$；
2. 验证者随机挑战 $b \in \{0, 1\}$；
3. 证明者发送 $\sigma$ 使 $\sigma(G_b) = H$；
4. 验证者检查。

模拟器：对 $b=0$ 选随机 $\sigma$，设 $H = \sigma(G_0)$；对 $b=1$ 类似。

### 3.2 例子：离散对数的知识证明

证明者知道 $x$ 使 $y = g^x$。Schnorr 协议：

1. 证明者选随机 $r$，发送 $t = g^r$；
2. 验证者发送挑战 $c$；
3. 证明者发送 $s = r + cx$；
4. 验证者检查 $g^s = t \cdot y^c$。
