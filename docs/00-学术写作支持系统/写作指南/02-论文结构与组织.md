# 论文结构与组织

## 1. 标准论文结构

数学论文遵循相对固定的结构模式：

```
┌─────────────────────────────────────┐
│              标题                    │
├─────────────────────────────────────┤
│              作者信息                │
├─────────────────────────────────────┤
│              摘要                    │
├─────────────────────────────────────┤
│  1. 引言                             │
│  2. 预备知识/背景                    │
│  3. 主要内容（可分多节）             │
│  4. 应用/例子（可选）                │
│  5. 结论与展望                       │
├─────────────────────────────────────┤
│            参考文献                  │
├─────────────────────────────────────┤
│           附录（可选）               │
└─────────────────────────────────────┘
```

## 2. 各部分详解

### 2.1 标题

**要求**：

- 准确反映论文内容
- 简洁有力（一般不超过15个词）
- 避免过于宽泛或过于狭窄

**示例**：

- ✅ "Galois理论在五次方程不可解性中的应用"
- ❌ "关于数学的一些研究"

### 2.2 摘要

**功能**：

- 独立概述全文
- 帮助读者判断是否阅读全文
- 用于检索系统

**结构**：

1. 研究背景/动机（1-2句）
2. 主要内容/方法（2-3句）
3. 主要结果（1-2句）
4. 意义/应用（可选，1句）

**长度**：

- 说明性论文：100-150词
- 学期论文：150-250词
- 毕业论文：200-300词

### 2.3 引言

**核心任务**：

1. **引入主题**：从读者熟悉的内容开始
2. **陈述问题**：明确论文要解决的问题
3. **文献回顾**：概述相关工作
4. **论文贡献**：说明本文的主要贡献
5. **论文结构**：概述后续章节安排

**写作建议**：

- 第一段要引人入胜
- 避免过于技术性的细节
- 明确论文的独创性（如果是原创性论文）

### 2.4 预备知识

**内容**：

- 必要的定义
- 相关的已知定理
- 必要的记号说明

**原则**：

- 只包含必要的背景
- 假设读者具有本科数学基础
- 明确引用标准教材中的结果

### 2.5 主要内容

**组织策略**：

**按主题组织**（适合综合类论文）：

```
3. 基本概念
4. 主要定理
5. 证明技术
```

**按逻辑顺序组织**（适合证明类论文）：

```
3. 引理准备
4. 主要定理证明
5. 推论与应用
```

**按历史发展组织**（适合历史性论文）：

```
3. 早期工作
4. 突破性进展
5. 现代发展
```

### 2.6 结论

**内容**：

1. 总结主要结果
2. 讨论局限性和未解决问题
3. 展望未来研究方向

**避免**：

- 简单重复摘要
- 引入新结果
- 过度谦虚或夸大

## 3. 长度分配建议

| 部分 | 说明性论文 | 学期论文 | 毕业论文 |
|------|----------|----------|----------|
| 摘要 | 150词 | 200词 | 300词 |
| 引言 | 1页 | 2-3页 | 5-8页 |
| 预备知识 | 1-2页 | 3-4页 | 10-15页 |
| 主要内容 | 3-5页 | 10-12页 | 30-40页 |
| 应用/例子 | 1页 | 2-3页 | 5-10页 |
| 结论 | 0.5页 | 1-2页 | 3-5页 |

## 4. 段落组织原则

### 4.1 主题句原则

每个段落应该有一个明确的主题句，通常位于段首。

### 4.2 段落长度

- 过短（<3行）：可能过于琐碎
- 过长（>10行）：可能包含过多内容
- 理想：3-8行

### 4.3 过渡与衔接

使用过渡词和短语：

- 递进：Furthermore, Moreover, In addition
- 转折：However, Nevertheless, On the other hand
- 因果：Therefore, Thus, Consequently
- 举例：For example, For instance
- 总结：In summary, To conclude

## 5. 章节编号系统

### 5.1 标准编号

```
1 引言
2 预备知识
  2.1 基本概念
    2.1.1 定义
    2.1.2 例子
  2.2 主要引理
3 主要结果
  ...
```

### 5.2 定理环境编号

推荐采用连续编号：

```
定理 2.1
引理 2.2
命题 2.3
定理 2.4
```

或按章节编号：

```
定理 2.1.1
定理 2.1.2
定理 2.2.1
```

## 6. 检查清单

- [ ] 标题准确反映内容
- [ ] 摘要独立完整
- [ ] 引言吸引读者并明确问题
- [ ] 预备知识充分但不过度
- [ ] 各章节比例适当
- [ ] 结论总结到位
- [ ] 章节过渡自然
- [ ] 整体结构逻辑清晰
