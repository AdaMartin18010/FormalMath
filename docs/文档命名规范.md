# FormalMath 项目文档命名规范

## 1. 概述

本文档定义了 FormalMath 项目中所有 Markdown 文档的统一命名规范，旨在提高文档的可读性、可维护性和可检索性。

## 2. 命名规范总览

### 2.1 基本格式

```
[序号]-[名称]-[版本/补充].md
```

### 2.2 各部分说明

| 部分 | 说明 | 示例 |
|------|------|------|
| 序号 | 2-3位数字，用于排序和分类 | `00-`, `01-`, `10-` |
| 名称 | 文档主题，使用 kebab-case 或中文 | `群论基础`, `course-mapping` |
| 版本 | 可选，表示文档版本或类型 | `深度版`, `v2`, `2026年4月` |

## 3. 详细规范

### 3.1 序号前缀规范

| 序号范围 | 用途 | 示例 |
|----------|------|------|
| `00-` | 项目报告、索引、说明文档 | `00-项目总览.md` |
| `01-` | 核心数学内容 - 基础数学 | `01-集合论基础.md` |
| `02-` | 核心数学内容 - 代数结构 | `02-群论基础.md` |
| `03-` | 核心数学内容 - 分析学 | `03-实分析.md` |
| `04-` | 核心数学内容 - 几何学 | `04-欧几里得几何.md` |
| `05-` | 核心数学内容 - 拓扑学 | `05-点集拓扑.md` |
| `06-` | 核心数学内容 - 数论 | `06-初等数论.md` |
| `07-` | 核心数学内容 - 离散数学 | `07-图论基础.md` |
| `08-` | 核心数学内容 - 应用数学 | `08-数值分析.md` |
| `09-` | 核心数学内容 - 概率统计 | `09-概率论基础.md` |

### 3.2 名称规范

#### 3.2.1 中文名称
- 使用简洁的中文描述文档主题
- 避免过长，建议不超过10个字符
- 示例：`群论基础`, `实分析`, `微分几何`

#### 3.2.2 英文名称 (kebab-case)
- 全部小写
- 使用连字符 `-` 连接单词
- 示例：`group-theory`, `real-analysis`, `differential-geometry`

### 3.3 版本/补充标识

| 标识 | 含义 | 示例 |
|------|------|------|
| `-深度版` | 深入详细的内容版本 | `01-群论基础-深度版.md` |
| `-增强版` | 内容增强版本 | `01-群论基础-增强版.md` |
| `-完整版` | 完整内容版本 | `01-项目总览-完整版.md` |
| `-YYYY年M月` | 时间版本标记 | `00-项目报告-2026年4月.md` |
| `-vN` | 版本号标记 | `01-API文档-v2.md` |

## 4. 特殊文档命名

### 4.1 标准文档

以下标准文档保持传统命名：

| 文档名 | 说明 |
|--------|------|
| `README.md` | 项目根目录说明 |
| `INDEX.md` | 索引文档 |
| `CHANGELOG.md` | 变更日志 |
| `LICENSE` | 许可证文件 |
| `CONTRIBUTING.md` | 贡献指南 |
| `CODE_OF_CONDUCT.md` | 行为准则 |
| `SECURITY.md` | 安全政策 |

### 4.2 目录内文档

子目录内的文档遵循相对序号：

```
docs/
  00-README.md                    # 目录主说明
  01-基础数学/
    00-README.md                  # 子目录说明
    01-集合论基础-深度版.md
    02-数理逻辑入门.md
```

## 5. 命名转换规则

### 5.1 英文转 kebab-case

| 原格式 | 转换后 |
|--------|--------|
| `CourseMapping` | `course-mapping` |
| `MIT Course` | `mit-course` |
| `Lean4形式化` | `lean4-formalization` |

### 5.2 特殊字符处理

| 字符 | 处理方式 | 示例 |
|------|----------|------|
| 空格 | 替换为 `-` | `课程 对齐` → `课程对齐` |
| `/` | 替换为 `-` | `分析/代数` → `分析-代数` |
| `&` | 替换为 `-and-` | `R&D` → `r-and-d` |
| `+` | 替换为 `-plus-` | `C++` → `c-plus-plus` |
| `%` | 替换为 `pct` | `100%` → `100pct` |

## 6. 目录分类与序号分配

### 6.1 根目录文档 (00-前缀)

主要用于：
- 项目整体报告
- 国际课程对齐报告
- 质量审查报告
- 完成状态报告

### 6.2 docs 目录

```
docs/
  00-*/      # 导航、索引、指南
  01-基础数学/
  02-代数结构/
  03-分析学/
  04-几何学/
  05-拓扑学/
  ...
```

### 6.3 concept 目录

```
concept/
  00-*/      # 项目文档、归档
  01-*/      # 思维导图、知识矩阵
  02-*/      # 主题概念
  03-*/      # 认知工具
  核心概念/   # 保持原名
```

### 6.4 project 目录

```
project/
  00-*/      # 项目规划、进度报告
  各子系统/   # 根据功能命名
```

## 7. 检查清单

在命名或重命名文档时，请确认：

- [ ] 序号前缀符合分类规范
- [ ] 名称简洁明确
- [ ] 使用正确的分隔符 `-`
- [ ] 英文部分使用 kebab-case
- [ ] 版本标识清晰（如有）
- [ ] 无特殊字符（除 `-` 和 `.md`）

## 8. 示例

### 8.1 良好示例

```
00-项目全面推进完成报告-2026年4月.md
01-集合论基础-深度版.md
02-群论基础-增强版.md
03-实分析.md
course-mapping-mit-18.100a.md
formal-math-user-guide.md
```

### 8.2 避免示例

```
00_项目报告.md           # 使用 _ 而非 -
集合论基础.md             # 缺少序号
01群论基础.md             # 缺少分隔符
01-GroupTheory.md         # 英文未使用 kebab-case
01-群论_基础.md           # 使用 _
```
