## 一、部署前准备

### 1.1 硬件要求

| 环境 | CPU | 内存 | 磁盘 | 网络 |
|------|-----|------|------|------|
| **开发环境** | 4核 | 8GB | 50GB SSD | 10Mbps |
| **测试环境** | 4核 | 8GB | 100GB SSD | 50Mbps |
| **生产环境** | 8核+ | 16GB+ | 200GB SSD | 100Mbps+ |

### 1.2 软件要求

| 软件 | 版本 | 用途 |
|------|------|------|
| Git | 2.40+ | 版本控制 |
| Python | 3.11+ | 后端服务 |
| Node.js | 18+ | 前端构建 |
| Lean | 4.29.0 | 形式化证明 |
| elan | 最新 | Lean工具链管理 |
| lake | 随Lean | Lean项目管理 |
| Docker | 24+ | 容器化部署 |

### 1.3 操作系统支持

- **推荐**: Ubuntu 22.04 LTS / Debian 12
- **支持**: macOS 13+ / Windows 11 (WSL2)
- **服务器**: CentOS Stream 9 / Rocky Linux 9

### 1.4 网络要求

```
端口说明:
├── 80/443    # HTTP/HTTPS (Web服务)
├── 8000      # FastAPI开发服务
├── 3000      # React开发服务
├── 8080      # 管理后台
└── 22        # SSH (管理)
```

### 3.5 Docker 部署

#### 创建 Dockerfile

```dockerfile
# Dockerfile
FROM ubuntu:22.04

# 安装基础依赖
RUN apt-get update && apt-get install -y \
    git curl wget python3.11 python3.11-venv python3-pip \
    nodejs npm \
    && rm -rf /var/lib/apt/lists/*

# 安装elan和Lean
RUN curl https://raw.githubusercontent.com/leanprover/elan/master/elan-init.sh[需更新[需更新]] -sSf | bash -s -- -y
ENV PATH="/root/.elan/bin:${PATH}"

# 设置工作目录
WORKDIR /app

# 复制项目文件
COPY . .

# 安装Python依赖
RUN python3.11 -m venv /opt/venv
ENV PATH="/opt/venv/bin:${PATH}"
RUN pip install --upgrade pip
RUN pip install -r requirements.txt

# 构建Lean项目
RUN cd docs/09-形式化证明/Lean4 && lake build

# 暴露端口
EXPOSE 8000 8001 8002 8003

# 启动脚本
COPY docker-entrypoint.sh /usr/local/bin/
RUN chmod +x /usr/local/bin/docker-entrypoint.sh
ENTRYPOINT ["docker-entrypoint.sh"]
```

#### 创建 docker-compose.yml

```yaml
version: '3.8'

services:
  # 主应用
  formalmath:
    build: .
    ports:
      - "8000:8000"
    volumes:
      - ./data:/app/data
      - ./logs:/app/logs
    environment:
      - ENV=production
      - DATABASE_URL=sqlite:///app/data/formalmath.db
    restart: unless-stopped

  # 认知诊断系统
  cognitive-diagnosis:
    build: ./tools/cognitive-diagnosis
    ports:
      - "8001:8000"
    volumes:
      - ./data:/app/data
    environment:
      - DATABASE_URL=sqlite:///app/data/cognitive.db
    restart: unless-stopped

  # 评估系统
  assessment:
    build: ./tools/assessment-system
    ports:
      - "8002:8000"
    volumes:
      - ./data:/app/data
    environment:
      - DATABASE_URL=sqlite:///app/data/assessment.db
    restart: unless-stopped

  # 自适应系统
  adaptive-learning:
    build: ./tools/personalized-recommendation
    ports:
      - "8003:8000"
    volumes:
      - ./data:/app/data
    environment:
      - DATABASE_URL=sqlite:///app/data/adaptive.db
    restart: unless-stopped

  # Nginx 反向代理
  nginx:
    image: nginx:alpine
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./nginx.conf:/etc/nginx/nginx.conf
      - ./ssl:/etc/nginx/ssl
    depends_on:
      - formalmath
      - cognitive-diagnosis
      - assessment
      - adaptive-learning
    restart: unless-stopped

  # 监控 (可选)
  prometheus:
    image: prom/prometheus
    ports:
      - "9090:9090"
    volumes:
      - ./prometheus.yml:/etc/prometheus/prometheus.yml
    restart: unless-stopped
```

#### 启动Docker服务

```bash
# 构建镜像
docker-compose build

# 启动服务
docker-compose up -d

# 查看状态
docker-compose ps

# 查看日志
docker-compose logs -f

# 停止服务
docker-compose down
```

    worker_connections 1024;

        location /api/adaptive/ {

### 4.3 系统服务配置

创建 Systemd 服务文件：

```ini
# /etc/systemd/system/formalmath-cognitive.service
[Unit]
Description=FormalMath Cognitive Diagnosis System
After=network.target

[Service]
Type=simple
User=formalmath
Group=formalmath
WorkingDirectory=/opt/formalmath/tools/cognitive-diagnosis
Environment=PATH=/opt/formalmath/venv/bin
Environment=DATABASE_URL=sqlite:///opt/formalmath/data/cognitive.db
ExecStart=/opt/formalmath/venv/bin/uvicorn main:app --host 0.0.0.0 --port 8001
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

```bash
# 启用服务
sudo systemctl daemon-reload
sudo systemctl enable formalmath-cognitive
sudo systemctl start formalmath-cognitive

# 查看状态
sudo systemctl status formalmath-cognitive
sudo journalctl -u formalmath-cognitive -f
```

## 六、回滚方案

### 6.1 代码回滚

```bash
# 查看历史版本
git log --oneline -10

# 回滚到指定版本
git reset --hard <commit-hash>

# 或创建回滚提交
git revert <commit-hash>
```

### 6.2 数据库回滚

```bash
# 停止服务
sudo systemctl stop formalmath-cognitive

# 恢复备份
cp /backup/cognitive.db.$(date -d yesterday +%Y%m%d) /opt/formalmath/data/cognitive.db

# 重启服务
sudo systemctl start formalmath-cognitive
```

### 6.3 Docker 回滚

```bash
# 查看历史镜像
docker images --filter reference=formalmath

# 回滚到指定版本
docker-compose down
docker tag formalmath:<old-version> formalmath:latest
docker-compose up -d
```

### 6.4 紧急回滚脚本

```bash
#!/bin/bash
# emergency-rollback.sh

BACKUP_DIR="/backup/$(date +%Y%m%d)"
DATA_DIR="/opt/formalmath/data"

echo "开始紧急回滚..."

# 停止所有服务
sudo systemctl stop formalmath-*
docker-compose down

# 恢复数据库
cp $BACKUP_DIR/*.db $DATA_DIR/

# 恢复代码
cd /opt/formalmath
git reset --hard HEAD~1

# 重启服务
sudo systemctl start formalmath-*
docker-compose up -d

echo "回滚完成"
```

### B. 常见问题

**Q: Lean构建失败怎么办？**
A: 检查网络连接，尝试 `lake exe cache get` 获取缓存。

**Q: 服务启动失败怎么办？**
A: 检查日志 `journalctl -u formalmath-cognitive -n 50`。

**Q: 数据库锁定怎么办？**
A: 检查是否有其他进程占用，使用 `lsof` 查找。
