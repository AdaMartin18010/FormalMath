## 二、故障分级

### 2.1 P0 - 严重故障

**定义**: 生产环境完全不可用，核心业务中断

**判定标准**:
- 所有服务无法访问
- 数据丢失或损坏
- 安全入侵确认
- 超过50%用户受影响

**响应时间**: 5分钟内

**升级条件**: 15分钟内无法恢复

### 2.2 P1 - 高优先级故障

**定义**: 核心功能受损，部分业务中断

**判定标准**:
- 主要服务不可用
- 性能严重下降 (响应>5s)
- 数据访问异常
- 10-50%用户受影响

**响应时间**: 15分钟内

**升级条件**: 30分钟内无法恢复

### 2.3 P2 - 中优先级故障

**定义**: 非核心功能异常，业务可继续

**判定标准**:
- 次要服务不可用
- 轻微性能下降
- 少量用户受影响 (<10%)

**响应时间**: 1小时内

### 2.4 P3 - 低优先级故障

**定义**: 轻微问题，不影响业务

**判定标准**:
- 界面显示问题
- 日志异常
- 优化建议类

**响应时间**: 1工作日内

## 四、具体场景预案

### 4.1 场景一: 服务完全不可用 (P0)

#### 症状
- 所有API返回502/503
- 网站无法访问
- 大量用户投诉

#### 应急步骤

```bash
#!/bin/bash
# emergency_p0_service_down.sh

echo "=== P0 服务不可用应急响应 ==="

# 1. 确认故障范围
echo "[1/6] 确认故障范围..."
for port in 8001 8002 8003 80 443; do
    if curl -s --max-time 5 http://localhost:$port/health > /dev/null; then
        echo "  端口 $port: 正常"
    else
        echo "  端口 $port: 异常"
    fi
done

# 2. 检查系统资源
echo "[2/6] 检查系统资源..."
free -h | grep Mem
df -h | grep -E "Filesystem|/dev/"
uptime

# 3. 检查服务状态
echo "[3/6] 检查服务状态..."
systemctl status formalmath-* --no-pager

# 4. 快速恢复尝试
echo "[4/6] 尝试快速恢复..."
echo "选项:"
echo "  1) 重启所有服务"
echo "  2) 回滚到上一版本"
echo "  3) 切换到备用实例"
echo "  4) 启动容灾站点"
read -p "选择恢复方案 [1-4]: " choice

case $choice in
    1)
        echo "重启所有服务..."
        sudo systemctl restart formalmath-*
        sudo systemctl restart nginx
        ;;
    2)
        echo "执行回滚..."
        /opt/formalmath/tools/rollback.sh
        ;;
    3)
        echo "切换流量到备用实例..."
        # 修改负载均衡配置
        ;;
    4)
        echo "启动容灾站点..."
        # 执行DR计划
        ;;
esac

# 5. 验证恢复
echo "[5/6] 验证恢复..."
sleep 5
for port in 8001 8002 8003; do
    if curl -sf http://localhost:$port/health > /dev/null; then
        echo "  ✓ 端口 $port: 已恢复"
    else
        echo "  ✗ 端口 $port: 仍异常"
    fi
done

# 6. 通知
echo "[6/6] 发送恢复通知..."
echo "故障已处理，服务已恢复" | mail -s "[RESOLVED] FormalMath服务恢复" ops@formalmath.org

echo "=== 应急处理完成 ==="
```

### 4.2 场景二: 数据库损坏 (P0)

#### 症状
- 数据库查询报错
- 应用日志显示数据库错误
- `database disk image is malformed`

#### 应急步骤

```bash
#!/bin/bash
# emergency_p0_db_corruption.sh

DB_NAME=$1

if [ -z "$DB_NAME" ]; then
    echo "用法: $0 <database_name>"
    exit 1
fi

echo "=== P0 数据库损坏应急响应 ==="
echo "受影响数据库: $DB_NAME"

# 1. 立即停止写入
echo "[1/7] 停止相关服务..."
sudo systemctl stop formalmath-*

# 2. 评估损坏程度
echo "[2/7] 评估损坏..."
DB_PATH="/opt/formalmath/data/$DB_NAME"
if [ -f "$DB_PATH" ]; then
    sqlite3 $DB_PATH "PRAGMA integrity_check;"
else
    echo "错误: 数据库文件不存在"
    exit 1
fi

# 3. 备份损坏的数据库
echo "[3/7] 备份损坏的数据库..."
CORRUPT_BACKUP="/backup/corrupt/${DB_NAME}.$(date +%Y%m%d_%H%M%S)"
mkdir -p /backup/corrupt
cp $DB_PATH $CORRUPT_BACKUP
echo "已备份到: $CORRUPT_BACKUP"

# 4. 尝试修复
echo "[4/7] 尝试修复..."
REPAIRED="/tmp/${DB_NAME}.repaired"
sqlite3 $DB_PATH ".dump" > /tmp/dump.sql 2>/dev/null

if [ $? -eq 0 ]; then
    sqlite3 $REPAIRED < /tmp/dump.sql
    if sqlite3 $REPAIRED "PRAGMA integrity_check;" | grep -q "ok"; then
        echo "修复成功"
        mv $REPAIRED $DB_PATH
    else
        echo "修复失败，需要恢复备份"
        rm -f $REPAIRED
    fi
else
    echo "导出失败，需要恢复备份"
fi

# 5. 如果修复失败，从备份恢复
if [ ! -f "$DB_PATH" ] || ! sqlite3 $DB_PATH "PRAGMA integrity_check;" | grep -q "ok"; then
    echo "[5/7] 从备份恢复..."
    read -p "输入备份日期 (YYYYMMDD): " backup_date
    /opt/formalmath/tools/restore_database.sh $DB_NAME $backup_date
fi

# 6. 验证
echo "[6/7] 验证数据库..."
if sqlite3 $DB_PATH "PRAGMA integrity_check;" | grep -q "ok"; then
    echo "✓ 数据库验证通过"
else
    echo "✗ 数据库验证失败"
    exit 1
fi

# 7. 恢复服务
echo "[7/7] 恢复服务..."
sudo systemctl start formalmath-*
sleep 3
systemctl status formalmath-* --no-pager

echo "=== 应急处理完成 ==="
```

### 4.3 场景三: 安全入侵 (P0)

#### 症状
- 异常进程运行
- 未知文件出现
- 异常网络连接
- 日志显示入侵痕迹

#### 应急步骤

```bash
#!/bin/bash
# emergency_p0_security_breach.sh

echo "=== P0 安全入侵应急响应 ==="

# 1. 立即隔离
echo "[1/8] 隔离受影响系统..."
sudo iptables -P INPUT DROP
sudo iptables -P OUTPUT DROP
sudo iptables -P FORWARD DROP

# 允许SSH (用于远程恢复)
sudo iptables -A INPUT -p tcp --dport 22 -j ACCEPT
sudo iptables -A OUTPUT -p tcp --sport 22 -j ACCEPT

# 2. 保留证据
echo "[2/8] 保留证据..."
EVIDENCE_DIR="/forensics/$(date +%Y%m%d_%H%M%S)"
mkdir -p $EVIDENCE_DIR

# 内存转储 (如果安装了lime)
if [ -f /tmp/lime.ko ]; then
    insmod /tmp/lime.ko path=$EVIDENCE_DIR/memory.lime format=lime
fi

# 网络连接
netstat -tulpn > $EVIDENCE_DIR/netstat.txt
ss -tulpn > $EVIDENCE_DIR/ss.txt

# 进程列表
ps auxf > $EVIDENCE_DIR/processes.txt

# 最近修改的文件
find / -mtime -1 -type f 2>/dev/null > $EVIDENCE_DIR/recent_files.txt

# 3. 终止恶意进程
echo "[3/8] 终止可疑进程..."
# 这里需要根据具体情况判断
# 不要盲目终止，先分析

# 4. 创建系统镜像
echo "[4/8] 创建系统镜像..."
dd if=/dev/sda of=$EVIDENCE_DIR/disk_image.img bs=1M status=progress &

# 5. 分析入侵路径
echo "[5/8] 分析入侵路径..."
grep "Failed password" /var/log/auth.log > $EVIDENCE_DIR/failed_logins.txt
grep "Accepted" /var/log/auth.log > $EVIDENCE_DIR/success_logins.txt

# 6. 通知安全团队
echo "[6/8] 通知安全团队..."
echo "安全事件告警: 系统可能已被入侵" | \
    mail -s "[CRITICAL] Security Incident" \
    -A $EVIDENCE_DIR/processes.txt \
    security@formalmath.org

# 7. 准备干净环境
echo "[7/8] 准备恢复..."
echo "需要从干净备份重建系统"

# 8. 等待进一步指示
echo "[8/8] 等待安全团队指示..."
echo "证据已保存到: $EVIDENCE_DIR"
echo "请等待安全团队分析后再进行恢复操作"

# 保持隔离状态，等待人工介入
echo "=== 系统已隔离，等待进一步指示 ==="
```

```python
#!/usr/bin/env python3
# improvement_tracking.py

import json
from datetime import datetime

IMPROVEMENTS_FILE = "/opt/formalmath/docs/ops/improvements.json"

def add_improvement(incident_id, description, owner, due_date):
    """添加改进项"""
    try:
        with open(IMPROVEMENTS_FILE, 'r') as f:
            improvements = json.load(f)
    except FileNotFoundError:
        improvements = []
    
    improvements.append({
        "id": f"IMP-{len(improvements)+1:04d}",
        "incident_id": incident_id,
        "description": description,
        "owner": owner,
        "due_date": due_date,
        "status": "pending",
        "created_at": datetime.now().isoformat(),
        "completed_at": None
    })
    
    with open(IMPROVEMENTS_FILE, 'w') as f:
        json.dump(improvements, f, indent=2)

def list_improvements(status=None):
    """列出改进项"""
    with open(IMPROVEMENTS_FILE, 'r') as f:
        improvements = json.load(f)
    
    if status:
        improvements = [i for i in improvements if i["status"] == status]
    
    return improvements

if __name__ == '__main__':
    # 示例: 添加改进项
    add_improvement(
        "INC-202604-001",
        "增加数据库备份频率",
        "ops@formalmath.org",
        "2026-04-30"
    )
```
