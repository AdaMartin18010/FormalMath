## 一、故障排除流程

### 1.1 标准故障处理流程

```
┌──────────────┐
│   发现问题    │
└──────┬───────┘
       │
       ▼
┌──────────────┐     是    ┌──────────────┐
│ 是否为已知故障 │────────►│ 应用已知方案  │
└──────┬───────┘          └──────┬───────┘
       │ 否                      │
       ▼                         ▼
┌──────────────┐          ┌──────────────┐
│  收集信息    │          │  验证修复    │
│ - 日志       │          └──────┬───────┘
│ - 指标       │                 │
│ - 配置       │                 ▼
└──────┬───────┘          ┌──────────────┐
       │                 │  记录文档    │
       ▼                 └──────────────┘
┌──────────────┐
│  分析根因    │
└──────┬───────┘
       │
       ▼
┌──────────────┐
│  实施修复    │
└──────────────┘
```

### 1.2 紧急响应分级

| 级别 | 定义 | 响应时间 | 升级条件 |
|------|------|----------|----------|
| **P0** | 服务完全不可用 | 5分钟 | 15分钟未恢复 |
| **P1** | 核心功能受损 | 15分钟 | 30分钟未恢复 |
| **P2** | 非核心功能异常 | 1小时 | 2小时未恢复 |
| **P3** | 轻微问题 | 1工作日 | 1周未解决 |

## 二、常见故障

### 2.1 Lean 构建失败

#### 症状

```
error: no such file or directory (error code: 2)
  file: ./././FormalMath/Basic.lean
```

#### 可能原因与解决方案

**原因1: 依赖缺失**

```bash
# 解决方案
lake update
lake exe cache get
lake build
```

**原因2: 缓存损坏**

```bash
# 清理缓存重新构建
rm -rf .lake/build
rm -rf ~/.cache/mathlib
cd docs/09-形式化证明/Lean4
lake exe cache get
lake build
```

**原因3: Lean版本不匹配**

```bash
# 检查版本
elan show

# 安装指定版本
elan install leanprover/lean4:v4.29.0
elan default leanprover/lean4:v4.29.0

# 重新构建
lake build
```

#### 诊断脚本

```bash
#!/bin/bash
# diagnose_lean.sh

echo "=== Lean 环境诊断 ==="

echo "1. Lean版本:"
lean --version

echo -e "\n2. Lake版本:"
lake --version

echo -e "\n3. Elan工具链:"
elan show

echo -e "\n4. 项目配置:"
cat lakefile.toml | grep -A5 "lean_lib\|require"

echo -e "\n5. 构建缓存:"
du -sh .lake/build 2>/dev/null || echo "无缓存目录"
du -sh ~/.cache/mathlib 2>/dev/null || echo "无mathlib缓存"

echo -e "\n6. 测试构建:"
cd docs/09-形式化证明/Lean4
timeout 120 lake build 2>&1 | tail -20
```

### 2.2 Python 服务启动失败

#### 症状

```
ImportError: No module named 'fastapi'
ERROR:    Application startup failed
```

#### 可能原因与解决方案

**原因1: 虚拟环境未激活**

```bash
# 解决方案
source /opt/formalmath/venv/bin/activate
# 或
conda activate formalmath
```

**原因2: 依赖缺失**

```bash
# 重新安装依赖
pip install -r requirements.txt

# 验证安装
pip list | grep fastapi
```

**原因3: 端口占用**

```bash
# 查找占用进程
lsof -i :8001
# 或
netstat -tulpn | grep 8001

# 终止进程
kill -9 <PID>

# 或更换端口
uvicorn main:app --port 8004
```

**原因4: 数据库权限**

```bash
# 检查数据库文件权限
ls -la /opt/formalmath/data/*.db

# 修复权限
chown -R formalmath:formalmath /opt/formalmath/data
chmod 644 /opt/formalmath/data/*.db
```

### 2.3 文档访问异常

#### 症状

- 文档无法加载
- 图片显示失败
- 链接失效

#### 解决方案

```bash
# 检查文档存在
ls -la docs/01-基础数学/

# 检查文件权限
find docs -type f ! -readable

# 修复权限
chmod -R 644 docs/*
find docs -type d -exec chmod 755 {} \;

# 验证链接完整性
python tools/check_cross_links.py
```

### 2.4 内存不足 (OOM)

#### 症状

```
OutOfMemoryError: killed process 1234 (python)
```

#### 解决方案

```bash
# 1. 查看内存使用
free -h
ps aux --sort=-%mem | head -10

# 2. 重启服务释放内存
sudo systemctl restart formalmath-cognitive

# 3. 配置swap (临时方案)
sudo fallocate -l 2G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile

# 4. 优化服务配置
# 修改 /etc/systemd/system/formalmath-cognitive.service
[Service]
MemoryLimit=1G
Restart=always
```

## 三、服务故障

### 3.1 认知诊断系统故障

#### 故障代码: CD-001 (服务无响应)

**症状**:

```
curl: (7) Failed to connect to localhost port 8001: Connection refused
```

**排查步骤**:

```bash
# 1. 检查服务状态
sudo systemctl status formalmath-cognitive

# 2. 查看日志
sudo journalctl -u formalmath-cognitive -n 100 --no-pager

# 3. 检查端口监听
ss -tulpn | grep 8001

# 4. 检查配置文件
cat /opt/formalmath/tools/cognitive-diagnosis/.env

# 5. 手动启动查看错误
cd /opt/formalmath/tools/cognitive-diagnosis
source ../../venv/bin/activate
uvicorn main:app --host 0.0.0.0 --port 8001
```

**解决方案**:

```bash
# 重启服务
sudo systemctl restart formalmath-cognitive

# 如果数据库锁定
sudo rm /opt/formalmath/data/cognitive.db-journal
sudo systemctl restart formalmath-cognitive

# 重置数据库 (数据丢失风险!)
sudo systemctl stop formalmath-cognitive
sudo cp /opt/formalmath/data/cognitive.db /opt/formalmath/data/cognitive.db.bak
sudo rm /opt/formalmath/data/cognitive.db
sudo -u formalmath python init_db.py
sudo systemctl start formalmath-cognitive
```

## 四、数据故障

### 4.1 数据库损坏

#### 症状

```
sqlite3.DatabaseError: database disk image is malformed
```

#### 修复步骤

```bash
# 1. 停止服务
sudo systemctl stop formalmath-*

# 2. 备份损坏的数据库
cp /opt/formalmath/data/cognitive.db /opt/formalmath/data/cognitive.db.corrupt.$(date +%Y%m%d)

# 3. 尝试修复
sqlite3 /opt/formalmath/data/cognitive.db.corrupt ".dump" > /tmp/dump.sql
sqlite3 /opt/formalmath/data/cognitive.db.new < /tmp/dump.sql

# 4. 验证修复
sqlite3 /opt/formalmath/data/cognitive.db.new "PRAGMA integrity_check;"

# 5. 替换并重启
mv /opt/formalmath/data/cognitive.db.new /opt/formalmath/data/cognitive.db
sudo systemctl start formalmath-*
```

### 4.2 数据不一致

#### 检测脚本

```python
#!/usr/bin/env python3
# check_data_consistency.py

import sqlite3
import json

def check_cognitive_data():
    """检查认知诊断数据一致性"""
    conn = sqlite3.connect('/opt/formalmath/data/cognitive.db')
    cursor = conn.cursor()

    issues = []

    # 检查孤立记录
    cursor.execute("""
        SELECT d.id FROM diagnoses d
        LEFT JOIN users u ON d.user_id = u.id
        WHERE u.id IS NULL
    """)
    orphaned = cursor.fetchall()
    if orphaned:
        issues.append(f"孤立诊断记录: {len(orphaned)}条")

    # 检查无效分数
    cursor.execute("""
        SELECT id, score FROM diagnoses
        WHERE score < 0 OR score > 100
    """)
    invalid_scores = cursor.fetchall()
    if invalid_scores:
        issues.append(f"无效分数记录: {len(invalid_scores)}条")

    # 检查重复记录
    cursor.execute("""
        SELECT user_id, COUNT(*) as cnt
        FROM diagnoses
        GROUP BY user_id
        HAVING cnt > 1
    """)
    duplicates = cursor.fetchall()
    if duplicates:
        issues.append(f"重复诊断: {len(duplicates)}个用户")

    conn.close()
    return issues

if __name__ == '__main__':
    issues = check_cognitive_data()
    if issues:
        print("发现数据问题:")
        for issue in issues:
            print(f"  - {issue}")
    else:
        print("数据一致性检查通过")
```

### 5.3 内存泄漏

#### 诊断

```python
#!/usr/bin/env python3
# memory_profiler.py

import psutil
import os
import tracemalloc

def monitor_memory():
    process = psutil.Process(os.getpid())

    # 启动内存跟踪
    tracemalloc.start()

    # 记录初始状态
    initial_mem = process.memory_info().rss / 1024 / 1024  # MB
    snapshot1 = tracemalloc.take_snapshot()

    # 执行操作
    # ... 你的代码 ...

    # 记录最终状态
    final_mem = process.memory_info().rss / 1024 / 1024  # MB
    snapshot2 = tracemalloc.take_snapshot()

    # 比较快照
    top_stats = snapshot2.compare_to(snapshot1, 'lineno')

    print(f"内存使用: {initial_mem:.1f}MB -> {final_mem:.1f}MB")
    print("\n内存增长TOP10:")
    for stat in top_stats[:10]:
        print(stat)

if __name__ == '__main__':
    monitor_memory()
```

### 6.1 异常访问检测

#### 症状

- 大量404错误
- 异常IP访问
- 登录失败激增

#### 处理

```bash
# 1. 分析访问日志
grep "404" /var/log/nginx/access.log | awk '{print $1}' | sort | uniq -c | sort -rn | head -20

# 2. 查找暴力破解尝试
grep "Failed login" /var/log/formalmath/app/security.log | awk '{print $4}' | sort | uniq -c | sort -rn

# 3. 封禁IP
sudo iptables -A INPUT -s <SUSPICIOUS_IP> -j DROP

# 或配置fail2ban
sudo fail2ban-client set nginx-http-auth banip <IP>
```

### 6.2 数据泄露响应

#### 应急响应流程

```
1. 立即响应 (5分钟内)
   - 停止相关服务
   - 隔离受影响系统
   - 通知安全团队

2. 评估影响 (30分钟内)
   - 确定泄露范围
   - 识别受影响数据
   - 评估风险等级

3. 遏制措施 (1小时内)
   - 修复安全漏洞
   - 重置受影响凭证
   - 加强监控

4. 恢复服务 (2小时内)
   - 验证修复
   - 逐步恢复服务
   - 持续监控

5. 事后分析 (24小时内)
   - 根因分析
   - 改进措施
   - 更新应急预案
```

## 七、诊断工具

### 7.1 综合诊断脚本

```bash
#!/bin/bash
# comprehensive_diagnose.sh

REPORT_FILE="/tmp/formalmath_diagnosis_$(date +%Y%m%d_%H%M%S).txt"

echo "FormalMath 综合诊断报告" > $REPORT_FILE
echo "========================" >> $REPORT_FILE
echo "时间: $(date)" >> $REPORT_FILE
echo "" >> $REPORT_FILE

# 系统信息
echo "## 系统信息" >> $REPORT_FILE
echo "主机名: $(hostname)" >> $REPORT_FILE
echo "操作系统: $(uname -a)" >> $REPORT_FILE
echo "运行时间: $(uptime)" >> $REPORT_FILE
echo "" >> $REPORT_FILE

# 资源使用
echo "## 资源使用" >> $REPORT_FILE
echo "CPU负载: $(cat /proc/loadavg)" >> $REPORT_FILE
echo "内存使用:" >> $REPORT_FILE
free -h >> $REPORT_FILE
echo "磁盘使用:" >> $REPORT_FILE
df -h >> $REPORT_FILE
echo "" >> $REPORT_FILE

# 服务状态
echo "## 服务状态" >> $REPORT_FILE
for service in formalmath-cognitive formalmath-assessment formalmath-adaptive; do
    echo "服务 $service:" >> $REPORT_FILE
    systemctl is-active $service >> $REPORT_FILE 2>&1
    systemctl status $service --no-pager 2>&1 | head -10 >> $REPORT_FILE
    echo "" >> $REPORT_FILE
done

# 端口检查
echo "## 端口监听" >> $REPORT_FILE
ss -tulpn | grep -E "800[0-3]" >> $REPORT_FILE
echo "" >> $REPORT_FILE

# 健康检查
echo "## 健康检查" >> $REPORT_FILE
for port in 8001 8002 8003; do
    echo "端口 $port:" >> $REPORT_FILE
    curl -s -o /dev/null -w "%{http_code} %{time_total}s\n" \
         >> $REPORT_FILE 2>&1
done
echo "" >> $REPORT_FILE

# Lean环境
echo "## Lean环境" >> $REPORT_FILE
lean --version >> $REPORT_FILE 2>&1
echo "Lake版本:" >> $REPORT_FILE
lake --version >> $REPORT_FILE 2>&1
echo "" >> $REPORT_FILE

# 最近错误
echo "## 最近错误日志" >> $REPORT_FILE
echo "应用错误:" >> $REPORT_FILE
grep -i "error\|exception\|failed" /var/log/formalmath/app/*.log 2>/dev/null | tail -20 >> $REPORT_FILE
echo "" >> $REPORT_FILE

echo "诊断报告已保存到: $REPORT_FILE"
cat $REPORT_FILE
```
