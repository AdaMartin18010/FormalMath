## 系统概述

FormalMath 是一个面向数学教育的综合性知识库系统，包含以下核心能力：

| 能力域 | 描述 | 状态 |
|--------|------|------|
| 内容管理 | 6,500+ 数学文档的结构化存储与管理 | ✅ 生产级 |
| 智能学习 | 认知诊断、学习路径推荐、能力评估 | ✅ 生产级 |
| 形式化证明 | Lean4 定理证明与验证 | ✅ 生产级 |
| 知识图谱 | 概念依赖关系与可视化 | ✅ 生产级 |
| 交互式前端 | React 驱动的学习界面 | ✅ 生产级 |

## 架构原则

### 1. 模块化设计 (Modularity)

```
┌─────────────────────────────────────────────────────────┐
│                      系统边界                            │
├─────────────────────────────────────────────────────────┤
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌─────────┐ │
│  │ 内容模块  │  │ 学习模块  │  │ 评估模块  │  │ 可视化  │ │
│  └──────────┘  └──────────┘  └──────────┘  └─────────┘ │
│       │              │              │            │      │
│       └──────────────┴──────────────┴────────────┘      │
│                      核心接口层                          │
└─────────────────────────────────────────────────────────┘
```

- 每个模块独立开发、测试、部署
- 模块间通过明确定义的接口通信
- 支持模块的热插拔

### 2. 数据驱动 (Data-Driven)

- 内容、配置、元数据分离存储
- 统一的元数据标准 (YAML Frontmatter)
- 支持多格式导出 (Markdown, HTML, JSON, PDF)

### 3. 可扩展性 (Extensibility)

- 插件式工具架构
- 标准化 API 接口
- 支持第三方系统集成

### 4. 质量优先 (Quality First)

- 自动化质量检查流程
- 多层次审核机制
- 持续集成/持续部署 (CI/CD)

## 核心组件

### 1. 内容管理组件

| 组件 | 职责 | 输入 | 输出 |
|------|------|------|------|
| 文档解析器 | 解析 Markdown 文档结构 | .md 文件 | AST / 结构化数据 |
| 元数据提取器 | 提取 YAML Frontmatter | .md 文件 | 元数据字典 |
| 链接管理器 | 管理内部/外部链接 | 文档集合 | 链接图谱 |
| 版本控制器 | 管理文档版本 | Git 历史 | 版本信息 |

### 2. 学习系统组件

| 组件 | 职责 | 算法/技术 |
|------|------|-----------|
| 认知诊断引擎 | 诊断学习者知识状态 | HCD (Hierarchical Cognitive Diagnosis) |
| 路径推荐引擎 | 生成个性化学习路径 | A* + 启发式搜索 |
| 评估引擎 | 多维度能力评估 | 五维数学能力模型 (MAA) |
| 反馈生成器 | 生成个性化学习反馈 | 规则引擎 + 模板 |

### 3. 形式化证明组件

| 组件 | 职责 | 技术 |
|------|------|------|
| Lean4 编译器 | 编译验证证明 | Lean4 + Mathlib4 |
| 定理映射器 | 映射到 Mathlib4 | 语义匹配 |
| 证明可视化 | 展示证明结构 | 交互式编辑器 |

### 4. 知识图谱组件

| 组件 | 职责 | 算法 |
|------|------|------|
| 概念抽取器 | 从文档抽取概念 | NLP + 规则 |
| 关系构建器 | 构建概念依赖关系 | 图算法 |
| 图谱可视化 | 生成可视化图表 | D3.js / Mermaid |

## 数据流

### 文档处理流程

```
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│  创建    │ -> │  解析    │ -> │  验证    │ -> │  处理    │ -> │  存储    │
│  文档    │    │  文档    │    │  质量    │    │  元数据  │    │  索引    │
└─────────┘    └─────────┘    └─────────┘    └─────────┘    └─────────┘
     │              │              │              │              │
     ▼              ▼              ▼              ▼              ▼
  Markdown    提取 Frontmatter  质量检查     更新知识图谱    生成索引
  编辑器       解析内容结构     MSC 验证     构建链接关系    全文搜索
```

### 学习流程

```
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│ 学习者   │ -> │ 认知    │ -> │ 路径    │ -> │ 学习    │ -> │ 评估    │
│ 注册    │    │ 诊断    │    │ 推荐    │    │ 过程    │    │ 反馈    │
└─────────┘    └─────────┘    └─────────┘    └─────────┘    └─────────┘
     │              │              │              │              │
     ▼              ▼              ▼              ▼              ▼
  创建画像     评估知识状态    生成学习路径   跟踪学习进度   生成能力报告
  设置目标     识别薄弱点     优化学习顺序   记录交互数据   提供改进建议
```

### 容器化部署

```dockerfile
# Dockerfile 示例
FROM node:18-alpine AS frontend
WORKDIR /app
COPY FormalMath-Interactive/ .
RUN npm ci && npm run build

FROM python:3.11-slim AS tools
WORKDIR /tools
COPY tools/ .
RUN pip install -r requirements.txt

FROM nginx:alpine
COPY --from=frontend /app/dist /usr/share/nginx/html
COPY nginx.conf /etc/nginx/conf.d/default.conf
EXPOSE 80
```
