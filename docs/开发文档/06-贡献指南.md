## 开发环境设置

### 基础环境

```bash
# 1. 克隆仓库
git clone https://github.com/formalmath/FormalMath.git
cd FormalMath

# 2. 设置 Git 钩子
# 预提交检查
ln -s ../../tools/contributor-workflow/pre_commit_check.py .git/hooks/pre-commit
chmod +x .git/hooks/pre-commit

# 3. 创建 Python 虚拟环境
python -m venv venv
source venv/bin/activate  # Linux/Mac
# 或
venv\Scripts\activate  # Windows
```

### Python 开发环境

```bash
# 安装开发依赖
pip install -r requirements-dev.txt

# 运行测试
pytest tests/unit

# 运行代码检查
pylint tools/
black tools/ --check
mypy tools/
```

### 前端开发环境

```bash
cd FormalMath-Interactive

# 安装依赖
npm install

# 启动开发服务器
npm run dev

# 运行测试
npm run test

# 代码检查
npm run lint
npm run type-check
```

### TypeScript/React 代码规范

```typescript
// components/math/ConceptCard.tsx
import React, { useCallback, useMemo } from 'react';
import { cn } from '@/lib/utils';

/**
 * 概念卡片属性
 */
interface ConceptCardProps {
  /** 概念 ID */
  id: string;
  /** 概念名称 */
  name: string;
  /** 难度等级 */
  difficulty: 'L0' | 'L1' | 'L2' | 'L3' | 'L4';
  /** 是否已掌握 */
  mastered?: boolean;
  /** 点击回调 */
  onClick?: (id: string) => void;
  /** 自定义类名 */
  className?: string;
}

/**
 * 概念卡片组件
 * 
 * 显示概念基本信息，支持点击交互。
 * 
 * @example
 * <ConceptCard
 *   id="group-theory"
 *   name="群论"
 *   difficulty="L2"
 *   onClick={(id) => console.log(id)}
 * />
 */
export const ConceptCard: React.FC<ConceptCardProps> = ({
  id,
  name,
  difficulty,
  mastered = false,
  onClick,
  className,
}) => {
  // 计算难度颜色
  const difficultyColor = useMemo(() => {
    const colors = {
      L0: 'bg-green-100 text-green-800',
      L1: 'bg-blue-100 text-blue-800',
      L2: 'bg-yellow-100 text-yellow-800',
      L3: 'bg-orange-100 text-orange-800',
      L4: 'bg-red-100 text-red-800',
    };
    return colors[difficulty];
  }, [difficulty]);

  // 点击处理
  const handleClick = useCallback(() => {
    onClick?.(id);
  }, [id, onClick]);

  return (
    <div
      className={cn(
        'p-4 rounded-lg border cursor-pointer transition-shadow',
        'hover:shadow-md',
        mastered && 'border-green-500 bg-green-50',
        className
      )}
      onClick={handleClick}
      data-testid={`concept-card-${id}`}
    >
      <div className="flex items-center justify-between">
        <h3 className="font-semibold text-lg">{name}</h3>
        <span className={cn('px-2 py-1 rounded text-sm', difficultyColor)}>
          {difficulty}
        </span>
      </div>
      {mastered && (
        <div className="mt-2 text-sm text-green-600">
          ✓ 已掌握
        </div>
      )}
    </div>
  );
};

export default ConceptCard;
```

### Lean4 代码规范

```lean
/-
# 定理名称：拉格朗日定理

## 作者：你的名字
## 创建日期：YYYY-MM-DD
## 关联 Issue：#123

## 数学背景

拉格朗日定理是群论中的基本定理，描述了有限群中子群的阶与群的阶之间的关系。

## 实现说明

使用 Mathlib 中的 Fintype 和 Subgroup 类型。
-/

import Mathlib

namespace FormalMath

/--
拉格朗日定理：有限群中子群的阶整除群的阶。

## 定理陈述

设 $G$ 是一个有限群，$H$ 是 $G$ 的子群，则 $|H|$ 整除 $|G|$。

## 证明概要

1. 考虑 $H$ 在 $G$ 中的左陪集分解
2. 每个陪集的大小等于 $|H|$
3. 陪集划分 $G$，因此 $|G| = |H| \cdot [G:H]$
-/
theorem lagrange_theorem {G : Type*} [Group G] [Fintype G]
    (H : Subgroup G) [Fintype H] :
    Fintype.card H ∣ Fintype.card G := by
  -- 证明步骤 1：建立陪集与群的基数关系
  have h1 : Fintype.card G = Fintype.card H * Fintype.card (G ⧸ H) := by
    rw [← Fintype.card_prod]
    exact Fintype.card_congr (quotientGroupEquivQuotient H)
  
  -- 证明步骤 2：由等式直接得出整除关系
  exact Dvd.intro _ (Eq.symm h1)

end FormalMath
```

## 提交规范

### 提交信息格式

```
<type>(<scope>): <subject>

<body>

<footer>
```

**类型 (type)**：

| 类型 | 说明 | 示例 |
|------|------|------|
| `feat` | 新功能 | `feat(assessment): add API endpoint` |
| `fix` | 错误修复 | `fix(metadata): handle missing fields` |
| `docs` | 文档更新 | `docs(api): update examples` |
| `style` | 格式调整 | `style(tools): format with black` |
| `refactor` | 代码重构 | `refactor(engine): optimize query` |
| `test` | 测试相关 | `test(unit): add missing tests` |
| `chore` | 构建/工具 | `chore(deps): update requirements` |

**范围 (scope)**：

- `assessment` - 评估系统
- `metadata` - 元数据系统
- `frontend` - 前端
- `api` - API 接口
- `docs` - 文档
- `tools` - 工具
- `core` - 核心

**示例**：

```bash
# 简单提交
git commit -m "feat(assessment): add formative assessment API"

# 详细提交
git commit -m "feat(assessment): implement formative assessment endpoint

- Add POST /api/v1/assessments/formative
- Implement scoring algorithm based on MAA standards
- Add comprehensive unit tests
- Update API documentation

Closes #234"

# 修复提交
git commit -m "fix(metadata): correct YAML parsing for special characters

- Handle Unicode characters in metadata fields
- Add validation for malformed YAML
- Add test cases for edge cases

Fixes #456"
```

## 常见问题

### Q: 如何调试失败的测试？

```bash
# 显示详细输出
pytest tests/unit/test_module.py -v -s

# 只运行特定测试
pytest tests/unit/test_module.py::TestClass::test_method -v

# 使用 pdb 调试
pytest tests/unit/test_module.py --pdb

# 生成 HTML 报告
pytest tests/ --html=report.html
```

### Q: 如何处理合并冲突？

```bash
# 1. 获取最新代码
git fetch upstream
git rebase upstream/main

# 2. 解决冲突
# 编辑冲突文件，保留需要的更改

# 3. 标记已解决
git add <resolved-files>
git rebase --continue

# 4. 强制推送 (如果是 rebase)
git push origin feature/branch --force-with-lease
```

### Q: 预提交检查失败怎么办？

```bash
# 查看详细错误
python tools/contributor-workflow/pre_commit_check.py --verbose

# 自动修复格式问题
black tools/
npm run lint:fix

# 跳过检查 (紧急情况)
git commit --no-verify -m "紧急修复"
```
