# FormalMath文档格式标准

**制定日期**: 2026年01月02日
**适用范围**: 所有Markdown文档
**状态**: ✅ 已生效

---

## 📋 格式标准概述

所有FormalMath项目的文档必须遵循统一的格式标准，确保文档结构清晰、排版一致、易于阅读和维护。

---

## 🎯 文档结构标准

### 1. 文档头部

**必须包含**：

```markdown
# [文档标题]

[可选：简短描述或引言，不超过3行]

## 📋 目录

- [文档标题](#文档标题)
  - [一级章节](#一级章节)
    - [二级章节](#二级章节)
      - [三级章节](#三级章节)
```

**说明**：
- 一级标题使用 `#`（单个#）
- 目录使用 `## 📋 目录` 作为二级标题
- 目录必须包含所有一级和二级章节的链接
- 目录使用emoji 📋 标识

---

### 2. 章节编号标准

#### 2.1 一级章节（主章节）

**格式**：使用中文数字编号

```markdown
## 一、[章节名称]

## 二、[章节名称]

## 三、[章节名称]
```

**说明**：
- 使用"一、二、三、四、五..."编号
- 章节名称简洁明了，不超过15字
- 章节之间用 `---` 分隔

#### 2.2 二级章节（子章节）

**格式**：使用数字编号

```markdown
### 1.1 [子章节名称]

### 1.2 [子章节名称]

### 2.1 [子章节名称]
```

**说明**：
- 使用"主章节号.子章节号"格式（如1.1, 1.2, 2.1）
- 子章节名称简洁明了

#### 2.3 三级章节（小节）

**格式**：使用数字编号或项目符号

```markdown
#### 1.1.1 [小节名称]

或

**核心概念**：

**定义**：

**定理**：
```

**说明**：
- 可以使用"1.1.1"格式，或使用加粗文字标识
- 根据内容选择合适格式

---

### 3. 分隔线标准

**使用场景**：

1. **章节之间**：使用 `---`（三个短横线）

```markdown
## 一、[章节名称]

内容...

---

## 二、[章节名称]
```

2. **文档末尾**：使用 `---` 分隔正文和元信息

```markdown
---

**文档状态**: ✅ 完成
**字数**: 约X,XXX字
**最后更新**: YYYY年MM月DD日
```

**禁止**：
- ❌ 使用 `***` 或 `___` 作为分隔线
- ❌ 在章节内部使用过多分隔线

---

### 4. 代码块格式标准

#### 4.1 数学公式

**行内公式**：使用 `$...$`

```markdown
设 $f(x) = x^2 + 1$，则...
```

**独立公式**：使用 `$$...$$` 或 LaTeX块

```markdown
费马小定理：

$$a^{p-1} \equiv 1 \pmod{p}$$

或

\[
a^{p-1} \equiv 1 \pmod{p}
\]
```

#### 4.2 代码示例

**格式**：使用代码块，指定语言类型

```markdown
```python
def example():
    return "Hello"
```

**文本块**：使用 `text` 或 `text` 类型

```markdown
```text
定义：
- 点：...
- 线：...
```
```

---

### 5. 列表格式标准

#### 5.1 无序列表

**格式**：使用 `-` 或 `*`

```markdown
- 第一项
- 第二项
- 第三项
```

#### 5.2 有序列表

**格式**：使用数字编号

```markdown
1. 第一项
2. 第二项
3. 第三项
```

#### 5.3 嵌套列表

**格式**：使用缩进

```markdown
- 第一级
  - 第二级
    - 第三级
```

---

### 6. 表格格式标准

**格式**：使用Markdown表格语法

```markdown
| 列1 | 列2 | 列3 |
|-----|-----|-----|
| 数据1 | 数据2 | 数据3 |
| 数据4 | 数据5 | 数据6 |
```

**说明**：
- 对齐方式：左对齐（默认）或使用 `:---`, `:---:`, `---:`
- 表头必须清晰
- 表格前后留空行

---

### 7. 引用格式标准

#### 7.1 引用块

**格式**：使用 `>`

```markdown
> 这是一段引用文字。
```

#### 7.2 强调文字

**格式**：
- **加粗**：使用 `**文字**`
- *斜体*：使用 `*文字*`
- `代码`：使用 `` `代码` ``

---

### 8. 文档尾部标准

**必须包含**：

```markdown
---

**文档状态**: ✅ 完成 / ⏳ 进行中 / 🔄 待补充
**字数**: 约X,XXX字（或X,XXX词）
**最后更新**: YYYY年MM月DD日
```

**状态说明**：
- ✅ 完成：文档内容完整，达到质量标准
- ⏳ 进行中：文档正在补充中
- 🔄 待补充：文档需要补充内容

---

## 📐 排版标准

### 1. 标题层级

**标准层级结构**：

```
# 文档标题（一级）
## 📋 目录（二级）
## 一、主章节（二级）
### 1.1 子章节（三级）
#### 1.1.1 小节（四级）
**加粗标识**（五级）
```

**禁止**：
- ❌ 跳过层级（如从二级直接跳到四级）
- ❌ 使用过多层级（超过5级）

---

### 2. 段落格式

**标准**：
- 段落之间留一个空行
- 段落首行不缩进
- 段落长度适中（3-8行）

**示例**：

```markdown
这是第一段文字。内容充实，表达清晰。

这是第二段文字。与第一段之间有空行分隔。
```

---

### 3. 列表和代码块

**标准**：
- 列表前后留空行
- 代码块前后留空行
- 代码块使用正确的语言标识

**示例**：

```markdown
以下是关键点：

- 第一点
- 第二点
- 第三点

代码示例：

```python
def example():
    pass
```

## 🚫 禁止格式

### 1. 禁止的标题格式

- ❌ `## 第一章`（应使用"一、"）
- ❌ `## Chapter 1`（应使用中文）
- ❌ `## 1. [标题]`（一级章节应使用中文数字）

### 2. 禁止的目录格式

- ❌ 目录中缺少章节链接
- ❌ 目录格式不一致
- ❌ 目录不完整（缺少部分章节）

### 3. 禁止的内容格式

- ❌ 模板文字（如"该应用的核心要点和意义"）
- ❌ 占位符（如"待补充"、"TODO"）
- ❌ 重复的通用描述
- ❌ 空泛的总结

```text

f(x) = λ∫K(x,y)f(y)dy

```

希尔伯特空间H是满足以下条件的复向量空间：

```text
1. 内积：⟨·,·⟩: H×H → ℂ
2. 范数：‖u‖ = √⟨u,u⟩
3. 完备性：任意Cauchy列收敛
```
