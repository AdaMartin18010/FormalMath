# FormalMath项目目录结构规范

**制定日期**: 2025年12月29日
**版本**: v1.0
**适用范围**: 项目所有目录
**状态**: 正式执行

---

## 📋 规范概述

本文档定义FormalMath项目中所有目录的标准结构，确保项目目录结构的一致性、清晰性和可维护性。

---

## 🎯 结构原则

1. **层次性**: 清晰的目录层级关系
2. **一致性**: 同类模块使用统一的结构
3. **可扩展性**: 预留扩展空间
4. **可导航性**: 易于查找和导航
5. **标准化**: 遵循国际标准实践

## 📁 标准目录结构

### 1. 根目录结构

```
FormalMath/
├── README.md                          # 项目主文档
├── LICENSE                            # 许可证
├── 00-项目状态.md                     # 项目状态（最新）
├── 00-文档索引.md                     # 文档索引
├── 00-归档/                           # 归档目录
│   ├── 2025年12月/
│   ├── 2025年11月/
│   └── 历史版本/
├── docs/                              # 核心文档目录
├── concept/                           # 核心概念目录
├── 数学家理念体系/                    # 数学家理念体系
├── research/                          # 研究文档目录
├── project/                           # 项目管理目录
├── Books/                             # 书籍资源目录
├── view/                              # 视图文档目录
├── Critique/                          # 批判性评价目录
└── tools/                             # 工具脚本目录（待创建）
```

### 2. 模块标准结构

**格式**: 每个主要模块应遵循以下结构

```
模块名/
├── 00-管理文档/                       # 管理文档目录（可选）
│   ├── 00-项目状态.md
│   ├── 00-文档索引.md
│   └── README.md
├── 01-核心内容/                       # 核心内容目录
│   ├── 01-子模块1/
│   │   ├── 01-文档1.md
│   │   └── 02-文档2.md
│   └── 02-子模块2/
├── 02-扩展内容/                       # 扩展内容目录（可选）
├── 03-应用案例/                       # 应用案例目录（可选）
├── 99-归档文档/                       # 归档文档目录（可选）
└── README.md                          # 模块说明文档
```

**说明**:

- `00-管理文档/`: 存放项目状态、索引等管理文档（可选，也可直接在模块根目录）
- `01-核心内容/`: 存放核心内容文档（必需）
- `02-扩展内容/`: 存放扩展内容（可选）
- `99-归档文档/`: 存放归档文档（可选）

### 3. 子模块结构

**格式**: 子模块应遵循以下结构

```
子模块名/
├── 00-README.md                       # 子模块说明（可选）
├── 01-文档1.md                        # 内容文档
├── 02-文档2.md
└── ...
```

## 🚫 禁止使用的结构

### 1. 禁止的目录命名

- ❌ 使用特殊字符: `目录<>名称/`
- ❌ 使用空格: `目录 名称/` （使用`-`或`_`代替）
- ❌ 使用中文标点: `目录，名称/`
- ❌ 过长的名称: `这是一个非常非常长的目录名称/`

### 2. 禁止的结构模式

- ❌ 扁平结构: 所有文档放在一个目录
- ❌ 过深嵌套: 超过5层嵌套
- ❌ 混乱编号: 不连续的编号（如01, 03, 05, 跳过02）
- ❌ 重复结构: 同一内容出现在多个位置

### 3. 推荐的结构模式

- ✅ 清晰的层级: 2-4层嵌套
- ✅ 连续编号: 01, 02, 03, ...
- ✅ 统一命名: 同类目录使用统一命名
- ✅ 预留空间: 预留编号空间（如01-05, 10-15）
