## 📚 概述

并行计算是利用多个处理单元同时执行计算任务的技术，通过将大规模问题分解为可并发执行的子问题来加速求解。在科学计算、人工智能和大数据分析等领域，并行计算已成为处理海量数据和复杂模型的核心技术。

本教学文档系统介绍并行计算的理论基础，包括可扩展性定律、并行算法设计、域分解方法和负载均衡策略，着重阐述大规模科学计算中的应用实践。

**核心问题**：给定计算任务，如何在 $p$ 个处理器上高效并行执行，实现接近线性的加速比？

### 早期并行计算 (1960-1980)

1960年代，超级计算机开始采用向量处理和流水线技术。1972年，ILLIAC IV成为第一台大规模并行处理机，采用 $8 \times 8$ 处理器阵列。

关键发展：
- **1966年**：Flynn提出计算分类法（SISD、SIMD、MISD、MIMD）
- **1974年**：Kuck建立并行复杂度理论
- **1978年**：Amdahl发表著名定律，指出并行加速的理论限制

### MPP与集群时代 (1990-2005)

1990年代，大规模并行处理机(MPP)和集群计算兴起：
- **1992年**：MPI标准发布，统一消息传递编程接口
- **1997年**：IBM Deep Blue击败国际象棋世界冠军，展示并行计算威力
- **1997年**：OpenMP标准发布，简化共享内存并行编程

**Top500里程碑**：
- 1993年：首台Teraflop级超级计算机（Intel Paragon）
- 2004年：首台10 Teraflop级系统（IBM BlueGene/L）

### 众核与异构计算 (2006-至今)

2006年后，GPU计算和异构架构成为主流：
- **2006年**：NVIDIA发布CUDA，开启通用GPU计算时代
- **2008年**：Intel推出Larrabee，探索众核架构
- **2012年**：AlexNet使用GPU训练，开启深度学习革命

**现代超级计算机**：

- Fugaku (2020)：442 Petaflop，基于ARM架构

## 📐 核心理论

### 1. Amdahl定律与Gustafson定律

#### Amdahl定律

1967年，Gene Amdahl提出并行加速比的理论上限。

**设定**：程序中串行部分比例为 $f$，并行部分比例为 $1-f$。

**定理（Amdahl定律）**：
使用 $p$ 个处理器的最大加速比为：
$$S(p) = \frac{1}{f + \frac{1-f}{p}}$$

**渐近行为**：
$$\lim_{p \to \infty} S(p) = \frac{1}{f}$$

**数值示例**：

| 串行比例 $f$ | 处理器数 $p$ | 加速比 $S(p)$ | 效率 $E(p)$ |
|------------|------------|--------------|------------|
| 0.01 | 100 | 50.3 | 50.3% |
| 0.01 | 1000 | 90.9 | 9.1% |
| 0.10 | 100 | 9.2 | 9.2% |
| 0.10 | 1000 | 9.9 | 1.0% |

**启示**：即使只有1%的串行代码，最大加速比也被限制在100倍以内。

#### Gustafson定律

1988年，John Gustafson提出弱可扩展性观点。

**定理（Gustafson定律）**：
假设问题规模随处理器数增加而增长，固定每个处理器的计算量，则：
$$S(p) = p - f(p-1)$$

**推导**：设串行时间为 $f$，并行时间为 $1-f$，则总工作量 $W(p) = f + p(1-f)$。

加速比：
$$S(p) = \frac{W(p)}{W(1)} = \frac{f + p(1-f)}{1} = p - f(p-1)$$

**数值示例**（$f = 0.01$）：
- $p = 100$：$S(100) = 100 - 0.99 = 99.01$
- $p = 1000$：$S(1000) = 1000 - 9.9 = 990.1$

**启示**：对于可扩展问题（问题规模随处理器增加），可以实现接近线性的加速比。

#### 对比分析

| 维度 | Amdahl定律 | Gustafson定律 |
|-----|-----------|---------------|
| 问题规模 | 固定 | 随处理器增加 |
| 关注点 | 强可扩展性 | 弱可扩展性 |
| 适用场景 | 固定工作量的加速 | 大规模问题求解 |
| 结论 | 加速比有上限 | 加速比可线性增长 |

### 2. 并行矩阵乘法算法

#### Cannon算法

对于 $n \times n$ 矩阵乘法 $C = AB$，在 $\sqrt{p} \times \sqrt{p}$ 处理器网格上执行。

**算法步骤**：

1. **初始对齐**：
   - 矩阵 $A$ 的第 $i$ 行向左循环移位 $i$ 次
   - 矩阵 $B$ 的第 $j$ 列向上循环移位 $j$ 次

2. **计算与通信**：重复 $\sqrt{p}$ 次
   - 局部矩阵相乘累加到 $C$
   - $A$ 向左循环移位1位
   - $B$ 向上循环移位1位

**伪代码**：

```

// 初始对齐
for i = 0 to √p-1:
    shift_left(A[i][*], i)
for j = 0 to √p-1:
    shift_up(B[*][j], j)

// 计算循环
for k = 0 to √p-1:
    C_local += A_local * B_local
    shift_left(A_local, 1)
    shift_up(B_local, 1)

```

#### 复杂度分析

**串行复杂度**：$O(n^3)$

**并行复杂度**（Cannon算法）：
- **计算时间**：$T_{comp} = O(n^3/p)$
- **通信时间**：$T_{comm} = O(n^2/\sqrt{p})$
- **总时间**：$T(p) = O(n^3/p + n^2/\sqrt{p})$

**效率分析**：
当 $n \gg p$（细粒度并行），$T(p) \approx O(n^3/p)$，效率接近100%。

**其他算法**：
- **SUMMA算法**：更灵活的广播方案
- **Strassen算法**：$O(n^{2.81})$ 计算复杂度，但通信开销大
- **3D算法**：利用额外维度减少通信

### 3. 域分解方法

域分解(Domain Decomposition)是求解大规模偏微分方程的核心并行策略。

#### Schwarz交替法

**基本思想**：将全域 $\Omega$ 分解为重叠子域 $\Omega_1, \Omega_2, \ldots, \Omega_N$，交替求解。

**算法**：

```

初始化 u⁰
for k = 0, 1, 2, ...:
    // 并行求解各子域
    for i = 1 to N in parallel:
        求解 Lu_i^{k+1} = f 在 Ω_i 上
        边界条件: u_i^{k+1} = u^k 在 ∂Ω_i ∩ Ω 上
    
    // 更新全局解
    u^{k+1} = 组合各子域解

```

**收敛性**：对于对称正定椭圆问题，Schwarz方法收敛。

#### FETI与Neumann-Neumann方法

**FETI (Finite Element Tearing and Interconnect)**：
- 将子域间界面视为约束
- 通过拉格朗日乘子将问题转化为鞍点问题
- 使用预条件共轭梯度法求解界面问题

**数学形式**：
$$\min_{u_i} \sum_{i=1}^N \frac{1}{2}u_i^T K_i u_i - f_i^T u_i$$
$$\text{s.t.} \quad B_i u_i = 0 \text{ (界面连续性)}$$

**Neumann-Neumann方法**：
- 使用Neumann边界条件求解子域问题
- 适合非均匀介质，具有良好的可扩展性

### 4. 负载均衡策略

#### 静态负载均衡

**划分策略**：
1. **块划分**：将数据均匀分配给处理器
   - 1D划分：每处理器 $n/p$ 行
   - 2D划分：每处理器 $(n/\sqrt{p}) \times (n/\sqrt{p})$ 块

2. **循环划分**：处理器 $i$ 获得索引 $i, i+p, i+2p, \ldots$
   - 适合负载变化大的情况

3. **图划分**：使用METIS等工具最小化边切割
   - 目标：平衡顶点数，最小化跨域边数

**复杂度**：
- 块划分通信：$O(n/\sqrt{p})$（2D）
- 循环划分通信：$O(n)$（更高，但负载更均衡）

#### 动态负载均衡

**适用场景**：
- 自适应网格细化
- 粒子模拟（分子动力学、N体问题）
- 任务执行时间不确定

**策略**：
1. **任务窃取(Work Stealing)**：空闲处理器从繁忙处理器窃取任务
2. **任务池(Task Pool)**：中央任务调度器分配任务
3. **扩散算法(Diffusion)**：根据邻居负载差异迁移任务

**质量度量**：
$$\text{负载不均衡度} = \frac{\max_i T_i}{\bar{T}} - 1$$
其中 $T_i$ 是处理器 $i$ 的执行时间，$\bar{T}$ 是平均时间。

## 🔍 实例表征

### 例1：MPI并行矩阵乘法

**问题**：计算 $C = AB$，其中 $A, B, C \in \mathbb{R}^{n \times n}$，使用 $p$ 个进程。

**Cannon算法实现**（简化版）：

| 步骤 | 操作 | 通信量 | 计算量 |
|-----|------|-------|-------|
| 初始对齐 | 循环移位 | $O(n^2)$ | - |
| 迭代 $\sqrt{p}$ 次 | 局部乘法+移位 | $O(n^2\sqrt{p})$ | $O(n^3/p)$ |
| 结果收集 | 聚合 | $O(n^2)$ | - |

**性能模型**：
$$T(n, p) = \frac{2n^3}{p \cdot R_{peak}} + \frac{3n^2 \sqrt{p}}{B_{network}} + 2\sqrt{p} \cdot L_{latency}$$

其中 $R_{peak}$ 是峰值计算速率，$B_{network}$ 是网络带宽，$L_{latency}$ 是网络延迟。

**数值实验**（$n = 4096$，双精度）：

| $p$ | 计算时间(s) | 通信时间(s) | 总时间(s) | 加速比 |
|-----|-----------|-----------|---------|-------|
| 1 | 137.4 | - | 137.4 | 1.0 |
| 4 | 34.4 | 0.8 | 35.2 | 3.9 |
| 16 | 8.6 | 1.2 | 9.8 | 14.0 |
| 64 | 2.2 | 2.1 | 4.3 | 31.9 |

### 例2：OpenMP并行for循环

**问题**：并行计算向量点积 $s = \sum_{i=1}^n a_i b_i$。

```c
// 串行版本
double dot_product(double *a, double *b, int n) {
    double s = 0.0;
    for (int i = 0; i < n; i++)
        s += a[i] * b[i];
    return s;
}

// OpenMP并行版本
#include <omp.h>
double dot_product_omp(double *a, double *b, int n, int num_threads) {
    double s = 0.0;
    omp_set_num_threads(num_threads);
    
    #pragma omp parallel for reduction(+:s)
    for (int i = 0; i < n; i++)
        s += a[i] * b[i];
    
    return s;
}

```

**性能分析**（$n = 10^8$）：

| 线程数 | 时间(ms) | 加速比 | 效率 |
|-------|---------|-------|------|
| 1 | 285 | 1.0 | 100% |
| 2 | 148 | 1.93 | 96% |
| 4 | 76 | 3.75 | 94% |
| 8 | 42 | 6.79 | 85% |
| 16 | 28 | 10.18 | 64% |

效率下降原因：内存带宽瓶颈、缓存竞争、同步开销。

### 例3：域分解求解泊松方程

**问题**：在 $\Omega = [0,1]^2$ 上求解 $-\nabla^2 u = f$，使用5点差分格式。

**域分解设置**：
- 将 $n \times n$ 网格分为 $2 \times 2$ 子域
- 每个子域大小 $(n/2+1) \times (n/2+1)$（含重叠）

**Schwarz迭代**（$n = 512$）：

| 迭代 $k$ | 残差 $||r_k||_2$ | 残差比 $||r_k||/||r_{k-1}||$ |
|---------|----------------|---------------------------|
| 0 | 1.000e+00 | - |
| 1 | 2.345e-01 | 0.235 |
| 2 | 5.678e-02 | 0.242 |
| 3 | 1.234e-02 | 0.217 |
| 4 | 2.567e-03 | 0.208 |
| 5 | 5.123e-04 | 0.200 |

收敛因子约为 0.22，符合理论预测 $O(1 - 1/\sqrt{Hh}})$，其中 $H$ 是子域大小，$h$ 是网格间距。

## 🧠 大规模科学计算应用

### 1. 气候模拟

**社区地球系统模型(CESM)**：
- **空间分辨率**：1°（约100km）→ 0.25°（约25km）
- **时间步长**：30分钟（大气），1小时（海洋）
- **计算规模**：10,000+ 核心
- **模拟周期**：百年尺度气候预测

**并行策略**：
- 谱元方法 + 区域分解
- 时间并行（并行时间积分）
- I/O并行（并行NetCDF）

### 2. 分子动力学

**LAMMPS和GROMACS**：
- **系统规模**：10^9 原子
- **力场计算**：短程（邻居列表）+ 长程（PME）
- **并行方案**：
  - 空间分解：每个处理器负责空间区域
  - 力分解：每个处理器计算特定原子对的力
  - 原子分解：每个处理器负责特定原子

**性能**：在GPU集群上达到 Petaflop 级别。

### 3. 计算流体力学

**OpenFOAM和CFD++**：
- **控制方程**：Navier-Stokes方程
- **网格规模**：10^8 单元
- **并行方法**：
  - 网格分解（METIS/ParMETIS）
  - 隐式求解器 + Krylov子空间方法
  - 代数多重网格(AMG)预条件

**典型应用**：飞机气动设计、发动机燃烧模拟。

## 🔧 技术实现表征

### MPI实现

```c
#include <mpi.h>
#include <stdio.h>
#include <stdlib.h>

// 并行向量加法
void vector_add_parallel(double *a, double *b, double *c, int n_local) {
    for (int i = 0; i < n_local; i++)
        c[i] = a[i] + b[i];
}

int main(int argc, char **argv) {
    int rank, size;
    int n = 1000000;  // 总向量大小
    int n_local;
    double *a_local, *b_local, *c_local;
    
    MPI_Init(&argc, &argv);
    MPI_Comm_rank(MPI_COMM_WORLD, &rank);
    MPI_Comm_size(MPI_COMM_WORLD, &size);
    
    // 计算本地数据量
    n_local = n / size;
    if (rank < n % size) n_local++;
    
    // 分配本地数组
    a_local = (double*)malloc(n_local * sizeof(double));
    b_local = (double*)malloc(n_local * sizeof(double));
    c_local = (double*)malloc(n_local * sizeof(double));
    
    // 初始化数据（实际应用中从文件读取）
    for (int i = 0; i < n_local; i++) {
        a_local[i] = 1.0;
        b_local[i] = 2.0;
    }
    
    // 并行计算
    double start = MPI_Wtime();
    vector_add_parallel(a_local, b_local, c_local, n_local);
    double end = MPI_Wtime();
    
    if (rank == 0)
        printf("Parallel time: %f seconds\n", end - start);
    
    free(a_local); free(b_local); free(c_local);
    MPI_Finalize();
    return 0;
}

```

### CUDA实现

```cuda
#include <cuda_runtime.h>

// CUDA核函数：向量加法
__global__ void vector_add_kernel(double *a, double *b, double *c, int n) {
    int idx = blockIdx.x * blockDim.x + threadIdx.x;
    if (idx < n)
        c[idx] = a[idx] + b[idx];
}

// 主机函数
void vector_add_gpu(double *a_host, double *b_host, double *c_host, int n) {
    double *a_dev, *b_dev, *c_dev;
    size_t size = n * sizeof(double);
    
    // 分配设备内存
    cudaMalloc(&a_dev, size);
    cudaMalloc(&b_dev, size);
    cudaMalloc(&c_dev, size);
    
    // 拷贝数据到设备
    cudaMemcpy(a_dev, a_host, size, cudaMemcpyHostToDevice);
    cudaMemcpy(b_dev, b_host, size, cudaMemcpyHostToDevice);
    
    // 启动核函数
    int threadsPerBlock = 256;
    int blocksPerGrid = (n + threadsPerBlock - 1) / threadsPerBlock;
    vector_add_kernel<<<blocksPerGrid, threadsPerBlock>>>(a_dev, b_dev, c_dev, n);
    
    // 拷贝结果回主机
    cudaMemcpy(c_host, c_dev, size, cudaMemcpyDeviceToHost);
    
    // 释放设备内存
    cudaFree(a_dev);
    cudaFree(b_dev);
    cudaFree(c_dev);
}

```
