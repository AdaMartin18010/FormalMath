## 📏 质量标准

### 数学内容标准

#### 准确性（Accuracy）

- ✅ **定义准确**：数学定义符合国际通用标准
- ✅ **定理正确**：定理陈述无遗漏条件
- ✅ **证明严谨**：证明逻辑严密，无跳跃
- ✅ **引用可靠**：引用来源权威、可追溯

#### 完整性（Completeness）

| 文档类型 | 必需元素 |
|----------|----------|
| 基础版 | 定义、直观解释、例子 |
| 增强版 | + 定理陈述、证明概要、应用 |
| 深度版 | + 完整证明、推广、历史背景 |
| 国际标准版 | + 多语言术语、课程对齐 |

#### 一致性（Consistency）

- 术语使用一致
- 符号表示统一
- 风格格式规范
- 引用格式统一

### 文档质量标准

#### 可读性

- 语言清晰简洁
- 逻辑结构合理
- 段落长度适中（< 200字）
- 适当使用列表、表格

#### 可访问性

- 链接可正常访问
- 图片有替代文本
- 数学公式可渲染
- 支持屏幕阅读器

## 🤖 自动化检查

### 检查项目清单

- [ ] 代码可编译

## 👥 人工审核

### 审核者指南

#### 数学内容审核

**检查清单：**
- [ ] 定义是否准确？
- [ ] 定理条件是否完整？
- [ ] 证明是否正确？
- [ ] 例子是否恰当？
- [ ] 引用是否可靠？

**常见问题：**
1. **定义不严谨**：缺少边界条件
2. **定理条件遗漏**：导致结论不成立
3. **证明跳跃**：关键步骤缺乏解释
4. **符号不一致**：同一概念不同表示

#### 语言表达审核

**检查清单：**
- [ ] 语言是否清晰？
- [ ] 术语是否一致？
- [ ] 是否有歧义？
- [ ] 标点是否正确？

#### 格式规范审核

**检查清单：**
- [ ] 是否符合模板？
- [ ] 标题层级是否正确？
- [ ] 数学公式格式是否统一？
- [ ] 引用格式是否规范？

### 审核反馈模板

```markdown
## 审核意见

### 总体评价
[简要评价 PR 质量]

### 详细反馈

#### 数学内容
- [ ] 准确无误
- [ ] 需要修正：[具体说明]

#### 语言表达
- [ ] 清晰流畅
- [ ] 需要改进：[具体说明]

#### 格式规范
- [ ] 符合规范
- [ ] 需要调整：[具体说明]

### 建议
[具体改进建议]

### 结论
- [ ] 可以直接合并
- [ ] 需要小幅修改
- [ ] 需要大幅修改

---

日期：YYYY-MM-DD

```

## 🐛 问题处理

### 问题分类

| 优先级 | 标识 | 说明 | 响应时间 |
|--------|------|------|----------|
| P0 | 🔴 | 数学概念根本性错误 | 24小时内 |
| P1 | 🟠 | 影响理解的关键错误 | 3天内 |
| P2 | 🟡 | 一般性问题 | 1周内 |
| P3 | 🟢 | 改进建议 | 按需处理 |

### 问题报告流程

```

发现问题 → 创建 Issue → 分类标记 → 分配处理 → 修复验证 → 关闭 Issue

```

### 常见错误速查

#### 数学错误

| 错误类型 | 示例 | 修正 |
|----------|------|------|
| 定理条件遗漏 | "素数p" 未说明 p > 1 | 补充完整条件 |
| 符号误用 | $\subset$ 应为 $\subseteq$ | 使用正确符号 |
| 证明逻辑错误 | 循环论证 | 重构证明 |
| 引用错误 | 引用不存在的定理 | 修正引用 |

#### 格式错误

| 错误类型 | 示例 | 修正 |
|----------|------|------|
| 标题层级跳跃 | # 后直接 ### | 补充 ## |
| 公式未闭合 | $x = 1（缺少$） | $x = 1$ |
| 链接失效 | link | 修正路径 |
| 缺少 frontmatter | 无元数据 | 添加 --- 块 |

## 🎓 审核者培训

### 成为审核者

1. **积累经验**：提交 5+ 个高质量 PR
2. **学习规范**：熟读本手册
3. **实践审核**：协助审核 10+ 个 PR
4. **获得认证**：由维护者授予审核权限

### 审核者职责

- 及时响应审核请求（3天内）
- 提供建设性反馈
- 保持客观公正
- 持续学习提升
