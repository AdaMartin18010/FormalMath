## 🌐 翻译范围

### 翻译类型

| 类型 | 说明 | 示例 |
|------|------|------|
| **中译英** | 将中文内容翻译为英文 | 中文定理证明 → 英文版本 |
| **英译中** | 将英文教材翻译为中文 | MIT OCW 课程 → 中文整理 |
| **多语言** | 翻译为其他语言 | 法文、德文、日文等 |

### 优先级

1. **高优先级**：核心数学概念、定理证明
2. **中优先级**：应用案例、历史背景
3. **低优先级**：辅助性说明、脚注

## 📜 翻译原则

### 核心原则

1. **准确性第一**：数学术语翻译必须准确
2. **保持一致性**：相同术语统一译法
3. **尊重惯例**：遵循数学界通用译法
4. **可逆性**：翻译应能从译文还原原文含义

### 翻译策略

#### 直译 vs 意译

| 情况 | 策略 | 示例 |
|------|------|------|
| 术语 | 直译为主 | group → 群 |
| 定义 | 准确直译 | "A group is..." → "群是..." |
| 解释 | 适当意译 | 文化相关内容 |
| 证明 | 严格直译 | 保持逻辑结构 |

## 🔄 翻译流程

### 标准翻译流程

```

1. 原文分析 -> 2. 术语准备 -> 3. 初译 -> 4. 校对 -> 5. 审核 -> 6. 定稿

```

### 详细步骤

#### 1. 原文分析

- 理解原文数学内容
- 识别专业术语
- 确定翻译难度
- 预估工作量

#### 2. 术语准备

- 建立术语表
- 查证标准译名
- 标记不确定的术语

#### 3. 初译

- 保持原文结构
- 标注疑问之处
- 使用占位符标记未确定的术语

#### 4. 校对

- 检查术语一致性
- 验证数学准确性
- 润色语言表达
- 检查格式规范

#### 5. 审核

- 数学专家审核内容准确性
- 语言专家审核表达流畅度
- 根据反馈修改

#### 6. 定稿

- 最终格式检查
- 生成翻译记录
- 提交 PR

- [ ] 链接正确转换
- [ ] 元数据完整

### 常见翻译错误

#### 术语不一致

```

❌ 错误：在同一段落中交替使用「正规子群」和「不变子群」
✅ 正确：统一使用「正规子群」

```

#### 直译痕迹过重

```

❌ 错误：「让 G 是一个群」（英文直译）
✅ 正确：「设 G 是一个群」（中文习惯）

```

#### 数学符号错误

```

❌ 错误：将 $G/N$ 译作「G/N」而不使用 LaTeX
✅ 正确：使用 $G/N$ 保持数学格式

```

#### 漏译

```

❌ 错误：省略了原文的脚注或说明
✅ 正确：完整翻译，必要时添加译者注

```
