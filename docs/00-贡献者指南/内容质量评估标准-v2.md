# FormalMath 内容质量评估标准 v2

> 本标准用于替代以「文档数量」「覆盖率百分比」为核心的旧 KPI，转向可验证、可审计的语义质量指标。

---

## 一、三层内容金字塔

```
                    /\
                   /  \
                  / 金 \    研究生级深度专题
                 / 字  \   - 严格证明、原始文献、前沿进展
                /  塔   \  - 形式化代码深度嵌入
               /----------\
              /   银 层    \  本科高年级核心课程
             /              \ - 完整定理证明链、典型例题、习题解答
            /----------------\
           /      铜 层       \ 基础概念与入门路径
          /                    \ - 直观解释、多表征方式、常见误区
         /----------------------\
```

| 层级 | 目标读者 | 核心要求 |
|------|---------|---------|
| **铜层** | 初学者、自学者 | 直观解释、多例子、常见误区；不追求严格公理化 |
| **银层** | 本科生、考研/竞赛学生 | 严格定义、完整定理证明链、配套例题与习题解答 |
| **金层** | 研究生、研究者 | 原始文献、前沿视角、与 Stacks Project / nLab / Kerodon 对齐 |

**Phase 1 重点**：把 5 门银层核心课程做到「逐章、逐定理、逐习题」完整覆盖。

## 二、五级质量指标（取代旧 KPI）

### 指标 1：定理-证明覆盖率

**定义**：课程大纲或权威教材中列出的核心定理，在项目文档中具有等价表述和完整证明（或详细证明思路）的比例。

| 得分 | 标准 |
|------|------|
| A+ | ≥ 90% 核心定理有完整证明 |
| A  | 80%~89% |
| B  | 60%~79% |
| C  | < 60% |

### 指标 2：定义严格性得分

**定义**：项目文档中的定义与国际权威资源（MIT OCW、Stacks Project、nLab、Wikipedia、MathWorld）在数学上严格等价。

| 得分 | 标准 |
|------|------|
| A+ | ≥ 90% 核心定义已验证等价 |
| A  | 70%~89% |
| B  | 50%~69% |
| C  | < 50% |

### 指标 3：习题-解答对数量

**定义**：每个核心章节配套的、带有详细解答或提示的习题数量。

| 得分 | 标准 |
|------|------|
| A+ | 每章 ≥ 8 对 |
| A  | 每章 5~7 对 |
| B  | 每章 3~4 对 |
| C  | 每章 < 3 对 |

### 指标 4：原始文献引用密度

**定义**：每千字正文中包含的精确引用（含 DOI / ISBN / arXiv ID / MR Number / 具体章节号）数量。

| 得分 | 标准 |
|------|------|
| A+ | ≥ 3 条 / 千字 |
| A  | 2~3 条 / 千字 |
| B  | 1~2 条 / 千字 |
| C  | < 1 条 / 千字 |

### 指标 5：形式化-自然语言桥梁度

**定义**：自然语言文档中嵌入或引用对应 Lean4 代码片段的比例（Phase 1 为可选鼓励指标）。

| 得分 | 标准 |
|------|------|
| A+ | ≥ 50% 核心章节含 Lean 片段 |
| A  | 30%~49% |
| B  | 10%~29% |
| C  | < 10% |

## 三、六级语义对齐验证清单

对每一个声称「已对齐」的国际课程或权威资源，必须完成以下六级验证：

| 级别 | 验证问题 | 关键产出 |
|------|---------|---------|
| **L1** | 课程大纲的章节标题是否已映射到项目文档路径？ | 课程-文档路径映射表 |
| **L2** | 课程中的核心概念是否在项目文档中有对应条目？ | 概念对照表 |
| **L3** | 课程教材中的定义与项目文档中的定义在数学上是否严格等价？ | 定义等价性说明 |
| **L4** | 课程大纲中的核心定理在项目文档中是否有等价表述和严格证明？ | 定理对齐表 |
| **L5** | 课程的 Problem Sets / Exams / 作业在项目文档中是否有对应习题与解答？ | 习题映射表 |
| **L6** | 课程强调的证明技巧、常见错误、教学哲学是否在项目文档中系统提炼？ | 教学法注解 |

**要求**：Phase 1 的 5 门核心课程必须至少达到 L4，并补齐 L5、L6。

## 四、银层文档必备结构

每篇银层文档必须包含以下模块（允许根据主题调整顺序，但核心要素不可缺）：

1. **学习目标**（1~3 条可测量的目标）
2. **前置知识**（列出 prerequisite 概念/定理，并附链接）
3. **动机与直观**（为什么需要这个概念？）
4. **严格定义**（与权威资源等价的表述）
5. **核心定理与证明**（完整证明或详细证明思路）
6. **典型例题**（≥ 2 道，含详细解答）
7. **习题与解答**（≥ 5 道，难度梯度）
8. **常见误区与诊断**（至少 2 个常见错误及纠正）
9. **与其他概念的关联**（交叉引用）
10. **参考文献**（精确到章节/页码/DOI/ISBN/arXiv ID）

## 五、参考文献标准

### 5.1 必须字段

| 资源类型 | 必须字段 |
|---------|---------|
| 书籍 | 作者、书名、版次、出版社、年份、ISBN、引用章节/页码 |
| 期刊论文 | 作者、标题、期刊、卷号、年份、页码、DOI 或 MR Number |
| 预印本 | 作者、标题、arXiv ID、年份、分类 |
| 在线课程/讲义 | 机构、课程代码、讲师、URL、访问日期 |
| 百科/知识库 | 作者/机构、条目名、URL、版本/访问日期 |

### 5.2 引用格式

采用 AMS 格式，示例：

```
[1] W. Rudin, Principles of Mathematical Analysis, 3rd ed., McGraw-Hill, 1976.
    ISBN: 978-0-07-054235-8. Chapter 1, Theorem 1.19.

[2] T. Tao, The prime number theorem in arithmetic progressions,
    arXiv:math/0309380 [math.NT], 2003.

[3] The Stacks Project Authors, The Stacks Project,
    https://stacks.math.columbia.edu, Tag 00EU, accessed June 2026.
```
