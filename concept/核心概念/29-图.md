## 形式化定义

**定义（图）**：**图** $G$ 是有序对 $(V, E)$，其中 $V$ 是顶点集，$E$ 是边集：
- **无向图**：$E \subseteq \binom{V}{2}$（$V$ 的2元子集族）
- **有向图**：$E \subseteq V \times V$
- **多重图**：允许重边和自环

**度**：顶点 $v$ 的**度** $\deg(v)$ 是与 $v$ 关联的边数

**符号说明**：
- $G = (V, E)$：图
- $V(G)$ 或 $V$：顶点集
- $E(G)$ 或 $E$：边集
- $n = |V|$：顶点数
- $m = |E|$：边数
- $\deg(v)$ 或 $d(v)$：顶点的度

## 1. 📋 概述 (编号: C.CORE.029.01)

图是离散数学的基础概念，由顶点和边组成，用于建模关系和网络。图论在计算机科学、运筹学、网络分析等领域有广泛应用。

本文档从**数学认知学**、**教育学**、**数学家**三个视角深入展开图概念，避免简单的概念堆垒。

## 🧠 认知学视角：如何理解图 (编号: C.CORE.029.02)

### 认知起点 (编号: C.CORE.029.02.01)

**学习者已有的知识基础**:

- 集合的概念
- 关系的概念
- 日常经验中的"网络"、"连接"概念

**日常经验中的类似概念**:

- "网络"：节点和连接
- "关系"：对象之间的关系
- "连接"：点之间的连接

### 认知过程 (编号: C.CORE.029.02.02)

#### 阶段1：直观理解阶段 (编号: C.CORE.029.02.02.01)

**具体例子**:

- 例子1：完全图 - 每对顶点都有边
- 例子2：树 - 连通无环图
- 例子3：二分图 - 顶点可以分成两部分

**形象类比**:

- **网络类比**: 图就像"网络"
  - 顶点是"节点"
  - 边是"连接"
  - 图描述"关系"

- **关系类比**: 图就像"关系图"
  - 顶点是"对象"
  - 边是"关系"
  - 图描述"关系结构"

**可视化表示**:

```text
图:
    ●───●
    │   │
    ●───●
    顶点和边
```

#### 阶段2：概念形成阶段 (编号: C.CORE.029.02.02.02)

**从例子中抽象出共同特征**:

- 所有例子都涉及"顶点和边"
- 顶点是"对象"
- 边是"关系"

**识别关键属性**:

1. **顶点集**: $V$是有限集合
2. **边集**: $E$是边的集合
3. **结构**: 图描述关系结构

**建立概念边界**:

- **什么是图**: 由顶点集和边集组成的结构
- **什么不是图**:
  - 只有顶点没有边（不是图）
  - 只有边没有顶点（不是图）

#### 阶段3：形式化阶段 (编号: C.CORE.029.02.02.03)

**严格定义**:

- 图定义：$G = (V, E)$，其中$V$是顶点集，$E$是边集
- 无向图：边是无序对
- 有向图：边是有序对

**公理化表述**:

- 条件1：顶点集
- 条件2：边集
- 条件3：边连接顶点

**逻辑结构**:

- 图是离散数学的基础
- 图可以建模关系和网络
- 图是算法设计的基础

### 认知障碍 (编号: C.CORE.029.02.03)

**常见误解**:

1. **误解1**: 认为图就是图形
   - **纠正**: 图是数学结构，不是图形

2. **误解2**: 认为所有图都相同
   - **纠正**: 不同的图有不同的结构

3. **误解3**: 混淆无向图和有向图
   - **纠正**: 无向图边无序，有向图边有序

**理解难点**:

1. **难点1**: 图的抽象性
   - **原因**: 图比较抽象
   - **解决方法**: 从具体例子开始，逐步抽象

2. **难点2**: 图的性质
   - **原因**: 图的性质比较多
   - **解决方法**: 用具体例子，逐步掌握性质

3. **难点3**: 图的算法
   - **原因**: 图算法比较复杂
   - **解决方法**: 用具体例子，逐步掌握算法

**认知陷阱**:

- **图形**: 需要理解图是数学结构
- **结构**: 需要理解图的结构

### 实际计算示例

#### 示例1: 图遍历
**问题**: 对下图进行DFS和BFS遍历。
```
    1 --- 2
    |     |
    3 --- 4
```

**步骤**:
**DFS**（从1开始）: 1 → 2 → 4 → 3
**BFS**（从1开始）: 1 → 2 → 3 → 4

---

#### 示例2: 最短路径
**问题**: 求图中节点A到D的最短路径。
```
    A --2-- B
    |      /|
    3    1  4
    |   /   |
    C --3-- D
```

**步骤**:
- A→B→D: 2 + 4 = 6
- A→B→C→D: 2 + 1 + 3 = 6
- A→C→D: 3 + 3 = 6

**最短路径**: 长度6

---

#### 示例3: 社交网络分析
**问题**: 分析朋友关系网络中的中心性。

**步骤**:
- 度中心性: 朋友数量
- 中介中心性: 作为桥梁的次数
- 应用: 识别关键影响者

>
> - **抽象**：将区域抽象为点（顶点），将桥抽象为线（边）
> - **图**：得到图$G = (V, E)$，其中$V = \{A, B, C, D\}$，$E = \{AB, AC, AD, BC, BD, CD, CD\}$
> - **欧拉路径**：存在欧拉路径当且仅当图中最多有两个顶点的度数为奇数

>

## 💡 数学解释：为什么图是这样定义的 (编号: C.CORE.029.05)

### 一、直观解释：图是什么？

#### 1.1 具体例子

**生活中的例子**：

- **社交网络**：朋友关系可以用图表示
  - 例如：顶点表示人，边表示朋友关系
  - **为什么这样定义**：因为我们需要一个概念来表示"关系和连接"
- **交通网络**：道路系统可以用图表示
  - 例如：顶点表示城市，边表示道路
  - **为什么这样定义**：因为我们需要一个概念来表示"网络结构"
- **网页链接**：互联网可以用图表示
  - 例如：顶点表示网页，边表示超链接
  - **为什么这样定义**：因为我们需要一个概念来表示"链接关系"

**数学中的例子**：

- **简单图**：$G = (V, E)$，其中$V$是顶点集，$E$是边集
  - **为什么这样定义**：我们需要一个简单的方式来表示"顶点和边的结构"
- **完全图**：$K_n$是$n$个顶点的完全图（每对顶点都有边）
  - **为什么这样定义**：我们需要一个概念来表示"所有顶点都连接的图"
- **树**：无圈的连通图
  - **为什么这样定义**：我们需要一个概念来表示"层次结构"

#### 1.2 形象类比

**网络类比**：

- **图就像"网络"**
  - 网络有节点（顶点）和连接（边）
  - 图有顶点和边
- **为什么这样类比**：这个类比帮助我们理解图的"网络结构"性质

**地图类比**：

- **图就像"地图"**
  - 地图有地点（顶点）和道路（边）
  - 图有顶点和边
- **为什么这样类比**：这个类比帮助我们理解图的"连接关系"

#### 1.3 几何直观

**图形表示**：

- 使用图形表示图
- 例如：用圆圈表示顶点，用线段表示边
- **为什么使用图形**：图形提供几何直观，帮助我们理解图的结构

**邻接矩阵表示**：

- 使用邻接矩阵表示图
- 例如：$A_{ij} = 1$如果顶点$i$和$j$之间有边，否则$A_{ij} = 0$
- **为什么使用邻接矩阵**：邻接矩阵提供代数表示，帮助我们计算

#### 1.4 操作体验

**图运算的操作**：

- **图的并**：$G_1 \cup G_2$是两个图的并
  - **为什么这样操作**：因为我们需要一个运算来表示"两个图的合并"
- **图的交**：$G_1 \cap G_2$是两个图的交
  - **为什么这样操作**：因为我们需要一个运算来表示"两个图的共同部分"
- **子图**：$H$是$G$的子图如果$V(H) \subseteq V(G)$且$E(H) \subseteq E(G)$
  - **为什么这样操作**：因为我们需要一个工具来表示"图的部分"

### 二、知性解释：图的本质是什么？

#### 2.1 概念定义

**图的内涵**：

- **图是由顶点集和边集组成的数学结构**
  - **顶点**：$V$是顶点集，表示对象
  - **边**：$E$是边集，表示关系
  - **结构**：$G = (V, E)$是图
- **为什么这样定义**：这些特征使得图成为一个完整的离散数学工具，支持所有关系研究

**图的外延**：

- **简单图**：没有自环和重边的图
- **有向图**：边有方向的图
- **加权图**：边有权重的图
- **为什么这样分类**：不同的图类型有不同的性质和用途

#### 2.2 分类体系

**按连通性分类**：

- **连通图**：任意两个顶点之间都有路径
- **非连通图**：存在两个顶点之间没有路径
- **为什么这样分类**：连通性是图的基本特征

**按结构分类**：

- **树**：无圈的连通图
- **森林**：无圈的图（多个树）
- **完全图**：每对顶点都有边
- **为什么这样分类**：结构是图的重要特征

#### 2.3 抽象结构

**图的运算结构**：

- **图的并**：$G_1 \cup G_2 = (V_1 \cup V_2, E_1 \cup E_2)$
- **图的交**：$G_1 \cap G_2 = (V_1 \cap V_2, E_1 \cap E_2)$
- **为什么有这些运算**：这些运算使得图具有代数结构

**图的关系结构**：

- **同构**：两个图如果顶点和边可以一一对应，则同构
- **子图**：$H$是$G$的子图如果$V(H) \subseteq V(G)$且$E(H) \subseteq E(G)$
- **为什么有这个关系**：这个关系帮助我们研究图的结构

#### 2.4 知识体系

**图在数学体系中的位置**：

- **基础地位**：图是离散数学的基础
  - 图可以建模关系和网络
  - 图是算法设计的基础
- **为什么是基础**：图提供了统一的框架来研究关系和网络

**图与其他概念的关系**：

- **图与集合**：图是集合的特殊结构
- **图与算法**：图是算法设计的基础
- **图与组合**：图是组合数学的研究对象
- **为什么有这些关系**：这些关系揭示了图与其他数学概念的内在联系

### 三、理性解释：图的公理化定义

#### 3.1 公理体系

**图的定义**：

- **定义**：$G = (V, E)$，其中$V$是顶点集，$E \subseteq V \times V$是边集
- **为什么需要这个定义**：这个定义提供了图的严格数学基础

**图的公理**：

- **顶点公理**：顶点集是非空的有限集合
- **边公理**：边集是顶点对的集合
- **为什么这样定义**：这些公理将图放在更一般的框架中

#### 3.2 形式化证明

**图性质的证明**：

##### 定理1：握手定理

**定理陈述**：对于图$G = (V, E)$，所有顶点的度数之和等于边数的两倍，即$\sum_{v \in V} \deg(v) = 2|E|$。

**形式化表述**：
$$\forall G = (V, E), \quad \sum_{v \in V} \deg(v) = 2|E|$$

**证明**：
1. **度数定义**：对于顶点$v$，$\deg(v)$是与$v$关联的边的数量
2. **双重计数**：每条边$(u, v)$在$\deg(u)$和$\deg(v)$中各被计数一次
3. **求和**：$\sum_{v \in V} \deg(v) = \sum_{(u,v) \in E} (1 + 1) = 2|E|$
4. **结论**：所有顶点的度数之和等于边数的两倍

**为什么这样证明**：利用双重计数原理和度数的定义。

---

##### 定理2：Euler公式（平面图）

**定理陈述**：对于连通平面图$G = (V, E, F)$，顶点数、边数、面数满足$V - E + F = 2$。

**形式化表述**：
$$\forall G = (V, E, F) \text{ 连通平面图}, \quad |V| - |E| + |F| = 2$$

**证明**（数学归纳法）：
1. **基础情况**：对于树（$F = 1$），有$|E| = |V| - 1$，因此$|V| - |E| + |F| = |V| - (|V| - 1) + 1 = 2$
2. **归纳假设**：假设对于所有边数小于$n$的连通平面图，公式成立
3. **归纳步骤**：对于有$n$条边的连通平面图$G$：
   - 如果$G$是树，则$|V| - |E| + |F| = 2$（基础情况）
   - 如果$G$不是树，则存在圈，删除圈上的一条边$e$得到$G'$
   - $G'$是连通的，且$|E'| = |E| - 1$，$|F'| = |F| - 1$（删除边减少一个面）
   - 由归纳假设：$|V| - |E'| + |F'| = 2$
   - 因此：$|V| - (|E| - 1) + (|F| - 1) = 2$，即$|V| - |E| + |F| = 2$
4. **结论**：对所有连通平面图，$V - E + F = 2$

**为什么这样证明**：利用数学归纳法和平面图的结构性质（删除边不破坏连通性）。

---

##### 定理3：奇度顶点个数为偶数

**定理陈述**：图中度数为奇数的顶点个数为偶数。

**形式化表述**：
$$\forall G = (V, E), \quad |\{v \in V : \deg(v) \text{ 为奇数}\}| \equiv 0 \pmod{2}$$

**证明**：
1. **握手定理**：$\sum_{v \in V} \deg(v) = 2|E|$（偶数）
2. **奇偶性**：偶数等于所有度数之和
3. **奇数和**：奇数个奇数相加为奇数，偶数个奇数相加为偶数
4. **结论**：度数为奇数的顶点个数必须为偶数

**为什么这样证明**：利用握手定理和奇偶数的性质。

## 🎯 一、Tall三个世界理论分析 / Tall's Three Worlds Theory Analysis (编号: C.CORE.029.MULTI.01)

### 1.1 感知-操作世界（Embodied World）

**图在感知-操作世界中的理解 / Understanding Graphs in the Embodied World**：

- **具体经验 / Concrete Experience**：
  - 通过"连接"的日常经验理解图：点与点之间的连接 / Understanding graphs through daily experience of "connections": connections between points
  - 例如：社交网络、交通网络、电路图 / Examples: social networks, transportation networks, circuit diagrams
  - 通过"路径"理解图的遍历：从一个点到另一个点的路径 / Understanding graph traversal through "paths": paths from one point to another
  - 例如：地图导航、网络路由 / Examples: map navigation, network routing

- **身体动作 / Bodily Actions**：
  - 通过"画图"的动作理解图 / Understanding graphs through the action of "drawing graphs"
  - 通过"遍历"的动作理解路径 / Understanding paths through the action of "traversal"
  - 通过"标记"的动作理解图的着色 / Understanding graph coloring through the action of "marking"

- **直观理解 / Intuitive Understanding**：
  - 图是"点和边的集合" / A graph is a "collection of vertices and edges"
  - 图描述"关系"的结构 / Graphs describe the structure of "relationships"
  - 图可以用来建模实际问题 / Graphs can be used to model real-world problems

**教学建议**：

- 使用具体例子（社交网络、交通网络）
- 通过画图演示理解图
- 使用图形可视化图的结构

### 1.2 概念-符号世界（Symbolic World）

**图在概念-符号世界的理解 / Understanding Graphs in the Symbolic World**：

- **符号表示 / Symbolic Representation**：
  - 使用图符号：$G = (V, E)$、$G(V, E)$ / Using graph notation: $G = (V, E)$, $G(V, E)$
  - 使用顶点符号：$v \in V$、$|V| = n$ / Using vertex notation: $v \in V$, $|V| = n$
  - 使用边符号：$e \in E$、$e = \{u, v\}$ / Using edge notation: $e \in E$, $e = \{u, v\}$

- **概念理解 / Conceptual Understanding**：
  - 图是顶点集和边集的二元组 / A graph is an ordered pair of vertex set and edge set
  - 图有各种性质（连通性、平面性、着色性） / Graphs have various properties (connectivity, planarity, colorability)
  - 图可以用来研究组合问题 / Graphs can be used to study combinatorial problems

- **抽象操作 / Abstract Operations**：
  - 通过符号操作研究图 / Studying graphs through symbolic operations
  - 通过图的性质研究图的结构 / Studying graph structure through graph properties
  - 通过图算法解决实际问题 / Solving real-world problems through graph algorithms

**教学建议**：

- 逐步引入图符号
- 通过图的性质练习理解图
- 使用图算法解决实际问题

### 1.3 形式-公理世界（Formal World）

**图在形式-公理世界的理解 / Understanding Graphs in the Formal World**：

- **公理系统 / Axiomatic System**：
  - 图的严格定义：$G = (V, E)$ 其中 $E \subseteq \binom{V}{2}$ / Strict definition of graph: $G = (V, E)$ where $E \subseteq \binom{V}{2}$
  - 通过集合论严格定义图 / Strictly defining graphs through set theory
  - 通过公理推导演分性质 / Deriving properties through axioms

- **形式化定义 / Formal Definition**：
  - 图的形式化定义 / Formal definition of graph
  - 图同构的形式化定义 / Formal definition of graph isomorphism
  - 图性质的形式化证明 / Formal proof of graph properties

- **逻辑推理 / Logical Reasoning**：
  - 通过逻辑推理证明图性质 / Proving graph properties through logical reasoning
  - 通过形式化方法研究图论 / Studying graph theory through formal methods
  - 通过组合数学研究图的一般性质 / Studying general properties of graphs through combinatorics

**教学建议 / Teaching Suggestions**：

- 介绍图定义 / Introduce graph definition
- 通过形式化证明理解图性质 / Understand graph properties through formal proofs
- 研究图在图论中的作用 / Study the role of graphs in graph theory

### 1.4 三个世界的转换 / Transitions Between Worlds

**转换路径 / Transition Paths**：

1. **从感知-操作到概念-符号 / From Embodied to Symbolic**：

   - 将图概念形式化为集合论定义 / Formalizing graph concepts into set theory definition
   - 从符号运算到逻辑推理 / From symbolic operations to logical reasoning
   - 例如：从"$G = (V, E)$"到"$E \subseteq \binom{V}{2}$" / Example: from "$G = (V, E)$" to "$E \subseteq \binom{V}{2}$"

**转换支持 / Transition Support**：

- **识别转换困难 / Identifying Transition Difficulties**：
  - 识别从具体网络到抽象图的困难 / Identifying difficulties in transitioning from concrete networks to abstract graphs
  - 识别从符号运算到形式化定义的困难 / Identifying difficulties in transitioning from symbolic operations to formal definitions

**各理论在图学习中的互补作用 / Complementary Roles of Theories in Graph Learning**：

- **Tall理论 / Tall's Theory**：提供三个世界的理解框架 / Provides understanding framework of three worlds
  - 感知-操作世界：连接关系的直观理解 / Embodied world: intuitive understanding of connection relationships
  - 概念-符号世界：图符号和运算 / Symbolic world: graph notation and operations
  - 形式-公理世界：图定义和理论 / Formal world: graph definitions and theory

- **APOS理论 / APOS Theory**：提供认知发展路径 / Provides cognitive development path
  - Action：图操作 / Action: graph operations
  - Process：图过程 / Process: graph process
  - Object：图作为对象 / Object: graphs as objects
  - Schema：图图式 / Schema: graph schema

- **Sfard理论 / Sfard's Theory**：提供交流视角 / Provides communication perspective
  - 图作为交流工具 / Graphs as communication tools
  - 通过实践参与学习 / Learning through practice participation

- **Piaget理论 / Piaget's Theory**：提供认知发展阶段 / Provides cognitive development stages
  - 具体运算阶段：具体图 / Concrete operational stage: concrete graphs
  - 形式运算阶段：抽象图概念 / Formal operational stage: abstract graph concepts

- **Vygotsky理论 / Vygotsky's Theory**：提供社会文化视角 / Provides sociocultural perspective
  - 最近发展区：学习支持 / Zone of proximal development: learning support
  - 社会互动：合作学习 / Social interaction: cooperative learning
  - 文化工具：数学文化 / Cultural tools: mathematical culture

### 6.2 整合教学策略 / Integrated Teaching Strategies

**基于多理论的整合教学策略 / Integrated Teaching Strategies Based on Multiple Theories**：

1. **多层次理解 / Multi-Level Understanding**：
   - 从感知-操作世界开始（连接关系） / Start from embodied world (connection relationships)
   - 过渡到概念-符号世界（图符号） / Transition to symbolic world (graph notation)
   - 最终达到形式-公理世界（图定义） / Finally reach formal world (graph definition)

2. **渐进式发展 / Progressive Development**：
   - 从Action开始（图操作） / Start from Action (graph operations)
   - 发展为Process（图过程） / Develop into Process (graph process)
   - 对象化为Object（图作为对象） / Objectify into Object (graphs as objects)
   - 构建Schema（图图式） / Construct Schema (graph schema)

3. **实践参与 / Practice Participation**：
   - 通过数学实践学习图 / Learning graphs through mathematical practice
   - 使用图语言交流 / Using graph language for communication
   - 解决实际问题 / Solving real-world problems

4. **认知发展支持 / Cognitive Development Support**：
   - 根据认知发展阶段设计教学 / Design teaching according to cognitive development stages
   - 提供适当的支持和指导 / Provide appropriate support and guidance
   - 促进认知发展 / Promote cognitive development

5. **社会文化融入 / Sociocultural Integration**：
   - 通过社会互动学习 / Learning through social interaction
   - 使用文化工具 / Using cultural tools
   - 理解数学文化 / Understanding mathematical culture

- **认知理解 / Cognitive Understanding**：
  - 集合：元素的无序组合 / Sets: unordered collections of elements
  - 图：集合配备关系结构 / Graphs: sets equipped with relational structure

- **教学建议 / Teaching Suggestions**：

### 7.2 与函数的关系 / Relationship with Functions

- **认知理解 / Cognitive Understanding**：

- **教学建议 / Teaching Suggestions**：

  - 对比图与函数的异同 / Compare similarities and differences between graphs and functions

**图在离散数学中的地位 / Position of Graphs in Discrete Mathematics**：

### 8.4 元认知理论分析 / Metacognitive Theory Analysis

**元认知知识 / Metacognitive Knowledge**：

- **概念知识 / Conceptual Knowledge**：
  - 图的定义：由顶点和边组成的数学结构 / Graphs are mathematical structures consisting of vertices and edges
  - 图的性质：连通性、可平面性、着色性 / Properties: connectivity, planarity, colorability
  - 图的表示方法：邻接矩阵、邻接表、图的可视化 / Representation methods: adjacency matrices, adjacency lists, graph visualization
  - 图与其他概念的关系：图是离散数学的基础，是研究关系的工具 / Relationship: graphs are foundations of discrete mathematics and tools for studying relationships

- **策略知识 / Strategy Knowledge**：
  - 学习策略：从具体例子（社交网络、交通网络）到抽象定义，通过画图掌握图 / Learning strategies: from concrete examples (social networks, transportation networks) to abstract definitions, master graphs through drawing
  - 理解策略：从具体到抽象，从简单到复杂 / Understanding strategies: from concrete to abstract, from simple to complex

  - 任务难度：基础任务（图构造）→ 中级任务（图遍历）→ 高级任务（图算法） / Task difficulty: basic (graph construction) → intermediate (graph traversal) → advanced (graph algorithms)

  - 能否识别图构造的错误？ / Can I identify errors in graph construction?
  - 能否判断图的性质？ / Can I determine the properties of graphs?

- **计划调节 / Planning Regulation**：

  - 调整学习计划：根据学习情况调整计划 / Adjust learning plan: adjust plan based on learning situation

- **策略调节 / Strategy Regulation**：

  - 如果遇到困难，尝试不同的策略 / If encountering difficulties, try different strategies

  - 从错误中学习：分析错误原因，避免类似错误 / Learn from errors: analyze error causes, avoid similar errors

**教学建议 / Teaching Suggestions**：

- 教授元认知知识：帮助学生建立关于图的元认知知识 / Teach metacognitive knowledge: help students build metacognitive knowledge about graphs
- 培养监控能力：引导学生自我监控理解过程 / Develop monitoring ability: guide students to self-monitor their comprehension process
- 训练调节能力：教授学生如何调节学习策略 / Train regulation ability: teach students how to regulate learning strategies

### 5.1 完整学习流程示例

**阶段1：入门理解**:

1. 使用学习决策导图 → 确定学习路径：基础概念 → 图定义 → 基本性质 → 基本应用
2. 使用总体思维导图 → 了解图在离散数学中的位置
3. 阅读[图-三视角版](./29-图.md) → 理解图的定义和基本性质
4. 使用论证过程 → 理解图的直观意义（顶点、边、关系、网络）

**阶段2：深入分析**:

1. 使用多维矩阵 → 分析图的多维度特征（知识层次L0基础层、知识领域D7离散数学、学习难度中等）
2. 阅读[图-三视角版](./29-图.md) → 深入理解图算法和重要定理
3. 使用形式化证明 → 学习Euler定理的形式化证明
4. 使用分支关联思维导图 → 了解图与集合、关系、算法的关系

**阶段3：应用研究**:

1. 使用研究决策导图 → 确定研究方向：图论理论
2. 使用知识关联网络 → 发现图与组合、代数、拓扑的关系
3. 使用形式化证明系统 → 构建图论理论体系
4. 阅读[图-三视角版](./29-图.md) → 了解图的历史发展和开放问题
