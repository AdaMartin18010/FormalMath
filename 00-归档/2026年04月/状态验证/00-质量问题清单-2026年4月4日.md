## P1 - 高优先级问题 (需立即修复)

### P1-001: SyntaxError in multi_format_exporter.py

| 属性 | 内容 |
|------|------|
| **文件** | `tools/doc-generator/multi_format_exporter.py` |
| **位置** | 第248行 |
| **问题** | SyntaxError: invalid decimal literal |
| **描述** | CSS样式字符串中的`@page`规则使用了Python字符串格式化语法，导致语法错误 |
| **影响** | 该文件无法被Python编译器正确解析，相关功能不可用 |
| **修复建议** | 将CSS字符串改为原始字符串或使用三引号包裹，避免与Python格式化语法冲突 |
| **修复代码示例** | ```python
css = '''
@page {{ size: A4; margin: 2cm; }}
body {{ font-family: "Noto Sans CJK SC", sans-serif; }}
'''

# 改为:

css = r'''
@page { size: A4; margin: 2cm; }
body { font-family: "Noto Sans CJK SC", sans-serif; }
'''

``` |

### P1-002: SyntaxWarning in fix_data_integrity.py

| 属性 | 内容 |
|------|------|
| **文件** | `tools/fix_data_integrity.py` |
| **位置** | 第170行 |
| **问题** | SyntaxWarning: invalid escape sequence '\.' |
| **描述** | 字符串中使用了反斜杠转义序列，但未使用原始字符串前缀 |
| **影响** | 虽不影响功能，但在未来Python版本中可能成为错误 |
| **修复建议** | 使用原始字符串r前缀或双反斜杠 |
| **修复代码示例** | ```python
# 原代码:
"Get-ChildItem -Recurse -Filter '*.backup' | ForEach-Object { Move-Item $_.FullName ($_.FullName -replace '\.backup$', '') -Force }"
# 改为:
r"Get-ChildItem -Recurse -Filter '*.backup' | ForEach-Object { Move-Item $_.FullName ($_.FullName -replace '\.backup$', '') -Force }"
``` |

## P3 - 低优先级/已知限制

### 链接检查误报 (32,548个)

**说明**: 大量链接错误实际上是检查工具的误报，包括：

1. **外部链接**: GitHub Issues、Discussions、Pull Requests等
   - 示例: `../../issues`, `../../discussions`

2. **锚点链接**: 指向文档内部锚点的链接
   - 示例: `file.md#section-name`

3. **模板文件**: 包含占位符链接的模板
   - 示例: `.github/pull_request_template.md`

4. **动态生成的链接**: 某些链接在运行时才能确定

**建议**: 改进链接检查工具，添加例外规则或白名单机制

### 数学公式括号警告 (7,134个)

**说明**: 公式检查工具对数学公式中的括号使用产生了大量误报，如：

1. **区间表示法**: `[a, b)`, `(-∞, +∞]` 中的方括号
2. **函数表示法**: `f: X × X → [0, ∞)` 中的方括号
3. **集合构造器**: `{x ∈ ℝ | a ≤ x < b}` 中的花括号

这些在数学公式中是合法的用法。

**建议**: 改进公式检查工具，识别数学上下文中的合法括号使用
