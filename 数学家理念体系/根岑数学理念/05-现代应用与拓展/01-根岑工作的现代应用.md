## 一、自动定理证明

### 1.1 归结原理与根岑方法

约翰·艾伦·罗宾逊（John Alan Robinson）在1965年提出的归结原理源于根岑的序列演算。

**归结规则**：

$$\frac{C_1 \vee L \quad C_2 \vee \neg L}{C_1 \vee C_2}$$

**与序列演算的联系**：
归结可以看作是序列演算中切规则的特例。

**归结作为切消除**：
$$\frac{\Gamma \vdash \Delta, L \quad L, \Sigma \vdash \Pi}{\Gamma, \Sigma \vdash \Delta, \Pi} \text{ Cut}$$

当 $\Delta$ 和 $\Sigma$ 为空时，就得到归结规则。

### 1.2 反链法（Connection Method）

沃尔夫冈·比布尔（Wolfgang Bibel）的反链法直接基于根岑的切消定理。

**核心思想**：
通过寻找公式中的"连接"来证明有效性，避免了完整的证明搜索。

**数学表述**：
对于公式 $\varphi$，寻找一组替换 $\sigma$ 使得：

$$\text{连接矩阵 } M_\varphi \text{ 在 } \sigma \text{ 下存在覆盖路径}$$

## 二、逻辑编程

### 2.1 Prolog的根岑基础

Prolog语言的执行机制直接源于根岑的序列演算。

**Horn子句与序列演算**：

Horn子句：
$$A \leftarrow B_1, B_2, \ldots, B_n$$

对应于序列演算中的右规则：

$$\frac{\Gamma \vdash B_1, \Delta \quad \cdots \quad \Gamma \vdash B_n, \Delta}{\Gamma \vdash A, \Delta}$$

**SLD归结**：
Prolog的SLD归结可以看作是受限的切消除过程。

### 2.2 约束逻辑编程

根岑的序数分析方法被扩展到约束逻辑编程。

**CLP(ℝ)示例**：
```prolog
?- {X + Y = 5, X - Y = 1}.
X = 3.0,
Y = 2.0.
```

约束求解器使用类似根岑切消的算法来简化约束系统。

## 三、程序验证

### 3.1 Hoare逻辑的序列演算基础

Hoare逻辑的推理规则可以表示为序列演算的形式。

**Hoare三元组作为序列**：

$$\{P\} C \{Q\} \quad \equiv \quad P \vdash [C]Q$$

其中 $[C]Q$ 表示"执行 $C$ 后 $Q$ 成立"。

**组合规则**：

$$\frac{\{P\} C_1 \{R\} \quad \{R\} C_2 \{Q\}}{\{P\} C_1; C_2 \{Q\}}$$

这对应于序列演算中的切规则。

### 3.2 分离逻辑与资源推理

分离逻辑扩展了Hoare逻辑，处理指针和内存，其推理规则也遵循根岑的结构。

**分离合取规则**：

$$\frac{\{P_1\} C \{Q_1\} \quad \{P_2\} C \{Q_2\}}{\{P_1 * P_2\} C \{Q_1 * Q_2\}}$$

其中 $*$ 是分离合取。

## 四、线性逻辑与资源敏感性

### 4.1 吉拉尔的线性逻辑

让-伊夫·吉拉尔（Jean-Yves Girard）在1987年提出的线性逻辑直接源于对根岑序列演算的深刻分析。

**结构规则的区分**：

吉拉尔区分了两种结构规则：

**结构规则**（控制资源使用）：
- 弱化：$A \vdash B \Rightarrow A, C \vdash B$
- 收缩：$A, A \vdash B \Rightarrow A \vdash B$

**非结构规则**（保持逻辑内容）：
- 切规则
- 逻辑规则

**线性蕴涵**：

$$A \multimap B \quad \text{（单次使用 } A \text{ 得到 } B \text{）}$$

### 4.2 资源敏感性计算

线性逻辑启发了资源敏感性的编程语言设计。

**Rust语言的所有权系统**：

Rust的所有权系统直接受线性逻辑启发：

```rust
fn main() {
    let s1 = String::from("hello");
    let s2 = s1;  // s1 被移动（move）到 s2
    // println!("{}", s1);  // 错误！s1 不再有效
    println!("{}", s2);     // 正确
}
```

**线性类型的数学基础**：

$$\Gamma \vdash M : A \multimap B \quad \text{表示 } M \text{ 恰好使用其输入一次}$$

## 五、证明网络与并行计算

### 5.1 证明网络（Proof Nets）

吉拉尔的证明网络是序列演算的几何化表示，消除了语法上的冗余。

**证明网络的构造**：

$$\begin{array}{ccc}
A & & B \\
\downarrow & & \downarrow \\
& \otimes & \\
& A \otimes B &
\end{array}$$

**正确性标准**：
通过Danos-Regnier几何条件判断证明网络是否对应于有效证明。

### 5.2 并行性提取

证明网络的并行结构使其成为并行计算的理论模型。

**无收缩证明的并行执行**：

在证明网络中，独立的子证明可以并行执行：

$$\frac{\vdash \Gamma, A \quad \vdash \Delta, B}{\vdash \Gamma, \Delta, A \otimes B}$$

两个前提可以并行证明。
