## 一、历史背景：1936年的突破

### 1.1 Gödel的原始递归（1931）

在Kurt Gödel证明不完备定理（1931年）时，他引入了**原始递归函数**（Primitive Recursive Functions）来形式化数学中使用的可计算概念。

Gödel需要一种严格的方式来定义可计算函数，以便在形式系统内表达元数学概念。

### 1.2 Church与Kleene的贡献（1936）

**Alonzo Church** 和 **Stephen Kleene** 在1936年独立发展了递归函数理论：

- **Church**：通过λ演算定义λ可定义函数
- **Kleene**：通过原始递归加μ-算子定义一般递归函数

两人证明了这些定义与**图灵机**（Turing, 1936）的等价性，奠定了**Church-Turing论题**的基础。

### 1.3 递归函数的三种等价形式

| 形式 | 提出者 | 核心思想 |
|------|--------|----------|
| λ可定义函数 | Church, 1936 | 函数作为λ项 |
| 一般递归函数 | Kleene, 1936 | 初始函数+构造规则 |
| 图灵可计算函数 | Turing, 1936 | 图灵机计算 |

## 二、初始函数：递归的基石

### 2.1 三个初始函数

**定义 2.1**（初始函数）：

所有递归函数都从以下三个**初始函数**构造：

**1. 零函数**（Zero Function）：
$$Z: \mathbb{N} \to \mathbb{N}, \quad Z(x) = 0$$

**2. 后继函数**（Successor Function）：
$$S: \mathbb{N} \to \mathbb{N}, \quad S(x) = x + 1$$

**3. 投影函数**（Projection Functions）：
$$P_i^k: \mathbb{N}^k \to \mathbb{N}, \quad P_i^k(x_1, x_2, \ldots, x_k) = x_i$$

其中 $1 \leq i \leq k$，$k \geq 1$。

### 2.2 初始函数的可计算性

**定理 2.2**：所有初始函数都是图灵可计算的。

**证明**：

- $Z$：图灵机擦除输入，输出0
- $S$：图灵机在输入（一元表示）后添加一个1
- $P_i^k$：图灵机的第 $i$ 个输入就是输出

### 2.3 初始函数的作用

初始函数提供了：

1. **常数0**：通过 $Z$ 构造
2. **数后继**：通过 $S$ 构造所有自然数
3. **参数选择**：通过 $P_i^k$ 选择和重排参数

## 三、复合：函数的组装

### 3.1 复合的定义

**定义 3.1**（复合 / Composition）：

给定函数 $g_1, \ldots, g_m: \mathbb{N}^n \to \mathbb{N}$ 和 $h: \mathbb{N}^m \to \mathbb{N}$，定义函数 $f: \mathbb{N}^n \to \mathbb{N}$：

$$f(x_1, \ldots, x_n) = h(g_1(x_1, \ldots, x_n), \ldots, g_m(x_1, \ldots, x_n))$$

记作 $f = h \circ (g_1, \ldots, g_m)$。

### 3.2 复合的封闭性

**定理 3.2**：递归函数类在复合下封闭。

**证明思路**：

- 如果 $h, g_1, \ldots, g_m$ 都是图灵可计算的
- 则 $f$ 可以通过顺序执行 $g_1, \ldots, g_m$，然后执行 $h$ 来计算

### 3.3 复合示例

**示例 3.3**（常数函数）：

定义常数函数 $C_k(x) = k$：

- $C_0(x) = Z(x) = 0$
- $C_1(x) = S(Z(x)) = 1$
- $C_{k+1}(x) = S(C_k(x)) = k+1$

**示例 3.4**（后继函数的推广）：

$f(x) = x + 2 = S(S(x)) = S \circ S(x)$

## 四、原始递归：递归的模式

### 4.1 原始递归模式

**定义 4.1**（原始递归）：

给定 $g: \mathbb{N}^n \to \mathbb{N}$ 和 $h: \mathbb{N}^{n+2} \to \mathbb{N}$，定义 $f: \mathbb{N}^{n+1} \to \mathbb{N}$：

$$
\begin{aligned}
f(0, x_1, \ldots, x_n) &= g(x_1, \ldots, x_n) \\
f(k+1, x_1, \ldots, x_n) &= h(k, f(k, x_1, \ldots, x_n), x_1, \ldots, x_n)
\end{aligned}
$$

**直观理解**：

- **基础情况**：$f(0, \vec{x})$ 由 $g$ 直接给出
- **递归情况**：$f(k+1, \vec{x})$ 依赖于 $k$、$f(k, \vec{x})$ 和其他参数

### 4.2 原始递归函数的封闭性

**定理 4.2**：原始递归函数类在原始递归下封闭。

### 4.3 加法的定义

**示例 4.3**（加法 $+$）：

定义 $+(m, n)$：

$$
\begin{aligned}
+(0, n) &= P_1^1(n) = n \\
+(k+1, n) &= S(+(k, n)) = S(P_2^3(k, +(k, n), n))
\end{aligned}
$$

**验证**：

- $+(0, 5) = 5$ ✓
- $+(1, 5) = S(+(0, 5)) = S(5) = 6$ ✓
- $+(2, 5) = S(+(1, 5)) = S(6) = 7$ ✓

### 4.4 乘法的定义

**示例 4.4**（乘法 $\times$）：

定义 $\times(m, n)$：

$$
\begin{aligned}
\times(0, n) &= Z(n) = 0 \\
\times(k+1, n) &= +(\times(k, n), n)
\end{aligned}
$$

**验证**：

- $\times(0, 5) = 0$ ✓
- $\times(1, 5) = +(0, 5) = 5$ ✓
- $\times(2, 5) = +(5, 5) = 10$ ✓
- $\times(3, 5) = +(10, 5) = 15$ ✓

### 4.5 指数的定义

**示例 4.5**（指数 $exp(m, n) = n^m$）：

定义：

$$
\begin{aligned}
exp(0, n) &= S(0) = 1 \\
exp(k+1, n) &= \times(exp(k, n), n)
\end{aligned}
$$

## 五、μ-算子：从全函数到部分函数

### 5.1 μ-算子的定义

**定义 5.1**（μ-算子 / 最小化算子）：

给定 $g: \mathbb{N}^{n+1} \to \mathbb{N}$，定义 $f: \mathbb{N}^n \to \mathbb{N}$：

$$f(x_1, \ldots, x_n) = \mu y[g(x_1, \ldots, x_n, y) = 0]$$

其中：

- $\mu y[\ldots]$ 表示满足条件的最小 $y$
- 如果不存在这样的 $y$，则 $f(\vec{x})$ **无定义**

### 5.2 μ-算子的可计算性

**算法**：从 $y = 0$ 开始，逐个检查 $g(\vec{x}, y) = 0$：

```
μ(g)(x):
  y ← 0
  while g(x, y) ≠ 0:
    y ← y + 1
  return y
```

**注意**：这个算法可能不终止！

### 5.3 一般递归函数的定义

**定义 5.2**（一般递归函数 / μ-递归函数）：

函数 $f$ 是一般递归的，如果它可以：

1. 从初始函数 $Z, S, P_i^k$ 开始
2. 通过有限次应用**复合**、**原始递归**和**μ-算子**构造

### 5.4 部分函数与全函数

**定义 5.3**：

- **全函数**：对所有输入都有定义
- **部分函数**：对某些输入可能无定义

**定理 5.4**：

- 原始递归函数都是全函数
- 一般递归函数可以是部分函数

## 六、递归函数示例详解

### 6.1 阶乘函数

**示例 6.1**（阶乘 $fact(n) = n!$）：

**原始递归定义**：
$$
\begin{aligned}
fact(0) &= 1 \\
fact(n+1) &= \times(n+1, fact(n))
\end{aligned}
$$

**详细计算**：

- $fact(0) = 1$
- $fact(1) = 1 \times 1 = 1$
- $fact(2) = 2 \times 1 = 2$
- $fact(3) = 3 \times 2 = 6$
- $fact(4) = 4 \times 6 = 24$

### 6.2 斐波那契函数

**示例 6.2**（斐波那契数列 $fib$）：

原始递归需要配对编码：

定义辅助函数 $fibpair(n) = (fib(n), fib(n+1))$：

$$
\begin{aligned}
fibpair(0) &= (0, 1) \\
fibpair(n+1) &= (snd(fibpair(n)), fst(fibpair(n)) + snd(fibpair(n)))
\end{aligned}
$$

则 $fib(n) = fst(fibpair(n))$。

### 6.3 前驱函数

**示例 6.3**（前驱函数 $pred$）：

$$
\begin{aligned}
pred(0) &= 0 \\
pred(n+1) &= n
\end{aligned}
$$

### 6.4 截断减法

**示例 6.4**（截断减法 $m \dot{-} n = max(m-n, 0)$）：

$$
\begin{aligned}
m \dot{-} 0 &= m \\
m \dot{-} (n+1) &= pred(m \dot{-} n)
\end{aligned}
$$

### 6.5 除法

**示例 6.5**（整除 $div(m, n)$）：

使用μ-算子：
$$div(m, n) = \mu k[m < (k+1) \times n]$$

这需要定义 $<$（小于关系），它也是原始递归的。

### 6.6 素数判定

**示例 6.6**（素数判定 $isprime(n)$）：

$$isprime(n) = (n > 1) \wedge \forall d < n: (d \leq 1) \vee (d = n) \vee (n \mod d \neq 0)$$

所有使用的操作（$>, <, \wedge, \forall$ 在有界范围内）都是原始递归的。

## 七、原始递归函数的局限性

### 7.1 Ackermann函数

**定义 7.1**（Ackermann函数）：

$$
\begin{aligned}
A(0, n) &= n + 1 \\
A(m+1, 0) &= A(m, 1) \\
A(m+1, n+1) &= A(m, A(m+1, n))
\end{aligned}
$$

**性质**：

- $A(m, n)$ 对所有 $m, n$ 有定义（全函数）
- 增长速度极快：$A(4, 2)$ 是一个有19,729位的数

**定理 7.2**（Ackermann, 1928）：

Ackermann函数是**递归的但不是原始递归的**。

**证明思路**：

- 对任意原始递归函数 $f$，存在 $k$ 使得 $f(n) < A(k, n)$ 对所有足够大的 $n$ 成立
- Ackermann函数增长快于任何原始递归函数
- 因此Ackermann函数不是原始递归的

### 7.2 原始递归与总递归函数

**定理 7.3**：存在全递归函数不是原始递归的。

这表明**μ-算子是必要的**，即使我们只关心全函数。

## 八、现代应用：递归函数在证明助手中的实现

### 8.1 Lean 4中的原始递归

```lean4
-- 零函数
def Z (x : Nat) : Nat := 0

-- 后继函数（Lean内置）
def S (x : Nat) : Nat := x + 1

-- 投影函数
def P (i k : Nat) (xs : Fin k → Nat) : Nat := xs i

-- 加法（原始递归）
def add : Nat → Nat → Nat
  | 0, n => n
  | k+1, n => S (add k n)

-- 乘法（原始递归）
def mul : Nat → Nat → Nat
  | 0, n => 0
  | k+1, n => add (mul k n) n

-- 阶乘（原始递归）
def fact : Nat → Nat
  | 0 => 1
  | n+1 => (n+1) * fact n

-- 斐波那契（使用配对）
def fibpair : Nat → Nat × Nat
  | 0 => (0, 1)
  | n+1 => let (a, b) := fibpair n; (b, a + b)

def fib (n : Nat) : Nat := (fibpair n).1

-- 验证
#eval fact 5  -- 120
#eval fib 10  -- 55
```

### 8.2 μ-递归函数的表示

```lean4
-- 使用Option表示部分函数
def mu (g : Nat → Nat → Nat) (x : Nat) : Option Nat :=
  let rec find (y : Nat) : Option Nat :=
    if g x y = 0 then some y else find (y + 1)
  find 0

-- 整除：使用μ-算子
def divides (m n : Nat) : Bool := n % m = 0

def smallestDivisor (n : Nat) : Option Nat :=
  mu (λ n d => if d > 1 ∧ divides d n then 0 else 1) n
```

### 8.3 Church数与递归

```lean4
-- Church数（递归的另一种表示）
def ChurchNat : Type := (α : Type) → (α → α) → α → α
def ChurchZero : ChurchNat := λ _ _ x => x
def ChurchSucc (n : ChurchNat) : ChurchNat :=
  λ α f x => f (n α f x)

-- Church加法（递归定义）
def ChurchAdd (m n : ChurchNat) : ChurchNat :=
  λ α f x => m α f (n α f x)

-- 验证：2 + 3 = 5
#reduce ChurchAdd
  (ChurchSucc (ChurchSucc ChurchZero))
  (ChurchSucc (ChurchSucc (ChurchSucc ChurchZero)))
  Nat (λ n => n + 1) 0  -- 5
```
