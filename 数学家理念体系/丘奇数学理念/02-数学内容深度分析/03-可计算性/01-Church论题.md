## 一、历史背景与问题提出

### 1.1 Hilbert的挑战与Entscheidungsproblem

20世纪初，数学基础研究面临深刻危机。David Hilbert在1900年的著名演讲中提出了23个问题，其中**第10问题**（判定丢番图方程是否有整数解）引发了关于**算法可判定性**的思考。

**核心问题**：是否存在一个通用算法，可以判定任意数学命题的真假？

这个问题在1928年被Hilbert和Ackermann形式化为**Entscheidungsproblem**（判定问题）：

> **判定问题**：给定一阶逻辑中的一个公式，是否存在一个算法可以判定该公式是否普遍有效？

要回答这个问题，首先需要回答一个更基本的问题：**什么是算法？** 什么是"可计算"的精确含义？

### 1.2 1936年：可计算性理论的诞生

1936年成为可计算性理论的里程碑年份。三位杰出的逻辑学家独立提出了三种不同的可计算性形式化：

| 学者 | 形式化 | 核心思想 | 论文 |
|------|--------|----------|------|
| **Alonzo Church** | λ可定义函数 | 函数作为规则，通过抽象和应用构造 | 1936, *An unsolvable problem of elementary number theory* |
| **Alan Turing** | 图灵机 | 抽象计算机器，模拟人类计算过程 | 1936, *On computable numbers* |
| **Stephen Kleene** | 一般递归函数 | 从基本函数通过递归构造 | 1936, *General recursive functions of natural numbers* |

**关键发现**：这三种看似完全不同的形式化实际上是等价的！这一发现促成了Church-Turing论题的诞生。

## 二、可计算性的三种形式化

### 2.1 λ可定义函数（Church, 1936）

**定义**：函数 $f: \mathbb{N} \to \mathbb{N}$ 是**λ可定义**的，如果存在λ项 $F$ 使得对所有 $n \in \mathbb{N}$：

$$F \, \overline{n} =_\beta \overline{f(n)}$$

其中 $\overline{n}$ 是自然数 $n$ 的Church编码：

$$\overline{n} = \lambda f. \lambda x. f^n x$$

**核心思想**：计算就是函数应用。任何可计算函数都可以通过λ演算中的函数抽象和应用来表示。

### 2.2 图灵可计算函数（Turing, 1936）

**定义**：函数 $f$ 是**图灵可计算**的，如果存在图灵机 $M$ 使得：

- 对输入 $n$（以二进制或一元编码写在带上），$M$ 最终停机
- 停机时带上包含 $f(n)$ 的编码

**图灵机的基本组成**：

- 无限长的纸带（存储）
- 读写头（当前位置）
- 有限状态控制器（程序）
- 转移函数 $\delta: Q \times \Gamma \to Q \times \Gamma \times \{L, R\}$

**核心思想**：计算是状态的机械变换。任何人类计算者能完成的算法步骤都可以被图灵机模拟。

### 2.3 一般递归函数（Kleene, 1936）

**定义**：函数 $f$ 是**一般递归的**（或μ递归的），如果它可以从以下**初始函数**通过**构造规则**得到：

**初始函数**：

1. **零函数**：$Z(x) = 0$
2. **后继函数**：$S(x) = x + 1$
3. **投影函数**：$P_i^k(x_1, \ldots, x_k) = x_i$

**构造规则**：

1. **复合**：$f(\vec{x}) = h(g_1(\vec{x}), \ldots, g_m(\vec{x}))$
2. **原始递归**：
   $$f(0, \vec{x}) = g(\vec{x})$$
   $$f(n+1, \vec{x}) = h(n, f(n, \vec{x}), \vec{x})$$
3. **最小化（μ算子）**：
   $$f(\vec{x}) = \mu y [g(\vec{x}, y) = 0]$$
   （寻找使 $g(\vec{x}, y) = 0$ 的最小 $y$）

**核心思想**：计算可以通过基本的算术操作和递归构造来定义。

## 三、Church-Turing论题的陈述

### 3.1 论题的两种形式

**Church论题（Church's Thesis, 1936）**：

> 一个函数是"可有效计算"的当且仅当它是λ可定义的。

**Turing论题（Turing's Thesis, 1936）**：

> 一个函数是"可有效计算"的当且仅当它是图灵可计算的。

**Church-Turing论题**：

> 以下概念等价，且刻画了"可有效计算"的直观概念：
>
> - λ可定义函数
> - 图灵可计算函数
> - 一般递归函数
> - 部分递归函数

### 3.2 定理：三种形式化的等价性

**定理 3.1**（等价性定理）：

以下函数类是等价的：
$$\text{λ可定义函数} = \text{图灵可计算函数} = \text{一般递归函数}$$

**证明概要**：

1. **λ可定义 ⇒ 图灵可计算**：构造图灵机模拟β归约
2. **图灵可计算 ⇒ 一般递归**：将图灵机编码为递归函数
3. **一般递归 ⇒ λ可定义**：证明初始函数可λ定义，且构造规则保持λ可定义性

## 四、等价性证明思路

### 4.1 从λ演算到图灵机

**定理 4.1**：每个λ可定义函数都是图灵可计算的。

**证明思路**：

1. **编码λ项**：将λ项编码为自然数（Gödel编码）
2. **模拟β归约**：设计图灵机执行以下操作：
   - 识别可归约式（redex）：$(\lambda x.M)N$
   - 执行替换：$M[N/x]$
   - 处理α转换（变量重命名）
3. **计算过程**：
   - 输入：$F \, \overline{n}$ 的编码
   - 过程：反复进行β归约直到达到范式
   - 输出：范式对应的数值

**技术难点**：

- λ项的结构是树形的，需要图灵机能够处理树结构
- 替换操作可能增加项的大小，需要动态分配空间
- α转换需要生成新的变量名

### 4.2 从图灵机到递归函数

**定理 4.2**：每个图灵可计算函数都是一般递归的。

**证明思路**：

1. **编码图灵机状态**：
   - 将纸带内容编码为Gödel数
   - 将状态、带头位置编码为数字
   - 整个配置可以编码为单个自然数

2. **构造转移函数**：
   - 使用原始递归定义一步转移函数 $step: \mathbb{N} \to \mathbb{N}$
   - $step(c)$ = 配置 $c$ 经过一步后的配置

3. **模拟运行**：
   - 使用递归定义运行函数 $run(n, c)$ = 从配置 $c$ 运行 $n$ 步后的配置
   - 使用μ算子寻找停机点：$\mu n [halt(run(n, c))]$

4. **提取结果**：
   - 停机配置中提取输出

### 4.3 从递归函数到λ演算

**定理 4.3**：每个一般递归函数都是λ可定义的。

**证明思路**：

1. **初始函数的λ定义**：
   - 零函数：$Z = \lambda x. \overline{0}$
   - 后继函数：$S = \lambda n. \lambda f. \lambda x. f (n f x)$
   - 投影函数：$P_i^k = \lambda x_1. \ldots \lambda x_k. x_i$

2. **复合的保持**：
   - 如果 $g_1, \ldots, g_m, h$ 都可λ定义，则 $f(\vec{x}) = h(g_1(\vec{x}), \ldots, g_m(\vec{x}))$ 也可λ定义

3. **原始递归的保持**：
   - 使用不动点组合子 $Y$ 定义递归函数
   - 例：加法 $+$ 定义为：
     $$+ = Y(\lambda p. \lambda m. \lambda n. \text{if } (m = 0) \text{ then } n \text{ else } S(p(\text{pred } m)n))$$

4. **μ算子的实现**：
   - $\mu y[g(\vec{x}, y) = 0]$ 可以通过递归搜索实现
   - 使用 $Y$ 组合子定义搜索函数

## 五、论题的哲学地位

### 5.1 数学定理还是经验假设？

Church-Turing论题与数学定理有本质区别：

| 数学定理 | Church-Turing论题 |
|----------|-------------------|
| 可以从公理证明 | 涉及直观概念"可有效计算" |
| 真值确定 | 是经验性/假说性的 |
| 可证伪（原则上） | 可证伪（如果发现反例） |

**论题的性质**：

- **非数学定理**：因为"可有效计算"是直观概念，没有形式化定义
- **经验支持**：所有已知的计算模型（包括现代的高级模型如量子计算）都被证明与图灵机等价
- **鲁棒性**：论题对计算模型的微小变化不敏感

### 5.2 支持证据

1. **模型等价性**：所有提出的形式化（λ演算、图灵机、递归函数、Post系统、Markov算法等）都被证明等价

2. **无反例**：从未发现直观上可计算但非图灵可计算的函数

3. **鲁棒性**：即使对计算模型进行各种扩展（多带图灵机、非确定性、概率性），可计算函数类保持不变

### 5.3 物理可计算性

**物理Church-Turing论题**：

> 任何物理过程都可以被图灵机有效模拟。

这个更强的论题涉及物理学基础。现代研究表明：

- **经典物理**：似乎支持物理Church-Turing论题
- **量子计算**：在可计算性层面与经典计算等价（量子图灵机可模拟经典图灵机，反之亦然）
- **超计算模型**（如无限时间图灵机）：超越标准Church-Turing论题，需要物理上可实现

## 六、现代应用与发展

### 6.1 在编程语言理论中的应用

Church-Turing论题为现代编程语言提供了理论基础：

**函数式编程**：

- Haskell、OCaml、Scala等语言直接基于λ演算
- 惰性求值对应于归约策略的选择
- 类型系统（Hindley-Milner）保证类型安全

**命令式编程**：

- 图灵机模型对应于冯·诺依曼架构
- 循环、条件语句对应于图灵机的控制流

### 6.2 在定理证明中的应用（Lean 4示例）

现代证明助手Lean 4使用λ演算作为核心：

```lean4
-- Lean 4中的Church数定义
def ChurchNat : Type := (α : Type) → (α → α) → α → α

-- 零的定义
def ChurchZero : ChurchNat := λ _ f x => x

-- 后继函数
def ChurchSucc (n : ChurchNat) : ChurchNat :=
  λ α f x => f (n α f x)

-- 加法（通过β归约计算）
def ChurchAdd (m n : ChurchNat) : ChurchNat :=
  λ α f x => m α f (n α f x)

-- 将自然数转换为Church数
def natToChurch : Nat → ChurchNat
  | 0 => ChurchZero
  | n+1 => ChurchSucc (natToChurch n)
```

### 6.3 复杂性理论的扩展

**扩展的Church-Turing论题**：

> 任何物理上可实现的计算设备都可以在多项式时间内被概率图灵机模拟。

这一论题与P vs NP问题密切相关，也是量子计算研究的基础。

**量子计算的挑战**：

- 量子计算机可以在多项式时间内解决某些问题（如因子分解），而经典计算机需要指数时间
- 这**不违反**Church-Turing论题（可计算性层面），但**可能违反**扩展论题（复杂性层面）
