## 一、历史背景：丘奇的创造

### 1.1 1930年代 Princeton

Alonzo Church在1930年代的普林斯顿大学致力于解决**Entscheidungsproblem**（判定问题）：是否存在一个通用算法可以判定任意一阶逻辑公式的有效性？

为了形式化"算法"的概念，Church创建了**λ演算**（Lambda Calculus）。这一创造发表于：

- **1932年**：λ演算的初步提出
- **1936年**：*An unsolvable problem of elementary number theory* — 使用λ演算证明Entscheidungsproblem不可解
- **1941年**：*The Calculi of Lambda Conversion* — λ演算的系统阐述

### 1.2 与图灵的合作与竞争

几乎同时（1936年），英国的Alan Turing独立提出了**图灵机**模型。两人很快证明了两种形式化的等价性：

> **Church-Turing论题**：λ可定义函数 = 图灵可计算函数 = 直观可计算函数

这一等价性是现代计算理论的基石。

### 1.3 λ演算的核心思想

丘奇的λ演算基于三个核心思想：

1. **函数即规则**：函数不是集合，而是计算规则
2. **匿名函数**：函数可以没有名字，通过 $\lambda$ 抽象直接构造
3. **高阶函数**：函数可以作为参数和返回值

## 二、λ演算的语法

### 2.1 BNF语法定义

**定义 2.1**（λ项的语法）：

给定可数无穷的变量集合 $\mathcal{V} = \{x, y, z, x_1, x_2, \ldots\}$，λ项的集合 $\Lambda$ 由以下BNF定义：

$$M, N ::= x \mid \lambda x.M \mid MN$$

其中：

- **$x$**：**变量**（variable）
- **$\lambda x.M$**：**抽象**（abstraction），表示以 $x$ 为参数的函数
- **$MN$**：**应用**（application），表示将函数 $M$ 应用于参数 $N$

### 2.2 归纳定义

λ项的集合 $\Lambda$ 也可以通过归纳定义：

**基础**：每个变量 $x \in \mathcal{V}$ 都是λ项。

**归纳**：如果 $M, N$ 是λ项，$x$ 是变量，则：

- $\lambda x.M$ 是λ项
- $MN$ 是λ项

**极小性**：只有通过上述规则构造的才是λ项。

### 2.3 语法约定

**约定 2.2**：

1. **应用左结合**：$MNP$ 表示 $(MN)P$，而非 $M(NP)$
2. **抽象右结合**：$\lambda x.\lambda y.M$ 表示 $\lambda x.(\lambda y.M)$
3. **抽象作用域最大化**：$\lambda x.MN$ 表示 $\lambda x.(MN)$，而非 $(\lambda x.M)N$

**示例**：

| 写法 | 解析 | 含义 |
|------|------|------|
| $\lambda x.\lambda y.xy$ | $\lambda x.(\lambda y.(xy))$ | 二元函数 |
| $\lambda x.xy$ | $\lambda x.(xy)$ | $x$ 映射到 $x y$ |
| $xyz$ | $(xy)z$ | $x$ 应用于 $y$，结果应用于 $z$ |

## 三、自由变量与绑定变量

### 3.1 自由变量的定义

**定义 3.1**（自由变量 $FV$）：

项 $M$ 的自由变量集合 $FV(M)$ 递归定义如下：

$$
\begin{aligned}
FV(x) &= \{x\} \\
FV(\lambda x.M) &= FV(M) \setminus \{x\} \\
FV(MN) &= FV(M) \cup FV(N)
\end{aligned}
$$

**直观理解**：

- 变量的自由出现是不受 $\lambda$ 绑定的出现
- 在 $\lambda x.M$ 中，$M$ 中的 $x$ 被绑定

### 3.2 绑定变量与作用域

**定义 3.2**（绑定变量）：

在项 $\lambda x.M$ 中：

- **绑定者**（binder）：$\lambda x$
- **绑定变量**：$x$
- **作用域**：$M$ 中所有 $x$ 的自由出现都被绑定

**示例**：

- $FV(\lambda x.x) = \emptyset$（$x$ 被绑定）
- $FV(\lambda x.y) = \{y\}$（$y$ 是自由的）
- $FV((\lambda x.xy)z) = \{y, z\}$

### 3.3 闭项与开放项

**定义 3.3**：

- **闭项**（closed term）或**组合子**（combinator）：$FV(M) = \emptyset$
- **开放项**：$FV(M) \neq \emptyset$

**重要组合子**：

- $I = \lambda x.x$（恒等组合子）
- $K = \lambda x.\lambda y.x$（常量组合子）
- $S = \lambda f.\lambda g.\lambda x.fx(gx)$（替换组合子）

## 四、α-变换：绑定变量的重命名

### 4.1 α-等价的定义

**定义 4.1**（α-等价 $=_\alpha$）：

两个项是α-等价的，如果它们只在绑定变量的名称上不同：

$$\lambda x.M =_\alpha \lambda y.M[y/x]$$

其中：

- $y$ 不在 $M$ 中自由出现（或进行适当的重命名）
- $M[y/x]$ 表示将 $M$ 中自由出现的 $x$ 替换为 $y$

### 4.2 α-等价的性质

**定理 4.2**：α-等价是等价关系：

1. **自反性**：$M =_\alpha M$
2. **对称性**：如果 $M =_\alpha N$，则 $N =_\alpha M$
3. **传递性**：如果 $M =_\alpha N$ 且 $N =_\alpha P$，则 $M =_\alpha P$

### 4.3 Barendregt约定

**Barendregt约定**（变量命名约定）：

在同一个项中，所有绑定变量都使用不同的名称，且不同于任何自由变量。

**示例**：

原始项：$(\lambda x.x(\lambda x.x))x$

按照Barendregt约定重写：$(\lambda x.x(\lambda y.y))z$

**目的**：避免在替换时出现变量捕获问题。

## 五、替换与变量捕获

### 5.1 替换的定义

**定义 5.1**（替换 $M[N/x]$）：

将 $M$ 中所有**自由出现**的 $x$ 替换为 $N$：

$$
\begin{aligned}
x[N/x] &= N \\
y[N/x] &= y \quad (y \neq x) \\
(PQ)[N/x] &= (P[N/x])(Q[N/x]) \\
(\lambda y.P)[N/x] &= \begin{cases}
\lambda y.P & y = x \\
\lambda y.(P[N/x]) & y \neq x, y \notin FV(N) \\
\lambda z.((P[z/y])[N/x]) & y \neq x, y \in FV(N), z \text{ 新变量}
\end{cases}
\end{aligned}
$$

### 5.2 变量捕获问题

**问题示例**：

考虑 $(\lambda y.xy)[y/x]$：

- 直接替换得到 $\lambda y.yy$，但 $y$ 被错误地绑定了！
- 正确做法：先进行α变换，$(\lambda y.xy) =_\alpha (\lambda z.xz)$
- 然后替换：$(\lambda z.xz)[y/x] = \lambda z.yz$

### 5.3 替换引理

**引理 5.2**（替换引理）：

如果 $x \notin FV(L)$，则：
$$M[N/x][L/y] = M[L/y][N[L/y]/x]$$

## 六、β-归约：计算的本质

### 6.1 β-归约的定义

**定义 6.1**（β-归约 $\to_\beta$）：

$(\lambda x.M)N \to_\beta M[N/x]$

**术语**：

- **可归约式**（redex）：形如 $(\lambda x.M)N$ 的子项
- **合约**（contract）：将 redex 替换为 $M[N/x]$

### 6.2 β-归约的示例

**示例 6.2**（恒等函数）：

$$(\lambda x.x)y \to_\beta x[y/x] = y$$

**示例 6.3**（常量函数）：

$$(\lambda x.\lambda y.x)ab \to_\beta (\lambda y.a)b \to_\beta a$$

**示例 6.4**（自应用）：

$$(\lambda x.xx)(\lambda x.xx) \to_\beta (\lambda x.xx)(\lambda x.xx) \to_\beta \ldots$$

这是著名的 **$\Omega$ 组合子**，导致无限归约。

### 6.3 β-等价的定义

**定义 6.5**（β-等价 $=_\beta$）：

$M =_\beta N$ 当且仅当存在序列 $M = M_0, M_1, \ldots, M_n = N$，其中每一步是β-归约或β-归约的逆。

## 七、η-变换与外延性

### 7.1 η-归约的定义

**定义 7.1**（η-归约 $\to_\eta$）：

$$\lambda x.Mx \to_\eta M \quad (x \notin FV(M))$$

**直观理解**：如果 $M$ 是一个函数，且对所有 $x$ 有 $Mx = Nx$，则 $M = N$（外延性）。

### 7.2 η-等价与外延性

**定理 7.2**（外延性）：

如果对所有 $N$ 有 $MN =_\beta NN'$，则 $M =_{\beta\eta} N$。

**示例 7.3**：

- $\lambda x.fx =_\eta f$（如果 $x \notin FV(f)$）
- $\lambda x.(\lambda y.y)x =_\eta \lambda y.y =_\eta \lambda x.x$

## 八、归约策略与范式

### 8.1 范式的定义

**定义 8.1**（β-范式）：

项 $M$ 是β-范式，如果它不包含任何β-redex。

**示例**：

- $\lambda x.x$ 是范式
- $y$ 是范式
- $(\lambda x.x)y$ 不是范式（可归约到 $y$）

### 8.2 归约策略

**定义 8.2**（归约策略）：

1. **最左归约**（Normal Order）：总是归约最左边的 redex
2. **应用序归约**（Applicative Order）：先归约参数，再归约函数
3. **按值调用**（Call-by-Value）：应用序的变体，不允许归约抽象内部

**定理 8.3**（Church-Rosser定理）：

如果 $M \to_\beta N_1$ 且 $M \to_\beta N_2$，则存在 $P$ 使得 $N_1 \to_\beta P$ 且 $N_2 \to_\beta P$。

**推论**：

- 如果项有范式，则范式是唯一的（模α-等价）
- 最左归约总能找到范式（如果存在）

### 8.3 归约示例对比

**示例 8.4**：

考虑 $(\lambda x.\lambda y.y)\Omega$，其中 $\Omega = (\lambda x.xx)(\lambda x.xx)$。

**应用序归约**：

1. 先归约 $\Omega$ → 无限循环！

**最左归约**：

1. $(\lambda x.\lambda y.y)\Omega \to_\beta \lambda y.y$（停止）

**结论**：最左归约是**规范化**的。

## 九、现代应用：从Haskell到Lean 4

### 9.1 Haskell中的λ演算

Haskell直接基于λ演算：

```haskell
-- 匿名函数（lambda抽象）
\x -> x          -- 恒等函数
\x y -> x        -- K组合子
\f g x -> f (g x) -- 函数复合

-- β归约示例
(\x -> x + 1) 5  -- 归约到 6

-- 惰性求值对应于最左归约
-- 无限列表是可能的，因为不计算不需要的部分
ones = 1 : ones  -- 对应于不动点构造
```

### 9.2 Lean 4中的λ演算

Lean 4的核心是依赖类型λ演算：

```lean4
-- 基本λ项
#check λ x => x              -- ?m.2 → ?m.2（恒等函数）
#check λ x y => x            -- ?m.4 → ?m.5 → ?m.4（K组合子）

-- β归约示例
#reduce (λ x => x + 1) 5     -- 6

-- 组合子定义
def I (α : Type) : α → α := λ x => x
def K (α β : Type) : α → β → α := λ x y => x
def S (α β γ : Type) : (α → β → γ) → (α → β) → α → γ :=
  λ f g x => f x (g x)

-- Church数（与丘奇原始定义一致）
def ChurchNat : Type := (α : Type) → (α → α) → α → α
def ChurchZero : ChurchNat := λ _ f x => x
def ChurchSucc (n : ChurchNat) : ChurchNat :=
  λ α f x => f (n α f x)

-- β归约验证Church数
#reduce ChurchSucc ChurchZero Nat (λ n => n + 1) 0  -- 1
```

### 9.3 编程语言中的归约策略

| 语言 | 归约策略 | 特点 |
|------|----------|------|
| Haskell | 惰性求值（最左归约） | 支持无限数据结构 |
| OCaml | 按值调用 | 严格求值，效率优先 |
| Scheme | 应用序 | 严格函数式 |
| Lean 4 | 按需求值 | 支持计算与证明 |
