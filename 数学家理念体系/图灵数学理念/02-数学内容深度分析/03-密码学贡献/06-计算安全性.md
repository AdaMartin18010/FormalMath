## 二、基于复杂度的安全性

### 2.1 复杂度基础

**复杂度类**：

密码的安全性可以基于复杂度类：

- **P类**：多项式时间内可破解的密码不安全
- **NP类**：NP问题可能安全
- **EXPTIME类**：指数时间问题可能更安全

密码是**计算安全的**，如果破解密码需要超多项式时间：

- **多项式时间**：多项式时间内可破解的密码不安全
- **超多项式时间**：超多项式时间内可破解的密码可能安全

## 五、典型例题

### 5.1 例题1：分析基于复杂度的密码安全性

**问题**：

分析如何基于计算复杂度评估密码的安全性。

**解答**：

**复杂度分析**：

评估密码的安全性需要分析破解密码的复杂度：

- **破解算法**：设计破解密码的算法
- **复杂度分析**：分析算法的复杂度
- **安全性评估**：根据复杂度评估安全性

**形式化表述**：

密码是**计算安全的**，如果破解密码需要超多项式时间：

\[
T_{\text{破解}}(n) = \Omega(2^{n^k})
\]

其中 $n$ 是密钥长度，$k > 0$。

**安全性等级**：

- **低安全性**：多项式时间可破解（$T_{\text{破解}}(n) = O(n^k)$）
- **中等安全性**：超多项式时间可破解（$T_{\text{破解}}(n) = \Omega(n^{\log n})$）
- **高安全性**：指数时间可破解（$T_{\text{破解}}(n) = \Omega(2^{n^k})$）

**形式化定义**：

设密码系统为 $S$，密钥长度为 $n$，攻击算法为 $A$，则：

- **计算安全性**：$S$ 是计算安全的，如果对于所有多项式时间算法 $A$：
  \[
  P(A \text{ 破解 } S) < \epsilon(n)
  \]
  其中 $\epsilon(n)$ 是可忽略函数。

- **可忽略函数**：函数 $\epsilon: \mathbb{N} \to \mathbb{R}$ 是**可忽略的**，如果：
  \[
  \forall \text{多项式 } p, \exists n_0, \forall n > n_0, \epsilon(n) < \frac{1}{p(n)}
  \]

**安全性层级**：

\[
\text{完美安全} > \text{计算安全} > \text{统计安全} > \text{多项式安全}
\]

其中：

- **完美安全**：$P(\text{破解}) = 0$（信息论安全）
- **计算安全**：$P(\text{破解}) < \epsilon(n)$（计算安全）
- **统计安全**：$P(\text{破解}) < \epsilon$（统计安全）
- **多项式安全**：$T_{\text{破解}}(n) = \Omega(n^k)$（多项式安全）

### 5.2 例题2：分析不可判定性与密码安全性

**问题**：

分析不可判定性在密码安全性中的应用。

**解答**：

**不可判定问题**：

某些密码问题可能是**不可判定的**：

- **密钥存在性**：判断密钥是否存在可能是不可判定的
- **密码等价性**：判断两个密码是否等价可能是不可判定的
- **安全性判定**：判断密码是否安全可能是不可判定的

**安全性保证**：

不可判定性提供安全性保证：

- **理论保证**：不可判定性提供理论安全性保证
- **实际应用**：不可判定性在密码设计中有应用
- **安全性分析**：不可判定性用于安全性分析

**形式化表述**：

设密码问题为 $P$，则：

- **不可判定性**：如果 $P$ 是不可判定的，则不存在算法可以判定 $P$
- **安全性**：如果破解密码需要判定 $P$，且 $P$ 不可判定，则密码是安全的
- **理论保证**：不可判定性提供理论安全性保证

**应用示例**：

- **密钥存在性**：判断密钥是否存在可能是不可判定的
- **密码等价性**：判断两个密码是否等价可能是不可判定的
- **安全性判定**：判断密码是否安全可能是不可判定的

**意义**：

- **理论保证**：不可判定性提供理论安全性保证
- **实际应用**：不可判定性在密码设计中有应用
- **安全性分析**：不可判定性用于安全性分析
- **计算边界**：不可判定性揭示了计算的边界

**归约的传递性**：

如果密码系统 $S_1$ 的安全性归约到问题 $P_1$，且 $P_1$ 归约到问题 $P_2$，则 $S_1$ 的安全性也归约到 $P_2$：

\[
S_1 \leq_P P_1 \land P_1 \leq_P P_2 \Rightarrow S_1 \leq_P P_2
\]

**归约的详细构造**：

对于RSA加密，归约的详细构造如下：

1. **假设**：存在多项式时间算法 $A$ 可以破解RSA
2. **构造**：使用 $A$ 构造多项式时间算法 $B$ 分解大整数
3. **归约**：$B$ 调用 $A$，将分解问题转化为破解RSA问题
4. **矛盾**：如果大整数分解是困难的，则RSA是安全的

### 5.3 例题3：分析计算安全性的形式化定义

**问题**：

分析计算安全性的形式化定义，特别是多项式时间安全性。

**解答**：

**形式化定义**：

密码系统是**计算安全的**，如果对于所有多项式时间算法 $A$：

\[
P(A \text{ 破解密码}) < \epsilon(n)
\]

其中 $\epsilon(n)$ 是安全阈值（通常是可忽略函数）。

**可忽略函数**：

函数 $\epsilon: \mathbb{N} \to \mathbb{R}$ 是**可忽略的**，如果：

\[
\forall \text{多项式 } p, \exists n_0, \forall n > n_0, \epsilon(n) < \frac{1}{p(n)}
\]

**可忽略函数的性质**：

- **渐近性**：可忽略函数在 $n \to \infty$ 时趋于0
- **多项式界**：可忽略函数比任何多项式的倒数都小
- **示例**：$\epsilon(n) = 2^{-n}$、$\epsilon(n) = n^{-\log n}$ 都是可忽略函数

**安全性等级**：

- **完美安全性**：$P(\text{破解}) = 0$（信息论安全）
- **计算安全性**：$P(\text{破解}) < \epsilon(n)$（计算安全）
- **统计安全性**：$P(\text{破解}) < \epsilon$（统计安全）

**安全性比较**：

\[
\text{完美安全} > \text{计算安全} > \text{统计安全}
\]

其中：

- **完美安全**：无条件安全，不依赖计算假设
- **计算安全**：基于计算复杂度假设的安全
- **统计安全**：基于统计假设的安全

**意义**：

- **理论基础**：计算安全性为密码学提供理论基础
- **实际应用**：计算安全性在实际密码系统中有应用
- **安全性分析**：计算安全性用于安全性分析

### 5.4 例题4：分析计算安全性的归约证明

**问题**：

分析如何使用归约方法证明密码系统的计算安全性。

**解答**：

**归约证明**：

归约是证明计算安全性的标准方法：

- **归约定义**：将破解密码系统归约到解决困难问题
- **安全性证明**：如果困难问题无法解决，则密码系统是安全的
- **复杂度假设**：安全性基于某些复杂度假设

**形式化表述**：

设密码系统为 $S$，困难问题为 $P$，则：

- **归约**：如果存在多项式时间归约 $R$，使得破解 $S$ 可以归约到解决 $P$，则 $S$ 的安全性基于 $P$ 的困难性
- **安全性**：如果 $P$ 是困难的（如NP困难），则 $S$ 是计算安全的

**归约示例**：

RSA加密的安全性归约到大整数分解问题：

- **困难问题**：大整数分解是困难的（假设）
- **归约**：破解RSA可以归约到分解大整数
- **安全性**：如果大整数分解是困难的，则RSA是计算安全的

**意义**：

- **证明方法**：归约是证明计算安全性的标准方法
- **理论基础**：归约为计算安全性提供理论基础
- **实际应用**：归约在实际密码系统中有应用
