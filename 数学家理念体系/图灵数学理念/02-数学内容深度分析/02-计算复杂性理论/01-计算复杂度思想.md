## 一、复杂度的早期思想

### 1.1 图灵的复杂度思想

**图灵的贡献**：

图灵在1936年的论文《论可计算数》中已经考虑了计算的**复杂度**：

- **时间复杂度**：计算的步数
- **空间复杂度**：计算的存储空间
- **复杂度分析**：复杂度分析的方法

**历史意义**：

虽然图灵没有明确提出"复杂度理论"，但他的思想为后来的计算复杂性理论奠定了基础：

- **早期洞察**：图灵在1936年就意识到计算效率的重要性
- **理论基础**：图灵的复杂度思想为复杂性理论提供理论基础
- **方法启发**：图灵的复杂度分析方法启发了后来的研究

### 1.2 复杂度的重要性

**计算效率**：

图灵认识到，仅仅知道一个问题**可计算**是不够的，还需要考虑计算的**效率**：

- **可计算性**：问题是否可以计算
- **复杂度**：计算需要多少时间和空间
- **实用性**：复杂度决定计算的实用性

**现代发展**：

图灵的复杂度思想发展为现代计算复杂性理论：

- **P vs NP问题**：图灵的思想影响P vs NP问题
- **复杂度类**：影响复杂度类的定义
- **算法分析**：影响算法分析

## 二、时间复杂度的概念

### 2.1 时间复杂度的定义

**定义**：

图灵机 $M$ 在输入 $w$ 上的**时间复杂度** $T_M(w)$ 是 $M$ 在 $w$ 上运行所需的**步数**。

**形式化表述**：

设图灵机 $M$ 在输入 $w$ 上的计算序列为：

\[
C_0, C_1, C_2, \ldots, C_k
\]

其中 $C_0$ 是初始配置，$C_k$ 是终止配置，则：

\[
T_M(w) = k
\]

**最坏情况复杂度**：

对于输入长度 $n$，定义：

\[
T_M(n) = \max\{T_M(w) : |w| = n\}
\]

### 2.2 时间复杂度类

**多项式时间**：

语言 $L$ 属于**P类**（多项式时间），如果存在图灵机 $M$ 和多项式 $p(n)$，使得：

- $M$ 判定 $L$
- 对于任意输入 $w$，$T_M(w) \leqq p(|w|)$

**指数时间**：

语言 $L$ 属于**EXPTIME类**（指数时间），如果存在图灵机 $M$ 和常数 $c$，使得：

- $M$ 判定 $L$
- 对于任意输入 $w$，$T_M(w) \leqq 2^{c|w|}$

## 三、空间复杂度的概念

### 3.1 空间复杂度的定义

**定义**：

图灵机 $M$ 在输入 $w$ 上的**空间复杂度** $S_M(w)$ 是 $M$ 在 $w$ 上运行所需的**带单元数**（不包括输入带）。

**形式化表述**：

设图灵机 $M$ 在输入 $w$ 上的计算序列为：

\[
C_0, C_1, C_2, \ldots, C_k
\]

其中每个配置 $C_i$ 使用 $s_i$ 个带单元，则：

\[
S_M(w) = \max\{s_i : 0 \leqq i \leqq k\}
\]

**最坏情况复杂度**：

对于输入长度 $n$，定义：

\[
S_M(n) = \max\{S_M(w) : |w| = n\}
\]

### 3.2 空间复杂度类

**多项式空间**：

语言 $L$ 属于**PSPACE类**（多项式空间），如果存在图灵机 $M$ 和多项式 $p(n)$，使得：

- $M$ 判定 $L$
- 对于任意输入 $w$，$S_M(w) \leqq p(|w|)$

**对数空间**：

语言 $L$ 属于**L类**（对数空间），如果存在图灵机 $M$ 和常数 $c$，使得：

- $M$ 判定 $L$
- 对于任意输入 $w$，$S_M(w) \leqq c \log |w|$

- **时间换空间**：使用更多时间可以减少空间需求

- **L** ⊆ **P** ⊆ **NP** ⊆ **PSPACE** ⊆ **EXPTIME**

## 五、典型例题

### 5.1 例题1：分析加法算法的时间复杂度

**问题**：

分析图灵机计算 $f(x, y) = x + y$ 的时间复杂度。

**解答**：

**算法描述**：

1. 输入：$1^x 0 1^y$（一元编码）
2. 过程：删除一个 $1$（从第二个数），将分隔符 $0$ 替换为 $1$，重复直到第二个数变为空
3. 输出：$1^{x+y}$

**复杂度分析**：

- **步数**：需要 $y$ 步（删除 $y$ 个 $1$）
- **时间复杂度**：$T_M(x, y) = O(y)$
- **最坏情况**：当 $y$ 很大时，时间复杂度为 $O(n)$，其中 $n = x + y$ 是输入长度

**形式化分析**：

设输入长度为 $n = x + y$，则：

- **时间复杂度**：$T_M(n) = O(n)$（线性时间）
- **空间复杂度**：$S_M(n) = O(n)$（需要存储输入和输出）
- **复杂度类**：加法属于**P类**（多项式时间）

**正确性证明**：

使用循环不变量证明算法的正确性：

- **循环不变量**：在每次迭代后，带上的内容为 $1^{x+i} 0 1^{y-i}$，其中 $i$ 是已删除的 $1$ 的个数
- **初始**：$i = 0$，带上内容为 $1^x 0 1^y$
- **保持**：每次迭代删除一个 $1$，$i$ 增加1，不变量保持
- **终止**：当 $i = y$ 时，带上内容为 $1^{x+y}$，算法终止

**结论**：

加法算法的时间复杂度是**线性的**，即 $O(n)$，且算法是正确的。

### 5.2 例题2：分析乘法算法的时间复杂度

**问题**：

分析图灵机计算 $f(x, y) = x \times y$ 的时间复杂度。

**解答**：

**算法描述**：

1. 输入：$1^x 0 1^y$（一元编码）
2. 过程：重复 $x$ 次，每次复制 $y$ 个 $1$
3. 输出：$1^{x \times y}$

**复杂度分析**：

- **步数**：需要 $x \times y$ 步（复制 $x$ 次，每次复制 $y$ 个符号）
- **时间复杂度**：$T_M(x, y) = O(x \times y)$
- **最坏情况**：当 $x$ 和 $y$ 都很大时，时间复杂度为 $O(n^2)$，其中 $n = x + y$ 是输入长度

**结论**：

乘法算法的时间复杂度是**二次的**，即 $O(n^2)$。

### 5.3 例题3：分析空间复杂度

**问题**：

分析图灵机计算 $f(x, y) = x + y$ 的空间复杂度。

**解答**：

**空间需求**：

- **输入空间**：$O(x + y)$（存储输入）
- **工作空间**：$O(1)$（只需要常数空间进行运算）
- **输出空间**：$O(x + y)$（存储结果）

**总空间复杂度**：

- **空间复杂度**：$S_M(x, y) = O(x + y) = O(n)$

**空间复杂度分析**：

- **输入空间**：$O(n)$（存储输入 $1^x 0 1^y$）
- **工作空间**：$O(1)$（只需要常数空间进行运算）
- **输出空间**：$O(n)$（存储结果 $1^{x+y}$）
- **总空间**：$S_M(n) = O(n)$（线性空间）

**空间优化**：

可以使用原地算法（in-place algorithm）将空间复杂度降低到 $O(1)$（不包括输入和输出），但基本算法的空间复杂度是 $O(n)$。

**结论**：

加法算法的空间复杂度是**线性的**，即 $O(n)$。

### 5.4 例题4：分析时间-空间权衡

**问题**：

分析算法的时间-空间权衡，特别是如何在时间和空间之间找到平衡。

**解答**：

**时间-空间权衡**：

算法的时间复杂度和空间复杂度之间存在**权衡关系**：

- **时间换空间**：使用更多时间可以减少空间需求
- **空间换时间**：使用更多空间可以减少时间需求

**形式化表述**：

设算法 $M$ 在输入 $w$ 上的时间和空间复杂度分别为 $T_M(w)$ 和 $S_M(w)$，则：

\[
T_M(w) \cdot S_M(w) \geqq \Omega(|w|)
\]

**最优权衡**：

寻找时间和空间的最优权衡：

\[
\min_{M} \alpha T_M(n) + \beta S_M(n)
\]

其中 $\alpha$ 和 $\beta$ 是权重。

**意义**：

- **算法设计**：时间-空间权衡指导算法设计
- **资源优化**：时间-空间权衡用于资源优化
- **实际应用**：时间-空间权衡在实际应用中有重要意义
