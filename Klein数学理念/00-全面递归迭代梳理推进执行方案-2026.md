## 一、执行工作流程

### 1.1 单文档梳理流程

#### 步骤1：文档扫描与评估（30分钟）

**任务清单**：

- [ ] 读取文档全文
- [ ] 提取核心概念列表（至少5个）
- [ ] 识别关键定理和方法
- [ ] 评估当前状态：
  - [ ] 权威对齐度（0-100%）
  - [ ] 思维表征度（0-100%）
  - [ ] 内容完整度（0-100%）
  - [ ] 现代性（0-100%）
- [ ] 填写文档评估表（见模板）

**产出**：

- 文档评估表
- 核心概念清单
- 改进需求清单

#### 步骤2：权威信息对齐（2-3小时）

**任务清单**：

**2.1 概念对齐**（每个概念15-20分钟）：

- [ ] 在Wikipedia检索概念条目
- [ ] 在nLab检索（如适用）
- [ ] 在MathWorld检索（如适用）
- [ ] 对比项目定义与权威定义
- [ ] 记录差异和缺失
- [ ] 补充或修正定义
- [ ] 添加权威引用

**2.2 术语对齐**（30分钟）：

- [ ] 检查术语是否使用标准表述
- [ ] 修正非标准术语
- [ ] 添加术语说明（如需要）

**2.3 历史事实对齐**（30分钟）：

- [ ] 检查历史事实是否准确
- [ ] 在权威来源验证历史事实
- [ ] 补充或修正历史事实
- [ ] 添加历史引用

**2.4 现代发展对齐**（1小时）：

- [ ] 检索2020-2026最新研究（arXiv、MathSciNet）
- [ ] 识别相关最新研究
- [ ] 补充最新发展内容
- [ ] 添加最新研究引用

**2.5 教育标准对齐**（如适用，1小时）：

- [ ] 检查是否与ICMI对齐
- [ ] 检查是否与OECD对齐
- [ ] 检查是否与NCTM对齐
- [ ] 检查是否与新课标对齐
- [ ] 添加教育标准对照表

**产出**：

- 权威对齐报告
- 更新后的文档（含权威引用）
- 权威引用清单

#### 步骤3：思维表征补充（2-3小时）

**任务清单**：

**3.1 多维矩阵创建**（每个矩阵30-45分钟）：

- [ ] 识别需要矩阵对比的核心概念
- [ ] 确定矩阵维度（定义、属性、关系、应用等）
- [ ] 确定对比概念（至少5个）
- [ ] 创建矩阵表格
- [ ] 填写矩阵内容
- [ ] 添加矩阵说明

**3.2 概念属性树图创建**（每个树图30-45分钟）：

- [ ] 识别需要树图的核心概念
- [ ] 确定树图结构（定义、属性、关系、解释、论证、证明）
- [ ] 创建树图（Mermaid格式）
- [ ] 添加树图说明

**3.3 决策树图创建**（如适用，每个30-45分钟）：

- [ ] 识别需要决策树的教学主题或问题
- [ ] 确定决策分支和条件
- [ ] 创建决策树图（Mermaid格式）
- [ ] 添加决策树说明

**3.4 推理树图创建**（如适用，每个45-60分钟）：

- [ ] 识别需要推理树的重要定理
- [ ] 确定推理步骤（前提、推理、结论）
- [ ] 创建推理树图（Mermaid格式）
- [ ] 添加推理树说明

**3.5 层次树图创建**（如适用，每个30-45分钟）：

- [ ] 识别需要层次树的知识体系
- [ ] 确定层次结构（抽象→具体）
- [ ] 创建层次树图（Mermaid格式）
- [ ] 添加层次树说明

**产出**：

- 思维表征补充报告
- 更新后的文档（含思维表征）
- 思维表征清单

#### 步骤4：内容完善（2-3小时）

**任务清单**：

**4.1 内容完整性检查**（1小时）：

- [ ] 检查概述是否完整
- [ ] 检查定义是否清晰
- [ ] 检查性质是否全面
- [ ] 检查应用是否丰富
- [ ] 检查历史是否准确
- [ ] 检查现代发展是否最新
- [ ] 补充缺失内容

**4.2 论证充分性检查**（1小时）：

- [ ] 检查重要观点是否有论据支持
- [ ] 检查论据是否有证据
- [ ] 检查论证链条是否完整
- [ ] 补充论据和证据
- [ ] 完善论证链条

**4.3 数学严格性检查**（1小时）：

- [ ] 检查定义是否严格
- [ ] 检查定理是否有证明
- [ ] 检查推理是否完整
- [ ] 补充严格定义
- [ ] 补充证明或证明思路
- [ ] 完善推理步骤

**产出**：

- 内容完善报告
- 更新后的文档

#### 步骤5：质量检查（1小时）

**任务清单**：

- [ ] 使用质量评估矩阵评估文档
- [ ] 计算综合评分
- [ ] 识别不合格项
- [ ] 制定改进计划
- [ ] 执行改进（如需要）
- [ ] 填写质量检查表

**产出**：

- 质量检查报告
- 质量评估表
- 最终文档
