# FormalMath 数学知识本体框架

**版本**: v1.0  
**目标**: 建立严格的数学知识本体，支持语义检索与推理  
**核心**: 概念-属性-关系-证明网络

---

## 一、知识本体架构

### 1.1 核心实体类型

```
数学知识本体
│
├─ 概念 (Concept)
│   ├─ 定义 (Definition)
│   ├─ 属性 (Property)
│   └─ 示例 (Example)
│
├─ 定理 (Theorem)
│   ├─ 条件 (Condition)
│   ├─ 结论 (Conclusion)
│   └─ 证明 (Proof)
│
├─ 方法 (Method)
│   ├─ 算法 (Algorithm)
│   ├─ 技巧 (Technique)
│   └─ 策略 (Strategy)
│
├─ 问题 (Problem)
│   ├─ 习题 (Exercise)
│   ├─ 反例 (Counterexample)
│   └─ 开放问题 (OpenProblem)
│
└─ 资源 (Resource)
    ├─ 文献 (Reference)
    ├─ 课程 (Course)
    └─ 形式化代码 (FormalCode)
```

### 1.2 关系类型体系

| 关系类型 | 描述 | 示例 |
|----------|------|------|
| **isA** | 概念层级 | 群 isA 代数结构 |
| **dependsOn** | 依赖关系 | 商群 dependsOn 正规子群 |
| **implies** | 蕴含关系 | 一致收敛 implies 点态收敛 |
| **equivalentTo** | 等价关系 | 柯西序列 equivalentTo 收敛序列 |
| **contrastsWith** | 对比关系 | 逐点收敛 contrastsWith 一致收敛 |
| **proves** | 证明关系 | 中值定理 proves 导数零点存在 |
| **appliedIn** | 应用关系 | 谱序列 appliedIn 层上同调 |
| **generalizes** | 推广关系 | 范畴论 generalizes 集合论 |

## 二、概念本体Schema

### 2.1 概念元数据

```yaml
concept:
  # 标识
  id: "GRP-001"
  name: "群"
  english_name: "Group"
  aliases: ["Group", "抽象群", "群结构"]
  
  # 分类
  msc_primary: "20A05"
  msc_secondary: ["20-01", "08A99"]
  branch: "代数结构"
  topic: "群论"
  difficulty: 2  # 1-5
  
  # 定义
  definition:
    formal: |
      群是一个有序对 $(G, \cdot)$，其中 $G$ 是集合，$\cdot: G \times G \to G$ 是二元运算，满足：
      1. 结合律: $(a \cdot b) \cdot c = a \cdot (b \cdot c)$
      2. 单位元: $\exists e \in G, \forall a \in G: e \cdot a = a \cdot e = a$
      3. 逆元: $\forall a \in G, \exists a^{-1} \in G: a \cdot a^{-1} = a^{-1} \cdot a = e$
    intuitive: "群是带有可逆运算的对称结构"
    
  # 属性
  properties:
    - property: "阶"
      notation: "|G| 或 #G"
      description: "群的元素个数（可能无限）"
    - property: "交换性"
      notation: "G 是阿贝尔群"
      condition: "$\forall a,b \in G: a \cdot b = b \cdot a$"
      
  # 关系
  relationships:
    isA:
      - " magma"
      - "半群"
      - "幺半群"
    generalizesTo:
      - "环"
      - "域"
    related:
      - "子群"
      - "商群"
      - "群同态"
      
  # 历史
  history:
    first_introduced_by: "Évariste Galois"
    year: 1830
    context: "多项式方程的可解性研究"
    
  # 参考文献
  references:
    textbooks:
      - title: "Algebra"
        author: "Michael Artin"
        chapter: "2.1"
      - title: "Abstract Algebra"
        author: "Dummit & Foote"
        chapter: "1.1"
    papers:
      - title: "Mémoire sur les conditions de résolubilité des équations par radicaux"
        author: "Évariste Galois"
        year: 1830
    stacks_tags:
      - "01FN"  # 群的定义
    nlab:
      - "https://ncatlab.org/nlab/show/group"
    mathworld:
      - "http://mathworld.wolfram.com/Group.html"
      
  # 形式化
  formalization:
    lean4: |
      class Group (G : Type*) extends Mul G, One G, Inv G where
        mul_assoc : ∀ a b c : G, (a * b) * c = a * (b * c)
        one_mul : ∀ a : G, 1 * a = a
        mul_one : ∀ a : G, a * 1 = a
        inv_mul_cancel : ∀ a : G, a⁻¹ * a = 1
        mul_inv_cancel : ∀ a : G, a * a⁻¹ = 1
    isabelle: "..."
    coq: "..."
    
  # 文档位置
  documents:
    - "docs/02-代数结构/02-核心理论/群论/01-群论.md"
    - "docs/02-代数结构/02-核心理论/群论/01-群论-国际标准深度扩展版.md"
    
  # 习题
  exercises:
    - id: "GRP-001"
      count: 25
      
  # 思维表征
  representations:
    - type: "概念图谱"
      file: "concept-maps/group-concept-map.mmd"
```

## 三、知识图谱构建

### 3.1 图谱节点类型

| 节点类型 | 属性 | 示例 |
|----------|------|------|
| 概念节点 | id, name, msc, difficulty | 群、环、拓扑空间 |
| 定理节点 | id, name, proof_status, importance | 中值定理、Sylow定理 |
| 方法节点 | id, name, applicability | ε-δ方法、归纳法 |
| 问题节点 | id, type, difficulty, solution | ANA-001, CEX-001 |
| 资源节点 | id, type, url, access_date | Stacks Tag 00DV |

### 3.2 图谱边类型

```cypher
// Cypher查询语言示例

// 概念依赖
MATCH (c:Concept {name: "商群"})-[:DEPENDS_ON]->(pre:Concept)
RETURN pre.name
// 返回: 正规子群、群、陪集

// 定理蕴含
MATCH (t:Theorem)-[:IMPLIES]->(implied:Theorem)
WHERE t.name = "Lagrange定理"
RETURN implied.name
// 返回: Euler定理, Fermat小定理

// 概念到文档
MATCH (c:Concept)-[:DOCUMENTED_IN]->(d:Document)
WHERE c.name = "群"
RETURN d.path

// 定理证明状态
MATCH (t:Theorem)
WHERE t.lean4_status = "COMPLETE"
RETURN t.name, t.lean4_file
```

## 六、查询接口设计

### 6.1 概念查询

```graphql
# GraphQL查询示例

# 查询概念详情
query GetConcept($id: ID!) {
  concept(id: $id) {
    name
    definition {
      formal
      intuitive
    }
    properties {
      name
      description
    }
    relationships {
      type
      target {
        name
      }
    }
    theorems {
      name
      statement
    }
    documents {
      path
      type
    }
    exercises {
      id
      difficulty
    }
    formalization {
      lean4
    }
  }
}

# 查询依赖路径
query GetDependencyPath($from: ID!, $to: ID!) {
  dependencyPath(from: $from, to: $to) {
    nodes {
      name
      type
    }
    edges {
      type
    }
  }
}
```
