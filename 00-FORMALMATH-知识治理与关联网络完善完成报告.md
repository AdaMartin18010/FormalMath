## 六、可持续维护建议

虽然本次治理已实现 100% 完成，但知识库是动态生长的。建议建立以下轻量级维护机制：

### 6.1 每月运行一次健康检查脚本

```bash
python tools/generate_project_stats.py
python tools/add_cross_references.py --dry-run
python tools/link_course_mappings.py --dry-run
```

- 监控是否有新增孤岛文档
- 监控是否有新增失效链接

### 6.2 新增文档的强制规范

- 任何新增的 `.md` 文件，末尾必须包含 "## 相关链接" 区块
- 任何新增的 docs/ 子目录，必须同时创建 `README.md`
- 禁止创建与现有目录语义重复的新目录

### 6.3 Lean4 代码的维护原则

- 不再追求"定理数量扩张"
- 若新增 .lean 文件，必须在 `docs/09-形式化证明/Lean4/00-定理形式化状态总览.md` 中同步登记
- 优先补全现有 A/B 类代码，而非新增 C 类骨架

### 6.4 统计报告的版本控制

- `docs/00-项目统计总览-v2.md` 为当前主版本
- 未来更新时，建议直接覆盖 v2 文件，或生成 v3，但需同步更新 README.md 的链接指向
