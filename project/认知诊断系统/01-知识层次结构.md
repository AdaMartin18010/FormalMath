## 2. L0: 基础概念层

### 2.1 层次定义

**目标**: 建立对数学概念的直观理解和基本认知

**特征**:
- 非形式化定义，强调直观理解
- 通过具体例子认识概念
- 了解基本性质，不涉及严格证明
- 能够识别和描述简单情况

### 2.2 能力要求

| 能力维度 | 描述 | 示例 |
|----------|------|------|
| 概念识别 | 能够识别概念的基本形式 | 识别一个对象是否是集合、函数 |
| 直观理解 | 理解概念的直观含义 | 理解函数是"输入-输出"关系 |
| 例子识别 | 能举出简单例子 | 举出有限集、空集的例子 |
| 基本操作 | 进行基本运算 | 求集合的并、交、补 |

### 2.3 各分支核心概念

#### 代数学 (Algebra)

```yaml
概念: 群 (Group)
  定义要点:
    - "一个集合配上一个二元运算"
    - "满足封闭性、结合律、单位元、逆元"
  
  直观理解:
    - "对称操作的集合"
    - "可逆变换的集合"
  
  基本例子:
    - "整数加法群 (Z, +)"
    - "循环群 Z_n"
    - "二面体群 D_n (正n边形的对称)"
  
  基本性质:
    - "单位元唯一"
    - "逆元唯一"
    - "消去律成立"
```

```yaml
概念: 环 (Ring)
  定义要点:
    - "一个集合配上两个二元运算(加法和乘法)"
    - "加法构成交换群，乘法满足结合律"
    - "乘法对加法满足分配律"
  
  直观理解:
    - "具有加减乘运算的代数结构"
    - "数的推广"
  
  基本例子:
    - "整数环 Z"
    - "多项式环 R[x]"
    - "矩阵环 M_n(R)"
```

#### 分析学 (Analysis)

```yaml
概念: 极限 (Limit)
  定义要点:
    - "当自变量趋近于某点时，函数值趋近于某值"
    - "ε-δ语言的直观理解"
  
  直观理解:
    - "无限接近的过程"
    - "稳定趋势"
  
  基本例子:
    - "lim(x→0) sin(x)/x = 1"
    - "lim(n→∞) 1/n = 0"
  
  基本性质:
    - "唯一性"
    - "保号性"
    - "夹逼定理"
```

```yaml
概念: 连续性 (Continuity)
  定义要点:
    - "函数在某点的极限等于函数值"
    - "图像没有断开"
  
  直观理解:
    - "一笔画成，不抬笔"
    - "没有突变"
  
  基本例子:
    - "多项式函数连续"
    - "sin(x), cos(x), e^x 连续"
  
  基本性质:
    - "连续函数的和、差、积、商(分母非零)仍连续"
    - "复合函数连续"
```

#### 几何与拓扑 (Geometry & Topology)

```yaml
概念: 拓扑空间 (Topological Space)
  定义要点:
    - "一个集合配上开集族"
    - "开集满足特定公理"
  
  直观理解:
    - "研究'接近'概念的抽象框架"
    - "橡皮几何学"
  
  基本例子:
    - "实数集R上的标准拓扑"
    - "离散拓扑"
    - "平凡拓扑"
  
  基本性质:
    - "开集的任意并仍是开集"
    - "有限个开集的交仍是开集"
```

```yaml
概念: 连续映射 (Continuous Map)
  定义要点:
    - "开集的原像是开集"
    - "拓扑空间之间的结构保持映射"
  
  直观理解:
    - "不撕裂的变形"
    - "邻近点映射到邻近点"
  
  基本例子:
    - "常值映射"
    - "恒等映射"
    - "包含映射"
```

### 2.4 L0层次诊断要点

**诊断目标**: 评估学生是否具备学习正式数学内容的基础认知

**诊断方式**:
1. **概念识别测试**: 给出定义，识别概念类型
2. **例子判断**: 判断给定对象是否属于某概念
3. **基本计算**: 进行简单的代数运算
4. **直观解释**: 用自己的语言解释概念含义

## 3. L1: 定义理解层

### 3.1 层次定义

**目标**: 掌握严格的形式化定义和基本定理

**特征**:
- 形式化定义，使用严格的数学语言
- 理解定理的陈述和条件
- 掌握基本证明技巧
- 能够应用标准方法解决问题

### 3.2 能力要求

| 能力维度 | 描述 | 示例 |
|----------|------|------|
| 形式化定义 | 理解并使用严格定义 | 使用ε-δ语言定义极限 |
| 定理理解 | 理解定理的条件和结论 | 理解Lagrange中值定理 |
| 基本证明 | 完成简单证明 | 证明群的基本性质 |
| 标准应用 | 应用标准方法解题 | 使用导数求极值 |

#### 代数学

## 6. 层次间依赖关系

### 6.1 依赖图

```
L3: 综合证明层
├─ 依赖 L2: 深入定理理解
├─ 依赖 L1: 形式化推理能力
└─ 依赖 L0: 概念基础

L2: 定理应用层
├─ 依赖 L1: 标准方法掌握
└─ 依赖 L0: 概念理解

L1: 定义理解层
└─ 依赖 L0: 直观认知

└─ 基础层(无依赖)
```

### 6.2 约束规则

```yaml
约束规则:
  
  先修约束:
    规则: "学习高层次内容必须先掌握低层次内容"
    强度: 1.0  # 强约束
    示例: "L3题目需要L0-L2的知识支撑"
  
  能力约束:
    规则: "高层次能力评估受低层次能力影响"
    强度: 0.8
    公式: "ability_Ln ≤ ability_L(n-1) + (1 - strength)"
    
  诊断约束:
    规则: "诊断结果必须符合层次依赖关系"
    强度: 0.9
    应用: "若L0诊断结果差，则L1-L3能力上限受限"
```

## 7. 与CPFS结构的对齐

### 7.1 CPFS理论回顾

CPFS (Concept-Proposition-Formula-System) 结构理论将数学认知分为：
- **概念域 (CF)**: Concept Field
- **概念系 (CS)**: Concept System  
- **命题域 (PF)**: Proposition Field
- **命题系 (PS)**: Proposition System

### 7.2 与L0-L3的映射

| CPFS组件 | 主要对应层次 | 诊断任务类型 |
|----------|--------------|--------------|
| 概念域 CF | L0-L1 | 概念定义任务、概念联想任务 |
| 概念系 CS | L1-L2 | 概念关系分析、概念网络构建 |
| 命题域 PF | L1-L2 | 命题表述任务、定理应用任务 |
| 命题系 PS | L2-L3 | 证明构造任务、综合推理任务 |

### 7.3 诊断任务设计

```yaml
基于CPFS的诊断任务:
  
  概念定义任务 (L0-L1):
    描述: "给出概念的准确定义"
    示例: "请给出'群'的形式化定义"
    评估维度: ["完整性", "准确性", "形式化程度"]
  
  概念联想任务 (L0-L2):
    描述: "给定概念，联想相关概念和例子"
    示例: "提到'连续函数'，你能想到哪些相关概念？"
    评估维度: ["联想广度", "联想深度", "例子适当性"]
  
  命题表述任务 (L1-L2):
    描述: "准确表述定理的条件和结论"
    示例: "请完整表述Lagrange中值定理"
    评估维度: ["条件完整性", "结论准确性", "条件必要性理解"]
  
  证明构造任务 (L2-L3):
    描述: "构造完整的数学证明"
    示例: "证明: 有限群的任一元素阶数整除群的阶"
    评估维度: ["逻辑严密性", "方法适当性", "论证完整性"]
```

### 8.1 诊断系统使用流程

```
1. 学生注册与初始评估
   └─ 进行L0水平测试，确定入学水平

2. 分层诊断
   ├─ L0测试: 基础概念识别
   ├─ L1测试: 定义理解与基本应用
   ├─ L2测试: 定理应用与综合问题
   └─ L3测试: 开放问题与研究潜力(可选)

3. 结果分析
   ├─ 识别薄弱环节
   ├─ 确定能力水平
   └─ 生成学习建议

4. 学习路径推荐
   ├─ 根据诊断结果推荐学习内容
   ├─ 规划学习顺序
   └─ 设定阶段性目标

5. 周期性复测
   ├─ 跟踪学习进展
   ├─ 调整学习路径
   └─ 验证学习效果
```

### 8.2 与内容系统的对接

```yaml
对接说明:
  
  内容标注:
    - 每篇内容文档标注对应知识层次(L0-L3)
    - 标注关键概念和先修知识
    - 提供自测要点
  
  诊断对接:
    - 诊断题目与内容知识点一一对应
    - 诊断结果直接关联推荐内容
    - 学习路径基于内容依赖关系生成
  
  反馈循环:
    - 学习内容后生成诊断报告
    - 根据学习行为调整诊断参数
    - 诊断结果指导后续内容推荐
```
