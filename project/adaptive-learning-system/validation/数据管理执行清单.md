# FormalMath 课堂验证数据管理执行清单

> 本清单基于 `project/课堂验证伦理审查与数据管理协议.md` 制定，是数据收集、存储、处理、销毁的实操指南。

## 一、数据收集流程图

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                        数据收集全流程（Data Collection Pipeline）           │
└─────────────────────────────────────────────────────────────────────────────┘

    ┌──────────────┐
    │   学生报名   │ ──→ 筛选问卷（姓名、学号、联系方式）
    └──────┬───────┘        ↓
           │           ┌──────────────────┐
           │           │  立即匿名化编码   │  姓名/学号 → 随机编号（VAL-2026-Q2-XXX）
           │           │  (ID Masking)    │  原始信息存入加密 lookup table
           │           └────────┬─────────┘
           │                    ↓
           │           ┌──────────────────┐
           │           │   分组脚本执行    │  A/B 随机分组 → 实验组/对照组标签
           │           └────────┬─────────┘
           │                    ↓
    ┌──────▼───────┐   ┌──────────────────┐   ┌──────────────────┐

    │   测试       │   │   (每周填写)     │   │   综合测试       │
    └──────┬───────┘   └────────┬─────────┘   └────────┬─────────┘
           │                    │                      │
           │    ┌───────────────┼──────────────────────┘
           │    │               ↓
           │    │      ┌──────────────────┐
           │    │      │  数据录入/扫描    │  纸质 → 电子（双人录入校验）
           │    │      │  (Data Entry)    │  在线 → 自动入库
           │    │      └────────┬─────────┘
           │    │               ↓
           │    │      ┌──────────────────┐
           │    │      │   质量检查节点    │  逻辑校验 + 异常值标记 + 缺失值处理
           │    │      │  (Quality Gate)  │  通过 → 进入分析库；不通过 → 复核
           │    │      └────────┬─────────┘
           │    │               ↓
           │    │      ┌──────────────────┐
           │    └────→│   匿名化分析库    │  仅含随机编号，无个人身份信息
           │           │  (Analysis DB)   │  用于统计分析和报告撰写
           │           └────────┬─────────┘
           │                    ↓
           │           ┌──────────────────┐
           │           │   焦点小组访谈    │  录音 → 转录 → 去标识化 → 销毁录音

           │
           ↓
    ┌────────────────────────────────────────────────────────────────────┐
    │                     退出处理分支（Withdrawal Branch）              │
    │  学生申请退出 → 立即停止数据收集 → 标记已有数据为"退出者"         │
    │  → 学生可选择：①保留匿名化数据 ②请求删除（30天内执行）          │
    └────────────────────────────────────────────────────────────────────┘
```

## 二、数据分类与收集规范

### 2.1 数据分类表

| 数据类别 | 敏感度 | 收集方式 | 匿名化时点 | 存储位置 |
|----------|--------|----------|------------|----------|
| 身份信息（姓名、学号、联系方式） | 🔴 高 | 筛选问卷、报名表 | **立即**（录入时即编码） | 加密 lookup table（与主数据物理隔离） |
| 测试分数（基线、章节、期末） | 🟡 中 | 纸质测试/在线系统 | 收集时直接使用编号 | 匿名化分析库 |
| 学习日志（周评分、开放题） | 🟡 中 | 在线问卷/纸质 | 收集时直接使用编号 | 匿名化分析库 |
| 问卷数据（满意度、自评） | 🟡 中 | 在线问卷 | 自动匿名 | 匿名化分析库 |
| 访谈录音 | 🔴 高 | 录音设备 | 转录后销毁 | 临时加密存储（30天） |
| 访谈转录文稿 | 🟡 中 | 人工转录+审核 | 转录时去除识别信息 | 匿名化分析库 |
| Lean4 编译日志 | 🟢 低 | 在线 Lean 环境 | 自动（仅记录尝试次数） | 匿名化分析库 |
| 平台行为日志（阅读时长等） | 🟢 低 | 学习平台 | 自动（脱敏后） | 匿名化分析库 |

### 2.2 收集操作 checklist

**每次测试前：**

- [ ] 准备足够数量的编号试卷（已预先打印随机编号，无姓名栏）
- [ ] 核对到场学生与编号映射表（仅协调教师持有）
- [ ] 宣读测试纪律：独立作答、禁用计算工具（除非题目要求）
- [ ] 收集试卷时按编号顺序整理，双人清点

**每周日志收集：**

- [ ] 周一上午发送日志填写提醒（邮件/微信群）
- [ ] 周三下午发送未填写提醒
- [ ] 周五截止，导出数据并备份
- [ ] 检查缺失率，若单周缺失率 > 20% 触发跟进流程

**访谈录音处理：**

- [ ] 访谈前告知录音目的，获得口头同意
- [ ] 使用专用录音设备，禁止个人手机录音
- [ ] 访谈结束后 24 小时内完成转录
- [ ] 转录完成后 7 天内完成去标识化审核（去除人名、地名、具体事件）
- [ ] 审核通过后 30 天内销毁原始录音文件（多次覆盖写入后删除）

## 三、匿名化与去标识化规范

### 3.1 编号规则

```
格式：[验证批次]-[课程代码]-[三位顺序号]

示例：
  VAL-2026-Q2-018-06-E    ← 2026年Q2验证，MIT 18.06，第18号，实验组
  VAL-2026-Q2-031-10A-C   ← 2026年Q2验证，MIT 18.100A，第31号，对照组

编码说明：
  - 06 = MIT 18.06 线性代数
  - 10A = MIT 18.100A 实分析
  - E = Experimental（实验组）
  - C = Control（对照组）
```

### 3.2 Lookup Table 管理

| 管理项 | 规范 |
|--------|------|
| 创建者 | 数据管理员（1人） |
| 存储位置 | 离线加密 USB + 项目组加密服务器双备份 |
| 访问权限 | 仅数据管理员 + 项目负责人（2人） |
| 加密方式 | AES-256，密码由项目负责人单独保管 |
| 使用场景 | 仅在需要联系学生（如发送提醒）时使用 |
| 销毁时间 | 研究结束后 6 个月内 |

### 3.3 开放题去标识化审核清单

学生问卷和访谈中的开放题回答，在纳入分析前必须经过以下审核：

- [ ] 删除真实人名（替换为「学生A」「受访者B」）
- [ ] 删除学校名称（替换为「合作院校」）
- [ ] 删除具体地名（如「在XX图书馆学习」→「在校内图书馆学习」）
- [ ] 删除可识别的具体事件（如「上周三李老师课上讲的」→「某次课上讲的」）
- [ ] 删除学号、手机号、邮箱等数字串
- [ ] 审核人签字确认：______________

## 四、数据存储规范

### 4.1 存储架构

```
加密服务器（FormalMath-DataVault）
│
├── raw-data/                        ← 原始数据（访问受限）
│   ├── lookup-table/
│   │   └── VAL-2026-Q2-lookup.enc   ← 编号-身份映射（AES-256）
│   ├── paper-scans/
│   │   └── [按日期归档扫描件]
│   └── audio-recordings/            ← 访谈录音（30天自动清理）
│       └── [临时存储，转录后删除]
│
├── anonymized-db/                   ← 匿名化分析库（核心研究团队可访问）
│   ├── mit-1806/
│   │   ├── baseline-tests.csv
│   │   ├── weekly-logs.csv
│   │   ├── chapter-quizzes.csv
│   │   ├── final-test.csv
│   │   └── survey-responses.csv
│   ├── mit-18100a/
│   │   └── [同上结构]
│   └── interviews/
│       ├── transcripts-deidentified/
│       └── coding-notes/
│
└── backups/                         ← 自动备份
    ├── daily/                       ← 每日增量备份
    ├── weekly/                      ← 每周全量备份
    └── offsite/                     ← 异地备份（加密云存储）
```

### 4.2 加密与访问控制

| 层级 | 加密方式 | 访问角色 | 访问方式 |
|------|----------|----------|----------|
| 原始身份数据 | AES-256 + 离线存储 | 数据管理员、项目负责人 | 物理隔离设备 |
| 匿名化分析库 | TLS 传输 + 数据库加密 | 核心研究团队（3人） | VPN + 双因素认证 |
| 备份数据 | 客户端加密后上传 | 系统管理员（运维） | 自动化脚本，无人为访问 |

### 4.3 备份策略

| 备份类型 | 频率 | 保留期限 | 存储位置 |
|----------|------|----------|----------|
| 增量备份 | 每日 02:00 | 30 天 | 本地加密服务器 |
| 全量备份 | 每周日 03:00 | 12 周 | 本地加密服务器 |
| 异地备份 | 每周日 04:00 | 3 年 | 加密云存储（不同地理区域） |
| 灾难恢复演练 | 每季度一次 | — | 验证备份可恢复性 |

## 五、数据处理规范

### 5.1 数据使用许可

匿名化分析库中的数据仅可用于以下目的：

1. ✅ 计算学习成效指标（前后测提升、Cohen's d 等）
2. ✅ 撰写课堂验证报告
3. ✅ 发表数学教育研究论文（仅汇报聚合统计结果）
4. ✅ 改进 FormalMath 文档质量

禁止行为：

- ❌ 将原始数据分享给第三方机构
- ❌ 在报告/论文中展示可识别个体的原始回答
- ❌ 将数据用于商业目的
- ❌ 超范围使用（如用于非教育研究项目）

### 5.2 统计分析规范

| 规则 | 说明 |
|------|------|
| 最小报告单元 | 任何报告中，分组样本量 ≥ 5 人才可汇报该组数据 |
| 引述规范 | 引用学生开放题回答时，必须使用「受访者 E-015」等匿名编号 |
| 聚合优先 | 优先汇报均值、标准差、百分比等聚合指标 |
| 效应量报告 | 所有组间比较必须报告效应量（Cohen's d 或 η²） |
| 原始数据不公开 | 原始匿名化数据集不随论文公开；可向期刊提供审核通道 |

### 5.3 数据质量检查节点

**节点 1：录入后（每周五）**

- [ ] 检查异常值（如测试满分 100 却出现 150 分）
- [ ] 检查缺失模式（如某学生连续 3 周未填日志）
- [ ] 双人交叉验证纸质数据录入准确性

**节点 2：分析前（期末测试后）**

- [ ] 检查基线-期末数据匹配完整性
- [ ] 标记退出者数据，确认是否从分析中排除
- [ ] 检验各组基线均衡性（独立样本 t 检验）

**节点 3：报告撰写前**

- [ ] 复核所有引用的学生原话已去标识化
- [ ] 确认所有表格/图表中的样本量 ≥ 5
- [ ] 项目负责人审核数据使用合规性

## 六、数据销毁时间表

### 6.1 销毁计划总览

| 数据类别 | 销毁触发条件 | 销毁方式 | 负责角色 | 监督角色 |
|----------|-------------|----------|----------|----------|
| 访谈原始录音 | 转录+去标识化审核通过后 30 天 | 多次覆盖写入（≥7次）后物理删除 | 数据管理员 | 项目负责人 |
| 身份 lookup table | 研究结束后 6 个月 | 加密设备格式化 + 物理销毁（可选） | 数据管理员 | 伦理审查委员会 |
| 纸质原始问卷 | 研究结束后 12 个月 | 碎纸机粉碎 | 协调教师 | 数据管理员 |
| 匿名化分析库 | 研究结束后 3 年（2029-12-31） | 安全擦除 + 存储介质销毁 | 数据管理员 | 项目负责人 |
| 异地备份 | 研究结束后 3 年 + 30 天 | 云存储端永久删除 + 删除确认回执 | 系统管理员 | 数据管理员 |

### 6.2 销毁执行 checklist

**每次销毁操作必须完成：**

- [ ] 填写《数据销毁登记表》，记录销毁时间、数据类别、预计条目数
- [ ] 执行销毁操作（软件删除/物理粉碎）
- [ ] 验证销毁结果（尝试恢复，确认不可恢复）
- [ ] 销毁执行人签字：______________
- [ ] 监督人签字：______________
- [ ] 将登记表扫描件存档

### 6.3 提前退出学生的数据处理

当学生在研究期间申请退出时：

| 学生选择 | 执行动作 | 时限 |
|----------|----------|------|
| "保留我的匿名化数据" | 标记该编号为"退出者"，已有数据保留在库中但不纳入后续分析 | 立即 |
| "删除我的所有数据" | 从分析库中删除该编号的所有记录；lookup table 中的映射关系删除；纸质问卷抽出销毁 | 30 天内 |

## 七、应急响应预案

### 7.1 数据泄露应急响应

若发生数据泄露事件（如服务器被入侵、设备丢失）：

1. **发现后 1 小时内**：
   - [ ] 断开受影响系统网络连接
   - [ ] 通知项目负责人和数据管理员
   - [ ] 启动事件日志保全

2. **发现后 24 小时内**：
   - [ ] 评估泄露范围和影响程度
   - [ ] 通知伦理审查委员会
   - [ ] 若涉及个人身份信息泄露，通知受影响学生

3. **发现后 72 小时内**：
   - [ ] 提交初步事件报告
   - [ ] 实施补救措施（更换凭证、强化加密等）
   - [ ] 启动外部安全审计

### 7.2 数据丢失应急响应

若发生数据丢失（如硬盘故障、误删除）：

1. **立即执行**：
   - [ ] 停止对受影响存储介质的任何写入操作
   - [ ] 从最近一次备份执行恢复

2. **恢复后**：
   - [ ] 验证恢复数据完整性
   - [ ] 分析丢失原因，修复系统漏洞
   - [ ] 更新备份策略（如增加备份频率）
