# 项目进度报告归档说明

**创建日期**: 2025年12月1日

---

## 📋 归档规则

### 1. 文件分类

项目根目录下的进度报告文件按以下规则分类：

- **文档深化报告** → `2025年12月/文档深化/`
  - 所有以 `00-文档深化` 开头的文件
  - 所有以 `00-文档内容深化` 开头的文件

- **阶段推进报告** → `2025年12月/阶段推进/`
  - 所有以 `00-阶段` 开头的文件

- **全面完成报告** → `2025年12月/全面完成/`
  - 所有以 `00-全面` 开头的文件
  - 所有以 `00-项目` 开头的文件
  - 所有以 `00-全部` 开头的文件
  - 所有以 `00-所有` 开头的文件

### 2. 归档时间

- 每月归档：将当月的进度报告归档到对应的月份文件夹
- 保留最新：最新的总览文件保留在根目录

### 3. 文件命名

归档后的文件保持原名，便于追溯历史。

---

## 📁 目录结构

```
project/00-项目进度报告/
├── README.md                          # 本说明文件
├── 项目状态总览.md                    # 最新项目状态
├── 归档说明.md                        # 本归档说明
└── 2025年12月/                       # 2025年12月的报告
    ├── 文档深化/                     # 文档深化相关报告
    ├── 阶段推进/                     # 各阶段推进报告
    └── 全面完成/                     # 全面完成相关报告
```

---

## 🔄 维护方式

1. **定期归档**：每月整理一次进度报告
2. **更新总览**：更新 `项目状态总览.md` 反映最新状态
3. **清理根目录**：将归档后的文件从根目录移除（可选）
