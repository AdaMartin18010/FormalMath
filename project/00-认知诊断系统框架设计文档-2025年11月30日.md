## 📋 文档概述

本文档设计FormalMath项目的认知诊断系统框架，基于层次约束感知的认知诊断框架（HCD），旨在准确评估学生在不同知识层次的能力表现，为个性化学习提供数据支持。

**设计目标**:

- 准确评估学生在不同知识层次的能力
- 识别学生的学习优势和不足
- 生成个性化的学习建议
- 支持自适应学习系统

**参考标准**:

- 层次约束感知的认知诊断框架（HCD）([arxiv.org](https://arxiv.org/abs/2412.04488))
- 符号化认知诊断方法（符号树和混合优化技术）

**与2026方案对齐**（`00-全面分析与改进方案-2026年2月2日.md`）:

- **L0–L3 知识层次**：与改进方案第五章 5.1.1 一致——L0（基础层 1000–2000 字）、L1（中级层 2000–4000 字）、L2（高级层 4000–6000 字）、L3（研究层 6000+ 字），用于层次约束与认知负荷管理。
- **CPFS 整合**：诊断维度可对接 CPFS 结构理论（概念域 CF、概念系 CS、命题域 PF、命题系 PS），见 `research/01-数学认知/认知过程/01-CPFS结构理论.md`；概念定义任务、概念联想任务、命题表述任务、证明构造任务可用于题目设计与能力评估。
- **HCD 框架**：本设计采用层次约束感知认知诊断（HCD），与 2026 方案 5.1.1 节目标一致，支持知识状态估计与个性化学习路径推荐。

**与内容侧正式对接**（2026年2月）：自测要点、认知负荷标记（L1–L3）、推荐阅读顺序与交错路径已在本项目 02-数学内容深度分析 中实施；诊断题开发与学习建议生成可据此对接。详见 [00-认知诊断与内容对接说明-2026年2月](00-认知诊断与内容对接说明-2026年2月.md)。

- 能够识别简单例子

#### L1: 中级层（严格定义、基本定理、证明思路）

**特点**:

**评估标准**:

- 能够理解形式化定义
- 能够掌握基本定理
- 能够理解证明思路

#### L2: 高级层（深入定理、应用、前沿内容）

**特点**:

- 深入定理
- 复杂应用
- 交叉应用
- 前沿内容

**示例概念**:

- 基数（无限基数）
- 连续统假设

**评估标准**:

- 能够掌握深入定理
- 能够进行复杂应用
- 能够理解前沿内容

#### L3: 研究层（开放问题、研究方向）

**特点**:

- 开放问题
- 研究方向
- 前沿研究
- 未解决问题

**示例概念**:

- 大基数公理
- 力迫法
- 内模型理论

**评估标准**:

- 能够理解开放问题
- 能够探索研究方向
- 能够进行前沿研究

### 2.2 层次约束关系

**约束规则**:

1. **先修约束**: L1需要先掌握L0，L2需要先掌握L1，L3需要先掌握L2
2. **能力约束**: 高级层次的能力评估需要考虑低层次的能力
3. **学习约束**: 学习高级层次内容需要先掌握低层次内容

**约束表示**:

```python
class HierarchyConstraint:
    lower_level: int
    upper_level: int
    constraint_type: str  # prerequisite, ability, learning
    strength: float  # 约束强度
```

## 🔬 四、认知诊断算法设计 (编号: DESIGN.001.04)

### 4.1 HCD算法框架

#### 算法原理

**层次约束感知的认知诊断框架（HCD）**:

- 考虑知识层次间的约束关系
- 利用层次约束提高诊断准确性
- 支持多层次能力评估

#### 算法流程

```
1. 输入: 学生答题响应、测试题信息
2. 初始化: 知识状态、能力水平
3. 迭代优化:
   a. 估计知识状态
   b. 评估能力水平
   c. 应用层次约束
   d. 更新参数
4. 输出: 知识状态、能力水平、学习建议
```

#### 算法实现框架

```python
class HCDAlgorithm:
    def __init__(self, knowledge_hierarchy, constraints):
        self.hierarchy = knowledge_hierarchy
        self.constraints = constraints

    def estimate_knowledge_state(self, responses, test):
        """估计知识状态"""
        # 1. 初始化知识状态
        knowledge_state = self.initialize_knowledge_state()

        # 2. 迭代优化
        for iteration in range(max_iterations):
            # 2.1 估计知识状态
            knowledge_state = self.update_knowledge_state(
                responses, test, knowledge_state
            )

            # 2.2 应用层次约束
            knowledge_state = self.apply_hierarchy_constraints(
                knowledge_state, self.constraints
            )

            # 2.3 检查收敛
            if self.check_convergence(knowledge_state):
                break

        return knowledge_state

    def assess_ability_level(self, knowledge_state):
        """评估能力水平"""
        ability_level = {}
        for level in [0, 1, 2, 3]:
            ability_level[level] = self.calculate_level_ability(
                knowledge_state, level
            )
        return ability_level

    def generate_learning_suggestions(self, knowledge_state, ability_level):
        """生成学习建议"""
        suggestions = []

        # 1. 识别薄弱环节
        weak_areas = self.identify_weak_areas(knowledge_state)

        # 2. 生成学习建议
        for area in weak_areas:
            suggestion = self.create_suggestion(area, ability_level)
            suggestions.append(suggestion)

        return suggestions
```

### 4.2 知识状态估计

#### 估计方法

**方法1: 基于IRT模型**

- 使用项目反应理论（IRT）模型
- 估计学生的能力参数
- 估计知识点的掌握概率

**方法2: 基于DINA模型**

- 使用确定性输入噪声与门（DINA）模型
- 估计学生的知识状态
- 考虑猜测和失误参数

**方法3: 基于HCD框架**

- 使用层次约束感知的方法
- 考虑层次间的约束关系
- 提高估计准确性

#### 估计流程

```
1. 收集学生答题响应
2. 分析答题模式
3. 估计知识状态概率
4. 应用层次约束
5. 输出知识状态
```

### 4.3 能力水平评估

#### 评估维度

**维度1: 知识掌握度**

- 每个知识点的掌握程度
- 每个层次的掌握程度
- 整体知识掌握度

**维度2: 问题解决能力**

- 基础问题解决能力
- 中等问题解决能力
- 复杂问题解决能力

**维度3: 学习能力**

- 概念理解能力
- 方法应用能力
- 知识迁移能力

#### 评估方法

```python
def calculate_level_ability(knowledge_state, level):
    """计算层次能力"""
    # 1. 获取该层次的知识点
    level_concepts = get_level_concepts(level)

    # 2. 计算平均掌握度
    mastery_scores = [
        knowledge_state[concept]
        for concept in level_concepts
    ]
    average_mastery = sum(mastery_scores) / len(mastery_scores)

    # 3. 考虑层次约束
    if level > 0:
        lower_level_ability = calculate_level_ability(
            knowledge_state, level - 1
        )
        average_mastery = apply_constraint(
            average_mastery, lower_level_ability
        )

    return average_mastery
```
