# FormalMath 质量审计评分标准与盲审框架

> **文档编号**：D001  
> **版本**：v1.0  
> **适用范围**：FormalMath 项目全量文档（6573+ 篇）质量审计  
> **目标**：打破"规模幻觉"，建立以知识密度与教学价值为核心的质量评估体系。

---

## 1. 引言与背景

FormalMath 项目经过多年扩张，已形成庞大的文档矩阵。然而，内部审查表明存在典型的**规模幻觉（Scale Illusion）**：文档数量与知识密度呈反比，大量条目为模板化填充产物——定义空泛、证明残缺、习题雷同、引用缺失、教学逻辑混乱。

本框架旨在：
1. **建立可量化的质量底线**：任何文档必须通过五维评分检验；
2. **引入盲审机制消除偏见**：评分者不知文档作者与来源，只看内容；
3. **提供清晰的处置指令**：根据总分给出保留、修订、重写、删除的明确建议。

---

## 2. 五大评分维度与精确判定标准

每个维度满分 5 分，最低 1 分。总分范围为 5–25 分。评分时必须对照下文的**逐分判定标准**，禁止随意取中间值。

### 2.1 定义严格性（Rigor of Definitions, RD）

衡量数学概念的表述是否精确、无歧义、符号统一，并区分了直观描述与形式化定义。

| 分数 | 判定标准 |
|------|----------|
| **1 分** | 仅有口语化描述，无数学定义；或核心术语使用日常词汇替代，存在明显歧义。 |
| **2 分** | 给出了定义尝试，但缺少前提条件（如定义域、量词、空间假设），符号不统一或混用。 |
| **3 分** | 定义基本完整，符号规范，但存在小瑕疵：如未说明特例（零向量、空集）、或未区分等价表述的细微差别。 |
| **4 分** | 定义严格、完整、符号统一，包含必要的前提条件与反例提示，区分了直观理解与形式定义。 |
| **5 分** | 在 4 分基础上，进一步指出该定义在文献中的等价变体、历史演变，或给出了该定义最一般化的上下文（如从度量空间到拓扑空间的推广）。 |

### 2.2 证明完整性（Completeness of Proofs, CP）

衡量定理/命题的证明是否充分、逻辑链是否闭合、关键步骤是否有交代。

| 分数 | 判定标准 |
|------|----------|
| **1 分** | 仅有定理陈述，无任何证明；或仅有"显然"/"易证"等空话。 |
| **2 分** | 给出了证明骨架，但省略了关键步骤（如收敛性的 ε-N 构造、存在性的具体选取），逻辑跳跃明显。 |
| **3 分** | 证明基本完整，但存在非关键步骤的省略；或引用了未在文档内给出的引理而未标注来源。 |
| **4 分** | 证明完整、逻辑闭合，每一步都有依据；对复杂证明进行了分步拆解，并标注了引用的内部/外部来源。 |
| **5 分** | 在 4 分基础上，进一步给出了证明思路的直观解释、替代证法（如初等证法与抽象证法对比）、或指出了证明中隐藏的非平凡假设。 |

### 2.3 习题丰富度（Richness of Exercises, RE）

衡量配套习题的层次性、多样性与教育价值，而非单纯数量。

| 分数 | 判定标准 |
|------|----------|
| **1 分** | 无习题；或习题与主题无关，纯粹凑数。 |
| **2 分** | 有 1–2 道直接套用定义的机械题，无解答或提示，缺乏层次。 |
| **3 分** | 习题覆盖基础计算与简单证明（至少 3 道），但题型单一，难度梯度不明显，部分仅有答案无过程。 |
| **4 分** | 习题层次分明（基础→进阶→探索），题型多样（计算、证明、反例构造、概念辨析），配备详细解答或思路提示。 |
| **5 分** | 在 4 分基础上，包含至少一道开放性/研究性问题，引导学生连接更高级主题；或提供历史名题及其变体，激发深度思考。 |

### 2.4 引用规范性（Quality of References, QR）

衡量文献引用、交叉链接、外部来源标注的规范性与可追溯性。

| 分数 | 判定标准 |
|------|----------|
| **1 分** | 无任何引用；或声称引用某经典教材但无页码/章节信息，无法追溯。 |
| **2 分** | 引用了少量来源，但格式混乱（如混用 APA、GB/T 7714、随意 URL），缺少内部交叉链接。 |
| **3 分** | 引用格式基本一致，关键定理有外部来源标注，内部概念有交叉链接，但存在断链或过时链接。 |
| **4 分** | 引用规范统一，内部链接完整且可点击跳转，外部引用具体到章节/页码/定理编号，优先使用经典教材与同行评审文献。 |
| **5 分** | 在 4 分基础上，引用覆盖了定义的历史文献、证明的原始论文、以及该主题在 FormalMath 其他模块中的关联位置，形成了知识网络。 |

### 2.5 教学有效性（Pedagogical Effectiveness, PE）

衡量文档作为教学材料的可读性、结构清晰度、动机铺垫与学习者支持。

| 分数 | 判定标准 |
|------|----------|
| **1 分** | 内容像术语堆砌或机器翻译，缺乏任何教学结构；读者无法理解作者想传达什么。 |
| **2 分** | 有基本结构（定义→定理→例子），但缺少动机铺垫，例子生硬，符号 unexplained，阅读体验差。 |
| **3 分** | 结构清晰，有简单例子辅助，但缺少"为什么需要这个定义/定理"的说明，对初学者仍有明显门槛。 |
| **4 分** | 教学结构优良：动机明确、例子典型、难度递进合理、有常见错误警示与学习建议；不同背景的读者都能找到入口。 |
| **5 分** | 在 4 分基础上，采用了多模态或多层次呈现（如直观图→形式化→计算实例→高阶视角），能引导读者自主发现；具备出版级教材水准。 |

## 3. 评分细则总表

| 维度 | 1 分 | 2 分 | 3 分 | 4 分 | 5 分 |
|------|------|------|------|------|------|
| **RD 定义严格性** | 无形式定义，口语化 | 定义尝试但缺前提/符号乱 | 基本完整，小瑕疵 | 严格、完整、统一 | 等价变体、历史演变、最一般化 |
| **CP 证明完整性** | 无证明或"显然" | 骨架有，关键步骤省略 | 基本完整，小省略 | 完整闭合，分步拆解 | 多证法、思路阐释、隐藏假设 |
| **RE 习题丰富度** | 无习题或无关 | 1–2 道机械题 | 3+ 道但题型单一 | 层次分明，题型多样 | 开放性问题，连接高级主题 |
| **QR 引用规范性** | 无引用或不可追溯 | 格式混乱，少内部链接 | 基本规范，偶有断链 | 格式统一，链接完整可追溯 | 历史文献+原始论文+知识网络 |
| **PE 教学有效性** | 术语堆砌，无结构 | 有结构但缺动机铺垫 | 清晰但缺"为什么" | 动机明确，有错误警示 | 多层次呈现，出版级教材水准 |

## 4. 盲审操作流程

### 4.1 随机抽样规则

为避免选择性审计带来的偏差，抽样必须遵循以下规则：

1. **分层随机抽样**：
   - 将 6573+ 篇文档按主题域分为 10±2 个层（如：分析、代数、几何、拓扑、数论、概率、逻辑、计算数学、离散数学、应用数学）；
   - 每层按文档编号随机抽取 `n = max(层内文档数 × 2%, 15)` 篇，确保小层也有足够样本。

2. **加抽高风险的"边缘文档"**：
   - 从最近 6 个月内批量导入的文档中，额外随机抽取 5% 作为"新生文档层"；
   - 从被标记为"Stub"/"草稿"/"待完善"的文档中，抽取 10% 作为"低质量嫌疑层"。

3. **抽样不可由评分者执行**：
   - 抽样由项目管理员或脚本完成，生成加密样本池；评分者只接收样本，无法获知总体分布。

### 4.2 来源隐藏机制（Blinding）

盲审的核心是消除作者身份、文档历史与元数据对评分的干扰：

| 隐藏项 | 操作方式 |
|--------|----------|
| **作者信息** | 移除文档头部的作者、贡献者、编辑历史；若文档内有签名式语句，一并脱敏。 |
| **文件路径** | 将文档复制到临时目录，以随机编号重命名（如 `AUDIT-0427.md`），不暴露原目录结构。 |
| **编辑时间戳** | 不显示创建时间、最后修改时间；版本号用内部审计版本替代。 |
| **评论与批注** | 若文档包含内嵌评论（如 `<!-- TODO: ... -->`），评分阶段暂不展示，避免预设偏见。 |
| **引用计数/点赞数** | 任何平台元数据（阅读量、点赞、转发）均不提供给评分者。 |

**例外**：若文档内明确引用了自己的其他 FormalMath 条目（内部链接），链接文本可保留，但目标文档标题中的作者信息需脱敏。

### 4.3 评分表格式

每篇文档由至少 2 名评分者独立评分。评分表如下：

```markdown
## 盲审评分表

- **审计批次**：2026-Q2-Audit
- **样本编号**：AUDIT-0427
- **评分者 ID**：R-03
- **评分日期**：YYYY-MM-DD

### 维度评分（1–5 分，禁止小数）

| 维度 | 得分 | 评分依据（必须填写，限 50–200 字） |
|------|------|-------------------------------------|
| RD 定义严格性 |    |                                     |
| CP 证明完整性 |    |                                     |
| RE 习题丰富度 |    |                                     |
| QR 引用规范性 |    |                                     |
| PE 教学有效性 |    |                                     |

### 总分与初步建议

- **总分**：__/25
- **初步建议**：□ 保留标杆  □ 局部修订  □ 大幅重写  □ 归档删除

### 关键问题摘要（选填）
1. 
2. 
```

### 4.4 多人评分差异的处理机制

1. **差异阈值**：
   - 若两名评分者的**总分差异 ≤ 3 分**，视为"一致通过"，取平均分（若总分差为奇数，向上取整）。
   - 若**总分差异 ≥ 4 分**，或**任意单一维度差异 ≥ 2 分**，触发"仲裁流程"。

2. **仲裁流程**：
   - 引入第三名评分者（资深仲裁员，不参与初审）独立评分；
   - 仲裁员可查看前两名评分者的评分依据（但不可查看评分者身份）；
   - 最终得分取三名评分者的中位数（Median）；若中位数不唯一，取最接近仲裁员评分值的整数。

3. **分歧记录与分析**：
   - 所有分歧案例进入"分歧数据库"，用于季度性的评分标准校准会议；
   - 若某维度连续出现高分歧率（>20% 的文档触发仲裁），说明该维度判定标准模糊，需在下个版本中修订。

## 5. 评分示例

以下示例基于典型数学教育文档的抽象特征编写，供评分者校准使用。

### 5.1 优秀文档示例：《度量空间的完备化》

**文档特征概述**：
- 从有理数序列不收敛的直观问题出发，引入柯西序列与完备化的动机；
- 严格定义了等价关系 `~` 与商空间上的度量 `d̃`，并证明了良定义性；
- 给出了标准嵌入映射 `ι: X → X̃` 的等距性证明，及唯一性定理（泛性质）的完整证明；
- 包含 5 道习题：计算实数作为柯西序列等价类的表示（基础）、证明 `ℚ` 的完备化与 `ℝ` 等距同构（进阶）、讨论不完备赋范空间的完备化是否保持范数（探索）；
- 引用了 Munkres《Topology》第 43 节、Conway《A Course in Functional Analysis》定理 1.5，并链接到 FormalMath 的"柯西序列"、"等距同构"、"巴拿赫空间"条目。

**评分结果**：

| 维度 | 得分 | 依据 |
|------|------|------|
| RD | **5** | 定义从最一般度量空间出发，严格证明了商度量良定义，并说明了与范数完备化的联系。 |
| CP | **5** | 标准嵌入与泛性质证明完整，额外提供了"用等价类 vs 用闭包"两种视角的对比。 |
| RE | **5** | 三道基础+进阶+探索题层次清晰，探索题引向巴拿赫空间，附详细提示。 |
| QR | **5** | 引用具体到章节与定理编号，内部链接形成完备化知识网络。 |
| PE | **4** | 动机铺垫充分，例子典型，有常见错误警示；若增加图示可达 5 分。 |

- **总分**：24/25
- **建议**：**保留并作为标杆文档**

---

### 5.2 平庸文档示例：《矩阵的秩》

**文档特征概述**：
- 定义："矩阵的秩就是矩阵中线性无关的行向量的最大个数。"——未区分行秩与列秩，未说明是在哪个数域上；
- 列出了 3 条性质（如 `rank(A) = rank(A^T)`），但无任何证明，仅有"由线性代数基本定理可知"；
- 习题：2 道，一道是"求下列矩阵的秩"（给出 2×2 矩阵），另一道是"证明满秩矩阵可逆"（无提示）；
- 引用：文末写了一行"参见任意线性代数教材"，无具体书目；无内部链接；
- 结构：定义→性质→习题，但没有解释"为什么秩是矩阵最重要的不变量之一"，例子全是 2×2/3×3 的数字矩阵，未触及几何意义。

**评分结果**：

| 维度 | 得分 | 依据 |
|------|------|------|
| RD | **3** | 定义基本可用，但未区分行秩/列秩，缺少数域假设，符号规范但不够严谨。 |
| CP | **2** | 性质罗列无证明，逻辑跳跃严重，仅有定理骨架。 |
| RE | **2** | 两道机械计算/证明题，无解答、无层次、无探索性。 |
| QR | **1** | 无任何可追溯引用，内部链接为零。 |
| PE | **3** | 结构清晰但缺动机，例子过于简单，未触及秩的几何解释。 |

- **总分**：11/25
- **建议**：**大幅重写**（需补证明、补引用、重构习题、增加几何直观）

### 5.3 不合格文档示例：《微分方程简介》

**文档特征概述**：
- 全文约 400 字，内容如下："微分方程是含有未知函数及其导数的方程。它在物理、工程、经济中有广泛应用。解微分方程就是求出未知函数。微分方程分为常微分方程和偏微分方程。常微分方程只含一个自变量，偏微分方程含多个自变量。学习微分方程需要掌握积分技巧。"
- 无任何数学符号、无任何具体方程示例、无任何定义域/解的存在性说明；
- 无习题；
- 无引用；
- 文本像是从百科引言拼凑而成，没有任何教学价值。

**评分结果**：

| 维度 | 得分 | 依据 |
|------|------|------|
| RD | **1** | 纯口语化描述，无形式定义，未说明阶、线性/非线性、初值问题等核心概念。 |
| CP | **1** | 无任何定理或证明。 |
| RE | **1** | 无任何习题。 |
| QR | **1** | 无任何引用。 |
| PE | **1** | 术语堆砌，无结构、无例子、无动机，不具备任何教学功能。 |

- **总分**：5/25
- **建议**：**归档或删除**（若保留，需从零重写，成本高于新建）

## 6. 评分后的行动建议矩阵

根据总分，文档进入不同的处置轨道。该矩阵为**强制性建议**，审计团队可在此基础上结合主题重要性进行微调，但需记录偏离理由。

| 总分区间 | 质量等级 | 行动建议 | 执行要求 |
|----------|----------|----------|----------|
| **≥ 20 分** | 优秀 | **保留并作为标杆** | 纳入"标杆库"，在其他同类文档重写时作为参考模板；每季度抽检一次防止退化。 |
| **15–19 分** | 良好 | **局部修订** | 指定责任编辑，在 2 周内针对低分维度进行修补（如补习题、补引用、细化证明），修补后复评。 |
| **10–14 分** | 及格边缘 | **大幅重写** | 退回到内容生产流程，由主题专家主导重写；原文档标记为"重写中"，重写完成前不对用户可见。 |
| **< 10 分** | 不合格 | **归档或删除** | 直接移入 `archive/low-quality/` 或删除；若主题重要且尚无替代文档，允许从零新建，但不允许在原稿上小修。 |

### 6.1 特殊情形处理

- **单科维度 1 分**：即使总分 ≥ 15，若任一维度得 1 分，必须针对该维度进行强制修订；若该维度为 RD 或 CP，且文档属于核心概念（如极限、群、向量空间），则升级处置为"大幅重写"。
- **仲裁后仍低于 10 分**：三名评分者一致认定为不合格，删除流程无需额外审批。
- **标杆文档的维护**：标杆文档（≥20 分）若被后续数学发展证明存在概念过时或符号不统一，可启动"标杆降级"流程，重新进入局部修订轨道。

```
[双盲评分] → 评分者 R-A, R-B 独立打分
    ↓
    ├─ 差异 ≤3 分 → 计算最终得分 → 进入行动矩阵
    └─ 差异 ≥4 分 → 仲裁员 R-C 介入 → 取中位数 → 进入行动矩阵
    ↓
[结果汇总] → 生成批次质量报告 → 反馈给内容团队
```

### 7.1 评分者禁忌清单

评分者在打分时**严禁**进行以下行为：
1. 搜索文档作者的历史贡献来判断"是否应该给高分"；
2. 因为文档主题"冷门"或"热门"而调整标准；
3. 使用 0 分或小数（如 3.5 分），本框架只允许 1–5 的整数分；
4. 在评分依据中使用人身评价（如"作者显然不懂这个主题"），应只评论内容本身。

### 7.2 快速校准问题

若评分者对某维度拿捏不准，可自问以下问题：

- **RD**：如果把这篇文档的符号和定义原封不动地交给一位严谨的数学家，他会皱眉吗？
- **CP**：一位勤奋但非天才的学生，能否根据文档内容独立补出完整的证明？
- **RE**：做完这些习题后，学习者是否觉得自己真正"会用了"，还是只会照猫画虎？
- **QR**：五年后有人读到这篇文档，还能根据引用找到原始出处吗？
- **PE**：如果这篇文档出现在一本教材里，读者会给出版社写投诉信吗？
