# 三层内容金字塔写作标准 v2.0

> **文档编号**: D005  
> **版本**: v2.0  
> **适用项目**: FormalMath  
> **生效日期**: 2026年4月  
> **编制角色**: 数学教育内容架构师  

---

## 执行摘要

FormalMath 项目当前拥有 **6573+ 篇文档**，但存在严重的结构性失衡：铜层（基础入门）内容过度膨胀、注水严重，而银层（本科核心）和金层（研究生前沿）几乎处于空白状态。本标准旨在停止盲目的文档数量扩张，建立清晰、可执行的三层内容分层体系，为后续的质量重写、文档迁移和新文档生产提供统一判据。

---

## 1. 三层金字塔模型详解

### 1.1 铜层 (Copper) — 基础概念与入门路径

**定位**: 金字塔底座。建立直觉、消除恐惧、铺设语言。

| 属性 | 说明 |
|------|------|
| **目标读者** | 高中生、大一新生、跨专业转码者、对数学感兴趣但缺乏系统训练的自学者。 |
| **核心使命** | 让读者觉得"我能看懂"，建立对学科的基本语感和信心。 |
| **内容深度** | 严格限制在"科普级 + 高中/大一预备知识"。禁止出现 ε-δ 语言、抽象代数结构、层/sheaf、概形/scheme 等高级对象。 |
| **写作风格** | 口语化、类比丰富、图示优先、历史叙事辅助。每引入一个新符号必须在 200 字内给出直观解释。 |
| **禁止事项** | ① 把铜层文档写成"压缩版银层"；② 用大量定义-定理-证明填充篇幅；③ 引入未解释的专业黑话；④ 超过 50% 的内容是转述而非原创梳理。 |
| **标杆示例** | "什么是向量：从箭头到空间的直觉之旅"、"集合论：数学家如何谈论'东西'" |

### 1.2 银层 (Silver) — 本科高年级核心课程

**定位**: 金字塔中坚。严格的数学训练发生在这里。

| 属性 | 说明 |
|------|------|
| **目标读者** | 数学/物理/计算机专业本科二至四年级学生、准备研究生入学考试的学生、需要夯实基础的低年级研究生。 |
| **核心使命** | 培养形式化思维习惯：能读证明、能写证明、能判断一个命题的层级位置。 |
| **内容深度** | 对应国际标准本科高年级核心课程（如实分析、抽象代数、点集拓扑、线性代数的高等形式）。必须有完整的定义-定理-证明链，例题必须展示解题策略而非仅给出答案。 |
| **写作风格** | 严谨但不僵化。证明必须完整，但允许在证明前加入"证明思路"段落。鼓励使用形式化工具（如 Lean）标注关键证明步骤。 |
| **禁止事项** | ① 用口语化叙述替代严格证明；② 定理陈述不完整（缺少前提条件）；③ 例题与定理脱节；④ 缺少与前序铜层概念的明确衔接。 |
| **标杆示例** | Rudin《数学分析原理》级别的实分析讲义、Axler《线性代数应该这样学》级别的线性代数重构。 |

### 1.3 金层 (Gold) — 研究生前沿专题

**定位**: 金字塔尖。连接经典知识与当代研究前沿。

| 属性 | 说明 |
|------|------|
| **目标读者** | 高年级研究生、博士后、青年教师、具备银层基础后希望进入特定研究领域的自学者。 |
| **核心使命** | 展示一个领域"从哪里来到哪里去"：呈现开放问题、最新进展、不同学派的方法论差异，以及形式化数学在该领域的可能切入点。 |
| **内容深度** | 对应研究生研讨课（graduate seminar）或博士资格考准备材料。必须引用近 15 年内的高质量文献（arXiv、顶刊）。允许使用原始论文中的记号体系，但必须在首次出现时建立与项目标准术语的映射。 |
| **写作风格** | 学术综述式。每节开头给出"本节要回答的核心问题"，结尾给出"尚未解决的问题"。鼓励使用原始文献的精确表述，并附带批判性评论。 |
| **禁止事项** | ① 把金层写成"更长的银层"（即仅罗列经典定理而缺乏前沿视角）；② 引用过时文献（>20 年且无经典地位的）；③ 缺少与形式化数学的关联讨论；④ 未指明所需前置知识的具体银层文档。 |
| **标杆示例** | 一篇关于 "Liquid Tensor Experiment" 的形式化概述、"导出代数几何在数论中的最新应用" 专题。 |

## 2. 每层的内容规格表

| 维度 | 铜层 (Copper) | 银层 (Silver) | 金层 (Gold) |
|------|---------------|---------------|-------------|
| **目标读者** | 高中生、大一新生、跨专业自学者 | 本科高年级学生、考研学生、低年级研究生 | 高年级研究生、博士后、青年教师 |
| **字数建议** | 1,500 – 4,000 字 | 4,000 – 12,000 字 | 8,000 – 25,000 字（允许系列文档） |
| **定义数量** | 3 – 8 个（必须极度直观） | 10 – 30 个（必须严格、完整） | 20 – 50+ 个（可引入专业领域特化定义） |
| **定理-证明对数量** | 0 – 2 对（可只有定理直觉，无严格证明） | 5 – 20 对（证明必须完整） | 10 – 30+ 对（允许引用原始文献中的证明，但需给出关键步骤解析） |
| **例题数量** | 2 – 5 道（侧重直观计算或概念辨析） | 5 – 15 道（必须展示证明技巧或解题策略） | 3 – 10 道（侧重研究级例子或反例，展示前沿方法） |
| **习题数量** | 3 – 8 道（基础巩固） | 10 – 25 道（难度分层：基础/进阶/挑战） | 5 – 15 道（开放性问题、文献追踪题、形式化实现题） |
| **引用密度** | 低。每篇 1 – 3 个引用，优先引用项目内铜层文档 | 中。每篇 5 – 15 个引用，必须引用经典教材和项目内银层前置文档 | 高。每篇 20 – 50+ 个引用，必须包含近 15 年文献 |
| **历史/哲学叙述占比上限** | ≤ 30% | ≤ 10% | ≤ 15%（但必须是"研究史"而非"八卦史"） |
| **允许使用的外部资源级别** | 科普视频、维基百科、3Blue1Brown、大众数学读物 | 经典教材（GTM/UTM/USTM 系列）、大学课程讲义、StackExchange 高赞解答 | 顶刊论文、arXiv 预印本、博士论文、国际会议讲义、著名数学家的博客/讲义 |

### 2.1 规格表使用说明

1. **字数建议**是指导性而非强制性。若某铜层文档字数超过 6,000 字，必须接受"是否存在银层内容下沉"的审查。
2. **定理-证明对**的计数方式：一个"定理 + 其严格证明"计为 1 对；仅有命题陈述而无证明不计；铜层的"定理直觉"不计入此列。
3. **引用密度**的引用格式须遵循项目统一的 `[^id]` 脚注体系。
4. 若某文档的指标跨层分布严重（如既有 20 个定义又有 0 个证明），则视为"分层不清"，退回重写。

## 3. 文档类型的归属判定与改造建议

当前项目存在四种后缀的文档变体。以下是它们的明确归属判定标准和改造路径。

### 3.1 `-基础版.md` → **铜层 (Copper)**

| 项目 | 判定标准 |
|------|----------|
| **归属层** | 铜层 |
| **核心特征** | 使用日常语言、大量比喻、少量公式、无严格证明、侧重"是什么"和"为什么重要"。 |
| **现存问题诊断** | 大量基础版文档实质是维基百科的机械翻译或搬运，缺少项目独有的教学叙事结构；部分文档偷偷塞入了银层内容以显得"有深度"。 |
| **改造建议** | ① 删除所有 ε-δ、所有抽象结构定义；② 将长段落拆分为"概念卡片"式短段落；③ 每篇必须包含至少 1 张示意图或 1 个生活化类比；④ 增加"如果你准备好了，下一步读什么"的银层跳转链接。 |
| **质量红线** | 铜层基础版文档的注水率（即与维基百科/百度百科重合度）不得超过 30%。 |

### 3.2 `-增强版.md` → **银层 (Silver)**

| 项目 | 判定标准 |
|------|----------|
| **归属层** | 银层 |
| **核心特征** | 在基础概念之上补充严格定义、完整证明、经典例题和系统性习题。 |
| **现存问题诊断** | 大量增强版文档停留在"比基础版多几个公式"的层面，证明不完整、例题流于计算、缺少与前导铜层文档的衔接。 |
| **改造建议** | ① 补齐所有定理的完整证明；② 例题必须标注所属"技巧类型"（如反证法、构造法、紧致性论证）；③ 明确列出前置铜层文档清单；④ 引入 Lean 形式化片段作为可选阅读模块。 |
| **质量红线** | 增强版文档若被判定为"铜层内容的简单加长"，必须拆分为铜层基础版 + 银层核心讲义两部分。 |

### 3.3 `-深度扩展版.md` → **金层 (Gold)** 或 **银层 (Silver)**

| 项目 | 判定标准 |
|------|----------|
| **归属层** | 视内容而定：若聚焦经典主题的深入技巧（如实分析中的覆盖定理变体），可归银层；若引入前沿研究视角，必须归金层。 |
| **核心特征** | 探讨一个主题的边界案例、高级变体、交叉应用或最新研究。 |
| **现存问题诊断** | 大量深度扩展版文档只是罗列了更多定理，缺乏对"为什么这些扩展重要"的论述；很多文档实际上停留在银层深度，却错误地标注为金层。 |
| **改造建议** | ① 若为金层：必须增加"研究前沿"章节，引用近 15 年文献，指出开放问题；② 若为银层：标题应改为 `-高级专题版.md` 并归入银层子目录，避免层属混淆；③ 无论归属哪层，必须增加"与形式化数学的关联"小节。 |
| **质量红线** | 没有近 15 年文献引用的深度扩展版文档，不得标注为金层。 |

### 3.4 `-国际标准版.md` → **银层 (Silver)** 或 **金层 (Gold)**

| 项目 | 判定标准 |
|------|----------|
| **归属层** | 视内容而定：若对应本科核心课程的国际讲义（如 MIT OCW、Harvard Math），一般归银层；若对应研究生研讨课或前沿讲座，归金层。 |
| **核心特征** | 以国际顶尖大学课程为蓝本，注重与全球标准对齐。 |
| **现存问题诊断** | 国际标准版文档往往陷入"对齐报告化"：大量篇幅用于描述"MIT 讲了什么、Harvard 讲了什么"，而缺少项目自身的体系化重构。部分文档只是课程大纲的翻译。 |
| **改造建议** | ① 删除纯描述性的课程大纲段落，将内容重构为"主题驱动的讲义"；② 明确标注每个定理/定义的对齐来源（如 `对齐: MIT 18.100A Lecture 5`）；③ 若为银层：补齐被国际课程省略但对中国学生必要的背景知识；④ 若为金层：补充该领域在国际上的最新进展，超越原课程的时间边界。 |
| **质量红线** | 国际标准版文档中，直接翻译或转述外部课程材料的占比不得超过 40%；必须有项目独创的整合、补充或批判。 |

### 3.5 类型归属速查表

| 文档后缀 | 默认归属 | 可上调条件 | 可下调条件 |
|----------|----------|------------|------------|
| `-基础版.md` | 铜层 | 不适用（铜层不可混入银层内容） | 不适用 |
| `-增强版.md` | 银层 | 包含前沿文献综述和开放问题 → 金层 | 仅罗列公式无严格证明 → 铜层 |
| `-深度扩展版.md` | 需个案判定 | 有近 15 年文献、研究视角 → 金层 | 仅为经典技巧深入 → 银层 |
| `-国际标准版.md` | 银层（本科课程）/ 金层（研究生课程） | 补充最新研究进展 → 金层 | 仅翻译大纲 → 降级为铜层科普或删除 |

## 4. 新旧文档迁移判定流程

针对项目现有 6573+ 篇文档，设计以下迁移决策流程。所有判定应由至少两名内容审核员执行，结果记录于项目迁移日志。

### 4.1 迁移判定决策树

```
开始：抽取现有文档
    │
    ▼
步骤1：文档是否属于数学内容？
    ├── 否（如会议记录、个人笔记、过期TODO）──► 删除
    │
    └── 是
        │
        ▼
步骤2：文档的主题是否有铜层/银层/金层的明确归属？
    ├── 否（如随机数学杂谈、无主题堆砌）──► 归档
    │
    └── 是
        │
        ▼
步骤3：该主题在项目金字塔中是否已有标杆文档？
    ├── 是（存在同主题的高质量标杆）
    │       │
    │       ▼
    │   当前文档质量是否达到标杆的 80% 以上？
    │       ├── 是 ──► 局部修订（补充引用、修正术语、优化结构）
    │       └── 否 ──► 归档（避免重复建设）
    │
    └── 否（该主题尚无标杆文档）
            │
            ▼
步骤4：评估文档的"核心信息密度"
    │
    ├── 信息密度 < 20%（大量灌水、搬运、过时内容）──► 删除
    │
    ├── 信息密度 20%–50%（有骨架但血肉不足）──► 大幅重写
    │
    ├── 信息密度 50%–80%（有缺陷但基础尚可）──► 局部修订
    │
    └── 信息密度 > 80%（结构清晰、内容扎实）───► 保留（作为标杆）
```

### 4.2 判定标准的详细说明

#### 4.2.1 "信息密度"评估细则

信息密度 = （有效数学内容字数 / 总字数）× 100%。有效数学内容指：严格的定义、定理、证明、有教学价值的例题/习题、与研究直接相关的历史叙述。

| 密度区间 | 判定 | 后续动作 |
|----------|------|----------|
| **> 80%** | 标杆级 | 保留。作为该主题在金字塔中的标杆文档，其他同主题文档向它对齐。 |
| **50% – 80%** | 可修复 | 局部修订。指派给熟悉该领域的编辑，限期 2 周内完成修订。 |
| **20% – 50%** | 骨架级 | 大幅重写。保留文档的目录结构和核心论点，但正文需要 50% 以上的改写。建议拆分为多篇分层文档。 |
| **< 20%** | 废品级 | 删除。无归档价值，直接删除以释放维护成本。 |

#### 4.2.2 保留（作为标杆）的额外条件

一篇文档即使信息密度高，也必须同时满足以下条件才能被认定为标杆：

1. **层属清晰**：文档的铜/银/金属性可被至少两名审核员一致认定。
2. **术语一致**：文档中使用的术语与项目标准术语表的差异率 < 5%。
3. **引用规范**：所有外部来源均有规范引用，无大段未标注的复制内容。
4. **前后链接**：文档开头列出前置文档，结尾列出后续推荐阅读。
5. **形式化标记**：至少包含一个指向项目 Lean/形式化代码库的链接或片段（铜层可放宽为"概念说明"而非代码）。

#### 4.2.3 归档与删除的区分原则

- **归档**：文档具有一定的历史参考价值，但不符合当前质量标准或主题重复。归档后保留只读访问，不再出现在主索引中。
- **删除**：文档属于垃圾内容（如测试文件、空白模板、明显侵权的复制粘贴、个人草稿）。删除后不再保留任何副本。

### 4.3 迁移判定执行表单（模板）

审核员在对每篇文档进行判定时，应填写以下表单：

| 字段 | 填写内容 |
|------|----------|
| 文档路径 | `concept/analysis/real-analysis-intro-基础版.md` |
| 主题分类 | 实分析 / 铜层 |
| 信息密度评分 | 35% |
| 层属清晰度 | 不清晰（混入银层证明） |
| 标杆文档 | `concept/analysis/real-numbers-基础版.md` |
| 判定结果 | 大幅重写 |
| 具体建议 | ① 删除 ε-δ 段落，移至银层新文档；② 补充 2 个直观类比；③ 将长段落拆分为概念卡片。 |
| 审核员签名 | @reviewer_a, @reviewer_b |

## 5. 质量门控标准

所有新文档在合并到主分支前，必须通过对应层级的质量门控（Quality Gate）。未通过门控的文档，无论作者是谁，一律退回修改。

### 5.1 铜层文档最低通过标准 (QG-Copper)

| 检查项 | 通过标准 | 检查方式 |
|--------|----------|----------|
| **QG-C01 读者定位** | 文档开头明确声明"本文适合没有 XX 基础的读者" | 人工审核 |
| **QG-C02 术语友好** | 所有专业术语在首次出现时 200 字内有直观解释 | 自动扫描 + 人工抽查 |
| **QG-C03 无银层入侵** | 文档中不含 ε-δ、抽象代数结构、层/sheaf、概形等银/金层专属对象 | 关键词自动检测 + 人工复核 |
| **QG-C04 证明限制** | 定理-证明对 ≤ 2 对；若超过，必须有充分的直觉化说明 | 自动统计 |
| **QG-C05 前后链接** | 开头有"前置阅读"，结尾有"延伸阅读"（至少各 1 条项目内链接） | 自动链接检测 |
| **QG-C06 无侵权风险** | 与任一外部百科词条的连续重合度 < 100 字，且总体重合度 < 30% | 查重工具 |
| **QG-C07 可读性** | Flesch 阅读易度（中文等效标准：句子平均长度 < 35 字，段落平均长度 < 150 字） | 自动分析 |
| **QG-C08 形式化关联** | 至少包含 1 个指向项目形式化概念或铜层可视化工具的链接 | 自动链接检测 |

### 5.2 银层文档最低通过标准 (QG-Silver)

| 检查项 | 通过标准 | 检查方式 |
|--------|----------|----------|
| **QG-S01 读者定位** | 文档开头明确声明所需前置知识（如"需要熟悉极限的 ε-δ 定义"） | 人工审核 |
| **QG-S02 定义完整性** | 所有核心定义必须包含：名称、符号、严格陈述、直观解释、反例或边界说明 | 人工审核 |
| **QG-S03 证明完整性** | 所有标有"定理"的陈述必须有完整证明；若引用外部证明，必须给出关键步骤解析和出处 | 人工审核 |
| **QG-S04 例题策略性** | 每道例题必须在解答前或解答后标注"本题使用的核心技巧/策略" | 人工审核 |
| **QG-S05 习题分层** | 习题必须明确标注难度：基础 (Basic) / 进阶 (Advanced) / 挑战 (Challenge) | 自动扫描 |
| **QG-S06 引用规范** | 至少 5 个规范引用，包含教材引用和项目内前置文档引用 | 自动统计 |
| **QG-S07 前后链接** | 开头列出前置文档清单（≥3 个），结尾列出后续推荐（≥2 个） | 自动链接检测 |
| **QG-S08 形式化关联** | 至少包含 1 个可运行的 Lean 代码片段或 1 个指向项目形式化库中对应定理的链接 | 人工 + 自动检测 |
| **QG-S09 历史占比** | 历史/哲学叙述占比 ≤ 10% | 自动统计 |

### 5.3 金层文档最低通过标准 (QG-Gold)

| 检查项 | 通过标准 | 检查方式 |
|--------|----------|----------|
| **QG-G01 读者定位** | 文档开头明确声明"本文假设读者已掌握 XX 和 YY"，并列出具体前置银层文档 | 人工审核 |
| **QG-G02 前沿视角** | 必须包含一个"研究前沿"或"开放问题"小节，引用近 15 年文献 ≥ 3 篇 | 人工审核 |

| **QG-G04 原始关联** | 必须讨论该主题与形式化数学的关联：哪些结果已被形式化？哪些尚未？形式化的主要障碍是什么？ | 人工审核 |
| **QG-G05 证明解析** | 对于引用原始文献的定理，必须给出"证明思路解析"，不能只罗列定理陈述 | 人工审核 |
| **QG-G06 研究级例子** | 至少包含 1 个研究级别的例子或反例（即不能是教材标准例题的简单重复） | 人工审核 |
| **QG-G07 引用密度** | 总引用数 ≥ 15 个 | 自动统计 |
| **QG-G08 前后链接** | 开头列出前置银层文档清单（≥5 个），结尾列出后续金层推荐或相关研究领域 | 自动链接检测 |
| **QG-G09 形式化关联** | 至少包含 1 个指向项目形式化库或 Mathlib 中相关发展的链接/代码片段 | 人工 + 自动检测 |

## 6. 五门攻坚课程的层属定位

以下五门课程是 FormalMath 项目重点攻坚对象。本标准明确它们在三层金字塔中的定位，以避免文档生产者之间的层属争议。

### 6.1 MIT 18.100A Real Analysis

| 属性 | 判定 |
|------|------|
| **归属层** | **银层 (Silver)** |
| **判定理由** | 18.100A 是 MIT 本科生的第一门实分析课程，覆盖实数构造、序列与级数、连续性、可微性、Riemann 积分。其难度对应中国大学数学系大二《数学分析》或《实变函数》入门。 |
| **内容边界** | 允许出现：ε-δ 语言、一致收敛、紧性、基本拓扑概念。不允许出现：Lebesgue 积分、抽象测度论、泛函分析工具。 |
| **标杆角色** | 作为银层实分析序列的第一门标杆课程，项目所有实分析相关铜层文档必须能指向它作为"下一本读物"。 |
| **与金层的衔接** | 在其结尾明确指向金层文档："从 Riemann 积分到 Lebesgue 积分"、"实分析在偏微分方程中的应用"。 |
| **文档命名建议** | `mit-18.100a-real-analysis-国际标准版.md` → 归入银层目录 `silver/analysis/` |

### 6.2 MIT 18.701 Algebra I

| 属性 | 判定 |
|------|------|
| **归属层** | **银层 (Silver)** |
| **判定理由** | 18.701 是 MIT 本科生的抽象代数入门，覆盖群论、环论、线性代数在抽象代数视角下的统一。难度对应中国大学数学系大二《近世代数》或《抽象代数》。 |
| **内容边界** | 允许出现：群、环、域、理想、商结构、群作用、Sylow 定理。不允许出现：模论、Galois 理论（属于 18.702）、表示论、同调代数。 |
| **标杆角色** | 银层代数序列的入口。项目的铜层文档（如"什么是群"）必须以它为升级目标。 |
| **与金层的衔接** | 结尾指向金层："Galois 理论前沿"、"代数几何中的交换代数"。 |
| **文档命名建议** | `mit-18.701-algebra-i-国际标准版.md` → 归入银层目录 `silver/algebra/` |

### 6.3 MIT 18.06 / 18.700 Linear Algebra

| 属性 | 判定 |
|------|------|
| **归属层** | **铜层-银层交界，最终归银层 (Silver)** |
| **判定理由** | 18.06 是 MIT 的"矩阵计算"导向课程（偏应用），18.700 是"线性代数理论"导向课程（偏证明）。这两门课覆盖了从高中向量到抽象向量空间的完整光谱。 |
| **分层建议** | 项目不应简单复制 MIT 的单一课程结构，而应拆分为：  <br>① **铜层**：《向量的直觉》（对应 18.06 的前 4 讲）；  <br>② **银层**：《抽象向量空间与线性变换》（对应 18.700 全部 + 18.06 的后半部分）。 |
| **标杆角色** | 银层线性代数文档的对齐标杆。特别地，18.700 的严格证明风格应作为项目银层线性代数文档的写作模板。 |
| **与金层的衔接** | 指向金层："无限维空间中的谱理论"、"代数几何中的层与向量丛"。 |
| **文档命名建议** | `mit-18.700-linear-algebra-国际标准版.md` → 银层  <br>`linear-algebra-intuition-基础版.md` → 铜层 |

### 6.4 Harvard Math 232br

| 属性 | 判定 |
|------|------|
| **归属层** | **金层 (Gold)** |
| **判定理由** | Math 232br 是 Harvard 的研究生代数几何课程（以 Hartshorne 为教材），面向已通过博士资格考的研究生。内容涵盖层 (sheaves)、概形 (schemes)、上同调 (cohomology)。 |
| **内容边界** | 允许出现：概形、层、导出函子、 étale 上同调（若涉及）。这是典型的金层内容。 |
| **标杆角色** | 金层代数几何子领域的标杆课程。项目所有关于代数几何的金层文档应向它对齐。 |
| **前置要求** | 必须在文档开头明确列出前置银层文档：交换代数、层论基础、复几何入门。 |
| **文档命名建议** | `harvard-math-232br-algebraic-geometry-国际标准版.md` → 归入金层目录 `gold/algebraic-geometry/` |

### 6.5 Stanford FOAG (Foundations of Algebraic Geometry)

| 属性 | 判定 |
|------|------|
| **归属层** | **银层-金层交界，最终归金层 (Gold)** |
| **判定理由** | FOAG（Ravi Vakil 的讲义）是一本兼具本科高年级和研究生水平的代数几何教材。前半部分（概形定义、态射、性质）可作为银层向金层过渡的桥梁；后半部分（上同调、曲线、曲面）明确属于金层。 |
| **分层建议** | 项目应将 FOAG 拆分为两篇标杆文档：  <br>① **银层**：《从代数簇到概形——FOAG 前半部分导读》（帮助学生建立概形直觉和基本技术）；  <br>② **金层**：《FOAG 高阶专题——上同调技术与现代发展》（面向研究生和研究人员）。 |
| **标杆角色** | 作为项目代数几何从银层跃迁到金层的官方通道。所有铜层/银层的代数几何文档在提到"下一步"时，应指向这两篇文档。 |
| **文档命名建议** | `stanford-foag-schemes-silver-国际标准版.md` → 银层  <br>`stanford-foag-cohomology-gold-国际标准版.md` → 金层 |

## 7. 实施路线图

### 第一阶段：标准宣贯（第 1-2 周）

- 召开全员内容生产者会议，逐条解读本标准。
- 在五门攻坚课程中各选 1 篇现有文档，按本标准进行公开评审（live review），形成可参照的判例。

### 第二阶段：标杆建设（第 3-8 周）

- 为每个学科方向（分析、代数、几何、拓扑、数论、概率、逻辑、计算数学）建立至少 1 篇铜层标杆、1 篇银层标杆、1 篇金层标杆。
- 标杆文档通过本标准全部门控后，冻结内容结构，仅允许修正错别字。

### 第三阶段：批量迁移（第 9-16 周）

- 组织审核员按第 4 章的迁移判定流程，对现有 6573+ 篇文档进行批量分类。
- 目标：删除/归档 30% 的低质量文档，对 40% 的文档进行局部修订，对 20% 的文档进行大幅重写，保留 10% 作为标杆。

### 第四阶段：常态化门控（第 17 周起）

- 所有新文档必须通过 CI 自动检测 + 人工审核 + 架构师签批的三级门控。
- 每月发布一次《质量门控执行报告》，公开通过率和主要驳回原因。

### 附录 B：常见问题 (FAQ)

**Q1: 一篇文档是否可以同时包含铜层、银层和金层内容？**  
A: **不可以。** 这是导致当前项目结构混乱的主要原因。若一个主题需要跨层讲解，必须拆分为多篇文档，每篇文档明确标注其层属，并通过前后链接串联。

**Q2: 铜层文档是否意味着可以写得随意、不严谨？**  
A: **不是。** 铜层要求的是"直觉上的严谨"——每一个概念都必须有清晰的边界说明，即使不使用 ε-δ 语言，也必须让读者明白"这个直觉在什么时候会失效"。

**Q3: 金层文档是否必须全部由项目原创？**  
A: **不必。** 金层文档本质上是"学术综述 + 形式化视角"。大量内容会来自对原始文献的整理和评述，但必须明确标注出处，且项目的原创性体现在：整合框架、术语映射、形式化关联分析、以及对不同学派观点的比较。

**Q4: 如果某主题目前只有铜层和金层文献，中间缺少银层怎么办？**  
A: **这是项目的重大缺口，必须优先填补。** 银层是连接铜层和金层的唯一桥梁。缺少银层会导致读者从"我能看懂"直接跌落到"我完全不懂"，这是极差的学习体验。此类缺口应标记为"银层攻坚任务"，优先分配资源。
