## 1. L1-L6 六级对齐模型详解

FormalMath 项目将课程对齐工作划分为六个层级，从粗粒度的结构映射到细粒度的语义等价。L1-L2 为项目现有能力，L3-L6 为 v2.0 质量重写阶段的核心攻坚目标。

### 1.1 层级总览

| 层级 | 名称 | 核心问题 | 对齐粒度 | 判定难度 |
|:----:|:-----|:---------|:---------|:--------:|
| L1 | 课程/条目映射 | 这门课程在项目中有无对应文档？ | 课程 ↔ 文档条目 | ⭐ |
| L2 | 章节/概念映射 | 课程章节中的核心概念是否被覆盖？ | 章节 ↔ 概念节点 | ⭐⭐ |
| L3 | 定义对齐 | 课程中的数学定义与项目文档是否等价？ | 定义 ↔ 定义 | ⭐⭐⭐ |
| L4 | 定理对齐 | 课程中的定理在项目中有无完整证明？ | 定理 ↔ 证明 | ⭐⭐⭐⭐ |
| L5 | 习题/作业对齐 | 课程的 Problem Sets 是否被映射和解答？ | 习题 ↔ 解答 | ⭐⭐⭐ |
| L6 | 思想方法对齐 | 课程强调的独特技巧与教学哲学是否被提炼？ | 思想 ↔ 方法论 | ⭐⭐⭐⭐⭐ |

### 1.2 各层级的判定标准、验证方法、时间估算与责任人

#### L1：课程/条目映射（Entry Mapping）

**判定标准**：
- 课程代码（如 MIT 18.100A）在项目 `docs/` 或 `concept/` 目录下有明确的顶层入口文档。
- 入口文档包含课程的基本信息（名称、学期、教材、讲师、课程主页链接）。
- 不存在"幽灵对齐"：即项目声称对齐了某课程，但仅有一个空壳引用页。

**验证方法**：
1. 检查课程官方 Syllabus 与项目对齐文档的 URL 对应关系。
2. 确认项目文档的 `course-alignment` YAML frontmatter 中包含该课程代码。
3. 统计该课程下至少存在一个非空的 Markdown 文档。

**所需时间估算**：每门课程 15-30 分钟（10-20 门/人日）。

**责任人角色**：课程对齐协调员（Curriculum Alignment Coordinator）。

---

#### L2：章节/概念映射（Chapter/Concept Mapping）

**判定标准**：
- 课程各章节的主要概念在项目概念图谱（Concept Graph）中有对应节点。
- 映射不是模糊的：一个课程概念对应到一个明确的项目概念 UUID，而非一个目录。
- 标注差异：若课程使用了项目未覆盖的子概念，必须显式标注为"缺失"。

**验证方法**：
1. 将课程讲义按章节拆解为概念列表。
2. 在项目 `concept/` 目录或 `multilang_concept_table.json` 中搜索对应词条。
3. 记录映射结果：直接命中 / 近似命中 / 缺失。
4. 对缺失概念生成待办清单（backlog）。

**所需时间估算**：每门课程 2-4 小时（2-4 门/人日）。

**责任人角色**：领域研究生（Graduate Student in Field）+ 课程对齐协调员复核。

---

#### L3：定义对齐（Definition Alignment）

**判定标准**：
- 课程中每个核心定义的数学表述与项目文档中的定义在逻辑上等价。
- 允许等价变形（如用邻域定义拓扑 vs 用开集公理定义拓扑），但必须显式给出等价性说明。
- 不允许"近似表述"作为对齐完成的标准。

**验证方法**：
1. 提取课程讲义/教材中所有以 "Definition" 标记的条目。
2. 在项目文档中找到最相关的定义段落。
3. 逐条比对：前置条件、主命题、符号约定、特例说明。
4. 使用 "严格等价 / 近似表述 / 缺失" 三档标注（详见第 2 章）。
5. 对"近似表述"给出修正建议，对"缺失"生成写作任务。

**所需时间估算**：每门课程 6-10 小时（1-1.5 门/人日）。

**责任人角色**：领域数学家（Research Mathematician）或高年级博士生。

---

#### L4：定理对齐（Theorem-Proof Alignment）

**判定标准**：
- 课程中的每个核心定理（Theorem / Lemma / Proposition）在项目中有对应的完整证明。
- 证明的关键步骤无跳跃（no logical gaps > 2 行 Lean4 或 3 行自然语言）。
- 对等价表述的定理，项目文档至少覆盖一种与课程一致的证明路径。

**验证方法**：
1. 提取课程中所有定理陈述。
2. 在项目文档中搜索定理名称或等价的数学表述。
3. 评估证明的完整性：完整证明 / 部分证明 / 仅陈述 / 缺失。
4. 标记证明中的逻辑跳跃点，给出补充建议。
5. 对"缺失"和"部分证明"生成写作任务。

**所需时间估算**：每门课程 10-16 小时（0.6-1 门/人日）。

**责任人角色**：领域数学家（Research Mathematician）。

---

#### L5：习题/作业对齐（Exercise Alignment）

**判定标准**：
- 课程的 Problem Sets、Exams、Recitation 习题在项目中有显式映射。
- 映射的习题包含：原题转述（或引用）、详细解答、所属概念标签。
- 至少覆盖课程官方发布习题的 60%（核心课程）或 40%（高级专题课程）。

**验证方法**：
1. 收集课程所有公开可获取的 Problem Sets 和 Exams。
2. 将习题按类型分类（计算型 / 证明型 / 反例构造型 / 概念辨析型）。
3. 在项目 `docs/` 或专门的 exercise 目录中搜索对应解答。
4. 记录映射结果：逐题对应 / 变体对应 / 无对应。
5. 对未覆盖习题生成 backlog。

**所需时间估算**：每门课程 8-14 小时（0.6-1 门/人日）。

**责任人角色**：数学教育研究生（Math Education Graduate Student）+ 领域数学家复核。

---

#### L6：思想方法对齐（Pedagogical & Methodological Alignment）

**判定标准**：
- 课程讲义/视频中强调的"独特证明技巧"被提炼并记录。
- 课程中反复出现的"常见错误模式"（common pitfalls）被整理为诊断条目。
- 课程的教学哲学（如构造主义 vs 经典主义、具体例驱动 vs 公理化驱动）在项目的"数学思想"或"数学家理念"文档中有对应阐述。

**验证方法**：
1. 阅读课程讲义的前言、教学说明、Lecture Notes 中的 "Remark" / "Strategy" / "Pitfall" 段落。
2. 观看（或阅读转录稿）关键课程视频，提取教师反复强调的方法论。
3. 在 Math StackExchange (MSE) 搜索与该课程相关的常见学生问题。
4. 将提炼的内容映射到项目 `数学家理念体系/` 或 `docs/数学思想/` 文档。
5. 评估映射结果：完整提炼 / 部分提炼 / 未提炼。

**所需时间估算**：每门课程 4-8 小时（1-2 门/人日）。

**责任人角色**：数学教育专家（Mathematics Education Specialist）+ 领域数学家联合签字。

## 2. L3 定义对齐的精确判定标准

### 2.1 三档判定规则

对于课程中的每一个定义，与项目文档进行比对后，必须给出以下三种判定之一：

#### A. 严格等价（Strictly Equivalent）

**判定规则**（需同时满足）：
1. **对象域一致**：定义适用的数学对象集合完全相同（如都在 $\mathbb{R}^n$ 上，或都在一般拓扑空间上）。
2. **条件逻辑等价**：课程定义的条件 $P$ 与项目定义的条件 $Q$ 满足 $P \iff Q$（在公共公理体系内可证）。
3. **符号约定兼容**：核心符号（如拓扑学中的 $\tau$、分析学中的 $\epsilon$-$\delta$）的使用不冲突；若存在不同记号，必须在文档中以显式对照表说明。
4. **边界情况一致**：对空集、零元、平凡结构等边界情况，两个定义的结论一致。
5. **等价性说明已记录**：项目文档或对齐记录中明确说明了两种定义的等价性（引用教材页码或给出证明提示）。

**示例**：
- 课程："函数 $f: X \to Y$ 连续，当且仅当对 $Y$ 中每个开集 $V$，$f^{-1}(V)$ 是 $X$ 中的开集。"
- 项目："函数 $f: X \to Y$ 连续，当且仅当对每点 $x \in X$ 和 $f(x)$ 的每个邻域 $N$，存在 $x$ 的邻域 $M$ 使得 $f(M) \subseteq N$。"
- 判定：**严格等价**，因为开集原像定义与邻域定义在点集拓扑中是标准等价表述，且项目文档在后续章节给出了等价性证明。

#### B. 近似表述（Approximate / Paraphrase）

**判定规则**（满足以下任一条件即判为此档）：
1. **对象域缩小或扩大**：项目定义适用的范围与课程定义不同（如课程在一般度量空间定义，项目仅在 $\mathbb{R}$ 上定义）。
2. **条件单向蕴含而非双向**：课程条件 $P$ 与项目条件 $Q$ 只满足 $P \implies Q$ 或 $Q \implies P$，而非等价。
3. **符号冲突且无说明**：两个定义使用同一符号表示不同概念，或不同符号表示同一概念但无对照说明。
4. **边界情况处理不同**：某一定义对空集、零元等特殊情况未作说明，而另一定义明确处理。
5. **非标准变体**：项目使用了课程未采用的等价变体（如用滤子定义极限），但未说明其与课程标准定义的等价性。

**近似表述的处理方式**：
- 必须在验证记录中标注 **GAP-3**（Definition Gap）。
- 给出修正建议："建议补充 $\mathbb{R}^n$ 版本" 或 "建议添加与邻域定义的等价性证明"。
- 若近似表述不影响后续定理证明的理解，可降级为"备注"；若影响，则必须重写。

#### C. 缺失（Missing）

**判定规则**（满足以下任一条件）：
1. 项目文档中完全不存在与课程定义相关的词条或段落。
2. 项目文档中存在相关概念，但缺少形式化的定义块（只有直观描述或例子，没有 "Definition" 段落）。
3. 课程定义是项目声称对齐的章节中的核心定义（非边缘概念）。

**缺失的处理方式**：
- 标注 **MISSING-3**，生成写作任务单，明确优先级（P0=核心定义，P1=辅助定义）。
- 在课程对齐 YAML 中标记 `missing_definitions` 列表。

### 2.2 同一概念的不同定义路径

数学中同一概念常存在多条等价的定义路径。L3 对齐不要求项目文档与课程使用**完全相同**的定义路径，但要求：

| 场景 | 对齐要求 |
|:-----|:---------|
| 课程与项目使用**标准等价**定义（如开集公理 vs 邻域系） | 判定为严格等价，但要求在项目文档或对齐注释中给出等价性说明 |
| 课程使用**路径 A**，项目使用**路径 B**，且 A→B 的等价性证明是课程本身要求掌握的内容 | 判定为严格等价，但要求项目文档包含该等价性证明 |
| 课程使用**路径 A**，项目使用**路径 B**，但 A→B 的等价性证明超出课程范围（如高阶工具） | 判定为近似表述，要求补充一个与课程路径一致的入口定义 |
| 课程使用**构造性定义**，项目使用**存在性定义**（或反之） | 判定为近似表述，要求根据课程哲学补充对应版本 |

**操作流程**：
1. 识别课程采用的定义路径（标记为 Course-Path）。
2. 识别项目文档中对应的定义路径（标记为 FM-Path）。
3. 判断 Course-Path 与 FM-Path 是否标准等价。
4. 若等价，检查项目是否给出等价性证明或引用。
5. 若不等价或不满足学生预期路径，生成补充定义任务。

### 2.3 L3 定义对齐检查清单（Checklist）

每一条定义对齐记录必须包含以下字段的确认：

- [ ] **ID**：课程定义的唯一标识（如 `MIT-18.100A-DEF-003`）。
- [ ] **Source**：课程定义原文摘录（含教材名称、页码、讲义 URL）。
- [ ] **Target**：项目文档中对应定义的绝对路径和段落锚点。
- [ ] **Domain Check**：适用对象域是否一致？是 / 否（说明差异）。
- [ ] **Logic Check**：逻辑条件是否等价？是 / 否（说明差异）。
- **Notation Check**：符号是否兼容？是 / 否（提供对照表）。
- [ ] **Edge Case Check**：边界情况是否一致？是 / 否（说明差异）。
- [ ] **Equivalence Note**：等价性说明是否已记录？是 / 近似 / 缺失。
- [ ] **Verdict**：最终判定 □ 严格等价 □ 近似表述 □ 缺失。
- [ ] **Action Item**：若不为"严格等价"，给出明确的修正任务。
- [ ] **Reviewer**：审校人签名与日期。

## 3. L4 定理对齐的判定标准

### 3.1 四档分级标准

对于课程中的每一个定理（Theorem / Lemma / Proposition / Corollary），评估项目文档中的覆盖情况：

| 等级 | 名称 | 判定标准 | 图标标记 |
|:----:|:-----|:---------|:--------:|
| L4-A | 完整证明 | 项目文档包含从定理假设到结论的完整证明，无超过容忍阈值的逻辑跳跃。 | 🟢 |
| L4-B | 部分证明 | 项目文档给出了证明的主要思路或部分步骤，但存在需要读者自行补全的明显跳跃；或只证明了定理的特例。 | 🟡 |
| L4-C | 仅陈述 | 项目文档中提到了该定理的名称或结论，但无任何证明或仅引用外部文献而未展开。 | 🔴 |
| L4-D | 缺失 | 项目文档中完全不存在该定理的陈述。 | ⚫ |

**逻辑跳跃的容忍阈值**：
- 自然语言证明：连续 3 行以上的推理缺少中间环节，即视为跳跃。
- Lean4 形式化证明：连续 2 个 tactic 之间的类型状态变化涉及非平凡的隐式引理，且该引理未在文档中提及，即视为跳跃。

### 3.2 定理等价形式的处理

课程与项目可能对同一数学事实使用不同的定理名称或略有不同的表述。处理原则：

#### 3.2.1 名称不同但内容相同

- **示例**：课程称 "Bolzano-Weierstrass Theorem"，项目称 "Weierstrass 聚点定理"。
- **处理**：只要数学内容完全一致，视为同一定理，记录别名映射。

#### 3.2.2 内容等价但表述不同

- **示例**：
  - 课程："$\mathbb{R}$ 中的有界无穷集必有聚点。"
  - 项目："$\mathbb{R}^n$ 中的有界序列必有收敛子列。"
- **处理**：
  1. 判断两者是否标准等价（在 $\mathbb{R}$ 上，聚点存在性与收敛子列存在性等价）。
  2. 若项目证明的是更一般的版本（如 $\mathbb{R}^n$），但课程只要求 $\mathbb{R}$，判定为 **L4-A（完整证明）**，但需在注释中说明课程限制在 $\mathbb{R}$。
  3. 若项目是更弱的版本（如只在闭区间 $[a,b]$ 上证明），而课程要求一般的有界集，判定为 **L4-B（部分证明）**。

#### 3.2.3 逆定理、等价刻画与推广

- 若课程要求证明定理 $T$ 的逆定理 $T'$，而项目只证明了 $T$：判定为 **L4-D（缺失）**，需单独记录 $T'$ 的 backlog。
- 若课程将某定理作为一组等价刻画之一讲授（如连续性的 5 种等价刻画），而项目只覆盖了其中 3 种：对缺失的 2 种判定为 **L4-D**，对已覆盖的 3 种分别评定 L4-A/B/C。

### 3.3 L4 定理对齐检查清单（Checklist）

每一条定理对齐记录必须包含：

- [ ] **ID**：课程定理的唯一标识（如 `MIT-18.100A-THM-012`）。
- [ ] **Source**：定理原文陈述（含假设、结论、教材页码/讲义 URL）。
- [ ] **Target**：项目文档中的对应位置（路径 + 锚点）。
- [ ] **Name Mapping**：定理名称是否一致？是 / 否（列出别名）。
- [ ] **Form Check**：数学表述是否等价？是 / 否（说明差异，引用等价性说明）。
- [ ] **Proof Coverage**：证明覆盖度 □ L4-A 完整证明 □ L4-B 部分证明 □ L4-C 仅陈述 □ L4-D 缺失。
- [ ] **Gap Analysis**：若为 L4-B 或 L4-C，列出具体的逻辑跳跃或缺失步骤。
- [ ] **Prerequisites Check**：证明所依赖的定义/引理是否已在 L3/L4 中对齐？是 / 否（列出未对齐依赖）。
- [ ] **Lean4 Check**：若为 Lean4 文档，代码是否通过编译？是 / 否 / 不适用。
- [ ] **Action Item**：若不为 L4-A，给出明确的补充/重写任务。
- [ ] **Reviewer**：审校人签名与日期。

## 4. L5 习题对齐的判定标准

### 4.1 Problem Sets 到项目文档的映射方法

课程习题通常以以下形式发布：
- Weekly Problem Sets（PDF）
- Recitation Handouts
- Past Exams（Midterm / Final）
- Textbook Exercises（指定教材的习题编号）

映射到项目文档时，采用以下三种对应关系：

| 对应关系 | 说明 | 示例 |
|:---------|:-----|:-----|
| **直接对应** | 项目文档中出现了与课程习题在数学内容上完全相同的题目。 | 课程 PS3-Q2：证明 $\sqrt{2}$ 无理；项目文档同名习题。 |
| **变体对应** | 项目文档中的习题是课程习题的参数变体、维度推广或结论弱化/强化。 | 课程：在 $\mathbb{R}$ 上证明；项目：在一般有序域上证明。 |
| **无对应** | 项目文档中不存在与课程习题相关的题目。 | — |

**映射操作流程**：
1. 为课程每道习题生成唯一 ID（如 `MIT-18.100A-PS03-Q02`）。
2. 提取习题的核心数学任务（忽略具体数值，抽象为任务类型）。
3. 在项目 `docs/`、`tests/`、专门的 exercise 目录中搜索相关解答。
4. 记录对应关系，并为每个对应关系标注习题类型标签。

### 4.2 四类习题的映射方法

#### 4.2.1 计算型习题（Computational）

**特征**：要求求出具体数值、极限、导数、积分、矩阵运算结果等。

**映射方法**：
- 核心关注点是"解题步骤的完整性和正确性"。
- 允许数值变体：若课程求 $\int_0^1 x^2 dx$，项目有 $\int_0^1 x^3 dx$，可判定为变体对应。
- 必须有明确的中间步骤和最终答案。

**L5 判定**：
- 直接对应 + 有详细解答：🟢 完整对齐
- 变体对应 + 有详细解答：🟡 变体对齐
- 无对应或只有答案无步骤：🔴 缺失

#### 4.2.2 证明型习题（Proof-based）

**特征**：要求证明一个数学命题，通常以 "Prove that..." 开头。

**映射方法**：
- 抽象出待证命题的核心结构（前置条件、结论、使用的主要工具）。
- 若项目文档中证明了同一命题（或标准等价命题），判定为直接对应。
- 若项目证明的是课程命题的推广/特例，判定为变体对应。

**L5 判定**：
- 直接对应 + 证明完整：🟢 完整对齐
- 变体对应 + 证明完整：🟡 变体对齐
- 只有证明思路（proof sketch）：🟠 部分对齐
- 无对应：🔴 缺失

#### 4.2.3 反例构造型习题（Counterexample）

**特征**：要求举一个反例说明某个命题不成立，或构造一个满足特殊条件的对象。

**映射方法**：
- 核心是"构造对象的性质"而非具体数值。
- 若项目文档中出现了相同性质的反例（如 Dirichlet 函数说明点点极限不连续但不可积），即为直接对应。
- 反例的具体形式可以不同（如用 $x^2 \sin(1/x)$  vs  $x \sin(1/x)$ 说明同一问题）。

**L5 判定**：
- 有直接对应的反例构造：🟢 完整对齐
- 有变体反例：🟡 变体对齐
- 无对应：🔴 缺失

#### 4.2.4 概念辨析型习题（Conceptual / True-False）

**特征**：判断命题真假、比较不同定义的适用范围、解释概念间关系。

**映射方法**：
- 往往出现在课程的 True-False 练习或概念检查（Concept Check）中。
- 映射到项目的"常见错误诊断"、"概念辨析"或 FAQ 文档中。
- 即使不是以习题形式出现，只要项目文档中对该概念辨析问题有明确讨论，即可判定为对应。

**L5 判定**：
- 项目文档中有明确的辨析讨论：🟢 完整对齐
- 有相关提及但不够系统：🟡 部分对齐
- 无对应：🔴 缺失

### 4.3 L5 习题对齐检查清单（Checklist）

- [ ] **ID**：课程习题的唯一标识（如 `MIT-18.100A-PS03-Q02`）。
- [ ] **Source**：习题原文或核心任务摘要。
- [ ] **Type**：习题类型 □ 计算型 □ 证明型 □ 反例构造型 □ 概念辨析型。
- [ ] **Target**：项目文档中的对应习题路径或解答路径。

## 5. L6 思想方法对齐的判定标准

### 5.1 "独特证明技巧"的提取方法

课程讲义或视频中往往会强调一些通用的证明策略或技巧。提取流程如下：

**步骤 1：识别信号词**
在讲义或转录稿中搜索以下信号表达：
- "The key idea is..."
- "A standard trick is..."
- "This is a typical example of..."
- "The proof strategy is to..."
- 中文对应："关键技巧在于..."、"一个标准的处理方式是..."、"这类问题的通法是..."

**步骤 2：抽象技巧模板**
将具体的证明步骤抽象为可迁移的模板：
```
技巧名称：[名称]
适用场景：[什么样的定理/问题]
核心步骤：
  1. ...
  2. ...
  3. ...
典型例子：[课程中的定理/习题编号]
在项目中的位置：[项目文档路径]
```

**步骤 3：映射到项目文档**
- 若项目文档中已有独立的"证明技巧"或"数学思想"章节，直接映射。
- 若项目文档中的证明隐含了该技巧但未显式提炼，判定为 **L6-B（部分提炼）**，建议增加"Remark"段落。
- 若项目文档中的证明路径与课程完全不同（使用了课程未涉及的高阶工具），判定为 **L6-C（方法论错位）**，建议补充课程路径的初等版本。

### 5.2 "常见错误模式"的整理方法

错误模式整理基于三个来源：
1. 课程讲义中显式标注的 "Common Mistake" / "Pitfall"。
2. 课程助教在 Recitation 或 Piazza 论坛上反复强调的易错点。
3. Math StackExchange (MSE) 上该课程相关的高赞问答。

**错误模式记录格式**：
```yaml
error_pattern:
  id: MIT-18.100A-ERR-001
  name: "混淆点态收敛与一致收敛的极限交换"
  course_ref: "Lecture 15, Slide 3"
  mse_ref: "https://math.stackexchange.com/q/12345"
  description: "学生常认为点态收敛的连续函数列的极限仍连续"
  why_wrong: "点态收敛不能保证极限函数继承连续性，需要一致收敛"
  correction: "使用一致收敛的定义验证 $\epsilon$-邻域的保持性"
  formalmath_doc: "docs/analysis/uniform-convergence.md"
  status: mapped / partial / missing
```

### 5.3 教学哲学的映射

不同课程可能有不同的教学哲学倾向。L6 对齐要求识别并映射这些哲学：

| 教学哲学维度 | 典型倾向 A | 典型倾向 B | 映射要求 |
|:-------------|:-----------|:-----------|:---------|
| 证明风格 | 构造主义（显式构造） | 存在主义（反证法/选择公理） | 若课程倾向 A，项目应有构造性证明入口 |
| 例子驱动 | 先大量例子再抽象 | 先公理化再举例 | 若课程倾向 A，项目文档的例子密度应足够 |
| 难度曲线 | 由易到难渐进 | 直面最一般的情形 | 项目文档的章节难度曲线应与课程匹配 |
| 应用导向 | 强调物理/工程应用 | 强调纯数学结构 | 项目文档应在相应位置补充应用实例 |
| 形式化 | 强调自然语言直观 | 强调 Lean4 形式化 | 若课程无形式化要求，项目不应以 Lean4 作为唯一入口 |

**L6 判定标准**：
- **L6-A（完整映射）**：课程的核心证明技巧、常见错误模式、教学哲学在项目文档中都有明确对应和提炼。
- **L6-B（部分映射）**：部分技巧或错误模式被覆盖，但缺乏系统性整理；或教学哲学与课程存在明显错位。
- **L6-C（缺失）**：课程强调的核心方法论在项目文档中完全不可见。

### 7.4 L3：定义对齐

- **对象域**：课程在 $\mathbb{R}$ 上定义，项目在一般度量空间上定义。对象域扩大，但 $\mathbb{R}$ 是度量空间的特例。

**L3 判定**：**🟡 近似表述**（对象域扩大，虽为推广但需在课程对齐注释中显式标注）。

**修正建议**：在项目文档顶部增加课程对齐注释框：

> **课程对齐注记**：MIT 18.100A 在 $\mathbb{R}$ 上定义序列紧致性。本页使用度量空间语言，但限制到 $\mathbb{R}$ 即与课程完全一致。

### 7.5 L4：定理对齐

#### 定理 T1：Bolzano-Weierstrass 定理（序列形式）

**课程陈述**：Every bounded sequence in $\mathbb{R}$ has a convergent subsequence.

**项目覆盖情况**：
- 项目文档 `docs/analysis/bolzano-weierstrass-theorem.md` 中包含了该定理的完整证明。
- 证明采用二分法（Bisection Method），与课程讲授的证明路径一致。

**证明步骤核查**：
1. 假设 $(x_n)$ 是 $\mathbb{R}$ 中的有界序列，则存在闭区间 $I_0 = [a, b]$ 包含所有 $x_n$。 ✅
2. 将 $I_0$ 二等分，至少有一个子区间包含无穷多项，记为 $I_1$。 ✅
3. 归纳地构造闭区间套 $I_0 \supset I_1 \supset I_2 \supset \cdots$，长度 $|I_k| = (b-a)/2^k$。 ✅
4. 从每个 $I_k$ 中选取一项 $x_{n_k}$，得到子列 $(x_{n_k})$。 ✅
5. 由闭区间套定理，$\bigcap_k I_k = \{x\}$，且 $x_{n_k} \to x$。 ✅

**逻辑跳跃检查**：
- 步骤 4 中"从每个 $I_k$ 中选取一项且保持下标递增"的构造细节在项目文档中明确写出。
- 步骤 5 中由 $|I_k| \to 0$ 推出收敛性的估计 $|x_{n_k} - x| \leq (b-a)/2^k$ 明确给出。✅

**L4 判定**：**🟢 L4-A（完整证明）**
