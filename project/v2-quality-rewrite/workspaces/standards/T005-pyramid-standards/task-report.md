## 3. 文档设计亮点

### 3.1 三层模型的差异化定位

- **铜层 (Copper)**: 强调"直觉上的严谨"，建立了明确的红线（禁止 ε-δ、禁止抽象结构、禁止黑话）。
- **银层 (Silver)**: 强调"形式化思维训练"，要求完整的定义-定理-证明链，并强制引入 Lean 形式化关联。
- **金层 (Gold)**: 强调"前沿综述视角"，强制要求近 15 年文献引用、开放问题讨论、以及与形式化数学的关联分析。

### 3.2 可量化的内容规格表

为每层设定了 8 个维度的量化指标（字数、定义数、定理-证明对、例题、习题、引用密度、历史占比、外部资源级别），并附带了使用说明，避免指标被机械套用。

### 3.3 文档类型的明确归属

针对项目现有的四种后缀（`-基础版.md`、`-增强版.md`、`-深度扩展版.md`、`-国际标准版.md`），分别给出了：
- 默认归属层
- 核心特征与现存问题诊断
- 具体改造建议
- 质量红线
- 可上调/下调的条件

### 3.4 可操作的迁移决策树

设计了从"抽取文档"到"删除/归档/保留/修订/重写"的完整决策树，并配套了"信息密度"评估细则、标杆额外条件、以及迁移判定执行表单模板。

### 3.5 三级质量门控 (QG)

分别为铜层 (QG-C01~C08)、银层 (QG-S01~S09)、金层 (QG-G01~G10) 设计了最低通过标准，并区分了自动检测和人工审核的检查方式，附带门控执行流程图。

### 3.6 五门攻坚课程的定位

对 MIT 18.100A、MIT 18.701、MIT 18.06/18.700、Harvard Math 232br、Stanford FOAG 进行了逐门分析，明确了归属层、内容边界、标杆角色、与相邻层的衔接方式，以及文档命名/目录建议。特别对跨度较大的课程（如 Linear Algebra 和 FOAG）给出了拆分为铜+银或银+金的建议。

## 4. 遇到的问题与处理

### 4.1 目录创建命令兼容性问题

**问题**: 初次使用类 Unix 命令 `mkdir -p` 在 Windows PowerShell 环境中执行失败，报错 "A positional parameter cannot be found"。

**处理**: 立即切换为 PowerShell 原生命令 `New-Item -ItemType Directory -Force`，成功创建目标目录。

### 4.2 文档内容的尺度拿捏

**问题**: 在设计标准时，需要在"足够详细以供执行"和"不过于繁琐以至于难以推行"之间找到平衡。例如，金层的"近 15 年文献"要求是否过于严格？铜层的"定理-证明对 ≤ 2"是否过于僵化？

**处理**: 在正文中增加了大量弹性条款（如"字数建议是指导性而非强制性"、"允许系列文档"、"个案判定"），并通过附录 FAQ 进一步澄清常见误解，确保标准既有约束力又不失灵活性。

### 4.3 五门课程的分层争议

**问题**: MIT 18.700 Linear Algebra 和 Stanford FOAG 的内容跨度较大，直接归入单层可能导致层属模糊。

**处理**: 对这两门课程明确建议了**拆分策略**：
- Linear Algebra → 拆分为铜层直觉版 + 银层严格版
- FOAG → 拆分为银层概形入门 + 金层上同调专题
