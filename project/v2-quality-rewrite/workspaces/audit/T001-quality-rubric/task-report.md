## 设计要点说明

### 1. 评分维度的选取逻辑

五个维度覆盖了数学知识质量的**内容面**（RD、CP）、**教学面**（RE、PE）与**可信度面**（QR），既回应了"模板化填充"的核心痛点，也兼顾了数学教育文档的特殊性（证明必须完整、定义必须严格）。

### 2. 盲审机制的关键设计

- **双盲+分层抽样**：避免热门主题或资深作者的文档被系统性高估；
- **加密样本编号**：阻断评分者通过文件路径推断文档来源；
- **仲裁流程**：以"中位数规则"处理分歧，既保护极端低分/高分意见，又确保结果稳健；
- **分歧数据库**：将评分争议转化为标准迭代的反馈回路。

### 3. 行动矩阵的严格性

- 设置 **单科维度 1 分的强制升级规则**：即使总分不低，核心维度（RD/CP）若得 1 分，也直接触发大幅重写，防止"总分虚高但根基腐烂"的情况。
- 设置 **<10 分直接归档/删除**：避免在低价值文档上浪费修订资源，符合"质量革命"从数量转向密度的战略。

## 遇到的问题与处理

### 问题 1：Windows PowerShell 的语法差异

**现象**：首次使用 `cd ... && mkdir -p ...` 命令时，因 PowerShell 不支持 `&&` 连接符而报错。

**处理**：改用 PowerShell 分号 `;` 分隔命令，并使用 `-Force` 参数替代 `-p`，成功创建嵌套目录。

**现象**：项目根目录的 `AGENTS.md` 为空文件，无法获取额外的写作风格或格式约定。
