## 一、"全项目文档数" 统计口径

### 1.1 纳入范围（必须同时满足）

| 条件 | 说明 |
|------|------|
| 文件扩展名 | `.md`（Markdown 文档） |
| 内容属性 | 具有独立知识单元（非纯导航、非空文件） |
| 目录范围 | `docs/`、`concept/`、`数学家理念体系/`、`project/` 四个一级目录 |

### 1.2 排除范围（强制剔除）

| 排除类别 | 典型示例 | 排除原因 |
|----------|----------|----------|
| 根目录报告/归档 | `00-归档/`、`00-FORMALMATH-*.md`、根目录各类"最终报告" | 属于过程性管理文档，非知识内容 |
| 纯导航/索引文件 | `README.md`、`INDEX.md`、`*索引*.md`、`*导航*.md` | 不含独立知识单元 |
| 自动化生成痕迹文件 | `*-生成报告.md`、空占位文件（< 200 字节） | 无法构成有效知识内容 |
| 重复/归档副本 | `*-副本.md`、版本后缀文件（如 `-v1.md`、`-old.md`） | 避免重复计数 |
| 工具/配置文档 | `.github/`、`tools/`、`scripts/`、`deploy/` 下的文档 | 属于工程配置，非数学知识 |

### 1.3 精确计算公式

```powershell
# 诚实文档数 = 四个核心目录下的有效 .md 文件数 - 排除文件数
$ honest_doc_count = (
    (Get-ChildItem -Path "docs","concept","数学家理念体系","project" -Recurse -Filter "*.md" |
     Where-Object { $_.Length -gt 200 } |
     Where-Object { $_.Name -notmatch "README|INDEX|索引|导航|生成报告|副本|v\d+\.md|old\.md" } |
     Where-Object { $_.DirectoryName -notmatch "00-归档|\.github|tools|scripts|deploy" }
    ) | Measure-Object
).Count
```

## 二、"Lean4 定理数" 分级定义

鉴于项目内 `.lean` 文件数量（约 201 个）与实际可编译证明定理数量存在巨大差异，**必须采用三级口径**，严禁混用。

### 2.1 三级口径表

| 级别 | 定义 | 当前数量 | 核查方式 |
|------|------|----------|----------|
| **L1: 完整证明定理** | 在 Lean 4.29.0 + Mathlib4 环境下可编译通过，且包含完整 `theorem` / `lemma` 语句及证明体的独立定理 | **20 个** | 运行 `lake build` 通过，且人工核查 `theorem` 关键字存在完整证明 |
| **L2: 陈述框架级定理** | 以 `theorem` / `lemma` / `def` 形式给出数学陈述或类型签名，但可能缺少完整证明（如 `sorry`、骨架代码、进行中证明） | 约 **120 个** | 脚本扫描 `.lean` 文件中的 `theorem` / `lemma` 声明数量，减去 L1 中已验证的 20 个 |
| **L3: 总 .lean 文件数** | 项目中所有 `.lean` 源代码文件总数（含教程、测试、 lakefile、框架草稿） | **201 个** | `Get-ChildItem -Recurse -Filter "*.lean" | Measure-Object` |

### 2.2 对外披露规范

- **README 徽章/简介**：只能使用 L1 数量，必须标注"可编译验证"。
  - ✅ 正确示例：`Lean4 完整证明定理 20 个（可编译）`
  - ❌ 错误示例：`Lean4 定理 62 个`
- **内部技术报告**：可同时列出三级数据，但必须明确区分 L1/L2/L3。
- **禁止行为**：将 L2（框架陈述）或 L3（文件数）直接称为"定理数"进行对外宣传。

### 2.3 验证脚本

```bash
# L1 验证：必须在 Lean4 项目根目录执行
cd docs/09-形式化证明/Lean4
lake build

# L2/L3 扫描：统计 theorem/lemma 声明数量
grep -r "\btheorem\b\|\blemma\b" --include="*.lean" . | wc -l
```

## 三、"完成度" 计算公式

### 3.1 原则

- **禁止使用无法验证的百分比**（如 99.995%、100%）。
- 若继续使用"完成度"概念，必须采用**多维度加权公式**，且每个维度都须可独立审计。

### 3.2 诚实完成度公式

```
总体完成度 = Σ(维度完成度_i × 权重_i)
```

| 维度 | 权重 | 计算公式 | 当前数值（2026-04） | 数据来源 |
|------|------|----------|---------------------|----------|
| 内容框架 | 20% | `已建目录数 / 计划目录数` | 100% | 目录结构脚本扫描 |
| 基础文档 | 20% | `已发布基础文档数 / 计划基础文档数` | ~85% | `docs/` 核心分支文档统计 |
| 习题/案例 | 15% | `现有习题数 / 2000` | 10% | `docs/00-习题示例反例库/` |
| 形式化证明 | 15% | `L1 定理数 / 50`（目标 50 个） | 40% | Lean4 编译验证 |
| 智能系统代码 | 15% | `已测试功能模块数 / 总规划模块数` | ~50% | 系统测试用例通过数 |
| 国际对齐映射 | 10% | `已逐章映射课程数 / 50` | ~54% | 课程映射文档人工核查 |
| 质量修复 | 5% | `已修复问题数 / (已修复 + 待修复)` | ~30% | Issue/质量报告跟踪 |

**代入计算示例**：

```
总体完成度 = 1.00×0.20 + 0.85×0.20 + 0.10×0.15 + 0.40×0.15 + 0.50×0.15 + 0.54×0.10 + 0.30×0.05
         = 0.20 + 0.17 + 0.015 + 0.06 + 0.075 + 0.054 + 0.015
         = 0.589 ≈ 59%
```

> **注意**：此前报告中 77% 的数值因未包含习题、形式化证明、质量修复等维度的真实进展而被高估。采用上述诚实公式后，当前总体完成度约为 **59%**。

### 3.3 披露规范

- 对外报告完成度时，必须同时披露：
  1. 使用的公式版本号（如 v1.0）；
  2. 各维度具体数值表格；
  3. 最近一次审计日期。
- 禁止单独使用一个百分比数字作为宣传语。

## 四、"质量等级" 定义

### 4.1 原则

- **禁止自评 A+**。
- 质量等级必须基于**可量化的盲审评分**，或通过明确的自动化指标阈值判定。

### 4.2 盲审评分方案（推荐）

每次质量评估需邀请 **至少 3 位未参与文档编写的独立审阅者**（可以是社区贡献者、外部数学学习者或自动化审阅 Agent），按以下量表打分：

| 评分项 | 满分 | 评估标准 |
|--------|------|----------|
| 数学正确性 | 25 | 无概念性错误、定理陈述准确 |
| 结构完整性 | 25 | 文档结构符合模板，逻辑清晰 |
| 引用与溯源 | 20 | 关键定理有出处，参考文献规范 |
| 可读性 | 20 | 语言表达清晰，适合目标读者 |
| 国际对齐度 | 10 | 与权威教材/课程定义一致 |
| **总分** | **100** | — |

### 4.3 等级映射表

| 等级 | 盲审平均分要求 | 最低单项分限制 | 适用说明 |
|------|---------------|----------------|----------|
| **A+** | ≥ 90 | 无单项低于 20 | 极少数深度文档可达到 |
| **A** | 80–89 | 无单项低于 15 | 优质文档 |
| **B+** | 70–79 | 无单项低于 12 | 当前项目平均水平 |
| **B** | 60–69 | 无单项低于 10 | 合格但需改进 |
| **C** | < 60 | — | 需重写或下线 |

### 4.4 自动化质量指标（辅助参考）

在盲审尚未覆盖全部文档前，可采用以下自动化指标作为过渡：

| 指标 | 当前值 | B+ 阈值 | A 阈值 |
|------|--------|---------|--------|
| 文档平均字数 | ~1,900 | ≥ 1,500 | ≥ 2,500 |
| MSC 编码覆盖率（docs 核心分支） | 59.9% | ≥ 50% | ≥ 80% |
| Lean4 编译通过率（L1 定理） | 100%（20/20） | 100% | 100% |
| 课程映射文档有逐章对应的比例 | ~30% | ≥ 20% | ≥ 50% |

> **当前诚实评级**：根据 `00-无限深化推进最新状态.md` 及自动化指标综合判断，项目整体质量处于 **B+ 级**（约 70–76 分区间），**并非 A+**。
