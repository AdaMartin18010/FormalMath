# 课堂验证效果评估指标体系

## 1. 量化指标

### 1.1 学习成效指标

| 指标 | 测量方式 | 目标值 | 数据来源 |
|------|---------|--------|----------|
| 后测平均提升 | (后测均值 - 前测均值) / 前测标准差 | Cohen's d ≥ 0.5 | 前后测试卷 |
| 高分率提升 | 后测得分 ≥ 80 分的学生比例 - 前测对应比例 | ≥ 15% | 前后测试卷 |
| 习题完成率 | 完成并提交配套习题的学生比例 | ≥ 85% | 作业系统 |
| 习题正确率 | 配套习题的平均正确率 | ≥ 70% | 作业批改 |

### 1.2 形式化学习指标

| 指标 | 测量方式 | 目标值 | 数据来源 |
|------|---------|--------|----------|
| Lean4 编译通过率 | 能成功编译至少一个示例的学生比例 | ≥ 60% | 在线 Lean 环境 |
| 双语对照理解度 | 问卷题"能否理解自然语言与 Lean4 的对应关系"平均分 | ≥ 3.5/5 | 学生问卷 |
| 关键 tactic 识别率 | 能正确解释 `apply` / `rw` / `intro` 含义的学生比例 | ≥ 50% | 后测附加题 |

### 1.3 文档质量指标

| 指标 | 测量方式 | 目标值 | 数据来源 |
|------|---------|--------|----------|
| 文档可用性评分 | SUS (System Usability Scale) 改编版平均分 | ≥ 68/100 | 学生问卷 |
| 证明完整性满意度 | "完整证明对学习有帮助"评分 | ≥ 4.0/5 | 学生问卷 |
| 常见错误分析有效性 | "常见错误模式分析帮助我避免错误"评分 | ≥ 3.8/5 | 学生问卷 |

## 2. 质性指标

### 2.1 学生焦点小组访谈提纲

1. 你觉得项目文档中哪些地方比传统教材更好？哪些地方更差？
2. "完整自然语言证明"对你的理解有帮助吗？会不会觉得太长太啰嗦？
3. 双语对照（自然语言 + Lean4）是帮助你理解还是增加了认知负担？
4. 你在使用文档时遇到的最大困难是什么？
5. 如果让你向下一届同学推荐这套文档，你会说什么？会提什么警告？

### 2.2 教师反馈表

1. 您认为本模块文档在数学严格性上是否达到可替代教材的标准？
2. 课堂上学生对双语 Lean4 代码的反应如何？（积极参与 / 困惑 / 漠不关心）
3. 您认为哪些定理或主题最需要补充或重写？
4. 您是否愿意在下一学期继续扩大使用范围？（是/否/视改进情况而定）
5. 其他意见和建议：

## 3. 数据收集时间表

| 时间点 | 活动 | 负责方 |
|--------|------|--------|
| 模块开始前 1 周 | 前测 + 知情同意书签署 | 合作教师 |
| 模块进行中 | 课堂观察 + Lean4 环境日志收集 | 项目组 |
| 模块结束后 1 周 | 后测 + 学生问卷 | 合作教师 |
| 模块结束后 2 周 | 焦点小组访谈（5-8 人） | 项目组 |
| 模块结束后 3 周 | 教师反馈访谈 | 项目组 |
| 模块结束后 4 周 | 数据分析与报告撰写 | 项目组 |

## 4. 成功标准

课堂验证被视为"成功"，当且仅当：

1. 学习成效 Cohen's d ≥ 0.5（中等及以上效应量）
2. 学生文档可用性评分 ≥ 68
3. 至少 70% 的学生愿意在后续课程中继续使用
4. 合作教师愿意在下一学期扩大使用范围
