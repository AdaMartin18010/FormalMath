## 5. 未来建议

### 5.1 新增文档规范

所有新增文档必须遵循以下命名规范：

```
[序号]-[名称]-[版本].md
```

示例：
```
01-群论基础-深度版.md
02-线性代数-完整版.md
course-mapping-mit-18.100a.md
```

### 5.2 逐步改进计划

建议在后续维护中逐步改进：

| 阶段 | 时间 | 行动 | 优先级 |
|------|------|------|--------|
| 1 | 即时 | 新增文档遵循规范 | 🔴 高 |
| 2 | 维护时 | 修改文档时同步重命名 | 🟡 中 |

## 6. 检查清单

### 6.1 新增文档检查项

创建新文档时，请确认：

- [ ] 添加了适当的序号前缀（如 `01-`, `02-`）
- [ ] 名称简洁明确，不超过 10 个中文字符
- [ ] 英文部分使用 kebab-case（小写+连字符）
- [ ] 版本标识清晰（如有）
- [ ] 无特殊字符（仅允许 `-` 和 `.md`）
- [ ] 无空格、无下划线

### 6.2 重命名决策树

```
是否需要重命名?
│
├─ 新增文档? → 必须遵循规范
│
├─ 修改文档? → 建议同时重命名
│
├─ 有外部链接? → 谨慎处理
│
└─ 核心项目文档? → 记录但不修改
```

### 7.2 常见问题

**Q1: 为什么中文日期格式不被强制替换？**
A: 中文日期格式（如 `2026年4月`）在项目中广泛使用，强制替换可能导致：
- 文档内部链接断裂
- 外部引用失效
- 历史记录不一致

**Q2: 子目录内的文档是否需要序号前缀？**
A: 建议添加，但非强制。子目录 README.md 可以无序号前缀。

**Q3: 如何处理已存在的外部链接？**
A: 如果文档已被外部引用，谨慎重命名。如必须重命名，请：
1. 创建重定向说明
2. 更新相关链接
3. 在 CHANGELOG 中记录
