## 六、常见问题处理

### 6.1 部署失败处理

```bash
# 1. 立即停止部署
./deploy.sh maintenance on

# 2. 检查日志
docker-compose logs -f backend
docker-compose logs -f frontend

# 3. 执行回滚
./rollback.sh rollback

# 4. 验证回滚结果
./deploy.sh health
```

### 6.2 健康检查失败处理

```bash
# 检查服务状态
docker-compose ps
kubectl get pods -n formalmath-prod

# 检查资源使用
docker stats
kubectl top pods -n formalmath-prod

# 检查日志
docker logs <container_id>
kubectl logs <pod_name> -n formalmath-prod
```
