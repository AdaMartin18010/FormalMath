## 修复策略详解

### 1. 自动修复规则

#### 1.1 目录链接修复
- **问题:** 链接指向目录(如 `docs/`、`project/`)
- **修复:** 自动映射到对应目录的README.md
- **成功修复:** 606个

#### 1.2 锚点前导连字符修复
- **问题:** 锚点名称以 `-` 开头(如 `#-目录`、`#-概述`)
- **修复:** 自动移除前导连字符
- **成功修复:** 4,807个

#### 1.3 文件路径映射
- **问题:** 旧版术语词典路径
- **修复:** 映射到新的标准术语表
- **成功修复:** 14个

### 2. 需人工处理的链接类型

#### 2.1 文件移动/重命名
- 文件结构调整后，旧链接失效
- 需要人工确认新位置

#### 2.2 复杂锚点问题
- 双连字符锚点
- 特殊字符锚点
- 重复锚点

#### 2.3 跨文件引用
- 需要理解文档间的语义关系
- 无法自动确定正确的目标文件

## 工具使用记录

### link_checker.py
```bash
python tools/link_checker.py .
```

**输出文件:**
- `output/link_check_report.json` - JSON格式详细报告
- `output/link_check_report.txt` - 文本格式报告

### link_fixer.py
```bash
# 干运行模式(预览修复)
python tools/link_fixer.py --quick

# 应用自动修复
python tools/link_fixer.py --apply --quick

# 交互式确认
python tools/link_fixer.py --apply --interactive
```

**输出文件:**
- `output/link_fix_report.md` - 修复报告
- `output/link_fix_report.json` - JSON格式详细报告
- `output/manual_review_report.md` - 需人工处理的问题报告

## 改进建议

### 1. 预防措施

#### 1.1 CI/CD集成
```yaml
# 建议添加到GitHub Actions
- name: Check Links
  run: |
    python tools/link_checker.py .
    if [ $? -ne 0 ]; then
      echo "发现无效链接，请修复后再提交"
      exit 1
    fi
```

#### 1.2 提交前检查
- 使用pre-commit hook自动检查链接
- 在PR模板中添加链接检查提醒
