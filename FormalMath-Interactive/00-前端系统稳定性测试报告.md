## 4. 待解决的问题

### 4.1 TypeScript 类型错误

存在大量类型错误，主要集中在：

1. **模块导入问题**: `@types/*` 路径别名配置不正确
2. **缺少依赖类型**: `react-i18next`, `i18next`, `framer-motion` 等
3. **类型不匹配**: `XRSession`, `XRFrame` 等 WebXR 类型冲突
4. **未使用变量**: 大量声明但未使用的变量和导入

### 4.2 ESLint 警告

- **未使用变量**: 约 100+ 处
- **React Hooks 依赖**: 约 30+ 处 `exhaustive-deps` 警告
- **any 类型**: 多处使用 `any` 类型

### 4.3 缺少的依赖

```json
{
  "framer-motion": "^10.x",
  "react-i18next": "^13.x",
  "i18next": "^23.x",
  "i18next-browser-languagedetector": "^7.x",
  "react-markdown": "^9.x",
  "react-syntax-highlighter": "^15.x",
  "remark-math": "^6.x",
  "rehype-katex": "^7.x"
```

## 5. 性能优化建议

### 5.1 代码分割优化

```javascript
// vite.config.ts
manualChunks: {
  // 现有配置...
  'mermaid-elk': ['mermaid/dist/mermaid.esm.mjs'],
  'three-js': ['three', '@react-three/fiber', '@react-three/drei'],
}
```

### 5.2 动态导入

对于非关键组件，建议使用动态导入：

```typescript
// 将静态导入改为动态导入
const ARViewer = lazy(() => import('./ARVisualization/ARViewer'));
const Graph3D = lazy(() => import('./visualizations/Graph3D'));
```

### 5.3 Tree Shaking

检查并移除未使用的依赖和代码。
