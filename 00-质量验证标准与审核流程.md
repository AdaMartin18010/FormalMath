## 一、质量验证总则

### 1.1 项目质量目标

FormalMath项目致力于建立国际一流的数学知识库，质量目标遵循"准确性、完整性、可用性"三大原则：

| 质量维度 | 目标要求 | 验证方法 |
|---------|---------|---------|
| **准确性** | 数学内容零错误 | 专家审核 + 形式化验证 |
| **完整性** | 知识体系覆盖完整 | 对标国际权威课程 |
| **可用性** | 学习者易于理解 | 用户测试 + 可读性评分 |
| **一致性** | 术语符号统一规范 | 自动化检查 + 人工审查 |
| **可追溯** | 所有结论有出处 | 参考文献核查 |

### 1.2 六级对齐验证标准（L1-L6）

项目建立六级对齐标准，逐级提升内容深度与对齐精度：

| 级别 | 标准名称 | 核心要求 | 验证要点 |
|-----|---------|---------|---------|
| **L1** | 概念存在性 | 概念在文档中有明确定义 | 定义完整、术语准确 |
| **L2** | 概念理解性 | 概念解释清晰，有示例说明 | 示例恰当、易于理解 |
| **L3** | 定义对齐性 | 与权威来源定义等价 | 定义对比、等价性说明 |
| **L4** | 定理对齐性 | 定理陈述与权威来源一致 | 定理对照、条件匹配 |
| **L5** | 证明完备性 | 证明完整严谨或引用可靠 | 证明检查、引用核实 |
| **L6** | 形式化验证 | 有Lean4形式化实现 | 代码编译、与Mathlib4对齐 |

### 1.3 文档质量标准

#### 1.3.1 基础质量标准（所有文档必须满足）

- [ ] 文档包含标准YAML Front Matter（msc_primary, title等）
- [ ] 概念定义准确，无歧义表述
- [ ] 数学公式使用LaTeX格式，渲染正确
- [ ] 术语使用符合项目术语词典规范
- [ ] 无错别字、语法错误
- [ ] 内部链接有效，无断链

#### 1.3.2 增强质量标准（核心概念文档必须满足）

- [ ] 包含至少一个具体示例
- [ ] 包含概念理解三问（是什么、为什么、怎么用）
- [ ] 标注认知层次（记忆/理解/应用/分析/评价/创造）
- [ ] 说明与其他概念的关系
- [ ] 列出常见错误与注意事项

#### 1.3.3 深度质量标准（重要定理与理论文档）

- [ ] 包含历史背景与发展脉络
- [ ] 包含现代应用与拓展
- [ ] 包含与其他分支的关联
- [ ] 包含参考文献与进一步阅读
- [ ] 包含多表征方式（代数/几何/分析视角）

## 二、定理证明审核清单

### 2.1 数学正确性检查点

#### 2.1.1 定理陈述审核

| 检查项 | 检查内容 | 合格标准 |
|-------|---------|---------|
| **条件完备性** | 定理前提条件是否完整 | 所有必要条件已明确列出 |
| **结论准确性** | 定理结论是否准确无误 | 结论与权威来源一致 |
| **符号规范性** | 符号使用是否符合规范 | 与项目术语词典一致 |
| **量词清晰性** | ∀、∃等量词使用是否清晰 | 无歧义，范围明确 |
| **特殊情况** | 是否考虑边界情况 | 边界情况已说明或排除 |

#### 2.1.2 证明过程审核

| 检查项 | 检查内容 | 合格标准 |
|-------|---------|---------|
| **逻辑链条** | 每一步推理是否严密 | 无逻辑跳跃，每步可验证 |
| **引理引用** | 引用的引理是否成立 | 引理已证明或可追溯到已知结果 |
| **归纳基础** | 归纳证明的基础是否成立 | 基础情形已验证 |
| **归纳步骤** | 归纳步骤是否覆盖所有情况 | 归纳假设使用正确 |
| **反设完备** | 反证法的反面假设是否完备 | 考虑了所有可能情况 |
| **构造明确** | 构造性证明是否具体明确 | 构造对象明确可验证 |

### 2.2 逻辑严密性检查点

```
证明逻辑审核流程：
┌─────────────────┐
│  1. 识别证明类型  │  → 直接证明/反证法/归纳法/构造法/对角线法
└────────┬────────┘
         ▼
┌─────────────────┐
│  2. 验证推理规则  │  → 每步使用逻辑规则正确
└────────┬────────┘
         ▼
┌─────────────────┐
│  3. 检查隐含假设  │  → 识别并验证所有隐含前提
└────────┬────────┘
         ▼
┌─────────────────┐
│  4. 确认等价转换  │  → 变形过程保持等价性
└────────┬────────┘
         ▼
┌─────────────────┐
│  5. 边界情况检查  │  → 空集、零、无穷等特殊情况
└─────────────────┘
```

**逻辑严密性检查清单：**

- [ ] 所有变量在使用前已定义
- [ ] 量词顺序正确（∀x∃y vs ∃y∀x）
- [ ] 等号使用正确（= 表示相等，≜ 表示定义）
- [ ] 蕴含方向正确（⇒ 与 ⇔ 区分清楚）
- [ ] 必要性与充分性均已证明（如适用）
- [ ] 无循环论证
- [ ] 无未证明的假设

### 2.3 可读性检查点

| 维度 | 检查项 | 优秀标准 |
|-----|-------|---------|
| **结构** | 证明分段清晰 | 每段有明确主题，段间过渡自然 |
| **动机** | 证明思路说明 | 在关键步骤前说明"为什么要这样做" |
| **标注** | 关键步骤解释 | 复杂步骤有额外解释说明 |
| **符号** | 符号引入说明 | 新符号首次出现时说明含义 |
| **图示** | 辅助可视化 | 几何证明包含图示，代数证明有表格/矩阵 |
| **回顾** | 证明小结 | 长篇证明后有要点回顾 |

**可读性评分标准：**

| 评分 | 描述 | 要求 |
|-----|------|------|
| ⭐⭐⭐⭐⭐ | 优秀 | 结构清晰，动机明确，示例丰富，易于自学 |
| ⭐⭐⭐⭐ | 良好 | 结构清晰，有解释说明，需一定基础 |
| ⭐⭐⭐ | 合格 | 逻辑正确，但解释不足，适合参考 |
| ⭐⭐ | 需改进 | 逻辑跳跃较多，难以理解 |
| ⭐ | 不合格 | 逻辑错误或严重不清晰 |

### 2.4 与Lean4形式化一致性检查

对于包含Lean4形式化的定理：

| 检查项 | 检查内容 | 验证方法 |
|-------|---------|---------|
| **定理对应** | Lean定理与文档定理对应 | 定理名称、陈述一致 |
| **条件匹配** | Lean假设与文档条件匹配 | 逐一对比前提条件 |
| **结论一致** | Lean结论与文档结论一致 | 结论表达式等价 |
| **证明策略** | 文档证明与Lean证明策略对应 | 证明思路一致 |
| **Mathlib引用** | 正确引用Mathlib4 | 引用路径正确，版本匹配 |
| **代码编译** | Lean代码可独立编译 | `lake build` 通过 |

**Lean4代码审核清单：**

- [ ] 代码遵循Mathlib4命名规范
- [ ] 所有定理有docstring文档注释
- [ ] 复杂证明有行内注释
- [ ] 导入语句精简，无冗余依赖
- [ ] 证明风格与Mathlib4一致（tactic使用）
- [ ] 无抱歉（sorry）残留
- [ ] 无警告（warning）产生

## 三、习题质量检查表

### 3.1 难度适宜性

#### 3.1.1 难度分级标准

| 等级 | 标识 | 描述 | 预计解题时间 |
|-----|-----|------|------------|
| **L0** | 🌱 | 基础巩固 | 5-10分钟 |
| **L1** | 🌿 | 概念理解 | 10-15分钟 |
| **L2** | 🍃 | 简单应用 | 15-25分钟 |
| **L3** | 🌳 | 综合分析 | 25-40分钟 |
| **L4** | 🌲 | 深度探索 | 40-60分钟 |
| **L5** | ⭐ | 研究级 | 60分钟以上 |

#### 3.1.2 难度适宜性检查

| 检查项 | 检查内容 | 合格标准 |
|-------|---------|---------|
| **难度标注** | 习题标注难度等级 | 每题有明确难度标识 |
| **难度递进** | 习题难度递进合理 | 由易到难，梯度适中 |
| **前置知识** | 所需前置知识明确 | 与难度等级匹配 |
| **计算量** | 计算量适中 | 不陷入纯计算，考察核心能力 |
| **陷阱设置** | 适当设置易错点 | 考察对概念的理解深度 |

### 3.2 解答完整性

| 检查项 | 检查内容 | 合格标准 |
|-------|---------|---------|
| **解答存在** | 每题都有解答或提示 | 无遗漏 |
| **步骤完整** | 解答步骤不跳跃 | 关键步骤不省略 |
| **最终答案** | 明确给出最终答案 | 答案正确，格式规范 |
| **多种解法** | 重要题目提供多种思路 | 一题多解，拓展思维 |
| **验证方法** | 提供答案验证方法 | 可自检答案正确性 |

**解答质量分级：**

| 等级 | 描述 | 要求 |
|-----|------|------|
| **完整解答** | 详细步骤+最终答案 | 适合自学 |
| **关键提示** | 给出关键步骤提示 | 引导思考，适合练习 |
| **思路点拨** | 给出解题方向 | 适合挑战 |
| **仅答案** | 只给出最终答案 | 适合快速检查 |

### 3.3 教学价值评估

| 评估维度 | 权重 | 评估标准 |
|---------|-----|---------|
| **概念覆盖** | 25% | 覆盖核心概念，无偏废 |
| **技能培养** | 25% | 培养计算/证明/建模能力 |
| **思维训练** | 20% | 训练逻辑/抽象/创新思维 |
| **应用导向** | 15% | 联系实际应用 |
| **进阶铺垫** | 15% | 为后续学习打下基础 |

**教学价值检查清单：**

- [ ] 习题与学习目标对应
- [ ] 避免与已有习题重复
- [ ] 包含不同认知层次的考察
- [ ] 设计开放性问题培养探究能力
- [ ] 融入数学史与数学家思想

### 3.4 常见错误覆盖

| 错误类型 | 覆盖要求 | 示例 |
|---------|---------|------|
| **概念误解** | 必须覆盖 | 混淆"极限存在"与"函数有界" |
| **符号误用** | 建议覆盖 | 混淆∈与⊂ |
| **条件遗漏** | 必须覆盖 | 使用中值定理忽略连续性条件 |
| **范围扩大** | 建议覆盖 | 将局部性质推广到整体 |
| **计算错误** | 适当覆盖 | 级数求和的常见计算错误 |
| **逻辑错误** | 必须覆盖 | 逆命题与原命题混淆 |

**常见错误提示模板：**

```markdown
> ⚠️ **常见错误警示**
> 
> 错误做法：{描述错误做法}
> 
> 错误原因：{解释为什么错}
> 
> 正确做法：{说明正确方法}
```

## 四、对齐准确性验证流程

### 4.1 L3定义对齐验证步骤

**定义对齐性验证流程：**

```
┌─────────────────────────────────────────────────────────────┐
│ Step 1: 识别权威来源                                          │
│ • 优先：数学教科书（Dummit-Foote, Munkres, Rudin等）            │
│ • 其次：权威数学百科（MathWorld, nLab, Wikipedia数学板块）       │
│ • 补充：Mathlib4文档                                           │
└─────────────────────────────────────────────────────────────┘
                              ▼
┌─────────────────────────────────────────────────────────────┐
│ Step 2: 提取标准定义                                          │
│ • 记录权威来源的完整定义陈述                                   │
│ • 标注定义的前提条件、符号约定                                  │
│ • 记录定义的出处（书名、章节、页码/链接）                        │
└─────────────────────────────────────────────────────────────┘
                              ▼
┌─────────────────────────────────────────────────────────────┐
│ Step 3: 对比分析                                              │
│ • 术语等价性：术语是否对应                                     │
│ • 条件完备性：条件是否一致                                     │
│ • 表述差异：表述差异是否本质                                   │
└─────────────────────────────────────────────────────────────┘
                              ▼
┌─────────────────────────────────────────────────────────────┐
│ Step 4: 等价性判定                                            │
│ • 严格等价：定义完全等价，可直接引用                           │
│ • 实质等价：表述不同但数学内涵一致                             │
│ • 条件差异：有额外条件或限制，需说明                           │
│ • 定义差异：实质不同，需讨论                                   │
└─────────────────────────────────────────────────────────────┘
                              ▼
┌─────────────────────────────────────────────────────────────┐
│ Step 5: 文档记录                                              │
│ • 在对齐报告中记录验证过程                                     │
│ • 更新概念映射表（如适用）                                     │
│ • 标注对齐等级（L3a严格/L3b实质/L3c条件差异）                   │
└─────────────────────────────────────────────────────────────┘
```

**L3定义对齐验证清单：**

- [ ] 已确定至少一个权威参考来源
- [ ] 已提取权威来源的完整定义
- [ ] 已对比术语使用的等价性
- [ ] 已对比条件假设的完备性
- [ ] 已判定等价性等级
- [ ] 已在文档中记录对齐信息
- [ ] 已更新YAML Front Matter对齐标签

### 4.3 等价性证明要求

对于标注为"实质等价"的定义或定理，需要满足：

| 等价性类型 | 证明要求 | 文档记录 |
|-----------|---------|---------|
| **双向蕴含** | 证明A⇒B和B⇒A | 给出简要证明或引用 |
| **同构等价** | 证明存在规范同构 | 说明同构映射 |
| **嵌入等价** | 证明可互相嵌入 | 说明嵌入方式 |
| **公理等价** | 证明在公理体系中等价 | 引用公理系统 |

**等价性证明检查清单：**

- [ ] 等价性声称有数学依据
- [ ] 证明思路清晰，无逻辑漏洞
- [ ] 关键步骤有解释说明
- [ ] 已考虑边界情况
- [ ] 证明结论明确

### 4.4 参考文献核查

**参考文献质量要求：**

| 检查项 | 要求 | 示例 |
|-------|------|------|
| **权威性** | 优先引用权威教材和期刊 | Dummit-Foote, Annals of Math等 |
| **可获取性** | 提供可访问的链接或DOI | arXiv, JSTOR, 出版社官网 |
| **完整性** | 包含作者、标题、出处、年份 | 符合学术引用规范 |
| **时效性** | 优先引用较新但经典的版本 | FOAG 2025版, Mathlib4最新版 |
| **一致性** | 引用格式统一 | 遵循项目引用规范 |

**参考文献核查清单：**

- [ ] 所有引用都有出处
- [ ] 引用链接可正常访问
- [ ] 引用内容与文档内容对应
- [ ] 引用格式统一规范
- [ ] 优先使用Mathlib4等开放资源

### 5.2 同行评审流程

**同行评审阶段：**

| 阶段 | 评审内容 | 评审人 | 输出 |
|-----|---------|-------|------|
| **初审** | 文档规范性、基础正确性 | 项目负责人 | 初审报告 |
| **数学审** | 数学内容准确性 | 数学专家 | 数学审核意见 |
| **对齐审** | 与权威来源对齐性 | 对齐专员 | 对齐验证报告 |
| **可读审** | 可读性与教学价值 | 教育专家 | 可读性评分 |

**同行评审检查表：**

| 评审维度 | 评审要点 | 评分 |
|---------|---------|------|
| **数学准确性** | 定义、定理、证明正确 | 1-5分 |
| **逻辑严密性** | 推理严密，无逻辑漏洞 | 1-5分 |
| **对齐准确性** | 与权威来源对齐 | 1-5分 |
| **可读性** | 易于理解，结构清晰 | 1-5分 |
| **教学价值** | 对学习者有帮助 | 1-5分 |
| **规范性** | 符合文档规范 | 1-5分 |

**评审意见处理流程：**

```
┌─────────────────┐
│   接收评审意见   │
└────────┬────────┘
         ▼
┌─────────────────┐     ┌─────────────────┐
│  分类整理意见    │────→│  严重问题（数学错误）│
└────────┬────────┘     │  → 必须修正       │
         │               └─────────────────┘
         │               ┌─────────────────┐
         ├──────────────→│  中等问题（改进建议）│
         │               │  → 建议修正       │
         │               └─────────────────┘
         │               ┌─────────────────┐
         └──────────────→│  轻微问题（格式优化）│
                         │  → 可选修正       │
                         └─────────────────┘
```

### 5.3 专家终审标准

**终审专家资质要求：**

| 审核级别 | 专家资质 | 适用内容 |
|---------|---------|---------|
| **A+级终审** | 数学教授/研究员，相关领域专家 | 核心定理、前沿内容 |
| **A级终审** | 数学博士，教学经验5年以上 | 重要概念、主要定理 |
| **B级终审** | 数学硕士，教学经验3年以上 | 一般内容、基础概念 |

**专家终审检查清单：**

- [ ] 数学内容经专业验证无误
- [ ] 与权威来源对齐性确认
- [ ] 教学价值评估通过
- [ ] 可读性达到标准
- [ ] 文档规范性符合要求
- [ ] 形式化代码编译通过（如适用）

**终审结论类型：**

| 结论 | 含义 | 后续操作 |
|-----|------|---------|
| **通过** | 内容质量达标 | 正式发布 |
| **有条件通过** | 需小修改 | 修改后重新审核 |
| **退回修改** | 需重大修改 | 修改后重新走完整流程 |
| **拒绝** | 内容不达标 | 重新撰写 |

### 5.4 问题反馈与修正机制

**问题反馈流程：**

```
问题发现
    │
    ├──→ 读者反馈 ──→ 问题记录 ──→ 分类评估 ──→ 分配处理
    │                                      │
    └──→ 审核发现 ──→ 审核报告 ──→─────────┤
                                           │
                              ┌────────────┼────────────┐
                              ▼            ▼            ▼
                           紧急修正      常规修正      计划修正
                           (24小时内)    (1周内)       (下次迭代)
```

**问题严重等级：**

| 等级 | 描述 | 响应时间 | 示例 |
|-----|------|---------|------|
| **P0-紧急** | 数学错误，严重误导 | 24小时内 | 定理条件错误导致结论不成立 |
| **P1-高** | 重要内容缺失或不清晰 | 1周内 | 关键证明步骤跳跃 |
| **P2-中** | 一般改进建议 | 下次迭代 | 增加更多示例 |
| **P3-低** | 格式或风格问题 | 批量处理 | 标点符号统一 |

**修正验证要求：**

| 修正类型 | 验证要求 |
|---------|---------|
| **数学修正** | 重新数学审核 |
| **对齐修正** | 重新对齐验证 |
| **格式修正** | 格式检查即可 |
| **内容补充** | 完整审核流程 |

## 六、质量等级评定

### 6.1 A+级标准

**A+级质量要求（卓越）：**

| 维度 | 要求 | 评分标准 |
|-----|------|---------|
| **数学准确性** | 数学内容完全正确，无瑕疵 | 5/5 |
| **L6形式化** | 有完整的Lean4形式化证明 | 完成 |
| **L4对齐** | 与权威来源严格对齐 | 100% |
| **完整性** | 深度版质量标准全部满足 | 100% |
| **可读性** | ⭐⭐⭐⭐⭐ 优秀 | 自学无障碍 |
| **教学价值** | 具有示范意义 | 可作为标杆 |

**A+级内容示例：**
- 格洛腾迪克理念体系文档
- 已完成Lean4形式化的核心定理
- 与FOAG严格对齐的代数几何内容

### 6.2 A级标准

**A级质量要求（优秀）：**

| 维度 | 要求 | 评分标准 |
|-----|------|---------|
| **数学准确性** | 数学内容正确 | 4-5/5 |
| **L4对齐** | 与权威来源实质对齐 | ≥90% |
| **完整性** | 增强版质量标准全部满足 | 100% |
| **可读性** | ⭐⭐⭐⭐ 良好 | 适合参考学习 |
| **教学价值** | 对学习者有显著帮助 | 高 |

**A级内容示例：**
- 核心数学分支的主要概念文档
- 重要定理的完整证明
- 高质量的工作示例

### 6.3 B级标准

**B级质量要求（良好）：**

| 维度 | 要求 | 评分标准 |
|-----|------|---------|
| **数学准确性** | 数学内容基本正确 | 3-4/5 |
| **L3对齐** | 与权威来源定义对齐 | ≥80% |
| **完整性** | 基础版质量标准满足 | 100% |
| **可读性** | ⭐⭐⭐ 合格 | 需要一定基础 |
| **教学价值** | 对学习有帮助 | 中 |

**B级内容示例：**
- 一般概念文档
- 基础示例与习题
- 需要进一步深化的内容

### 6.4 不合格标准

**不合格判定标准：**

| 问题类型 | 判定标准 | 处理措施 |
|---------|---------|---------|
| **数学错误** | 定义、定理或证明存在错误 | 立即下架，修正后重新审核 |
| **严重不对齐** | 与权威来源定义/定理不一致 | 退回修改，重新对齐验证 |
| **逻辑缺陷** | 证明存在逻辑漏洞 | 退回修改，重新数学审核 |
| **基础缺失** | 缺少必要的基础内容 | 补充后重新审核 |
| **格式混乱** | 严重不符合文档规范 | 格式修正后重新审核 |

**不合格内容处理流程：**

```
判定不合格
    │
    ▼
┌─────────────────┐
│  标记问题类型    │
└────────┬────────┘
         ▼
┌─────────────────┐
│  从公开索引移除  │
└────────┬────────┘
         ▼
┌─────────────────┐
│  通知作者修正    │
└────────┬────────┘
         ▼
┌─────────────────┐
│  修正后重新审核  │
└────────┬────────┘
         ▼
┌─────────────────┐
│  通过则恢复发布  │
└─────────────────┘
```
