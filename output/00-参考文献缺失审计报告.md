## 一、审计背景与目的

项目被批评缺少严格的参考文献系统：无 DOI、zbMATH、MathSciNet 引用，教材引用停留在书名级别。本次抽样审计旨在量化评估核心知识目录的参考文献现状，识别薄弱环节，并提出可执行的修复建议。

## 六、修复建议

### 6.1 立即行动项（短期）

1. **清理空壳与错位文档**
   - 删除或归档字数低于 100 且无任何实质内容的占位文档（如 `张量分析-基础.md`）。
   - 将项目执行进度报告、任务清单等管理文档迁移至 `project/` 或 `00-归档/` 目录。

2. **强制习题文档标注来源**
   - 所有习题类文档（尤其是代数几何、分析学分支）必须在 YAML frontmatter 中补充以下字段：
     - `source_textbook`
     - `source_chapter`
     - `source_exercise`（如 `Hartshorne II.6.2`）

### 6.2 中期规范项

1. **引入 `references` YAML 字段模板**
   建议在所有核心文档头部统一加入如下元数据模板：

   ```yaml
   references:
     - type: textbook
       citation: "Hartshorne, R. Algebraic Geometry. Springer, 1977."
       detail: "Ch. II, §6, Ex. 6.2, pp. 138-139"
       isbn: "978-0-387-90244-9"
     - type: paper
       citation: "Grothendieck, A. Éléments de géométrie algébrique."
       journal: "Publ. Math. IHÉS"
       volume: 4
       pages: "5-228"
       year: 1960
       doi: "10.1007/BF02684778"      # 如有
       zbmath: "zbMATH 0118.36206"    # 如有
       mrnumber: "MR0217083"          # 如有
     - type: arxiv
       arxiv_id: "arXiv:2401.12345"
   ```

2. **建立历史/传记文档的最低引用标准**
   - 所有涉及数学家生平、学术贡献、历史评价的文档，必须至少提供一部权威传记或综述论文的引用（如 *The Princeton Companion to Mathematics*、 zbMATH 上的传记条目）。

### 6.3 长期战略项

1. **对接国际数学数据库**
   - 在文档生成/导出流程中，增加 DOI、zbMATH、MathSciNet 链接的自动补全脚本（通过 API 根据论文标题或作者查询）。
   - 在 Web 前端为引用添加跳转链接（如 `https://zbmath.org/?q=an:`、`https://mathscinet.ams.org/mathscinet-getitem?mr=`）。

2. **纳入 CI 质量门禁**
   - 在文档合并检查（CI）中加入引用质量扫描：
     - 核心理论文档（字数 > 500）必须至少含有一个精确引用或明确的 `references` 字段；
     - 禁止新提交的文档出现 0 引用且字数 > 300 的情况。

## 七、结论

本次对 100 篇文档的抽样审计显示：

- **亮点**：项目约 **68%** 的核心文档已具备某种形式的精确引用（主要是教材章节/习题号），其中代数几何分支表现尤为突出（87%）。
- **严重缺陷**：**DOI、zbMATH、MathSciNet、arXiv ID 的检出率为 0%**；分析学和数学家理念体系中的庞加莱分支存在大量无引用或仅人名级别的文档；空壳文档与错位文档仍占有一定比例。
- **总体判断**：项目目前处于“**有课程对齐、无学术引用体系**”的状态。教材引用多停留在“书名+章节”级别，缺乏期刊论文的精确 bibliographic 信息，尚未达到可公开发表或供学术研究追溯的参考文献标准。

建议优先执行空壳文档清理、YAML `references` 模板落地、以及 CI 引用质量门禁三项措施，以在 1-2 个迭代周期内将“完全无引用”比例从 11% 降至 3% 以下，并将 DOI/zbMATH 等数据库引用从 0% 提升至可见水平。
