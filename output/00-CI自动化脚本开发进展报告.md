## 2. 已交付脚本清单

### 2.1 质量门禁检查脚本

**文件**：`scripts/ci_quality_gate.py`

**功能**：
1. **禁止词汇检查**：扫描 `README.md`、`INDEX.md` 等入口文档，检测 `"100% 完成"`、`"99.995%"`、`"A+ 质量"`、`"世界级标准"` 等虚假宣传用语。
2. **参考文献字段检查**：扫描 `docs/`、`concept/`、`数学家理念体系/` 下的所有 `.md` 文件，强制要求 YAML frontmatter 中包含 `references` 字段。
3. **Lean4 空文件检查**：扫描 `docs/09-形式化证明/Lean4/`，识别 0 字节的 `.lean` 文件。
4. **空壳文档检查**：扫描 `docs/`、`concept/`、`数学家理念体系/`，识别大小低于 200 字节的 `.md` 文件。

**运行方式**：

```bash
python scripts/ci_quality_gate.py
python scripts/ci_quality_gate.py -o report.json -v
```

**输出示例**：

```json
{
  "status": "passed",
  "total_violations": 0,
  "violations": [],
  "summary": {
    "entry_docs_checked": 3,
    "forbidden_word_violations": 0,
    "references_field_violations": 0,
    "empty_lean_file_violations": 0,
    "shell_document_violations": 0
  }
}
```

### 2.2 仪表盘自动更新脚本

**文件**：`scripts/ci_update_dashboard.py`

**功能**：
1. 递归统计全项目 `.md` 文件总数（排除 `.git`、`node_modules` 等目录）。
2. 递归统计 `docs/` 目录下的 `.md` 文件数。
3. 递归统计全项目 `.lean` 文件总数。
4. 自动更新 `output/00-质量仪表盘.md` 中"项目诚实状态看板"的对应行，并将最后更新日期设为当天。

**运行方式**：

```bash
# 实际更新仪表盘
python scripts/ci_update_dashboard.py

# 仅预览，不写入文件
python scripts/ci_update_dashboard.py --dry-run
```

### 2.3 仪表盘 HTML 渲染脚本

**文件**：`scripts/render_dashboard_html.py`

**功能**：
- 仅使用 Python 标准库，通过逐行解析将 Markdown 转换为 HTML。
- 支持标题、表格、引用块、列表、代码块、水平线、行内加粗/斜体/代码/链接等常见元素。
- 生成内嵌 CSS 的单页 HTML 文件 `output/quality-dashboard.html`，表格样式清晰可读。
- 页面顶部包含"诚实状态声明"横幅。

**运行方式**：

```bash
python scripts/render_dashboard_html.py
python scripts/render_dashboard_html.py -i output/00-质量仪表盘.md -o output/quality-dashboard.html
```

## 3. GitHub Actions 工作流

**文件**：`.github/workflows/quality-gate.yml`

**触发条件**：
- `push` 到 `main` 分支
- `pull_request` 到 `main` 分支

**执行步骤**：
1. 检出代码。
2. 设置 Python 3.11 环境。
3. 运行 `scripts/ci_quality_gate.py`，将 JSON 报告输出并上传为 Artifact。
4. 运行 `scripts/ci_update_dashboard.py --dry-run`，验证仪表盘更新逻辑是否正常。

**Artifact**：`quality-gate-report.json` 会在每次运行后保留，便于追溯历史门禁结果。

## 4. 设计决策与技术细节

### 4.1 跨平台兼容性

所有脚本均使用 `pathlib.Path` 处理文件路径，避免 Windows 与 Linux 下的路径分隔符差异。GitHub Actions 工作流使用 `ubuntu-latest` 运行 CI，但脚本本身在 Windows 本地亦可直接执行。

### 4.2 仅使用标准库

三个 Python 脚本均不依赖第三方包，确保在任何具备 Python 3.9+ 的环境中即可运行，无需额外的 `pip install` 步骤。

### 4.3 忽略目录策略

为避免扫描构建产物和依赖目录，脚本统一忽略以下目录：

- `.git`
- `node_modules`
- `__pycache__`
- `.pytest_cache`
- `dist`
- `build`
- `00-归档`

### 4.4 友好命令行接口

每个脚本均内置 `--help` 参数，输出清晰的使用说明：

```bash
python scripts/ci_quality_gate.py --help
python scripts/ci_update_dashboard.py --help
python scripts/render_dashboard_html.py --help
```
