## 我们的承诺

作为 FormalMath 项目的成员、贡献者和领导者，我们承诺为每个人创造一个无骚扰的参与体验，无论其年龄、体型、可见或不可见的残疾、种族、性特征、性别认同与表达、经验水平、教育背景、社会经济地位、国籍、个人外貌、种族、宗教或性取向如何。

我们承诺以开放、欢迎、多元、包容和健康的方式行动和互动，为数学知识的传播和学习创造一个积极的环境。

## 我们的标准

### ✅ 鼓励的行为

- **尊重他人**：尊重不同的观点和经验
- **接受建设性批评**：优雅地接受批评，并从中学习
- **关注社区利益**：将社区最佳利益置于个人利益之上
- **展现同理心**：理解其他社区成员可能面临不同的挑战
- **帮助他人**：积极帮助新手入门，耐心解答问题
- **保持专业**：在讨论中保持专业和学术态度
- **准确传播知识**：确保数学内容的准确性，避免传播错误信息

### ❌ 不可接受的行为

- **骚扰行为**：使用性化语言或图像，以及任何形式的性骚扰
- **攻击行为**：挑衅、侮辱或贬损性评论，以及个人或政治攻击
- **公开或私下骚扰**：包括但不限于不当的物理接触、跟踪或不受欢迎的注意
- **破坏行为**：发布他人私人信息（如物理地址或电子邮件）未经明确许可
- **不专业行为**：在学术讨论中进行人身攻击而非针对论点
- **歧视行为**：基于个人特征的歧视或贬低
- **破坏协作**：故意破坏项目协作或社区和谐的行为

## 责任范围

本行为准则适用于所有项目空间，包括：

- GitHub 仓库（Issues、Pull Requests、Discussions）
- 项目文档和代码注释
- 项目相关的公共或私人通信
n- 项目相关的线上或线下活动

此外，当个人在公共空间中代表项目或其社区时，本行为准则也适用。代表项目的例子包括：

- 使用官方项目电子邮件地址
- 通过官方社交媒体账户发布内容
- 作为指定代表参加线上或线下活动

## 执行责任

项目维护者负责明确可接受行为的标准，并应对任何不可接受行为采取适当和公平的纠正措施。

项目维护者有权利和责任：
- 删除、编辑或拒绝不符合本行为准则的评论、提交、代码、Wiki 编辑、Issues 和其他贡献
- 暂时或永久禁止任何贡献者的行为，如果他们认为其行为不适当、威胁、冒犯或有害

## 违规处理流程

### 举报流程

如果您目睹或经历了违反本行为准则的行为，请通过以下方式举报：

1. **GitHub 私信**：联系项目维护者
2. **电子邮件**：发送邮件至 formalmath-conduct@example.com（示例地址）

举报内容应包括：
- 您的联系信息
- 违规行为的描述
- 发生的时间和地点
- 任何相关的上下文或证据

### 处理流程

1. **确认收到**：维护者将在 48 小时内确认收到举报
2. **调查**：维护者将调查举报内容，可能需要与相关方沟通
3. **决定**：基于调查结果，维护者将做出决定
4. **执行**：采取适当的措施
5. **反馈**：向举报者反馈处理结果（在保护隐私的前提下）

### 可能的措施

根据违规的严重程度，可能采取的措施包括：

| 措施 | 适用情况 |
|------|----------|
| **警告** | 轻微违规，首次发生 |
| **要求道歉** | 影响他人的言语不当 |
| **临时禁言** | 重复违规或较严重违规（1-30天） |
| **永久禁言** | 严重违规或多次违规 |
| **内容删除** | 不当内容 |

### 申诉流程

如果您对处理结果有异议，可以在收到决定后的 7 天内提出申诉：

1. 发送申诉邮件至指定邮箱
2. 说明申诉理由
3. 提供新的证据（如有）

维护者将在 14 天内回复申诉结果。

## 指导原则

### 学术讨论准则

在 FormalMath 项目中，学术讨论是核心活动。我们遵循以下原则：

1. **对事不对人**：批评应针对观点和内容，而非个人
2. **数学严谨性**：鼓励质疑和验证，但需基于数学原理
3. **引用来源**：重要论断应提供可靠的参考来源
4. **承认错误**：发现错误时勇于承认并修正
5. **终身学习**：保持谦逊，承认每个人都有学习和成长的空间

### 新手友好原则

- **耐心解答**：对基础问题保持耐心
- **鼓励参与**：鼓励新手参与贡献，无论贡献大小
- **提供指导**：主动提供学习资源和指导
- **避免精英主义**：不因知识水平而区别对待

### 多语言包容

项目支持中英文内容，我们：

- 尊重语言偏好，不强迫使用特定语言
- 鼓励在能力范围内进行翻译贡献
- 提供多语言学习资源
