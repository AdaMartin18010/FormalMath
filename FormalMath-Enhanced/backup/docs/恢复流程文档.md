# FormalMath 数据恢复流程文档

## 概述

本文档描述了 FormalMath 系统的数据恢复流程，包括从备份文件恢复数据库、配置文件和数据的操作步骤。

## 前提条件

- PowerShell 5.1 或更高版本
- 备份文件（.zip 格式）
- 系统管理员权限（某些操作需要）

## 恢复类型

### 1. 完整恢复 (Full Restore)

恢复所有数据类型：数据库、配置文件、数据目录、知识图谱。

```powershell
# 交互模式（有确认提示）
.\scripts\restore.ps1 -BackupFile "backup_full_20250401_120000.zip"

# 强制模式（无确认提示）
.\scripts\restore.ps1 -BackupFile "backup_full_20250401_120000.zip" -Force
```

### 2. 仅恢复数据库 (Database Only)

仅恢复数据库文件，保留现有配置。

```powershell
.\scripts\restore.ps1 -BackupFile "backup_full_20250401_120000.zip" -RestoreType db
```

### 3. 仅恢复配置 (Config Only)

仅恢复配置文件。

```powershell
.\scripts\restore.ps1 -BackupFile "backup_full_20250401_120000.zip" -RestoreType config
```

### 4. 仅恢复数据目录 (Data Only)

仅恢复数据目录。

```powershell
.\scripts\restore.ps1 -BackupFile "backup_full_20250401_120000.zip" -RestoreType data
```

### 5. 仅验证 (Verify Only)

不执行恢复，仅验证备份文件完整性。

```powershell
.\scripts\restore.ps1 -BackupFile "backup_full_20250401_120000.zip" -VerifyOnly
```

### 6. 演示模式 (Dry Run)

显示将要执行的操作，但不实际执行。

```powershell
.\scripts\restore.ps1 -BackupFile "backup_full_20250401_120000.zip" -DryRun
```

## 标准恢复流程

### 步骤 1: 确认备份文件

```powershell
# 列出可用的备份文件
Get-ChildItem -Path "G:\_src\FormalMath\FormalMath-Enhanced\backup\archive" -Filter "backup_*.zip" |
    Sort-Object LastWriteTime -Descending |
    Select-Object Name, LastWriteTime, @{N="Size(MB)";E={[math]::Round($_.Length/1MB,2)}}
```

### 步骤 2: 验证备份文件

```powershell
# 验证备份完整性
.\scripts\verify_backup.ps1 -BackupFile "backup_full_20250401_120000.zip" -Detailed
```

### 步骤 3: 停止相关服务

在恢复前，确保停止所有相关服务：

```powershell
# 查找并停止 Python/Uvicorn 进程
Get-Process -Name "python" | Where-Object { $_.CommandLine -like "*uvicorn*" } | Stop-Process -Force
```

### 步骤 4: 执行恢复

```powershell
# 执行完整恢复
.\scripts\restore.ps1 -BackupFile "backup_full_20250401_120000.zip" -Force
```

### 步骤 5: 验证恢复结果

```powershell
# 检查数据库文件
Test-Path "G:\_src\FormalMath\FormalMath-Enhanced\api\formalmath.db"

# 检查关键配置
Test-Path "G:\_src\FormalMath\FormalMath-Enhanced\api\app\core\config.py"
```

### 步骤 6: 重启服务

根据系统部署方式，重启相关服务。

## 常见恢复场景

### 场景 1: 数据库损坏

```powershell
# 1. 停止服务
# 2. 仅恢复数据库
.\scripts\restore.ps1 -BackupFile "backup_full_20250401_120000.zip" -RestoreType db -Force
# 3. 重启服务
```

### 场景 2: 配置错误

```powershell
# 仅恢复配置文件
.\scripts\restore.ps1 -BackupFile "backup_full_20250401_120000.zip" -RestoreType config -Force
```

### 场景 3: 误删除数据

```powershell
# 恢复所有数据
.\scripts\restore.ps1 -BackupFile "backup_full_20250401_120000.zip" -RestoreType data -Force
```

### 场景 4: 迁移到新服务器

```powershell
# 1. 在新服务器上安装依赖
# 2. 完整恢复
.\scripts\restore.ps1 -BackupFile "backup_full_20250401_120000.zip" -RestoreType full -Force
# 3. 更新配置中的路径和连接信息
```

## 故障排除

### 问题 1: 权限不足

**症状**: 无法写入目标目录

**解决**: 以管理员身份运行 PowerShell

```powershell
# 以管理员身份运行
Start-Process PowerShell -Verb RunAs
```

### 问题 2: 文件被占用

**症状**: 无法覆盖现有文件

**解决**: 停止相关服务后再恢复

```powershell
# 停止所有相关进程
Get-Process -Name "python" | Stop-Process -Force
```

### 问题 3: 备份文件损坏

**症状**: 无法解压或验证失败

**解决**: 尝试使用其他备份文件

```powershell
# 验证所有备份
.\scripts\verify_backup.ps1 -CheckAll -Detailed
```

### 问题 4: 恢复后应用无法启动

**症状**: 应用启动失败或报错

**解决步骤**:

1. 检查日志文件
2. 验证数据库完整性
3. 检查配置文件格式
4. 确认依赖项已安装

## 注意事项

1. **恢复前备份**: 恢复操作会自动备份当前数据到 `pre_restore_<timestamp>` 目录
2. **不可逆操作**: 恢复会覆盖现有数据，请谨慎操作
3. **服务停止**: 恢复前确保相关服务已停止
4. **验证完整性**: 恢复后务必验证数据完整性
