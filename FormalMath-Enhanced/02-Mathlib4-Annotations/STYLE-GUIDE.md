# Mathlib4 教育注释风格指南

## 写作原则

### 1. 受众导向

- **目标读者**：数学专业高年级本科生、研究生及对形式化数学感兴趣的研究者
- **假设**：读者具备基础数学知识，但不一定熟悉 Lean 4

### 2. 清晰性与准确性

- 使用准确但通俗的语言解释数学概念
- 避免过度技术化，除非必要
- 每个数学术语首次出现时应给出定义或解释

### 3. 结构一致性

所有注释必须遵循统一结构，详见 [TEMPLATE.md](TEMPLATE.md)。

## 格式规范

### Markdown 规范

- 使用 ATX 标题格式（`#` 而非 `=`）
- 代码块标注语言（`lean` 用于 Lean 代码，`python` 用于 Python 示例等）
- 数学公式使用 LaTeX 格式（`$...$` 或 `$$...$$`）
- 使用 `-` 而非 `*` 表示无序列表

### 数学符号规范

| 含义 | 符号 | LaTeX |
|------|------|-------|
| 属于 | $\in$ | `\in` |
| 子集 | $\subseteq$ | `\subseteq` |
| 映射 | $f: A \to B$ | `f: A \to B` |
| 极限 | $\lim$ | `\lim` |
| 积分 | $\int$ | `\int` |
| 求和 | $\sum$ | `\sum` |
| 乘积 | $\prod$ | `\prod` |

### Lean 代码规范

```lean
-- 代码应有适当缩进（2个空格）
-- 关键步骤应有注释
-- 复杂表达式应分行
example (n : ℕ) : n + 0 = n := by
  simp  -- 使用 simp 简化
```

## 内容规范

### 数学背景

- 说明定理发现的历史背景
- 解释定理在数学体系中的地位
- 提及主要贡献者及其工作

**长度**：100-200 字

### 直观解释

- 使用类比和可视化描述
- 避免形式化符号
- 强调定理的"为什么"和"是什么"

**长度**：150-250 字

### 证明思路

- 概述关键步骤，非完整证明
- 解释核心洞察
- 指出技术难点

**长度**：100-200 字

### 示例

- 提供具体数值示例
- 展示定理的应用
- 如有必要，包含计算过程

**长度**：2-3 个示例

### 相关概念

- 列出 3-5 个相关概念或定理
- 使用链接形式（如 `群 (Group)`）

### 参考

- 优先引用权威教材
- 包含原始论文（如有）
- 可包含网络资源

格式：

```markdown
- [作者. 书名. 出版社, 年份. 章节]
- [作者. 论文标题. 期刊, 年份.]
```

## 语言风格

### 中文优先

- 主要使用简体中文
- 专有名词保留英文（如 Lean 4, Mathlib4）
- 数学术语使用标准翻译

### 时态与人称

- 使用现在时态
- 避免使用第一人称（"我"、"我们"）
- 使用被动语态或无人称句式

### 示例

✅ **推荐**：
> 拉格朗日定理指出，有限群的子群的阶整除群的阶。

❌ **不推荐**：
> 我认为拉格朗日定理告诉我们子群的阶能整除群的阶。

## 审核清单

提交注释前，请确认：

- [ ] 遵循模板结构
- [ ] 数学公式正确渲染
- [ ] Lean 代码可编译（或来自 Mathlib4 实际代码）
- [ ] 无拼写和语法错误
- [ ] 引用格式正确
- [ ] 链接有效

## 版本控制

- 文件名使用英文，空格用连字符替代
- 版本号遵循语义化版本（如 v1.0.0）
- 重大修改需更新版本号并记录变更日志
