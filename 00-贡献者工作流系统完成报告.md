## 🎯 核心功能

### 1. 贡献指南体系

#### 多维度贡献支持
- **内容贡献**：文档创建、完善、翻译
- **代码贡献**：Lean4 形式化、Python 工具、Web 开发
- **维护贡献**：Issue 分类、PR 审核、质量检查

#### 详细的规范文档
- 文档命名规范
- 内容结构标准（四层版本体系）
- 数学表达规范（LaTeX、符号标准）
- 元数据标准（MSC2020、L0-L4 知识层次）

### 2. 自动化检查体系

#### 多层次检查
```
预提交检查（本地）
    ↓
CI 自动检查（PR 时）
    ↓
定期质量检查（每周）
```

#### 检查项目
- ✅ Markdown 格式规范
- ✅ YAML Frontmatter 完整性
- ✅ MSC2020 编码有效性
- ✅ 数学公式语法
- ✅ 内部链接有效性
- ✅ Lean4 代码编译

### 3. 奖励机制

#### 积分系统
| 贡献类型 | 积分 | 额外奖励 |
|----------|------|----------|
| 新文档（基础版） | 50 | +20（完整证明） |
| 新文档（增强版） | 100 | +30（应用案例） |
| 新文档（深度版） | 200 | +50（形式化） |
| Lean4 定理证明 | 300 | +100（通过验证） |
| 错误修正 | 20 | +10（测试用例） |
| 翻译（每千字） | 50 | +20（专业领域） |

#### 等级体系
- ⭐ 新成员 (0-99)
- ⭐⭐ 贡献者 (100-499)
- ⭐⭐⭐ 活跃贡献者 (500-1999)
- ⭐⭐⭐⭐ 核心贡献者 (2000-4999)
- ⭐⭐⭐⭐⭐ 项目维护者 (5000+)

#### 徽章系统
- 🏅 首次贡献
- 📚 文档大师（50+ 文档）
- 🔬 形式化专家（10+ Lean4 证明）
- 🌐 翻译专家（5+ 万字翻译）
- 🐛 捉虫能手（20+ 错误修正）
- 👁️ 审核专家（50+ PR 审核）

### 4. 质量控制体系

#### 三级审核
1. **自动化检查**：格式、语法、链接
2. **同行审核**：数学准确性、内容完整性
3. **维护者审核**：最终把关、合并决策

#### 问题分类
- 🔴 P0：数学概念根本性错误（24小时内响应）
- 🟠 P1：影响理解的关键错误（3天内响应）
- 🟡 P2：一般性问题（1周内响应）
- 🟢 P3：改进建议（按需处理）

## 🚀 使用指南

### 新贡献者入门

1. **阅读贡献指南**
   ```bash
   cat CONTRIBUTING.md
   ```

2. **Fork 项目并克隆**
   ```bash
   git clone https://github.com/YOUR_USERNAME/FormalMath.git
   cd FormalMath
   ```

3. **创建功能分支**
   ```bash
   git checkout -b feature/my-contribution
   ```

4. **运行预提交检查**
   ```bash
   python tools/contributor-workflow/pre_commit_check.py
   ```

5. **提交并创建 PR**
   ```bash
   git add .
   git commit -m "docs(algebra): add new theorem"
   git push origin feature/my-contribution
   ```

### 维护者操作

1. **合并 PR 后自动计算积分**
   - 由 `contributor-rewards.yml` 自动处理

2. **查看贡献者排行榜**
   ```bash
   cat CONTRIBUTORS.md
   ```

3. **生成质量报告**
   ```bash
   python tools/contributor-workflow/generate_quality_report.py \
       --output quality_report.md
   ```
